Vergleiche

Mit der Analysefunktion Vergleiche können Sie zwei oder mehr Datensätze miteinander vergleichen, um festzustellen, ob die Informationen gleich oder unterschiedlich sind, je nachdem, was Sie ausgewählt haben. Mit diesem Test können Sie beispielsweise überprüfen, ob der für die Eingabe von Transaktionsdetails zuständige Benutzer sich von dem Benutzer unterscheidet, der diese Transaktionen autorisiert. Dieser Test kann bei der Überprüfung der Konsistenz zwischen den Eingangsberichten und den entsprechenden Rechnungen hilfreich sein.

Diese Analyse kann zur Identifizierung verwendet werden:

  • Aufdeckung von möglichen Versuchen zur Manipulation von Jahresabschlüssen

  • Bereiche der Finanzberichterstattung, die eine geringere Genauigkeit oder Präzision aufweisen

  • Systemische Probleme bei der Verarbeitung von Transaktionen in einer Abteilung

  • Unsachgemäß konfigurierte Systemeinstellungen, die möglicherweise verbessert und weiter ausgewertet werden müssen

  • Unzureichende interne Kontrollen

  • Schwache Überwachung und Aufdeckung von Aktivitäten

  • Mangelhafte Ausbildung

  • Risiko wesentlicher falscher Darstellungen aufgrund potenzieller Betrugsfälle.

Für die Analyse verwendete Felder

Die folgenden Felder sind für diese Analyse erforderlich:

  • Referenzfeld(er) - Eindeutige(s) Feld(er), das/die zur Erstellung einer eindeutigen Transaktions-ID verwendet wird/werden, wie z. B. das Feld Eintrags-ID für den Hauptbuch-Datensatz. Diese Spalten sind nicht Teil des Ergebnisses, sondern dienen der Identifizierung der Transaktionen, die Teil des Ergebnisses sind. Dieses Feld wurde bereits im Test definiert und kann nicht geändert werden.

  • Kernfelder - Zwei oder mehr Felder eines beliebigen Typs müssen für den Vergleich markiert werden. Die markierten Felder sollten vom gleichen Typ sein.

Parameter

Zur Durchführung dieses Tests müssen die folgenden Parameter eingestellt werden:

Vergleichstyp - Wählen Sie aus, wie die Daten in den ausgewählten Feldern miteinander verglichen werden sollen. Ihre Optionen sind:

  • Gleich

  • Ungleich

  • Größer als

  • Größer als oder gleich

  • Kleiner als

  • Kleiner als oder gleich

Test-Konfigurationen

Hinweis: Die verfügbaren Konfigurationen hängen von dem von Ihnen verwendeten Produkt ab.

Die folgenden Konfigurationen sind für diesen Test verfügbar:

  • Zeichenfeldvergleich - Vergleicht zwei Zeichenfelder und listet sie basierend darauf auf, ob sie gleich oder ungleich sind.

  • Datumsfeldvergleich - Vergleicht zwei Datumsfelder und listet sie basierend darauf auf, ob sie gleich oder ungleich sind.

  • Numerisches Feld - Vergleich - Vergleicht zwei numerische Felder und listet sie basierend darauf auf, ob sie gleich oder ungleich sind.

Technische Spezifikationen

Wenn Sie die Analysefunktion Vergleiche ausführen, werden die folgenden Schritte zur Durchführung des Tests durchgeführt:

  1. Legen Sie bei Bedarf Filter über die Daten, damit eine Teilmenge für die Analyse verwendet wird. Wenn kein Filter gesetzt ist, wird die Analyse für die gesamte Datei durchgeführt. Dieser Schritt kann auch als letzter statt als erster Schritt durchgeführt werden.

    Hinweis: Filter sind derzeit nicht verfügbar und werden erst in einer späteren Version enthalten sein.

  2. Überprüfen Sie, ob die erforderlichen Referenzfelder ausgewählt wurden. Wenn keine Felder ausgewählt wurden, erstellen Sie ein eindeutiges Referenzfeld. Dies ist nur gültig, wenn bestimmte Felder ausgewählt wurden. Wenn alle Spalten vorhanden sind, ist dies nicht erforderlich.

  3. Überprüfen Sie, ob zwei oder mehr Felder ausgewählt wurden und ob alle ausgewählten Felder vom gleichen Typ sind.

  4. Ermitteln Sie die Art des Vergleichs anhand der eingegebenen Parameter. Die Optionen sind:

    • Gleich (==)

    • Ungleich (!=)

    • Größer als oder gleich (>=)

    • Größer als (>)

    • Kleiner als oder gleich (<=)

    • Kleiner als (<)

  5. Führen Sie die Kernanalyse durch, indem Sie die mit der Vergleichsart ausgewählten Felder vergleichen.

    • Beim Vergleich zweier Felder werden alle Datensätze extrahiert, bei denen der Vergleich „wahr“ ergibt.

    • Beim Vergleich von drei oder mehr Feldern werden alle Datensätze extrahiert, bei denen der Vergleich zwischen mindestens zwei der Felder den Wert „wahr“ ergibt.

  6. Extrahieren Sie die vom Benutzer ausgewählten Ergebnisfelder. Alle Felder werden standardmäßig extrahiert.

    Hinweis: Die Möglichkeit, Ergebnisfelder auszuwählen, ist derzeit nicht verfügbar und wird in einer späteren Version aufgenommen.

  7. Optional können die Filter in Schritt 1 auch nach Abschluss der Kernanalyse auf die Daten angewendet werden, anstatt vor deren Abschluss.

    Hinweis: Filter sind derzeit nicht verfügbar und werden erst in einer späteren Version enthalten sein.
