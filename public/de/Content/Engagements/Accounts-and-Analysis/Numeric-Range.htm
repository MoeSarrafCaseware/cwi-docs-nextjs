Numerischer Bereich

Diese Analyse Numerischer Bereich identifiziert Transaktionen innerhalb eines definierten Wertebereichs. Ihr Hauptzweck besteht darin, Transaktionen mit hohen Werten zu kennzeichnen. Aber Sie haben die Möglichkeit, die Konfiguration so anzupassen, dass Transaktionen unter einem bestimmten Schwellenwert oder innerhalb bestimmter Wertbereiche ermittelt werden. Zudem können Sie auch wählen, ob Sie prozentuale Schwellenwerte anstelle von numerischen Schwellenwerten verwenden möchten. Sie können beispielsweise die Analyse so konfigurieren, dass das höchste 1 % der Transaktionen identifiziert wird.

Diese Analyse kann zur Identifizierung verwendet werden:

  • Transaktionen, die signifikante Schätzungen enthalten

  • Komplexe, nicht routinemäßige oder ungewöhnliche Geschäftsvorgänge, die weitere Untersuchungen rechtfertigen könnten

  • Wesentliche Transaktionen

  • Fehlende Transaktionen, die potenziell entwertet wurden

  • Bewertungsfragen

  • Wesentliche Transaktionen innerhalb der Grundgesamtheit

  • Transaktionen mit einem geringem Wert oder dem Wert Null innerhalb der Grundgesamtheit

  • Transaktionen oder Salden mit dem Wert Null

  • Ein Bereich von Werten, z. B. knapp unter einem Grenzwert

  • Sollkonten mit Habentransaktionen oder Habenkonten mit Solltransaktionen

  • Kunden mit hohen Kreditlimits

  • Forderungen über einem bestimmten Betrag

  • Verbindlichkeiten aus Lieferungen und Leistungen über einem bestimmten Betrag

Für die Analyse verwendete Felder

Diese Analyse verwendet ein numerisches Feld, das entweder Beträge oder Mengen enthält. Sie können auswählen, welches Feld bei der Einstellung der Analyseparameter verwendet werden soll.

Parameter

Es gibt mehrere Parameter, die für die Durchführung dieses Tests festgelegt werden müssen:

  • Felder auswählen - Wählen Sie ein oder mehrere numerische Felder aus der Liste aus, die für die Durchführung des Tests verwendet werden sollen.

  • Numerischer oder Prozentwert - Wählen Sie aus, ob der Test anhand der numerischen Werte der Felder oder anhand der Prozentwerte durchgeführt werden soll. Standardmäßig ist Numerisch ausgewählt. Die Option, die Sie hier auswählen, bestimmt die übrigen Parameter für diesen Test.

Wenn Sie Numerisch wählen, treffen Sie eine Auswahl für die folgenden Parameter:

  • Schwellenwert für hohe Werte - Geben Sie einen Wert in dieses Feld ein, um alle Transaktionen mit Beträgen über diesem Wert zu extrahieren.

  • Schwellenwert für niedrige Werte - Geben Sie einen Wert in dieses Feld ein, um alle Transaktionen mit Beträgen unter diesem Wert zu extrahieren.

  • Wertbereiche - Geben Sie einen oder mehrere Wertbereiche ein, um alle Transaktionen mit Beträgen in diesen Bereichen zu extrahieren.

  • Genaue Werte - Geben Sie einen genauen Wert ein, um alle Transaktionen mit diesem Wert zu extrahieren. Sie können diesen Parameter beispielsweise verwenden, um alle Transaktionen mit Nullwerten zu extrahieren.

Wenn Sie Prozentwert wählen, treffen Sie eine Auswahl für die folgenden Parameter:

  • Schwellenwert für hohen Prozentsatz - Geben Sie in dieses Feld einen Prozentsatz ein, um alle Transaktionen zu extrahieren, deren Prozentsatz höher als dieser Wert ist.

  • Schwellenwert für niedrigen Prozentsatz - Geben Sie in dieses Feld einen Prozentsatz ein, um alle Transaktionen zu extrahieren, deren Prozentsatz niedriger als dieser Wert ist.

  • Wertbereiche - Geben Sie einen oder mehrere Prozentbereiche ein, um alle Transaktionen mit Prozentsätzen in diesen Bereichen zu extrahieren.

  • Genaue Werte - Geben Sie einen genauen Prozentsatz ein, um alle Transaktionen mit diesem Prozentsatz zu extrahieren.

Test-Konfigurationen

Hinweis: Die verfügbaren Konfigurationen hängen von dem von Ihnen verwendeten Produkt ab.

Die folgenden Konfigurationen sind für diesen Test verfügbar:

  • Betragsbereich (Betragsspanne) - Identifiziert Transaktionen innerhalb eines definierten Bereichs für Betragsfelder basierend auf einem numerischen Wert oder einem Prozentwert.

  • Rabattbereich - Identifiziert Transaktionen innerhalb eines definierten Bereichs für Rabattfelder basierend auf einem numerischen oder prozentualen Wert.

  • Mengenbereich - Identifiziert Transaktionen innerhalb eines definierten Bereichs für Mengenfelder, basierend auf einem numerischen oder prozentualen Wert.

  • Fälligkeitsbereich - Identifiziert Transaktionen innerhalb eines definierten Bereichs für Fälligkeitsfelder auf der Grundlage eines numerischen oder prozentualen Werts.

  • Rabattbereich - Identifiziert Transaktionen innerhalb eines definierten Bereichs für Rabattfelder basierend auf einem numerischen oder prozentualen Wert.

  • Hoher Geldwert - Identifiziert Transaktionen, bei denen das Betragsfeld größer oder gleich dem definierten numerischen Wert ist.

Technische Spezifikationen

Wenn Sie die Analyse Numerischer Bereich ausführen, werden die folgenden Schritte zur Durchführung des Tests durchgeführt:

  1. Legen Sie bei Bedarf Filter über die Daten, damit eine Teilmenge für die Analyse verwendet wird. Wenn kein Filter gesetzt ist, wird die Analyse für die gesamte Datei durchgeführt. Dieser Schritt kann auch als letzter statt als erster Schritt durchgeführt werden.

    Hinweis: Filter sind derzeit nicht verfügbar und werden erst in einer späteren Version enthalten sein.

  2. Überprüfen Sie, ob die erforderlichen Referenzfelder ausgewählt wurden. Wenn keine Felder ausgewählt wurden, erstellen Sie ein eindeutiges Referenzfeld. Dieser Schritt wird nur ausgeführt, wenn bestimmte Felder ausgewählt wurden. Wenn alle Felder vorhanden sind, ist dieser Schritt nicht erforderlich.

  3. Überprüfen Sie, ob das für den Test ausgewählte Feld numerisch ist.

  4. Bestimmen Sie auf der Grundlage der vom Benutzer ausgewählten Parameter die Art der Mengenextraktion:

    • Gleich dem Wert

    • Größer als der Wert

    • Größer als und gleich dem Wert

    • Kleiner als der Wert

    • Kleiner als und gleich dem Wert

    • Nicht gleich dem Wert

    • Eine Reihe von Werten

  5. Erfragen Sie vom Benutzer, ob die Analyse auf einem Prozentsatz oder einem Wert basieren soll.

    • Wenn ein Wert ausgewählt wird, wird der Wert vom Benutzer abgefragt. Wenn die Option eine Bereichsextraktion ist, werden zwei Werte benötigt.

    • Wenn ein Prozentsatz ausgewählt wird, erhalten Sie den Prozentsatz vom Benutzer und berechnen dann den Wert, den der Prozentsatz darstellt. Dieser Wert hängt von der Art der Extraktion ab, d. h. 10 % für hohe Werte ergeben einen anderen Wert als 10 % für niedrige Werte. Wenn die Option eine Bereichsextraktion ist, werden zwei Werte benötigt.

  6. Extrahieren Sie die vom Benutzer ausgewählten Ergebnisfelder. Alle Felder werden standardmäßig extrahiert.

    Hinweis: Die Möglichkeit, Ergebnisfelder auszuwählen, ist derzeit nicht verfügbar und wird in einer späteren Version aufgenommen.
