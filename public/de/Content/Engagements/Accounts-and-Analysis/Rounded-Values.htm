Gerundete Werte

Die Analyse Gerundete Werte identifiziert numerische Werte, die möglicherweise gerundet wurden, z. B. 10, 500 oder 9000. In Rechnungslegungsdaten ist das Vorhandensein von gerundeten Beträgen in der Regel unüblich und kann auf eine mögliche Datenmanipulation hindeuten.

Diese Analyse kann zur Identifizierung verwendet werden:

  • Aufdeckung von möglichen Versuchen zur Manipulation von Jahresabschlüssen

  • Geringere Genauigkeit und Präzision von Bereichen der Rechnungslegung

  • Transaktionen, die signifikante Schätzungen enthalten

  • Risiko wesentlicher falscher Darstellungen aufgrund potenzieller Betrugsfälle.

Hinweis: In einigen Branchen oder Unternehmen kann es aufgrund der Art ihrer Tätigkeit zu vielen gerundeten Transaktionen kommen.

Für die Analyse verwendete Felder

Dieser Test verwendet ein oder mehrere numerische Felder, um gerundete Beträge oder gerundete Mengenwerte zu ermitteln. Sie können festlegen, welche Felder verwendet werden sollen, indem Sie den Parameter Felder auswählen ändern.

Parameter

Es gibt drei Parameter, die für die Durchführung dieses Tests festgelegt werden müssen:

  • Felder auswählen - Wählen Sie ein oder mehrere numerische Felder aus der Liste aus, die für die Durchführung des Tests verwendet werden sollen.

  • Gerundeter Wert - Geben Sie einen gerundeten ganzzahligen Wert ein, z. B. 10, 100 oder 1000, der für die Suche nach gerundeten Werten verwendet werden soll. Wenn Sie z. B. 10 eingeben, sucht der Test nach allen Werten, die ein Vielfaches von 10 sind, z. B. 10, 20 und 30. Dieser Test ist nicht auf Vielfache von 10 beschränkt. Sie können auch 5, 50, 500 oder einen anderen ganzzahligen Wert eingeben. Der Standardwert ist 100.

  • Dezimale ignorieren - Legen Sie fest, ob Dezimalwerte ignoriert werden sollen oder nicht. Dieses Feld legt fest, ob Dezimalwerte als gerundete Werte betrachtet werden. Wenn Sie z. B. Dezimalstellen ignorieren, wird der Wert 100,99 als gerundeter Wert betrachtet, aber wenn Sie die Dezimalstellen nicht ignorieren, handelt es sich nicht um einen gerundeten Wert. Der Standardwert ist so eingestellt, dass Dezimalstellen ignoriert werden.

Test-Konfigurationen

Hinweis: Die verfügbaren Konfigurationen hängen von dem von Ihnen verwendeten Produkt ab.

Die folgenden Konfigurationen sind für diesen Test verfügbar:

  • Gerundete Beträge - Listet Transaktionszeilen auf, bei denen der Betrag gerundet ist.

  • Gerundete Menge - Listet Transaktionszeilen auf, bei denen die Menge gerundet ist.

  • Gerundete Werte - Listet Transaktionszeilen auf, bei denen der Wert gerundet ist.

Technische Spezifikationen

Wenn Sie die Analyse Gerundete Beträge ausführen, werden die folgenden Schritte zur Durchführung des Tests durchgeführt:

  1. Überprüfen Sie, ob das/die analytische(n) Kernfeld(er) numerisch ist/sind; falls nicht, beenden Sie den Test.

  2. Legen Sie fest, ob Dezimalzahlen in die Analyse einbezogen werden sollen.

  3. Sie erhalten vom Benutzer den numerischen Wert, der für die Rundung verwendet wird, führen den Test durch und extrahieren alle Transaktionen, die gerundete Werte des numerischen Wertes für das Ergebnis sind. Berücksichtigen Sie, ob der Benutzer Dezimalstellen ein- oder ausgeschlossen hat. Wenn mehrere Spalten ausgewählt wurden und eine beliebige Spalte fehlschlägt, schlägt die Zeile beim Test fehl und wird in die Ergebnisse eingetragen.

  4. Extrahieren Sie die vom Benutzer ausgewählten Ergebnisfelder. Alle Felder werden standardmäßig extrahiert.

    Hinweis: Die Möglichkeit, Ergebnisfelder auszuwählen, ist derzeit nicht verfügbar und wird in einer späteren Version aufgenommen.
