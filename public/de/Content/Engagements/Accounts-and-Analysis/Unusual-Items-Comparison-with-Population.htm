Ungewöhnliche Elemente

Die Analyse Ungewöhnliche Elemente (Vergleich mit der Grundgesamtheit) identifiziert Transaktionen, die im Vergleich zu einer ausgewählten Gruppe von Transaktionen als ungewöhnlich gelten.

Dieser Test kann zum Beispiel dazu verwendet werden, Transaktionen zu identifizieren, die von Benutzern gebucht werden, die im Vergleich mit anderen Transaktionen selten buchen. Die Nutzer werden mit der Grundgesamtheit zusammengefasst, um einen Prozentsatz der Transaktionen zu erhalten, und alle Nutzer, die unter einem bestimmten Schwellenwert liegen, werden im Ergebnis angezeigt.

Sie können diesen Test auch für die Analyse von Datumsfeldern verwenden. Die Transaktionspopulation wird nach Datum zusammengefasst, und Daten mit einer geringen Anzahl von Transaktionen werden im Ergebnis gekennzeichnet.

Diese Analyse kann zur Identifizierung verwendet werden:

  • Fiktive oder nicht belegte Transaktionen

  • Systemische Probleme bei der Verarbeitung von Transaktionen in einer Abteilung

  • Unsachgemäß konfigurierte Systemeinstellungen, die möglicherweise verbessert und weiter ausgewertet werden müssen

  • Aufdeckung von möglichen Versuchen zur Manipulation von Jahresabschlüssen

  • Bereiche der Finanzberichterstattung, die eine geringere Genauigkeit und Präzision aufweisen

  • Schwächen bei internen Kontrollen, die weitere Auswirkungen haben, die es zu berücksichtigen gilt

  • Schwache Überwachung und Aufdeckung von Aktivitäten

  • Transaktionen, die außerhalb der normalen Geschäftstage stattfanden

  • Risiko wesentlicher falscher Darstellungen aufgrund potenzieller Betrugsfälle.

Für die Analyse verwendete Felder

Die folgenden Felder sind für diese Analyse erforderlich:

  • Referenzfeld(er) - Eindeutige(s) Feld(er), das/die zur Erstellung einer eindeutigen Transaktions-ID verwendet wird/werden, wie z. B. das Feld Eintrags-ID für den Hauptbuch-Datensatz. Diese Spalten sind nicht Teil des Ergebnisses, sondern dienen der Identifizierung der Transaktionen, die Teil des Ergebnisses sind. Dieses Feld wurde bereits im Test definiert und kann nicht geändert werden.

  • Ungewöhnliche(s) Feld(er) - Die Felder, deren Werte untersucht werden, um festzustellen, ob sie ungewöhnlich sind. Bei diesen Feldern kann es sich um numerische, Datums-, Zeit- oder Zeichenfelder handeln, die eine Art von wiederkehrendem Muster enthalten. Ein numerisches Feld, das Benutzer-IDs enthält, könnte zum Beispiel dazu verwendet werden, Transaktionen zu identifizieren, die von Benutzern gebucht werden, die nur selten buchen. Felder, die Beträge oder Mengen enthalten, eignen sich nicht so gut für diesen Test, da sie normalerweise kein wiederkehrendes Muster aufweisen.

Parameter

Zur Durchführung dieses Tests müssen die folgenden Parameter eingestellt werden:

  • Prozentwert - Geben Sie den Prozentwert ein, den der Test verwenden soll, um zu bestimmen, welche Werte als ungewöhnlich betrachtet werden sollen. Wenn Sie beispielsweise einen Prozentwert von 3 % eingeben, wird jedes Element in der Spalte, das 3 % oder weniger der Population ausmacht, als ungewöhnlich betrachtet und in die Testergebnisse einbezogen.

  • Typ des Prozentsatzes - der Typ gibt an, ob der Prozentsatz kleiner, kleiner oder gleich, gleich, größer oder größer oder gleich dem Prozentwert sein soll.

Test-Konfigurationen

Hinweis: Die verfügbaren Konfigurationen hängen von dem von Ihnen verwendeten Produkt ab.

Die folgenden Konfigurationen sind für diesen Test verfügbar:

  • Ungewöhnliche Elemente im Vergleich zur Grundgesamtheit

  • Ungewöhnliche Datumseinträge im Vergleich zur Grundgesamtheit

Technische Spezifikationen

Wenn Sie die Analysefunktion Ungewöhnliche Elemente ausführen, werden die folgenden Schritte zur Durchführung des Tests durchgeführt:

  1. Legen Sie bei Bedarf Filter über die Daten, damit eine Teilmenge für die Analyse verwendet wird. Wenn kein Filter gesetzt ist, wird die Analyse für die gesamte Datei durchgeführt. Dieser Schritt kann auch als letzter statt als erster Schritt durchgeführt werden.

    Hinweis: Filter sind derzeit nicht verfügbar und werden erst in einer späteren Version enthalten sein.

  2. Überprüfen Sie, ob die erforderlichen Referenzfelder ausgewählt wurden. Wenn keine Felder ausgewählt wurden, erstellen Sie ein eindeutiges Referenzfeld. Dieser Schritt wird nur ausgeführt, wenn bestimmte Felder ausgewählt wurden. Wenn alle Felder vorhanden sind, ist dieser Schritt nicht erforderlich.

  3. Beziehen Sie die Spalten, um die Teilmenge zu erstellen, für die diese Analyse durchgeführt werden soll. Diese Teilmenge kann auf einer oder mehreren Spalten basieren. Die Spalten können von beliebigem Typ und in beliebiger Reihenfolge sein.

  4. Ermitteln Sie den Prozentsatz zum Vergleich.

    • Wenn der Benutzer eine der folgenden Möglichkeiten gewählt hat, ist nur ein Prozentsatz erforderlich:

      • Kleiner als (<)

      • Kleiner als oder gleich (<=)

      • Gleich (==)

      • Größer als oder gleich (>=)

      • Größer als (>)

    • Wenn der Benutzer einen Bereich auswählt, muss es einen Prozentsatz für einen niedrigen und einen für einen hohen Wert geben. Der Bereich kann auf folgenden Faktoren basieren:

      • Größer als (>)

      • Größer als oder gleich (>=)

      • Kleiner als oder gleich (<=)

      • Kleiner als (<)

  5. Fassen Sie die Spalten der Teilmenge zusammen, um die Anzahl der Datensätze pro eindeutiger Teilmenge zu erhalten.

  6. Berechnen Sie den Prozentsatz jeder Teilmenge auf der Grundlage der Gesamtzahl der Datensätze in der Grundgesamtheit.

  7. Extrahieren Sie alle Teilmengen, die in den in Schritt 4 ausgewählten prozentualen Vergleich fallen.

  8. Extrahieren Sie die Transaktionen (Zeilen), die sich auf die in Schritt 7 extrahierten Teilmengen beziehen.

  9. Extrahieren Sie die vom Benutzer ausgewählten Ergebnisfelder. Alle Felder werden standardmäßig extrahiert.

    Hinweis: Die Möglichkeit, Ergebnisfelder auszuwählen, ist derzeit nicht verfügbar und wird in einer späteren Version aufgenommen.
