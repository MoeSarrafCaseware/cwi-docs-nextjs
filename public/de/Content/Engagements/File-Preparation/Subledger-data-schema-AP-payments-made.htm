Accounts payable—payments made described

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Accounts payable—payments made subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Feld Usage
Transaktions-ID
(ap_payment_made_transaction_id)

A cross-reference to the general ledger posting. Kann viele verschiedene Ebenen beinhalten, um die Transaktion zu identifizieren. Examples include cost centers such as company, division, region, group and branch/department.

Data type: Text

Required: Nein

Primary key: The value of this field is used to form the unique identifier of this entry.
Zahlungsnummer
(ap_payment_made_payment_id)

The unique identifier for each payment made. This identifier may need to be created by concatenating fields, for example, supplier and payment number to uniquely identify each transaction.

Data type: Text

Required: Ja

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>

Zahlungsdatum
(ap_payment_made_payment_date)

Datum, an dem die Zahlung geleistet oder die Gutschrift vorgenommen wurde.

Data type: Datum

Required: Nein

Zahlungsjahr
(ap_payment_made_fiscal_year)

Fiscal year in which Payment Date occurs.

Data type: Text

Required: Nein

Primary key: Nein
Zahlungsperiode
(ap_payment_made_payment_period)

Fiscal period in which Payment Date occurs.

Data type: Text

Required: Nein

Primary key: Nein
Betragstyp (Soll/Haben)
(ap_payment_made_amount_credit_debit_indicator)

Kennzeichen, ob es sich bei dem Betrag um einen Soll- oder Habenposten handelt.

Data type: Text

Required: Nein

Possible values:
C=credit;
D=debit
Zahlungsart
(ap_payment_made_payment_type)

Codewert oder Indikator für die Methode, mit der der Soll- oder Habenbetrag der Transaktion vom Lieferanten getilgt oder mit der Schuld verrechnet wurde. Beispiele sind Scheck, Überweisung, Bargeld usw.

Data type: Text

Required: Nein
Zahlungsbetrag
(ap_payment_made_payment_amount)

Transaktionsbetrag in der Haus- oder Konzernwährung.

Data type: Money

Required: Nein
Zahlungsbetrag (Berichtswährung)
(ap_payment_made_payment_amount_reporting)

Transaktionsbetrag in der Berichtswährung.

Data type: Money

Required: Nein
Local Payment Amount
(ap_payment_made_payment_amount_local)

Transaction monetary amount in the local currency.

Data type: Money

Required: Nein
Buchungsstatus
(ap_payment_made_posting_status)

Indicates whether the transaction has been posted to the General Ledger.

Data type: Text

Required: Nein

Possible values: Posted or Not Posted
Chargennummer
(ap_payment_made_batch_id)

A batch identifier generated by the source system.

Data type: Text

Required: Nein
Zahlungsbeschreibung
(ap_payment_made_description)

Beschreibung der Zahlung.

Data type: Text

Required: Nein
Herkunfts-ID
(ap_payment_made_source_id)

Buchungsquelle. Kennzeichnung des Systems, das den Buchungseintrag ursprünglich erzeugt hat (z. B. Verkaufsjournal, Kassenbuch, Hauptbuch, Lohnjournal, manuelle Eingabe durch den Buchhalter, Tabellenkalkulation und so weiter).

Data type: Text

Required: Nein
Supplier ID (ap_payment_made_supplier_account_id)

Identifikator des Lieferanten, bei dem die Zahlung eingegangen ist oder von dem die Gutschrift erfolgt ist.

Data type: Text

Required: Nein
Supplier Name
(ap_payment_made_supplier_account_name)

Name des Lieferanten.

Data type: Text

Required: Nein
Erfasser
(ap_payment_made_entered_by)

The user identifier of the person who created the record.

Data type: Text

Required: Nein
Erfassungsdatum
(ap_payment_made_entered_date)

The date the transaction was entered into the source system. Dies wird gelegentlich auch als Erstellungsdatum bezeichnet.

Data type: Datum

Required: Nein
Erfassungszeit
(ap_payment_made_entered_time)

The time this transaction was entered into the source system.

Data type: Date/time

Required: Nein
Genehmigt von

The user identifier of the person who approved the entry.

Data type: Text

Genehmigungsdatum

Datum, an dem der Eintrag freigegeben wurde.

Data type: Datum

Required: Nein
Genehmigungszeit
(ap_payment_made_approved_by_time)

Uhrzeit, zu der der Eintrag freigegeben wurde.

Data type: Date/time

Required: Nein
Geändert von
(ap_payment_made_last_modified_by)

The user identifier of the last person to modify this entry.

Data type: Text

Required: Nein
Änderungsdatum
(ap_payment_made_last_modified_date)

Datum, an dem der Eintrag zum letzten Mal geändert wurde.

Data type: Datum

Required: Nein
Änderungszeit
(ap_payment_made_last_modified_time)

Uhrzeit, zu der der Eintrag zum letzten Mal geändert wurde.

Data type: Date/time

Required: Nein
Rechnungsnummer
(ap_payment_made_invoice_number)

Identifikationsnummer einer intern oder extern erzeugten Transaktion. Examples include check number, wire transfer number, or original document identifier.

Data type: Text

Required: Nein

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Rechnungsdatum
(ap_payment_made_invoice_date)

Datum einer extern erzeugten Transaktion. Beispiele sind Scheckdatum oder Überweisungsdatum.

Data type: Datum

Required: Nein
Geschäftseinheitencode
(ap_payment_made_business_unit_code)

Dient zur Identifizierung der Geschäftseinheit, Region, Niederlassung usw. auf der Ebene, auf der der Jahresabschluss geprüft wird.

Data type: Text

Required: Nein
Sachkonto (Soll) Hauptbuch
(ap_payment_made_gl_debit_account_number)

The general ledger (GL) account number on which the debit side of the transaction has been posted.

Data type: Text

Required: Nein
Sachkonto (Haben) Hauptbuch
(ap_payment_made_gl_credit_account_number)

The general ledger (GL) account number on which the credit side of the transaction has been posted.

Data type: Text

Required: Nein
Zeilendetail - Konto-ID
(ap_payment_made_line_account_id)

The general ledger (GL) account code of the GL revenue account. This can include a subaccount identifier, as well as can many different levels to identify the Account, including cost centers, such as company, division, region, group and branch/department.

Data type: Text

Required: Nein
Zeilendetail - Kontobeschreibung
(ap_payment_made_line_account_description)

A description of the account whose identifier appears in Line Detail - Account ID.

Data type: Text

Required: Nein
Zahlungswährung
(ap_payment_made_payment_amount_currency)

Die Haus- oder Konzernwährung des Zahlungsbetrags.

Data type: Text

Required: Nein
Zahlungswährung (Berichtswährung)
(ap_payment_made_payment_amount_reporting_currency)

Berichtswährung bezogen auf den Zahlungsbetrag für die nicht konsolidierte Berichterstattung.

Data type: Text

Required: Nein
Zahlungswährung (lokale Währung)
(ap_payment_made_payment_amount_local_currency)

Währung für lokale Berichtsanforderungen.

Data type: Text

Required: Nein
Eintrags-ID
(ap_payment_made_journal_id)

Eindeutiger Identifikator eines Buchungseintrags.

Data type: Text

Required: Nein
