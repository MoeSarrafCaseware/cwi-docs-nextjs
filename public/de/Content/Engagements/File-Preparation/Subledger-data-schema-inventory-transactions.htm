Inventory transactions described

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Inventory transactions subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Feld Usage
Transaktions-ID
(inventory_transaction_transaction_id)

The transaction identifier associated with either the Procure to Pay or Order to Cash Subledger Audit Data Standards (ADS).

Data type: Text

Required: Nein

Primary key: The value of this field is used to form the unique identifier of this entry.
Transaktionsdatum
(inventory_transaction_transaction_date)

Datum der Aktivität pro zugehörigem Transaktionsbeleg, falls anwendbar.

Data type: Datum

Required: Nein

Primary key: Third alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Inventarprodukt-ID
(inventory_transaction_product_id)

The product identifier is a code that uniquely identifies an item that you carry in inventory.

Data type: Text

Required: Ja

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Seriennummer
(inventory_transaction_serial_number)

Im Falle einer Serienfertigung wird hier die Seriennummer erfasst. Obwohl Seriennummer der gängige Begriff ist, ist der Wert nicht auf numerische Werte beschränkt. Es ist zwar ungewöhnlich, dass ein Artikel sowohl einer Seriennummern- als auch einer Losnummernsteuerung unterliegt (die Seriennummer identifiziert den Artikel eindeutig), aber das schließt sich nicht unbedingt gegenseitig aus.

Data type: Text

Required: Nein

Primary key: Fourth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Losnummer
(inventory_transaction_lot_number)

Wenn es sich um ein Los handelt, wird hier die Losnummer erfasst. Obwohl Losnummer der gängige Begriff ist, ist der Wert nicht auf numerische Werte beschränkt.

Data type: Text

Required: No Primary key: Fifth alternative. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Transaktionstyp
(inventory_transaction_transaction_type)

Codes, die die wichtigsten bestandsspezifischen Aktionen beschreiben, die in Bestandstransaktionen im Zusammenhang mit der Bewegung von Waren, Handel, Mengenanpassungen aufgrund interner Geschäftsvorfälle und Kostenanpassungen aufgrund von Marktfaktoren oder Schäden vorkommen.

Data type: Enum

Required: Nein

Example values: Goods Received (GR), Goods Shipped (GS), Inventory Transfers (IT), Inventory Adjustments (IA), Other (OT)

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Transaktionstyp System
(inventory_transaction_transaction_type_system)

A transaction code that is local to the underlying accounting software system.

Data type: Text

Required: Nein
Transaktionszeit
(inventory_transaction_transaction_time)

Zeit, zu der die Aktivität stattgefunden hat, falls anwendbar pro zugehörigem Transaktionsbeleg.

Data type: Date/time

Required: Nein
Transaktionsmenge
(inventory_transaction_transaction_quantity)

Betroffene Menge oder Position, ausgedrückt in der Bestandseinheit des Lagerorts.

Data type: Decimal

Required: Nein
Transaktionskostenänderung
(inventory_transaction_transaction_cost_adjustment)

Anstieg (oder Rückgang) der Lagerhaltungskosten pro Einheit.

Data type: Decimal

Required: Nein
Transaktionsbeschreibung
(inventory_transaction_transaction_description)

Beschreibung der Transaktion.

Data type: Text

Required: Nein
Transaktionsbezogene Bestell-ID
(inventory_transaction_transaction_associated_order_id)

Debitorenbestellung, Kreditorenbestellung oder ein anderer mit der Transaktion verbundener Beleg. Should specify whether the number found in Transaction Associated Order Line ID is from the Procure to Pay or Order to Cash Subledger Audit Data Standards (ADS) by using the text P2P or "O2C”, respectively.

Data type: Text

Required: Nein
Transaktionsbezogene Bestellzeilen-ID
(inventory_transaction_transaction_associated_order_line_id)

Zeilennummer einer Debitorenbestellung oder Kreditorenbestellung zur Unterscheidung zwischen mehreren Positionen in einer einzigen Bestellung für unterschiedliche Mengen. Should specify whether the number found in Transaction Associated Order Line ID is from the Procure to Pay or Order to Cash Subledger Audit Data Standards (ADS) by using the text P2P or O2C, respectively.

Data type: Text

Required: Nein
Transaktionsbezogene Beleg-ID
(inventory_transaction_transaction_associated_document_id)

Kommissionierschein, Lieferschein oder ein anderer Beleg, der bei einer Bewegung erstellt wurde oder mit dieser verbunden ist. Should specify whether the number found in Transaction Associated Order Line ID is from the Procure to Pay or Order to Cash Subledger ADS by using the text P2P or O2C, respectively.

Data type: Text

Required: Nein
Transaktionsbezogene Belegzeilen-ID
(inventory_transaction_transaction_associated_document_line_id)

The line number for a document other than a Customer Order, or Vendor Purchase Order. The document should specify whether the number found in Transaction Associated Order Line ID is from the Procure to Pay or Order to Cash Subledger ADS by using the text P2P or O2C, respectively.

Data type: Text

Required: Nein
Bestände von Lagerstandort
(inventory_transaction_inventory_from_location)

The storage location from where the inventory originated.

Data type: Text

Required: Nein
Bestände zu Lagerstandort
(inventory_transaction_inventory_to_location)

Lagerort, an den die Bestände verbracht werden.

Data type: Text

Required: Nein
Bestandssystem oder externe Quelle
(inventory_transaction_inventory_system_or_external_source)

Dient zur Kennzeichnung von Eingabeinformationen bei Transaktionen, die von einem anderen ADS-Modul (z. B. P2P, O2C) oder aus dem Bestandssystem heraus angestoßen wurden. Wenn sie aus dem Bestandssystem heraus angestoßen wurden, werden die folgenden Felder zur Erfassung von Benutzer- und Hauptbuchinformationen verwendet.

Data type: Text

Required: Nein
Sachkonto Vorräte (Soll) Hauptbuch Zeileneintrag
(inventory_transaction_gl_line_debit_account_number)

Data type: Text

Required: Nein
Sachkonto Vorräte (Haben) Hauptbuch Zeileneintrag
(inventory_transaction_gl_line_credit_account_number)

The general ledger (GL) account number on which the credit side of the transaction has been posted.

Data type: Text

Required: Nein

Data type: Text

Bestandsorganisation Code
(inventory_transaction_inventory_organization_code)

Code der Bestandsorganisation. Die Bestandsorganisation bezieht sich auf die physische oder virtuelle organisatorische Einheit, in der die Bestandstransaktionen und -salden nachverfolgt und überwacht werden können. Sie liefert Bestandsinformationen an Module wie Einkauf und Verkauf. Die einfachste Form der Bestandsorganisation ist ein Lager.

Data type: Text

Required: Nein
Bestandsmengeneinheit Code
(inventory_transaction_inventory_stocking_uom_code)

The standardized measurement unit used to count stock items and to express them as specific quantities.

Data type: Text

Required: Nein
Bestandskostenmaßeinheit Code
(inventory_transaction_inventory_costing_uom_code)

Code der Maßeinheit für die Kosten des Bestands.

Data type: Text

Required: Nein
Bestandsanforderung gestellt von
(inventory_transaction_inventory_required_by)

Die Person oder Organisation, die eine Bestandsanforderung stellt.

Data type: Text

Required: Nein
Kostenorganisation Code
(inventory_transaction_cost_organization_code)

Code der Kostenorganisation, der sich auf die organisatorische Einheit/Ebene bezieht, auf der die Kosten für ausgehende Bestände berechnet werden. Die Berechnung kann auf Unternehmensebene, auf der Ebene der Lagerhaltungsorganisation oder an den einzelnen Standorten, an denen die Bestände gelagert werden, erfolgen.

Data type: Text

Required: Nein
Bestandskosten
(inventory_transaction_inventory_cost)

The cost per unit using the method found in Inventory Cost Method, in Inventory on Hand.

Data type: Text

Required: Nein
Geschäftseinheitencode
(inventory_transaction_business_unit_code)

Dient zur Identifizierung der Geschäftseinheit, Region, Niederlassung usw. auf der Ebene, auf der der Jahresabschluss geprüft wird.

Data type: Text

Required: Nein

Erfassungsdatum
(inventory_transaction_entered_date)

Date the transaction was entered into the source system. Dies wird gelegentlich auch als Erstellungsdatum bezeichnet. This should be a system-generated date (rather than user-entered date), when possible. Dieses Datum entspricht nicht unbedingt dem Datum der Transaktion selbst.

Data type: Datum

Required: Nein
