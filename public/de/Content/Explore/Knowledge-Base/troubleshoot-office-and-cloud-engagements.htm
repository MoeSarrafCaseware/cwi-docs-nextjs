Lösung von Problemen im Zusammenhang mit Microsoft Office und Cloud Engagements

Veröffentlicht: Mittwoch, 10. Januar 2024

Benötigen Sie Hilfe beim Hochladen von Office-Dokumenten in Caseware Cloud? Lösungen finden Sie unten.

Überprüfen Sie die Richtlinie für das formularbasierte Anmeldeverhalten, die im Microsoft Trust Center eingerichtet wurde

Bei diesem Verfahren wird davon ausgegangen, dass Sie ein Administrator sind und eine Abonnementversion von Microsoft Office haben, wie z. B. Microsoft 365 Apps für Unternehmen, um die Registerkarte Formularbasiertes Anmeldeverhalten im Microsoft Trust Center anzuzeigen.

Um die Richtlinie für das formularbasierte Anmeldeverhalten zu überprüfen:

  1. Öffnen Sie eine beliebige Office-Anwendung, z. B. Word oder Excel.

  2. Gehen Sie zu Datei | Optionen | Trust Center.

  3. Klicken Sie auf Trust Center-Einstellungen.

  4. Klicken Sie auf Formularbasierte Anmeldung.

  5. Wählen Sie unter Sign-in prompt behavior (Verhalten bei der Anmeldeaufforderung) die Option Ask me what to do for each host (Fragen Sie mich, was ich für jeden Host tun soll).

  6. Klicken Sie auf OK, um die Änderungen zu speichern.

  • Microsoft Office wird zukünftige Anmeldeaufforderungen nicht blockieren.
  • Die Liste der Hosts, die Anmeldeaufforderungen anzeigen dürfen wird auf der Grundlage Ihrer zukünftigen Aktionen automatisch ausgefüllt.

  1. Führen Sie auf der Seite Dokumente eines Auftrags einen der folgenden Schritte aus: 
    • Öffnen Sie eine vorhandene Microsoft Office-Datei. Weitere Informationen finden Sie unter Zugriff auf ein hochgeladenes Word- oder Excel-Dokument.
    • Oder fügen Sie eine neue Word- oder Excel-Datei hinzu. Weitere Informationen finden Sie unter Neue Word- und Excel-Dokumente hinzufügen.
  2. Nehmen Sie eine Änderung im Dokument vor, speichern Sie es und schließen Sie es.
  3. Aktualisieren Sie die Engagement-Seite in Ihrem Browser.
  4. Öffnen Sie das von Ihnen geänderte Microsoft Office-Dokument und rufen Sie die Trust Center-Einstellungen auf, wie zuvor in diesem Verfahren beschrieben.

  • Der folgende Eintrag sollte automatisch für Caseware Cloud in der Liste Hosts, die Anmeldeaufforderungen anzeigen dürfen:

app.casewarecloud.com

Überprüfen Sie, ob Add-Ins die Cloud beeinträchtigen

Beheben Sie Probleme mit Add-Ins, indem Sie sie nacheinander deaktivieren.

So deaktivieren Sie Add-Ins nacheinander:

  1. Öffnen Sie Excel.
  2. Wählen Sie Datei | Optionen | Add-ins.
  3. Wählen Sie in der Drop-down-Liste Verwalten die Option „Excel-Add-Ins“ und klicken Sie auf Los....
  4. Wenn Sie in der Liste Verwalten keine Add-Ins-Option für die Office-Anwendung finden können, wählen Sie stattdessen COM-Add-Ins.
  5. Deaktivieren Sie eines der Add-Ins in der Liste und klicken Sie auf OK.
  6. Versuchen Sie, unterstützende Word- oder Excel-Dokumente in ein Cloud-Engagement hochzuladen, und prüfen Sie, ob das Problem gelöst ist.
  7. Wiederholen Sie diese Schritte, bis Sie das Add-In gefunden haben, das das Problem verursacht.

Überprüfen des HttpClient unter Windows

HttpClient ist eine Klasse im .NET-Framework, die zum Senden von HTTP-Anforderungen und Empfangen von HTTP-Antworten von einer durch einen URI identifizierten Ressource verwendet wird und in verschiedenen Arten von Anwendungen zum Einsatz kommt. Sie kann den Informationsaustausch zwischen Microsoft Office und Cloud-Engagements beeinträchtigen.

So überprüfen Sie, ob der HttpClient ein Problem beim Informationsaustausch verursacht:

  1. Geben Sie in das Windows-Suchfeld services.msc ein
  2. Select Dienste.
  3. Suchen Sie unter Name nach WebClient.
  4. Führen Sie einen rechten Mausklick darauf aus und wählen Sie Eigenschaften.
  5. Wählen Sie bei Starttyp die Option Automatisch.
  6. Klicken Sie auf Übernehmen und anschließend auf OK.
  7. Verlassen Sie den Dialog.
  8. Überprüfen Sie, ob das Problem dadurch behoben wird.

Fehlerbehebung beim Speichern von Office-Dokumenten in Cloud Engagements

Überprüfen Sie, ob die automatische Speicherung deaktiviert wurde

Die automatische Speicherung muss deaktiviert werden, damit die Interaktion zwischen Microsoft Office und der Cloud ordnungsgemäß funktioniert.

Bitte beachten Sie: Sie dürfen keine Website eines Drittanbieters verwenden, um ein Microsoft Office-Dokument zu speichern oder zu sichern, das an ein Cloud-Engagement angehängt wurde.

So überprüfen Sie, ob die automatische Speicherung deaktiviert wurde:

  1. Öffnen Sie eine beliebige Office-Anwendung, z. B. Word oder Excel.
  2. Wählen Sie Datei | Optionen | Speichern.
  3. Deaktivieren Sie die Option Automatisch in der Cloud gespeicherte Dateien standardmäßig in Excel speichern.

Vergewissern Sie sich, dass Sie die Office-Dateien nicht aus dem standardmäßigen lokalen Dateispeicher hochladen

Für Office-Anwendungen ist der standardmäßige lokale Dateispeicherort wie folgt:

C:\Benutzer\{benutzer.name}\Dokumente

Standardspeicherort Ihrer lokalen Datei überprüfen:

  1. Öffnen Sie Excel.
  2. Wählen Sie Datei | Optionen | Speichern.
  3. Suchen Sie den Eintrag zu Lokaler Standardspeicherort für Datei und überprüfen Sie den Pfad. Stellen Sie sicher, dass Sie keine Dateien von diesem Speicherort in die Cloud hochladen.

Stellen Sie sicher, dass Sie oder andere Cloud Engagement-Benutzer die Freigabefunktion in Microsoft Office nicht verwenden

Vergewissern Sie sich, dass Sie oder andere Benutzer nicht die Microsoft Office-Freigabefunktion in Microsoft Office-Dokumenten verwenden, die Sie in Cloud-Engagements hochladen oder erstellen. Die Microsoft Office-Freigabefunktion wird in Verbindung mit Caseware Cloud nicht unterstützt:

  • Wenn zwei Benutzer auf dieselben Office-Dokumente zugreifen und diese gemeinsam nutzen, kann es zu Wettlaufsituationen kommen, die zu unerwarteten Ergebnissen führen.

  • Wenn ein Benutzer ein Office-Dokument über die Office-Freigabefunktion freigibt, ist eine Website erforderlich, um Synchronisierungsvorgänge zu unterstützen. Folglich müssen die Synchronisierungsvorgänge auf dem Computer des Benutzers stattfinden, der die Freigabe initiiert hat. Beim Speichern des gemeinsamen Dokuments können mehrere Szenarien eintreten, die alle zum Verlust der gespeicherten Informationen führen.

Um dies zu beheben:

  1. Entfernen Sie alle Freigaben für aktuelle Dokumente, indem Sie das Dokument an einen anderen Ort kopieren.
  2. Nehmen Sie alle erforderlichen Aktualisierungen an den Dokumenten vor, z. B. alle Änderungen, die bei aktivierter Freigabe nicht vorgenommen werden konnten.
  3. Ersetzen Sie die Datei, indem Sie die nicht freigegebene gespeicherte Version hochladen.

Bereitstellung von Protokollen auf Anfrage des technischen Supports

Die Bereitstellung dieser Protokolle hilft uns bei der weiteren Fehlersuche, wenn Sie sich an den technischen Support wenden.

Mac

Wenn Sie oder Ihr Kunde Mac OS verwenden und wir die Protokolle überprüfen müssen, benötigt unser technischer Support Ihre Hilfe, um sie zu sammeln.

Pfad, unter dem Sie die Protokolle finden können

/Users/user_name/Library/Containers/com.microsoft.Excel/Data/Library/Logs/Diagnostics

Um die Protokolle zu finden, müssen Sie möglicherweise die Protokolldateien ausblenden. Drücken Sie gleichzeitig die Tasten Shift + (⌘) + (.).

 

 

Weitere Informationen finden Sie unter Protokolldatei mit der Konsole auf dem Mac suchen.

Windows

Die folgenden Anweisungen gelten für Windows 10.

  1. Öffnen Sie den Windows Explorer.
  2. Navigieren Sie zu C:\Benutzer\{benutzer.name}\AppData\Local\Temp
  3. Suchen Sie den Ordner Diagnostics.

  • Suchen Sie den Ordner, der nach der Anwendung benannt ist, mit der Sie Probleme haben.

Löschen Sie den Microsoft Office-Cache

Siehe die Anweisungen für Ihr Betriebssystem:

Windows

  1. Löschen Sie die zwischengespeicherten Dateien manuell, und öffnen Sie die Dateien erneut.

Hinweis: Die aktuell in Excel geöffnete Datei wird ausgeschlossen. Dies sollte einen Download aus der Caseware Cloud auslösen.

  1. Wählen Sie in Ihrer Office App Datei | Optionen | Speichern.
  2. In Cache Settings, select Delete files from the Office Document cache when they are closed. This clears the file's cache when it is closed.

Mac OS

Go to the following location, which will change slightly depending on whether you use Excel or Word:

~/Library/Containers/MicrosoftExcel/Data/Library/Caches

For more information, see Clear the Office cache.
