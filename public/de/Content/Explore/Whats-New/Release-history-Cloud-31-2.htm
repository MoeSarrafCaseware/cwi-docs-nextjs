What's new - Cloud 31.2

Caseware Cloud 31.2 enhances your firm’s security with ADFS single sign-on integration, permission updates and more.

Single sign-on integration with ADFS

Caseware Cloud now supports single sign-on (SSO) integration with Microsoft Active Directory Federation Services (ADFS). Integration with ADFS simplifies the staff login process and improves security. To get started, see Integrate single sign-on with ADFS.

Separate staff and contact group permissions

We’ve split the permission for staff and contact groups to provide greater control over these areas. Users with the Staff Admin role now manage staff groups, while users with the Entities Admin role now manage contact groups. If you still require the previous functionality of these roles, you can create a custom role with both permissions.

Alternative Share permission

We’ve added an alternative Share permission to provide greater control over security in your Cloud site. This new Share permission (labeled S) restricts users from sharing entities or entity content that they themselves cannot access. The existing Share permission (relabeled to A) allows users to share all available entity or entity content permissions to themselves and other users, even if it would grant them a higher level of access than they currently possess. For more information and best practices, see Create custom security roles.

Multiple attachments in Other Engagements

You can now attach multiple documents to Other Engagements. This enables firms to use Caseware Cloud to track and manage a greater variety of engagements.

General enhancements and fixes

In addition to the latest features, Caseware Cloud 31.2 includes various enhancements and bug fixes to improve your overall workflow.
