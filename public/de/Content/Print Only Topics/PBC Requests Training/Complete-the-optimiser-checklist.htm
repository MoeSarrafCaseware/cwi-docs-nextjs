Complete the optimiser checklist

Initially, the Documents page displays one document, the Optimiser checklist. It consists of questions and inquiries that you need to respond to. The responses you provide automatically populate additional content and documents in the file. In this chapter, we'll look at how to modify and complete the checklist, as well as how to create a custom checklist.

Optimiser

Select the Optimiser checklist to open it.

Once you open the document, a number of options are available at the top of the page.

Aufgaben

Select the Issues () icon to add an issue to the document. See the Issues section in the previous chapter to learn more.

Print

Select the Print () icon to print the document. Once you click the icon, you can adjust your print settings before printing.

Review Tools

Select the Review Tools () icon to open the review tools pane.

Review tools apply visual markup to the document, making potential issues with content stand out on the page so you can complete your review more quickly.

The following review tools are available in the checklist:

Tool Beschreibung

Added

Content that was added by staff members. Ensure that it is relevant and necessary for the engagement.

Modified

Changes made to the default product content by staff members. Ensure that the changes are correct and that content still complies with guidelines and standards.

Overridden visibility

Content that has been manually hidden or forced visible by staff members. Confirm that the hidden content does not apply to the engagement. Confirm that the forced visible content does apply to the engagement.

Optimized out

Content that has been automatically hidden based on its visibility conditions. Confirm that the hidden content does not apply to this engagement.

Non optimal answers

Responses that may indicate a problem on the engagement. Confirm that the chosen response is correct and determine whether annotation or supporting documentation is required.

Modified after sign-off

Responses that were changed after a staff member had signed off on the query.

Not signed off

Steps that have not been signed off by staff members.

You can display different review content by selecting the appropriate checkboxes in the left pane.

Edit view

Select the Edit () icon to open edit view. In edit view, you can customize the checklist and the checklist settings.

Checklist settings

To review and modify the checklist settings, select the Settings icon () to open the Checklist Settings dialog. Once you're done, select Save to apply the changes to the checklist.

The following options are available in the Checklist Settings dialog.

Einstellung

Beschreibung

Guidance title

This is the title for the document guidance information. You can enter a guidance title to change the default title.
Risk title This is the title for the document risk information. You can enter a risk title to change the default title.

Sign-off procedures and conclusion

Select this check box to add a signoff option in all procedures and the conclusion rather than having one signoff for the whole document.

Conclusion available

Select this check box to add a conclusion section to the document. Once you select it, a text field displays so you can add a title to the conclusion section.

Enable assertions

Select this check box to add assertions in the document, if a procedure addresses any of them. Once selected, the assertions icon () displays at each procedure.

Use non-consolidated data

Select this check box to determine whether or not non-consolidated data can be used in the document.

Allow input notes in procedures

Select this check box to insert the Add note button below each response in the document which allows users to enter a note related to the response or procedure. You can also use this button to add a note placeholder.

Response

Select + Response to define the default response each procedure in this checklist. If you don't define a default response, you'll have to define a response every time you add a procedure in the document.

For information on how to add a response type, see Add a response type to a procedure.

Show responses beneath procedure

Select this check box to display responses below their respective procedures.

Analysis

Enable data analysis by selecting the desired chart option from the drop-down list. The data used for analysis is sourced from the trial balance.

Once you enable this option, the Analysis page is added in the document. The tab displays the selected charts for specific a group or as an Analysis document for an overall financial analysis. For information on how to use the Analysis page, see View account analysis in checklists.

Show risk information

Select this check box to enable the risks tab. You need to also define the data source for this tab using the Financial groups section.

Standardpositionen

Select one or more financial groupings to associate with the checklist. If you enabled the Risks tab, risks that apply to the selected financial groupings are displayed in the tab.

Add group

Groups are sections of the checklist document where you can add procedures. You can organize your procedures by adding them to groups and subgroups.

To add a group, select + Group.

The default title of the new group is New Group with a black font color. Sie können den Standardtitel auswählen, um ihn zu ändern.

Add subgroup

You can also add subgroups to a group.

To add a subgroup in a group:

  1. Go to the group that you want to add a subgroup for.

  2. Select More actions (), then choose Add subgroup.

A new subgroup is added with a default title New Group with a blue font color. Sie können den Standardtitel auswählen, um ihn zu ändern.

Add procedure

Procedures are the checklist items that users need to go through and provide responses for. You can only add procedures in groups or subgroups.

To add a procedure:

  1. Go to the group or subgroup that you want to add a procedure in.

  2. Select More actions () | Add Procedure or select + Procedure.

    A new procedure is added with a default description New Procedure and the default response type (if specified in the document settings).

  3. Select the default description to change it.

    Once you select the text field, the formatting toolbar displays. Use the toolbar to make your changes.

    You can also select the Summary field above the description to enter a short name for the procedure.

Edit visibility

Earlier in the training, we learned that OnPoint Collaborate displays different content based on the answers you provide. This is done through visibility settings.

Was sind Sichtbarkeitseinstellungen?

Visibility settings provide built-in automation so that only the relevant documents or content are available in your engagement file. This is achieved by setting up visibility conditions for various engagement items.

The application evaluates the conditions and if the conditions are met, some items are visible (available) in the engagement file. Otherwise, they're hidden (not available).

To set visibility for a procedure:

  1. Select the procedure that you want to set visibility for.

  2. Select More actions () | Settings.

  3. Select Edit Visibility to display the Visibility Settings dialog.

    4. You can also select the Visibility () | Settings ().

  4. Select Hide or Show from the blue drop-down list to specify the visibility action.

  5. Select all or any to determine whether all conditions must be met or one condition must be met for the visibility action (from Step 4) to apply.

  6. Select Add Condition.

  7. Choose a condition type from the list, then specify the condition parameters.

  8. Once you're done, select Save.

You can select Clear to clear all conditions and start over.

Hide procedure numbers within group

If you want to hide procedure numbering within a group, select More actions () and choose Hide procedure numbers within group.

Add 2nd level procedure

You can also add subprocedures to a procedure. Select the More actions () icon for the appropriate procedure and choose Add 2nd Level Procedure.

Sort procedures

You can move procedures in the checklist document to change the order of appearance.

Select the Move () icon for the procedure you want to move.

Arrows indicating the possible target locations where you can move the procedure display in the document.

Select the arrow next to the desired target location.

The procedure now displays in the new location.

Other Options

When you edit a checklist, you may want to see all items in the checklist, including hidden items. Select More Actions (), then select the Hidden items check box to display all hidden items in the checklist.

Annotation

You can add an annotation to each procedure in the checklist. Annotations are useful if you want to provide a status to a response, add commentary, or link to other relevant documents.

To add an annotation, go to the procedure or subprocedure that you want to annotate, then select the Add annotation () icon. In the annotation dialog you can select the purpose of the annotation, link to a document, and enter comments.

Aufgaben

Visibility

Guidance

If guidance information is available for a procedure, the Guidance () icon displays next to the response area. When you click the icon, the left navigation panel opens and displays the guidance information for the procedure.

Create a new optimiser checklist

Before we sign off on the Optimiser and move on to next steps, we're going to take a look at how to create our own checklist.

Select Documents in the top menu to return to the Documents page.

Select Add () next to the Start Here phase, then choose Checklist.

In the Document Properties dialog, name the checklist "My Optimiser" and select Save.

Select the checklist you have created to open it. The checklist opens in edit view.

Customize the checklist settings

The first thing we're going to do is customize the checklist settings.

Select the Settings icon () to open the Checklist Settings dialog.

In the Checklist Settings dialog, make the following modifications:

  1. Select the Sign off procedures and conclusions checkbox.

  2. Select +Response.

  3. Select the Edit() icon for the response.

  4. Select the Display inline check box.

  5. Select Save.

Add group

Before we add any procedures to our checklist, we need to add a group so we can add the procedure to it.

Select + Group to add a group. Name your group "Additional Items".

Add procedure

Next we're going to add a procedure.

To add a procedure:

  1. Next to the group, select More actions () | Add procedure.

  2. Enter "PBC Training Items" as the procedure description.

  3. Select More actions () for the procedure, then select Add 2nd level procedure.

  4. Enter "Request information about Property Plant and Equipment additions?" as the procedure description.

  5. Select More actions () for the procedure, then select Add 2nd level procedure to add another 2nd level procedure.

  6. Enter "Request information about related party transactions?" as the procedure description.

  7. Select More actions () for procedure 1a, then select Settings.

  8. Select the Override checklist settings check box.

  9. Select the Edit() icon for the response.

  10. Select Display inline.

  11. Delete the Not applicable response.

  12. Select Save.

  13. Repeat steps 7-12 to make the same changes for procedure 1b .

Complete the checklist

After we create the checklist, we can complete it.

Select the Edit () icon to toggle edit view off.

Respond Yes to both procedures 1a and 1b, then Sign off as the Preparer.

We're going to use this checklist later when we create a query.
