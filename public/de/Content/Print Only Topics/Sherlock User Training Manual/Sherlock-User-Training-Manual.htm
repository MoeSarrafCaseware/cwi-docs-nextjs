Überblick

Caseware Sherlock kann wegweisende Buchhaltungsinformationen für Unternehmen weltweit zur Verfügung stellen. Es handelt sich um eine leistungsfähige, sichere, cloud-basierte Business-Intelligence-Plattform mit einer übersichtlichen Benutzeroberfläche. Sherlock sammelt und speichert automatisch Daten von den aktuellen und historischen Caseware Engagements und bietet einen flexiblen Pfad, um diese großen Datenmengen in Echtzeit in praktisch umsetzbare Erkenntnisse umzusetzen, indem fortgeschrittene Analysen und Dashboards verwendet werden. Jetzt können Sie den Zustand Ihres Unternehmens überwachen, Risiken besser verwalten, Wachstumschancen identifizieren und werden mit diesem neuen Schlüssel grenzenloser Möglichkeiten zum vertrauenswürdigen strategischen Berater für Ihre Kunden.

Wie funktioniert das?

Sherlock führt automatisch und kontinuierlich Data-Mining in allen Engagements durch, die sich im Caseware System befinden. Engagementdaten werden extrahiert, transformiert, organisiert und im Sherlock Data Lake (Datensee) gespeichert, was eine unternehmensweite Engagementanalyse ermöglicht. Unternehmen können auf Engagementdaten zugreifen und diese abfragen, um Geschäftsentscheidungen sowohl für ihre Praxis als auch für ihre Kunden voranzutreiben.

Die Caseware Sherlock-Plattform besteht aus drei Hauptlösungen: Sherlock Reader, Sherlock Builder und Sherlock API.Sherlock Reader, Sherlock Builder und Sherlock API. Zusammen bieten diese Produkte eine unternehmensweite Lösung, die von Mitarbeitern mit unterschiedlichen Rollen verwendet werden kann.

Sherlock Reader

Sherlock Reader ermöglicht Mitarbeitern Schlüsseldaten zum Engagement aus Kundenentitäten in einfach zu verwendenden Dashboards zu präsentieren. Sie können diese Daten in Dashboards nutzen, um Einblicke in die Finanzen Ihrer Kunden zu gewinnen, Ihr eigenes Unternehmen besser zu verwalten, neue Einnahmemöglichkeiten mit Beratungsdienstleistungen zu eröffnen etc. Dashboards sind bei Ihrem Caseware Distributor erhältlich oder können von einem Ihrer Mitarbeiter (Firmenautor) mit Sherlock Builder unternehmensspezifisch entwickelt werden.

Erste Schritte in Sherlock Reader.

Sherlock Builder

Sherlock Builder ist ein Tool zur optischen Darstellung von Working Papers Engagementdaten in Dashboards und ermöglicht Firmenautoren, angepasste Vorlagen zu entwickeln.

Sherlock Builder besteht aus drei Komponenten:

  • Analysen sind anpassbare Vorlagen für Dashboards. Sie können eine Analyse für andere Sherlock Builder-Benutzer freigeben, um bei deren Entwicklung zusammenzuarbeiten.
  • Dashboards sind schreibgeschützte Versionen von Analysen, die Sherlock Reader-Benutzer für Kunden freigeben.
  • Datensätze beinhalten die gesammelten Daten, die die Analyse ausfüllen. Die Datensätze in Sherlock Builder sind vorinstalliert und können nicht geändert werden. Datensätze werden täglich aktualisiert.

Erste Schritte in Sherlock Builder. (ein MyCaseware-Konto und eine Lizenz von Sherlock Builder ist notwendig, bei Fragen wenden Sie sich an Ihren Kundenberater von CaseWare Germany GmbH)

Get started

Before staff can access Sherlock, an administrator must assign them a license for the applicable app (Sherlock Reader or Sherlock Builder).

Warnung: Do not change the email address or revoke the license of a Sherlock Builder user. Any analyses created by the user will become inaccessible to all users — including administrators.

To assign a Sherlock license to a staff member:

  1. Ensure you have the Staff Admin role or equivalent permissions.

  2. From the Cloud menu (), select Staff.

  3. Select the staff member that you want to assign the license to, then select Edit ().

  4. Select Apps, then choose the appropriate permissions (either Sherlock Reader or Sherlock Builder).

  5. Select Save.

Sherlock Builder: Eine Analyse erstellen

You can create an analysis for each dataset and customize how you want to present the engagement data in the visualization.

To create an analysis:

  1. From the Cloud menu (), select Sherlock Builder.

    An interactive frame powered by Amazon QuickSight displays.

  2. Select the Analyses tab.

    A list of existing analyses displays.

  3. Select New analysis.

  4. Select the applicable dataset for the analysis. Available datasets include:

    • accountsummary
    • cv (CaseView)
    • cvcontrol
    • cvmlp (management letter points)
    • cvrisk
    • groupings
    • hi (history)
    • mappings
    • metadata (engagement properties)
    • sh (document)
    • trialbalance
    • ts (issues)
    • user
    • usergroup

    Hinweise:

    • Dataset names are appended with the firm's short name (e.g. trialbalance-testfirm-001).
    • The entityrls dataset contains entity level security information and should not be used as the dataset for an analysis or shared with others.

  5. Select Create analysis.

  6. Customize the analysis.

    Example customizations for different scenarios can be found following this procedure.

  7. If you want to collaborate on the analysis with other Sherlock Builder users in your firm, select Share () | Share analysis.

    The Share analysis dialog displays.

  8. Enter each user's email and select Share.

Engagement progress

  • Dataset: SH.

  • Suggested visual type: Gauge chart ()

Field wells

Wert

  • Add a custom calculation field that counts the number of completed documents:

    1. On the top menu, select Add () | Add calculated field.

    2. Enter a name for the calculated field.

    3. Build a calculation that defines the logic of what your users would consider a "completed" document.

      For example, if a document is considered complete after Role 1 is signed off, the calculation would read:

      countIf(Role 1 Initials, isNotNull(nullIf(Role 1 Initials,'')))

Target value

  • Add a custom calculation field that counts the number of documents in the Document Manager:

    1. On the top menu, select Add () | Add calculated field.

    2. Enter a name for the calculated field.

    3. Build the following calculation:

      count({Description})

Additional considerations

  • Apply any entity and/or year-end filters to the widget, as necessary.

Customize an analysis for: Business advisory

Income statement - Revenue

  • Dataset: Mappings

  • Suggested visual type: Table ()

Field wells

Group by

  • Add a custom calculation field that filters the map numbers to only include revenue items:

    1. On the top menu, select Add () | Add calculated field.

    2. Enter a name for the calculated field.

    3. Build a calculation that filters out map numbers that aren't associated with revenue map numbers.

      For example, for revenue map number 2.1, the calculation would read:

      ifelse(substring(mapno,1,3)='2.1' AND strlen(mapno)<9, mapno, NULL)

  • mapdesc -Displays the mapping description used in engagements.

Wert

  • Add a custom calculation field that determines whether a row contains zero balances across all of the final, prior year 1, and prior year 2 columns:

    1. On the top menu, select Add () | Add calculated field.

    2. Enter a name for the calculated field.

    3. Build the following calculation:

      ifelse(finalbalance+py1+py2 = 0, 1, 0)

  • finalbalance - Displays the final balance value of a given map number.

  • py1 - Displays the prior year 1 balance value of a given map number.

  • py2 - Displays the prior year 2 balance value of a given map number.

Additional considerations

  • If you want to hide zero balance rows from the table:

    1. Create a filter based on the "zero balance" calculation field (created above).

    2. On the Edit filter panel, click the first drop-down and select Only this visual.

    3. Under Filter type:

      1. On the first drop-down, select Sum.

      2. On the second drop-down, select Does not equal.

      3. In the value field, enter 1.

      4. On the third drop-down, select Include nulls.

    4. Select Apply.

Sherlock Builder: Publish an analysis to a visualization

Once you finalize an analysis, publish it to a visualization to share with other staff members and clients. Note that they can only view the visualization — they cannot modify it.

Note: Purchased visualizations will display in the Dashboards tab if a license has been assigned to a Sherlock Builder user, but they cannot be modified or shared.

To publish a visualization:

  1. Select the Analyses tab.

    A list of existing analyses displays.

  2. Select the analysis that you want to publish to a visualization.

  3. Select Share () | Publish dashboard.

  4. In the Publish a dashboard dialog, select:

    • Publish new dashboard as to create a new visualization. Enter a title for the new visualization in the field.

    • Replace an existing dashboard to overwrite a visualization while maintaining existing share settings. Select the visualization to replace from the drop-down menu.

  5. If required, apply any additional options from the Advanced publish options expandable menu.

  6. Select Publish dashboard.

    The Share dashboard dialog displays.

  7. If you want to collaborate on the visualization with other Sherlock Builder users in your firm, enter each user's email and select Share.

    Note: Do not use the Share with everyone in this account option. It does not function correctly with Sherlock Builder.
