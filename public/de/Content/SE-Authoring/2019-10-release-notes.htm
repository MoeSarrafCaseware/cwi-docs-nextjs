2019-10 release notes

SE 2019-10 features a redesign of the risk module, support of Inline eXtensible Business Reporting Language (iXBRL) and more.

New user interface for the risk module

The Cloud Engagements simple risk module is switching to a new user interface to support more complex engagements, where the need for additional documentation and enhanced workflows related to risk identification, assessment and response is required.

Key features at a glance:

  • A centralized approach for risk identification and assessment.

  • A workflow to guide users through risk assessment, featuring a high level overview and drill-down options - users can view details as they work through each financial statement area.

  • Extending the risk documentation to scale the platform for different products or engagement types and jurisdictions.

  • Introducing product-level customization capabilities, where you can customize the terminology and level of complexity required for their market, such as field names and labels, assessment methods, Risk of Material Misstatement (RMM) mapping.

The new interface utilizes modern UI best practices, featuring:

  • risk cards and assessment cards with collapsed and expanded views.

  • data filtering and sorting.

  • an improved method for linking to checklist procedures for risk identification and response.

  • linkage to analysis tabs in checklists.

  • improved assertions tagging.

Scaling up to larger and more complex engagements called for numerous wording changes, for instance, the term Reference in the old interface has been reworded to Audit Response in the new interface.

To switch to the new risk module layout, go to the Product Settings and select Risks | General, then select the Enable new risk functionality check box.

Once you enable the new layout, the new Risks page becomes available in the template, where you can easily access the new risk module.

To complement the new risk module, we've enhanced the Risk of Material Misstatement (RMM) visibility condition to support more logic. In addition to the assertion type, the following parameters are now available:

  • Business cycle or area.

  • Logical operators.

  • RMM value.

For example, an RMM condition logic can now be: If 'Accuracy' for 'Inventory' has RMM 'less than' 'Medium', then hide this item.

Please note that:
- the additional RMM condition parameters are only available if the new risk module is enabled in the product.
- if you enable the new risk module in your product, any existing RMM visibility conditions in the old risk module do not migrate to the new layout. You must set your RMM conditions again using the enhanced parameter logic.
- we strongly advise that you thoroughly test the new risk module and verify everything works as expected, before you enable it in the production environment.

iXBRL support

We now support Inline eXtensible Business Reporting Language (iXBRL). iXBRL is an open standard that allows firms to prepare financial statements in a format that provides the structured data that regulators and analysts require, while allowing preparers to retain full control over the layout and presentation of their report. To learn more about iXBRL, see iXBRL.org.

For information on how to enable iXBRL, see the XBRL SDK tutorial.

Form values in visibility settings and data links

Values from SE Builder forms can now be used to drive visibility logic in the product. Form values can also be retrieved and inserted in various text fields within documents.

Hinweis: This feature is only available in the product if you have developed custom forms using SE Builder. To learn more, see Get started with SE Builder.

  • A new visibility condition Form Value is now available with the following parameters:

    • A list of the available forms in the product.

    • A list of the available fields for a selected form.

    • The applicable values for a selected form field. Note that the displayed values depend on the field types. For example, numeric fields would have different value options and than date fields.

    • Note: More then one form can be selected when setting up this condition.

  • A new data link () option Forms Value is now available. This option retrieves a specific value in a form and inserts it in text fields. For example, insert the value for Provincial Jurisdiction from a Tax form in a text section in the financial statements.

Queries integration with Working Papers

#primaryauthor#firmauthor

A new field Author ID is now available in query documents to allow Working Papers (WP) users to pull the uploaded files within a specific query document to designated locations in a WP file.

Note that Author ID is only available in query documents, not in query templates. It is an optional field that you only need to specify if you intend to integrate query documents in your product with Working Papers.

Here is a brief description of the workflow for Queries-WP integration:

  1. Set an Author ID value. You can set it for one or more response.

    2. Note: You must collaborate with WP authors to decide where you should add Author IDs in your query document.

  2. Working Papers authors must use the same ID when setting up a location in WP for the files received from an SE query document.

  3. Once a file is uploaded in the SE query document, WP users can click the Receive button to automatically pull the file from SE to its location in the WP file.

To learn more, see Add Author IDs.

New for end users

Here is what's new for end users in SE 2019-10.

Queries (new design)

We've introduced a few feature enhancements for query documents and the Query Dashboard.

  • The Query Dashboard now has an additional column Contacts that displays the number of assigned contacts to each query document in the dashboard.

    • Users can select the number to display a popup showing contact details. The popup also includes the Add contacts button so you users can create a contact, if needed.

  • In query documents,

    • users can now collapse or expand question sets in the left navigation pane.

    • query questions are no longer numbered.

  • A notification displays for contacts once they answer all questions in the document. The notification message features the Mark Responded button so contacts can easily proceed to the next step in the workflow. Similarly, a notification also displays for staff members once they accept all responses in the document. The notification message in this case features the Mark Complete button.

Data page

We've introduced a few UI improvements for end users, as well as functionality changes in the Accounts view page to support new features.

  • Users can now view prior year transactions, if they've imported the respective data. This feature introduces two changes in the current functionality in the Accounts view page.

    • To view transactions for a specific account in the current year, users now select accounts from the Preliminary column instead of the Final column to view account transactions. For a prior year account transactions, users select accounts from the corresponding prior year column.

    • Users can no longer edit the account name or number directly on the page. In this release, users select the Edit () icon to modify account information. Two additional fields are also available in the Account dialog to allow users to edit the current and prior balances, if needed.

    Weitere Informationen finden Sie in Kontotransaktionen anzeigen.

  • Opening and closing balances are now available in the Transactions page footer for the selected account. This helps users compare account transactions net total in a specific period.

  • Tax Groups and Prescribed Groups views no longer display for users in the left navigation pane in group views if you haven't set up Tax Code or Chart of Accounts groupings in the product.

    • Similarly, the Tax Code and Tax Adjustments options no longer display for users in the Columns drop-down in group views if you haven't set up Tax Code groupings and tax adjustments in the product.

Review Tools in financial statements

Two additional review tools are now available in financial statements to easily locate disagreeing balances and overridden balances in the document.

  • Balance check - marks all rows with identified balance discrepancies and all rows where balances agree (that is, all rows with a Diagnostics label).

  • Overwritten balances - marks all rows with overridden balances.

To learn more, see Review prepared documents (Review-Tools).

Fixes

The following listing features a brief description of the bug fixes in SE 2019-10.

Dynamic tables

Fixed issues in dynamic tables where:

  • custom calculations in dynamic table headings display incorrect values or NaN.

  • column descriptions with data links () do not display correctly.

  • users experience a delay in data display when they set up input rows.

  • the Account Details popup does not show adjustments for a prior year group.

  • column headers with long descriptions do not display properly on the page.

  • users are unable to reopen the Data Link dialog in table headers.

  • some trial balance line items are missing.

Checklist documents

Fixed issues in checklist documents where:

  • users get an error when they attempt to add a data link () in the Conclusions section in checklist documents.

  • content in multiple-row checklist procedures are misaligned.

Product authoring

Fixed product authoring issues where:

  • queries open in the old layout although the new layout has been enabled in the Product Settings.

  • the custom terms entered for Preliminary and Final column labels are not applied.

  • the Data page is enabled in the template although it is disabled in the Product Settings.

  • the order of data in analysis graphs does not follow the trial balance order.

Abfragen

Fixed issues in query documents where:

  • adding a comment in a question that has been responded to, produces multiple notification messages on the page.

  • recently accessed query documents do not show on the left navigation pane.

Data page

Fixed issues in the Data page where:

  • the Out of balance () indicator displays although all accounts are correctly balanced.

  • accounts are not sorted properly.

  • users have to refresh the page so they can search accounts properly in the Assign Accounts page.

Jahresabschluss

Fixed issues in financial statements related to:

  • the reconciliation feature, where rows with agreeing balances are identified and flagged as disagreeing in the document.

  • the financial statements notes, where adding page breaks in the Notes to the Financial Statements area generates duplicate entries in the Table of Contents.

Other fixes

The following general issues are fixed in SE 2019-10.

  • The word Trial Balance is not highlighted while users are actively working on the Trial Balance page.

  • The filenames for some files copied from Caseware Cloud to an engagement file do not display correctly.

  • GIFI codes do not change when reassigning financial grouping accounts.

  • Users experience delays while using the rounding feature in financial statements with large trial balance data set.

  • Internal links do not work when users attempt to access them in the left navigation pane of a document.

  • Carry Forward issue, where user-added sections in financial statements are not visible or available as a carry forward selection item after the file has been rolled forward.

  • The sign of an account balance changes when users override the balance in the Statement of Cash Flows.

  • The Suppress icon () displays as disabled in firm templates when it is actually enabled.

  • A few translation issues have been fixed.
