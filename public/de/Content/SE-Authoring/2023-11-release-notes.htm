SE 2023-11 release notes

The SE 2023-11 release introduces extensive enhancements to financial statements formatting, a new form of consolidation to provide flexibility for users, the ability to assign multiple dimension categories to financial groups and more.

Multiple dimension categories for financial groups

#enduser#primaryauthor#firmauthor #enduser #primaryauthor #firmauthor

You can now assign multiple dimension categories to a single financial group. This allows you to organize and present data in multiple ways in the financial statements. For example, you can have a dynamic table that breaks down your data by location, as well as a table that’s organized for fund reporting purposes.

Firm authors and end users also now have the ability to assign multiple dimension categories to a single financial group, as well as the ability to assign multiple dimensions to an account from the Account properties dialog. Note that only one dimension per dimension category can be assigned to an account at a time.

To learn more, see Add dimensions and Group properties.

Enhancements to financials statements presentation

#enduser#primaryauthor#firmauthor #enduser #primaryauthor #firmauthor

To further enhance the presentation of the financial statements, a number of improvements have been made to allow you greater customization capabilities.

New Formatting options dialog

The Formatting options dialog is now available in the financial statements to allow you to customize font sizing, margins, text padding, and indentations.

To learn more, see Customize font sizes and spacing in the financial statements.

To learn more, see Customize font sizes and spacing in the financial statements.

Table header print settings

The Display column headings on subsequent pages option has been added to the financial statements. When you select this option, dynamic tables that are larger than a page will display table column headings on each page the table is on.

To learn more, see Set up the print options in the financial statements.

To learn more, see Set up the print options in the financial statements.

Keep together paragraphs

The Keep notes headings with paragraphs and keep paragraphs together option is now available in the document print settings. Selecting this option ensures that note headings are not printed on separate pages from the note content and that paragraphs are not split across multiple pages unless the paragraph is longer than a page.

To learn more, see Set up the print options in the financial statements.

To learn more, see Set up the print options in the financial statements.

Headers for multiple page notes

In the Notes print settings, you can now select the Enable continued header text option. When this option is selected, you can enter a custom suffix for headers for notes that span multiple pages.

Wenn Sie beispielsweise das Suffix „Fortsetzung“ eingeben, wird eine Notiz mit der Überschrift „Wesentliche Bilanzierungs- und Bewertungsmethoden“ auf den Folgeseiten als „Wesentliche Bilanzierungs- und Bewertungsmethoden Fortsetzung“ angezeigt.

Note that the suffix you enter is customizable for each language in multi-language templates.

To learn more, see Set up the print options in the financial statements.

To learn more, see Set up the print options in the financial statements.

Cloud Connector materiality formulas

Two new formulas have been added to Cloud Connector to allow materiality information to be used in Cloud Connector formulas. The formulas are:

  • CWI.MATERIALITY - Returns the materiality information.

  • CWI.TRIVIALMATERIALITY - Returns the Clearly Trivial materiality.

New for end users

Here is what's new for end users in SE 2023-11.

External consolidation

A new form of consolidated engagements is now supported in SE to take into account consolidation engagements where users either do not receive individual balances for subsidiary entities from clients or are performing consolidation outside of SE.

Note that to make the external consolidation option available in your product, you must enable the Enable new workflow for CSV/Excel trial balance imports option in the product settings.

To mark an engagement as being externally consolidated, users can select the Externally consolidate option in the Engagement properties popover.

They can then import their data, ensuring that it includes both the consolidated and parent-only balances. Similarly to engagements that are consolidated using SE, users can filter their financial data by consolidated and parent-only values on the Data page.

To learn more, see Create an externally consolidated engagement.

Create new engagement based on existing file

Users can now create new engagement files based on existing files from either the More actions () menu for the existing engagement in Cloud or from the New engagement dialog.

To enable this feature, you must set the property this.canCreateEngagementFromSource = true in the collaborate.js file for your product.

Die folgenden Informationen werden aus dem Engagement-Quelle kopiert:

  • Anpassungen von Gruppierungsstrukturen

  • Anpassungen von Abfragen

  • Externe Dokumente, die dem Auftrag hinzugefügt werden, z. B. PDFs und Microsoft Office-Dateien

  • Anpassungen und Antworten zu Checklisten

  • Anpassungen des Jahresabschlusses

  • Sichtbarkeitseinstellungen

  • Risiken, Kontrollen und Risikobewertungen

  • Zu den Engagement-Einstellungen hinzugefügte Bereiche

  • Abzeichnungsverfahren und Rollen

Bitte beachten Sie:

  • Trial balance accounts and balances are not included

  • Links to internal and external documents in queries are not included

  • Gesperrte Engagements können nicht als Quelle für ein Engagement verwendet werden

A link to the source engagement is available in the About dialog of the new engagement.

To learn more, see Create a new engagement based on an existing file.

Mention contacts in issues

You can now mention contacts in client comment issues.

When you enter the ‘@’ symbol in an issue, a full list of both contacts and staff that have access to the engagement will display. You can then select the appropriate contact from the list.

To learn more, see Document issues in the engagement file.

Updates to PDF export of trial balance data

When end users export their data to PDF from either the Accounts or group tabs of the Data page, the exported PDF now accurately reflects the column settings the user has selected when choosing how to display their data in the Data page.

In consolidation engagements, the PDF will also reflect whether the user has chosen to view consolidated data, or data for a selected entity in the consolidation hierarchy.

To learn more, see Export trial balance data.
