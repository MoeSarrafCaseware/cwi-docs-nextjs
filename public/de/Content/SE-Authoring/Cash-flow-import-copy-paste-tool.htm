Cash flow import / copy-paste tool

Cash flow import / copy-paste tool has been developed to assist our users to efficiently transfer the cash flow statement balances from external sources (Excel or CSV worksheets) into dynamic tables within the financial statements.

Once the Cash flow import / copy-paste tool has been integrated with your solution, select Add () on your Documents page and add the following documents:

  • Cash flow import - The main form which includes all the functionality, for both the Indirect and the Direct Cash flow methods.

  • PA - Cash flow statement structures table - An SE Builder form that is required to support the functionality of the main form. You must add this worksheet to your Documents page but you can always hide it by using the visibility settings.

  • GC - Author mode visibility form - An SE Builder form that drives the visibility settings on the content of the form based on the type of author that is accessing it. You must add this worksheet to your Documents page but you can always hide it by using the visibility settings.

    Note: GC - Author mode visibility form is a form that is used by other Global Components forms. If it was previously added to your solution, there are no other changes you have to perform.

Required Primary Author changes

Financial reporting framework & Industry

If your app supports multiple Frameworks or Industries, insert the procedures IDs where this information is documented.

Note: If your app supports only one financial reporting framework or industry, leave these fields blank.

Cash flow statement reporting method

Perform the same steps to insert the procedure ID where the cash flow method type is documented in your solution (direct or indirect method).

Note: If your app supports only one cash flow method, leave this field blank. By default, the form will apply the indirect method.

Disable editing line’s description

Select Yes to disable the ability for end-users to edit the lines' descriptions.

Cash & Cash Equivalents financial groupings

Select the financial groupings that total the cash and cash equivalents balance, which is used for the cash flow statement. This balance will be used to cross check amounts in the summary section.

Supplemental cash flow information

Select Yes if a Supplemental cash flow information section applies for your Solution.

Cash flow structure table

Use the table to set up the Cash Flow structures for each financial reporting framework and industry supported in your app based on the information provided in the above Settings section:

  • Select each configuration applicable based on the financial reporting framework and industry supported by your app.

  • Specify the number of rows that should be included in each cash flow section (Operating / Investing or Financing).

    Note: The maximum number of rows for each section is defined by the SE Builder author through the values.json file. A maximum of 40 rows for each section have been defined in the master version of this form. Additional rows can be added by the SE Builder author by adding additional keys in the values.json.

  • Click Edit to define the line item descriptions for each of the cash flow structures required.

The Edit dialog provides the following options:

  • Clear all - Removes all line descriptions from all sections.

  • Restore original - Restores the line descriptions back to the default ones as defined in the values.json file.

  • Set new default - Saves the current values entered as the new default to be used for future structures.

  • Save - Saves the values that have been entered.

    Note:

    • If your app supports only one financial reporting framework or one industry only, an “Engagement default” row will be shown to set-up the structure for the cash flow.

    • The default cash flow statement structure included in the tool is based on the International Full IFRS XBRL taxonomy. Modify the structure based on your market specifics and ensure to save the changes.

    • To modify or translate the default values, refer to Add translation or wording updates

Authoring Financial Statements Dynamic Tables

The Cash flow import / copy paste tool saves the information using Form values IDs. To link the values between the SE Builder form and the Financial Statements, the primary author is required to author these Form values IDs into the dynamic tables.

  • To access a list of Form values IDs which are linked to the financials dynamic tables click here. Note that these are the default values shared and they could have already been modified by your solution SE developer.

  • Link each form value ID into its corresponding cell value in the dynamic table in financial statements.

End user workflow

As an end user, you are required to document the following information:

  1. Specify whether the amounts you want to import or copy are already rounded or not.

    Note:

    • Financial statements can be rounded to thousands, millions and billions.

    • Specify whether the amounts imported/copied are already rounded as per the financial statements rounding settings.

    • If the amounts are not rounded, the form will automatically apply the same rounding settings as specified in the financial statements rounding settings.

    • To enable rounding in the financial statements refer to

      Customize rounding in the financial statements.

  2. Select the method to insert the cash flow amounts into this form: CSV Import | Copy-Paste.

    Note: The cash flow import tool supports two methods to insert the amounts into this form:

    1. CVS Import: CSV Import: Import the amounts using the predefined CSV template file provided.

      Note: CSV Import supports loading the information from a CSV file with a predefined format/structure. An example file can be found here: CSV Import Template demo file.

    2. Copy-Paste: Allows you to directly copy and paste the amounts for each of the applicable cash flow sections.

  3. Select if you want to import Prior Year amounts or not.

  4. Review the Summary section to ensure the cash flow statement is in balance.
