List of product features (API)

The following list describes the product features that you can enable/disable in your SE product.

See How to author your product template to learn more about how to author your SE product template.

Additional Entity Fields

Define the customFields field to customize the fields in the Create Entity dialog.

Additional annotations

You can include the com.caseware.reference module in the product.json file to enable additional annotation types.

Areas

Allows you to summarize financial statement groups at a higher level for risk identification and assessment. For more information, see Add areas and cycles.

Assertions

Include the com.caseware.assertion module in the product.json to enable annotations in your Cloud app.

Automapping

Automatically maps sections of data files on import. To learn more, see Set up automapping.

See the importing and automapping SDK tutorial for more information on the automap property.

Automatic rounding

To enable automatic rounding in the financial statements, create a script called financial-rounding-setup.js and include it in the “js” array of your product.json file. In this script, include the following:

(function() {
     ‘use strict';
     wpw.financials.financialsSetup.roundingErrorGroupNumbers = ['A', 'B'];
})();

‘A’ and ‘B’ are strings representing the group numbers of the groups in the structure that should sum to zero. If required, you can include more than two groups.

For example, if the grouping structure contains two groups Assets and Liabilities that should sum to zero, then replace the ‘A’ string in the above function with the group number for Assets and replace the ‘B’ with the group number for Liabilities. Liabilities and Assets are then summed and checked for rounding errors.

Note that rounding differences are applied by default to the first non-zero account in the lowest child level that appears in the first group in the trial balance. For more information, see Change the default group for rounding differences.

Checkliste

Enables the checklist functionality.

For more information, see Add a checklist.

Contacts control

This property controls contacts' access to engagement. The default setting is that contacts cannot edit or view any engagement. To grant view access to contacts assigned to the engagement, set contactAccess to READ.

Note that once you grant contacts access to engagements, you cannot limit what areas of the engagement file they can access. We recommend using this feature for reporting products only and not audit products. We also recommend thoroughly reviewing the feature before releasing your product to ensure it works as you expect.

Cycles (business cycles)

Used to further organize the groups in the Trial Balance.

For more information, see Add areas and cycles.

Detailed Documents View

Makes the Detailed View in the Documents page the default view.

Dimensions

Dimensions are properties that can be assigned to accounts in an engagement. Each account can be assigned one or more dimensions.

For more information, see Add dimensions.

Engagement Properties Prompt

Display the Engagement Properties prompt each time users open a new engagement. For more information, see Set the engagement properties.

Engagement Search

Disables the search field at the Document Map in all engagement documents.

For more information, see Locate and access documents in the engagement files.

Envision

Enables the ability to add the out-of-the-box Envision data profile to your product. Envision data profiles are dashboards that display information about imported transactions in engagements. For more information, see Envision.

To enable Envision, add "com.caseware.an.e.analytics.ai" to the set array and set the property "envisionProfile": true.

You can then use the Add Document icon () in the Documents page to add the Envision Profile custom form to your product.

External life cycle

Set the externalLifecycle property to true if you want the engagements in your SE product to be created or deleted outside of Caseware Cloud. --NEED MORE INFOR OR AN EXAMPLE

Firm Customizations

Allows firm authors to customize the template used to create new engagements. This property is enabled by default.

Help

Allows you to provide your users with a link to a help page, such as your product information site.

Import Documents

Set the property "importDocuments": true to copy content from a different Caseware Cloud app template. To learn more, see Copy content from another template.

Imports

The imports array in the product .js file allows you to enable import options in the product, so that users can import client data.

Add the modules for the desired import options to the imports array. To learn more, see Enable Provider Bindings imports.

For end users, the enabled import options display in the Data page (or the Trial Balance page if you've not yet enabled the Data page in your product).

You can also build a custom integration for your product to import data from a 3rd party online accounting package. To learn more, see Provider Bindings SDK.

Enable the import of dataset types

Enables dataset type options for general ledger data, subledger transaction data and trial balance imports from CSV files. This allows users to select which dataset type applies to their data import.

To enable a dataset type, include its corresponding schema ID in the product.json file as shown in the code sample below. Note that you should adjust the code to remove schema IDs for any dataset types you do not want to enable for users. See the descriptions linked below for more information.

"importTaxonomy": {
					"id": "cwi.taxonomy.transactions",
					"version": "0",
					"schemaIds": [
					"cwi.schema.general_ledger",
					"cwi.schema.ap_invoices",
					"cwi.schema.ap_payments_made",
					"cwi.schema.ap_suppliers",
					"cwi.schema.ap_open_balances",
					"cwi.schema.ar_invoices",
					"cwi.schema.ar_cash_received",
					"cwi.schema.ar_customers",
					"cwi.schema.ar_open_balances",
					"cwi.schema.inventory_transactions",
					"cwi.schema.inventory_on_hand"
					]
			}

To enable users to import data that does not conform to any dataset type, you can add "schemaless": true to the "feature" section in the product.json file. As a result, users will have the option of selecting the Other dataset type.

 

Interim Reporting Period

Enables interim reporting.

For more information, see Create an engagement for an interim reporting period.

Aufgaben

Allows users to add issues.

For more information, see Add issues to an engagement file.

Job Number

You can hide the Job Number field in the Engagement Properties dialog.

Set the property "hiddenJobNumber": true to hide Job Number.

Wesentlichkeit

Enable materiality in your product. An accepted misstatement (due to omission or error) is considered material. Materiality is the accepted threshold (error margin) for misstatements.

For more information, see Set up Materiality and Summary of Misstatements.

Prior Years

By default, one prior period balance is supported and displays in the engagement - once users import the client's data.

To enable up to four prior period balances, set the showAllPriorPeriods property to true.

Product Class

This property is utilized by Working Papers users. It returns one of the following values:

  • PBC - To indicate that Working Papers integrated with a PBC app. Note that this integration is only available for Working Papers 2019 base release and higher.

  • ANALYTICS - To indicate that Working Papers is integrated with an AnalyticsAI app. Note that this integration is only available for Working Papers 2020 base release and higher.

Note: If queries are enabled in the AnalyticsAI app, this property returns "ANALYTICS".

Query

Allow users to create queries.

For more information, see Staff-Contact collaboration.

Erneut importieren

Allows update of account name and number during trial balance reimport.

Review Tools

Disables the Review Tools throughout the product for users.

For more information, see Review prepared documents with review tools.

Risikobeurteilung

To enable the Risk Assessment report in your product.

For more information, see how to set up Risk, Control and RMM.

Forms and Form sets

To include the risk assessments, risks, controls, materiality and misstatements reports in your Cloud app, include the general form set com.caseware.ca.general in the set property. You must also enable the following:

"feature": [ "cycles", "riskAssessment" ]

"module": [ "com.caseware.visualization.charts", "com.caseware.assertion" ]

If you don't want to include all the reports, you can include specific reports. Note that if you choose to include specific forms, you must enable the required property or module. For example, to enable the Controls report:

  • Include the com.caseware.ca.general.control-report in the form property

  • Enable the Cycles property

Roll Forward

Enables carry forward. The product.json file in the API needs to be modified to include the previous and next years when they exist. See the roll forward SDK tutorial for the detailed instructions.

For information on the end user experience for carry forward, see Carry forward an engagement.

Second Reviewer

Enables a second reviewer in your product so that the Final Review option is included in all signoffs and the Second Reviewer field is available in the Create Engagement dialog. To learn more, see Set up the engagement properties.

Hauptabschlussübersicht

Enable the Trial Balance tab in the engagement.

For more information, see Define groupings.

User Defined Financials Groups

Allows users to add their own groups.

For more information, see Add a group.

Workflow

To enable read-only workflow feature or to hide the workflow status at the engagement properties dialog.

For more information, see Workflows and Triggers in SE.

iXBRL

Enables Inline eXtensible Business Reporting Language (iXBRL) functionality throughout the product. iXBRL is an open standard that allows firms to prepare financial statements in a format that provides the structured data that regulators and analysts require, while allowing preparers to retain full control over the layout and presentation of their report.

A conversion support file and a pointer to it must be added in the API. See the XBRL SDK tutorial for the detailed instructions.

If you have at least one XBRL taxonomy set up, XBRL review mode is available to end users and enabled by default. If you want to hide an XBRL taxonomy from end users, you can set the "hideFromReviewMode" property to true in the XBRLTaxonomyDefinition.

Note that if all taxonomies are hidden, XBRL review mode is not available to end users.

To learn more about how to review and export XBRL documents, see XBRL review and export.
