Prepare a CSV file for import (Data page)

The CSV file must include the following columns They can have empty values but each column must be available in the file:

  • Map number
  • Kontoname
  • Postable
  • Typ
  • Sign

You can optionally include the following columns:

  • Weight
  • Allow
  • Contrary Name
  • Inconsistent Name
  • Dimension Category
  • Excluded Dimensions
  • Reassign Group for contrary accounts
  • Group Classification
  • Link to a Tax Group
  • Link to a Chart of Accounts Group
  • Prozess
  • Prüffeld
  • Components
  • Default group

You can use these columns to help set up automapping (automatic assignment of accounts to groupings) when users import a new trial balance.

The following listing describes the columns and the expected value for each column.

Map Number (mandatory)

Enter a map number for the grouping. Also known as group number in SE.

Expected value:

Number - can contain alphanumeric characters, decimal points, hyphens and underscores.

Kontoname

Enter a name for the grouping. Also known as group name in SE.

Expected value:

Text - can contain alphanumeric characters, decimal points, hyphens and underscores.

If this field is left blank, "[Untitled]" displays as the group name.

Postable

Indicate whether accounts can be assigned to the group.

Expected value:

  • Y or 1 or TRUE- accounts cannot be assigned to the group
  • N or 0 or FALSE - accounts can be assigned to the group

If this field is left blank, the system assumes that the accounts can be assigned to a group and one of the these values is added (N or 0 or FALSE).

Type (mandatory)

Indicates whether the grouping belongs to the balance sheet, income statement, comprehensive income, or net income.

Expected value:

  • B - Balance Sheet
  • I - for Income Statement
  • C - for Comprehensive Income
  • N - for Net Income

Sign

Indicate whether the grouping should be for debit accounts or credit accounts.

Expected value:

  • D- debit
  • C- credit

If this field is left blank, the system assumes debit is the grouping sign and the value D is applied.

Group Classification

Enter automapping classification for the grouping.

Expected value:

Text

Can contain alphanumeric characters, decimal points, hyphens and underscores.

Weight

Enter a weight for the grouping.

In cases where multiple groupings have the same classification, then any account with that classification will be assigned to the grouping with the highest weighting number. For example, groups with a positive weight are preferred over groups without a weight assigned.

Expected value:

Numerical value between -1 and +1.

If this field is left blank, the weight is assigned a value 0.

Allow

Indicate whether the grouping has any account balance restrictions.

Expected value:

  • * (Accounts with any balance to be assigned)
  • + (Only accounts with debit (positive) balance to be assigned)
  • - (Only accounts with credit (negative) balance to be assigned)

If this field is left blank, the system assumes that the accounts with any balance can be assigned to the group and the value * is applied.

Link to a Tax Group

Enter a group number from the Link to a group in the Tax Code grouping structure drop-down in the Group Properties dialog.

By adding this value, you automatically assign accounts to groups in the Tax grouping structure based on their initial group assignment.

Expected value:

Text

Link to a Chart of Accounts Group

Enter a group number from the Link to a group in the Prescribed grouping structure drop-down in the Group Properties dialog.

By adding this value, you automatically assign accounts to groups in the Prescribed grouping structure based on their initial group assignment.

Expected value:

Text

Prozess

Enter a cycle for the group (as defined in the Product Settings).

When you assign a group to a business cycle, items in that group will automatically be shown under this cycle in the Risk Assessment document, making it easier to audit by business cycle.

Expected value:

Text

Prüffeld

Enter an area for the group (as defined in the Product Settings).

When you assign a group to an area, items in that group will automatically be shown under this area in the Risk Assessment document, making it easier to audit by area.

Expected value:

Text

Components

Components provide a more market-driven, region-driven or product-driven visibility logic for trial balance groups. For example, if the entity operates within the Agriculture industry, then only display trial balance groups that are relevant to agriculture. To learn more, see Components.

Enter one or more component to assign to the group (as defined in the Product Settings).

Expected value:

Text

Use a colon to separate multiple components associated with a group (example: component1;component2;component3).

Contrary Name

If applicable, enter a description for group entries that are opposite to the group's normal sign.

For example, "Bank Overdraft" will appear instead of "Bank" when the account balance is a credit.

Expected value:

Text

Inconsistent Name

If applicable, enter an alternative description for the group when the current and prior year have opposite signs.

Expected value:

Text

Dimension Category

Enter a dimension category for the group (select from the list defined in Product Settings).

Expected value:

Text

Excluded Dimensions

Enter the dimensions that are excluded from the selected category.

Note: This property is only be available if a value for Dimension Category exists.

Expected value:

Text

Use a colon to separate multiple excluded dimensions (example: dimension1;dimension2)

Reassign Group for contrary accounts

If applicable, enter a group from the Financials grouping structure for accounts with a contrary balance.

If an account should automatically be assigned to a specific group but it does not match the selected sign (debit or credit) for this group, it is automatically assigned to the reassign target. For example, negative Asset -> positive Liability

Expected value:

Text

Default Group

Specify whether or not the group is a default group. To learn more, see Default Group.

Expected value:

  • Y or 1 or TRUE
  • N or 0 or FALSE
