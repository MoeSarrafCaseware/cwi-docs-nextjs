Recommendations for multi-language implementation

Here are the implementation recommendations for multi-language support:

  • To set up translated versions for note disclosures: Create multiple versions of the notes in different languages with visibility settings.

  • To set up translated versions for letters: Create multiple versions of the letter sections in different languages with visibility settings.

  • To set up translated query questions and checklist procedures: Create multiple versions of the procedures and query questions in different languages using dynamic text.

For detailed instructions, see Use- case: A template with French and English language support.

Bitte beachten Sie:

  • Multi-language support feature does not automatically translate content. You must provide the translations for the languages that you want your SE product to support.

  • If you delete a language, the translated text that you have added in the template is not deleted.

  • On carry forward, the selected content language is carried over to the next period's file.

  • Firm authors cannot add or delete languages but they can make all other changes.

  • Firm author changes for content language are not overridden when they update the firm template with the latest change from the product template.

    • However, there is one exception:

    If firm authors make a language active in the firm template, then you change it to inactive in the product template. In this case, your settings override what's in the firm template once firm authors update it - and the language becomes inactive.
