AnalyticsAI 2020.1

This highlights the major changes since the last release of this product.

AnalyticsAI changes

A number of improvements have been made in AnalyticsAI for this release.

Translations are now supported and implemented

You can now select AnalyticsAI’s language from the Language drop-down in the top-right of the Caseware Cloud login screen. You must be logged out to do this.

When you log back in, you will see the following AnalyticsAI elements in the selected language:

  • The login screen
  • All elements on the Results page
  • All elements on the Configuration page
  • The Export Data Locally dialog
  • The contents of the Analysis Summary PDF when exported

Improved export file name defaults

The default file name formats for exported data files have been improved.

Exported file type

File name format

Analysis Summary PDF

<year>_<month>_<day>_<entity>_AAi Summary.pdf

Results (CSV)

<year>_<month>_<day>_<entity>_AAi Audit Evidence.csv

All Data (CSV)

<year>_<month>_<day>_<entity>_AAi All Data.csv

For more information on exporting results, see Export AnalyticsAI results.

CSV export headers consistent

Header information for exported Results and All Data CSV files is now consistent.

Added approved_by property to Results CSV export

The CSV file that is created by a Results export from the Export Data Locally dialog now includes an approved_by column.

Data import changes

Improvements have been made to the process for importing trial balance CSV files.

Data layouts

This new functionality lets you save data layouts for transactions properties. This saves time by avoiding the need to map similar transactions files for multiple imports.

Now, you can map an imported transactions file in the usual manner, and then save the layout for future use.

See Import the client's transactions for more information.
