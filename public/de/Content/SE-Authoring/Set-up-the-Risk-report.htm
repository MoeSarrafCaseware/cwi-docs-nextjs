Set up the Risk report

Note: This topic is not applicable for SE apps that have the new risk module enabled.

The Risk report includes the identified risks that users (for example, auditors) have added throughout the engagement workflow.

To enable the Risk report in your product, specify the com.caseware.ca.general.risk-report in the form property in the product.json file. To learn more, see Enable or disable features in your product.

Once you enable the report in your product, you can add it in your draft and set it up.

To add the risk report, select Add () at the documents page, then choose Risk Report.

To set up the Risk report:

  1. Select which Risk attributes display on the page.

    2. Use the Risk attributes drop-down list on the top left.

  2. Add guidance information.

Select the text field to start adding content. Die Formatierungssymbolleiste wird angezeigt. Sie können die Symbolleiste verwenden, um die folgenden Änderungen vorzunehmen:

  1. Select Edit () to modify the settings for existing risks.

    1. Modify the Name and Description.

    2. Select the Type Settings () to mark the risk types that you want to be available.

  2. Once you're done, select Save or Save Risk Settings.
