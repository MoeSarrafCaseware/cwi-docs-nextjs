Optionale Ansprüche konfigurieren (Azure AD)

Bevor Sie den Integrationsprozess der Funktion Einmaliges Anmelden (Single Sign On/SSO) abschließen, müssen Sie das Manifest bearbeiten, um die Ansprüche (Claims) given_name und family_name einzuschließen. Wenn ein Mitarbeiter Einmaliges Anmelden verwendet, liefern diese Ansprüche den Vor- und Nachnamen an Cloud.

Optionale Ansprüche konfigurieren:

  1. Melden Sie sich an Ihrem Azure Portal an.

  2. Wählen Sie Azure Active Directory.

  3. Im linken Navigationsbereich wählen Sie App registrations.

  4. Wählen Sie die richtige App aus der Liste.

  5. Im linken Navigationsbereich wählen Sie Manifest.

  6. Bearbeiten Sie die Eigenschaft optionalClaims wie folgt:

    "optionalClaims": {
    
	"idToken": [
    
		{
    
			"name": "given_name",
    
			"essential": true
    
		},
    
		{
    
			"name": "family_name",
    
			"essential": true
    
		}
    
	]
    
},

    Hinweis: Sie müssen die obigen Informationen akkurat kopieren, um die optionalen Ansprüche richtig zu konfigurieren.

  7. Klicken Sie auf Speichern.
