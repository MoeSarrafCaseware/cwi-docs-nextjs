Geheimen Clientschlüssel generieren (Azure AD)

Um eine sichere Kommunikation mit Ihrer Active Directory-Domäne zu ermöglichen, müssen Sie einen geheimen Clientschlüssel generieren. Der geheime Clientschlüssel wird von Azure verwendet, um Caseware Cloud zu authentifizieren.

Einen geheimen Clientschlüssel generieren:

  1. Melden Sie sich an Ihrem Azure Portal an.

  2. Wählen Sie Azure Active Directory.

  3. Im linken Navigationsbereich wählen Sie App-Registrierungen.

  4. Wählen Sie die richtige App aus der Liste.

  5. Wählen Sie Certificates & secrets aus dem linken Navigationsbereichs.

  6. Wählen Sie New client secret.

  7. Geben Sie eine passende Key description ein und nehmen Sie eine Auswahl bei Duration vor.

    Hinweis: Wenn Ihr Clientschlüssel ausläuft, können sich die Benutzer nicht mehr in Caseware Cloud anmelden. Wir empfehlen, Ihren Clientschlüssel vor dem Ablauf zu erneuern, um die Möglichkeit von Störungen zu minimieren. Zur Erneuerung generieren Sie einen neuen geheimen Clientschlüssel und fügen ihn zum Feld Geheimer Clientschlüssel in Einstellungen | Einmaliges Anmelden | Identitätsanbieter zusammen mit der existierenden Anwendungs-ID und dem Metadatenendpunkt hinzu.

  8. Wählen Sie Hinzufügen.

    Hinweis: Kopieren Sie den Geheimen Clientschlüssel und fügen Sie sie zur späteren Verwendung in ein Textdokument ein. Nennen Sie das Textdokument Geheimer Clientschlüssel.

    Ihr geheimer Clientschlüssel wird dauerhaft verborgen, wenn Sie den Bereich Zertifikate & Geheime Clientschlüssel verlassen. Sie müssen Zugriff auf Ihren geheimen Clientschlüssel haben, um den Prozess zum Einrichten von Einmaliges Anmelden zu vervollständigen. Sollten Sie den Zugriff auf den Wert Ihres geheimen Clientschlüssels verlieren, müssen Sie Ihren geheimen Clientschlüssel löschen und anschließend neu erstellen.
