Einmaliges Anmelden integrieren

Caseware Cloud unterstützt die Funktion Einmaliges Anmelden (Single Sign-On oder SSO) mit Azure Active Directory (Azure AD), Active Directory Federation Services (ADFS) und Okta. Die Verwendung der Funktion Einmaliges Anmelden (SSO) mit Caseware Cloud reduziert die Anzahl der Kennwörter, die sich Mitarbeiter merken müssen und hilft Administratoren bei der Problemlösung im Bereich von Benutzerkonten (wie z. B. der Zurücksetzung von Kennwörtern) von einem Standort.

Bitte beachten Sie:

  • Das Autorisierungsprotokoll, das für die Integration von Einmaliges Anmelden verwendet wird, ist OpenID Connect.

  • Time Desktop ist nicht kompatibel mit Einmaliges Anmelden. Sobald Einmaliges Anmelden (SSO) aktiviert ist, ist es SSO-Benutzern nicht mehr länger möglich, sich in Time Desktop anzumelden.

Wenden Sie sich an den Caseware Support, um zu beantragen, dass Single Sign-On für Ihr Unternehmen aktiviert wird.

Nachdem Single Sign-On aktiviert wurde, finden Sie im Folgenden Anweisungen zur Einrichtung:

  • Einmaliges Anmelden in ADFS integrieren
  • Einmaliges Anmelden in Azure AD integrieren
  • Einmaliges Anmelden in Okta integrieren
