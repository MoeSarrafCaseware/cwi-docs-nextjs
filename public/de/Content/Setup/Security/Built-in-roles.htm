Vorgegebene Rollen

Caseware Cloud erhalten Sie mit einer Reihe von vorgegebenen Rollen. Diese Rollen bestimmen sowohl den Typ des Zugriffs als auch die Operationen, die ein Anwender in Cloud durchführen kann. Jede vorgegebene Rolle gilt entweder für Mitarbeiter oder Kontakte.

Um sich die vorgegebenen Rollen anzeigen zu lassen, wählen Sie Einstellungen und anschließend Zugriffschutz | Rollenberechtigungen.

Wählen Sie eine Rolle aus, um weitere Informationen im Menü auf der rechten Seite anzeigen zu lassen.

Hinweise:

  • Vorgegebene Rollen können nicht geändert oder gelöscht werden.

  • Jedes Unternehmen muss mindestens über einen Anwender mit der Rolle Administrator verfügen. Dieser Benutzer hat alle Berechtigungen und die Möglichkeit, alle Organisationseinstellungen zu ändern.

Mitarbeiterrollen

Zu den vorgegebenen Rollen gehören Unternehmensrollen und Inhaltliche Rollen. Mitarbeiterrollen können Mitarbeitern oder Mitarbeitergruppen zugewiesen werden.

Unternehmensrollen

Unternehmensrollen bieten unternehmensweite Rechte. Mitarbeiter mit diesen Rollen können die Cloud-Umgebung des Unternehmens gestalten und definieren.

  • Wenn eine Unternehmensrolle in einer Firma zugewiesen wird, die die Funktion Business Units verwendet, gilt sie für alle Business Units im Unternehmen.

  • Wenn eine Unternehmensrolle in einer Firma ohne Business Units zugewiesen wird, gilt sie für die gesamte Cloud-Instanz (d. h. systemweit).

In der folgenden Tabelle werden die integrierten Unternehmensrollen für Mitarbeiter beschrieben.

Rolle Anwendungsbereich Beschreibung
Administrator Unternehmensweit

Jedes Unternehmen muss mindestens über einen Mitarbeiter mit der Rolle Administrator verfügen. Administratoren verfügen über alle Berechtigungen und die Möglichkeit, alle Organisationseinstellungen zu ändern.

Hinweis: Sie müssen den Mitarbeitern mit der Rolle Administrator weiterhin Anwendungslizenzen (für Anwendungen, die sie benötigen) zuweisen.

Einstellungen-Administrator

 Unternehmensweit

Einstellungen-Administratoren können alle Systemeinstellungen bearbeiten.

Mitarbeiteradministrator

 Unternehmensweit

Mitarbeiteradministratoren können Personal und Personalgruppen erstellen, anzeigen, bearbeiten und löschen.

Inhaltliche Rollen

Inhaltliche Rollen bieten systemweite Rechte, Rechte auf Entitätsebene oder Rechte auf Entitätsinhaltsebene.

  • Wenn eine inhaltliche Rolle systemweit zugewiesen wird, gilt sie für alle Entitäten in der Cloud-Instanz (Firmen ohne Business Units) oder für einen einzelnen Business Unit (Firmen mit Business Units).

  • Wenn eine inhaltliche Rolle is assigned auf der Ebene Entität zugewiesen wird, gilt sie für die angegebene Entität.

  • Wenn eine inhaltliche Rolle is assigned auf der Ebene Inhalt zugewiesen wird, gilt sie für den angegebenen Inhalt in einer Entität.

In der folgenden Tabelle werden die vorgegebenen inhaltlichen Rollen für Mitarbeiter beschrieben.

Rolle Anwendungsbereich Beschreibung
Entitätsadministrator Systemweit Entitätsadministratoren können Kontakte, Kontaktgruppen, Entitäten und alle Inhalte innerhalb von Entitäten erstellen, bearbeiten und löschen.

Eigentümer

Systemweit, Entitäten und Inhalte innerhalb dieser Entitäten

Eigentümer können den Inhalt von Entitäten anzeigen, bearbeiten und löschen und anderen Mitarbeitern Rollen zuweisen, wenn sie diese Rolle haben.

Editor

Systemweit, Entitäten und Inhalte innerhalb dieser Entitäten

Editoren können Inhalte von Entitäten anzeigen und bearbeiten, wenn sie diese Rolle haben.

Beobachter

Systemweit, Entitäten und Inhalte innerhalb dieser Entitäten

Beobachter können Inhalte von Entitäten anzeigen, wenn sie diese Rolle haben.

Zugriff auf Entität

Systemweit, Entitäten und Inhalte innerhalb dieser Entitäten

Zugriff auf Entität - der Entitätszugriff ermöglicht es den Mitarbeitern, auf Entitäten zuzugreifen, in denen sie diese Rolle haben, so dass ihnen eine Sicherheitsrolle für Inhalte wie eine Aktivität oder eine Datei zugewiesen werden kann. Sie können keine Mitarbeiter, Kontakte oder andere Inhalte der Entität anzeigen oder bearbeiten, aber sie können die Zusammenfassung und Details der Entität anzeigen.

Diese Rolle wird normalerweise Mitarbeitern zugewiesen, die andernfalls nicht wissen würden, dass eine Entität existiert.

Kontaktrollen

Kontaktrollen bieten Rechte auf Entitätsebene (eine Entität) oder auf Inhaltsebene (Inhalt innerhalb einer Entität). Kontakte mit diesen Rollen können nur eingeschränkt oder gar nicht auf die Cloud zugreifen (mit der Rolle Zugewiesen).

In der folgenden Tabelle werden die vorgegebenen Rollen für Kontakte beschrieben.

Rolle Anwendungsbereich Beschreibung

Kontakt - Entitätsmitarbeiter

Entitäten und deren Inhalte

Kontakt - Entitätsmitarbeiter können neue Inhalte erstellen und auf vorhandene Inhalte in Entitäten zugreifen, in denen sie diese Rolle haben. Sie können Dateianforderungen empfangen.

Kontakt - Zugriff auf Entität

Entitäten und deren Inhalte

Kontakt - Zugriff auf Entität ermöglicht Kontakten den Zugriff auf vorhandene Inhalte in Entitäten, in denen sie diese Rolle haben. Es erlaubt ihnen nicht, Dateianforderungen zu empfangen oder irgendeine Kommunikation zu initiieren.

Zugewiesen

Entitäten und deren Inhalte

Zugewiesen ermöglicht die Zuordnung von Kontakten zu einer Entität. Sie können sich nicht bei der Cloud anmelden, keine Benachrichtigungen erhalten und auf keine Funktionen zugreifen.

Kontakt - Eigentümer

Entitäten und deren Inhalte

Kontakt - Eigentümer können Inhalte anzeigen, bearbeiten und löschen, wenn sie diese Rolle haben. Sie können keine Inhalte teilen.

Diese Rolle wird automatisch für Aktivitäten oder Dateien zugewiesen, die Kontakte erstellen.

Kontakt - Editor

Entitäten und deren Inhalte

Kontakt - Editoren können Inhalte anzeigen und bearbeiten, wenn sie diese Rolle haben. Sie können keine Inhalte teilen.

Kontakt - Beobachter

Entitäten und deren Inhalte

Kontakt - Beobachter können Inhalte anzeigen, für die sie diese Rolle haben. Sie können keine Inhalte freigeben.
