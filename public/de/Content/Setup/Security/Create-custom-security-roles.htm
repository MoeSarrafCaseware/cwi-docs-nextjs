Benutzerdefinierte Zugriffsrollen erstellen

Im laufenden Geschäftsbetrieb Ihrer Organisation ist es notwendig, dass Sie den Zugriff auf Ihre Mandantendateien kontrollieren. Verschiedene Engagementteams benötigen Zugriff auf ihre spezifischen Mandantendateien, aber der Zugriff auf andere Mandantenmaterialien ist ihnen verwehrt. Einige Mitarbeiter benötigen Zugriff auf alle Mandantendateien für ein Engagement, während andere nur Zugriff auf die ihnen zugewiesenen Dokumente brauchen.

Sie können den Zugriff auf die Engagements Ihres Unternehmens und die verbundenen Engagementmaterialien in Cloud mit Hilfe von Zugriffsrollen verwalten.

Die vorgegebenen Zugriffsrollen umfassen die häufigsten Typen von Zugriffschutz, die Sie den Benutzern gewähren möchten. Sie können aber auch benutzerdefinierte Rollen für Ihr Unternehmen erstellen.

Benutzerdefinierte Zugriffsrollen erstellen:

  1. Stellen Sie sicher, dass Sie über die Rolle Einstellungen-Administrator bzw. entsprechende Berechtigungen verfügen.

  2. Wählen Sie im Cloud Hauptmenü den Eintrag Einstellungen.

  3. Wählen Sie Zugriffschutz | Rollenberechtigungen.

  4. Wählen Sie Rolle hinzufügen.

  5. Füllen Sie die folgenden Felder aus:

    • Name

    • Beschreibung

    • Ist anwendbar auf: (Mitarbeiter oder Kontakte)

    • Anwendungsbereich (Systemweit, Inhalte innerhalb von Entitäten oder beides)

  6. Wählen Sie Weiter.

  7. Wählen Sie eine oder mehrere Berechtigungen, um diese zur Rolle hinzuzufügen. Wenn Sie fertig sind, wählen Sie OK.

    Best Practice: Bitte beachten Sie, dass es zwei Freigeben-Berechtigungen gibt: A und S. Die Berechtigung Freigeben (A) erlaubt dem Benutzer, alle verfügbaren Berechtigungen auf Entitäten oder Entitäteninhalte für sich und andere Benutzer freizugeben, selbst wenn es ihnen eine höhere Zugriffsebene als die bisher besessene gewährt. Wenn Sie z. B. eine benutzerdefinierte systemweite Rolle erstellen, die nur über die Berechtigung Freigeben (A) für Entitäten verfügt, können Benutzer mit dieser Rolle sich und anderen Benutzern die Rolle Eigentümer für jede Entität gewähren. Sollte Ihre Firma diesbezüglich Sicherheitsbedenken haben, empfehlen wir Ihnen, stattdessen die Berechtigung Freigeben (S) zu verwenden, die die gleiche Funktionalität bietet, aber verhindert, dass der Benutzer Entitäten oder Entitäteninhalte für Benutzer freigeben kann, auf die sie nicht zugreifen können.
