Benutzern und Gruppen den Zugriff auf eine Entität verweigern

Sie können Zugriffsrollen auf die gleiche Art und Weise wie beim Zuweisen auch wieder von Benutzern entfernen. Um die Rolle Eigentümer zu entfernen und den Eigentümer-Transfer zu vervollständigen, müssen Sie die Rolle im Dialog Freigabe deaktivieren.

Entitätsrollen von einem Benutzer oder einer Gruppe entfernen:

Die nachfolgenden Schritte beschreiben den Prozess zum Entfernen einer Entitätsrolle von einem Mitarbeiter. Sie können Kontakte oder Gruppen von einer Entität entfernen, indem Sie stattdessen Kontakte oder Gruppen wählen.

  1. Bitte stellen Sie sicher, dass Sie über die Rolle Eigentümer oder über gleichwertige Berechtigungen für die Entität verfügen.

  2. Wählen Sie im Cloud Hauptmenü den Eintrag Mitarbeiter.

  3. Wählen Sie den Benutzer, von dem Sie die Rolle entfernen möchten und anschließend Weitere Aktionen | Zuweisung zu Entitäten.

  4. Wählen Sie im Dialog Freigabe einen Benutzer und deaktivieren Sie die betreffenden Kontrollkästchen.

  5. Klicken Sie auf Freigeben, um Ihre Änderungen zu speichern.

Sie haben die Rolle Eigentümer für den gewählten Mitarbeiter entfernt. Diese können nicht mehr länger anderen Benutzern Zugriff auf die Entität oder deren Inhalt darunter gewähren. Diese können auch nicht Aktivitäten oder Dateien für diese Entität erstellen, bearbeiten oder löschen.
