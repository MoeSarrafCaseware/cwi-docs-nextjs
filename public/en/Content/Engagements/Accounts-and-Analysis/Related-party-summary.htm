Disclosure—Related party summary

Transactions with related parties must be properly identified and disclosed in the financial statements. A related party summary in the general ledger facilitates the identification of relevant transactions and balances for analysis and disclosure.

The objective of the test is to generate a summary that includes all transactions associated with related parties in the general ledger, detailing the affected accounts, amounts, nature of the transactions, and any relevant balances.

Fields used for analysis

Reference columns

Unique columns that create a unique transaction ID, such as entry_id for the General ledger. These columns will not be part of the result but may be needed in order to achieve the result.

Core columns

Numeric column - used to create the summary (could be disaggregated between debits and credits).

Related party column - used to summarize transactions involving related parties.

Parameters

None needed as this is a specific test and does not have any user input.

Output

The output of this test is a summary that presents related party transactions in the general ledger. This summary shows the affected accounts and the aggregated amounts for each related party.

Technical specification

  1. Identify all transactions related to specific parties in the General ledger.

  2. Aggregate the transaction amounts for each related party.
