Import data from SAP S4 HANA

You can import trial balance and general ledger data from SAP S4 HANA.

Export your data from SAP S4 HANA

If you want to import trial balance or general ledger data into your engagement from SAP S4 HANA, you must first export that data from SAP.

To export trial balance data from SAP:

  1. Log in to SAP.

  2. In the SAP menu bar, enter S_ALR_87012284 as the transaction code and then select Execute.

  3. Enter the correct information for the data you want to export in the following fields:

    • Chart of Accounts

    • Company code

    • Financial statement version

    • Language

    • Reporting year

    • Reporting periods

    • Comparison year

    • Comparison periods

  1. Select Execute.

  2. From the top menu, select System | List | Save | Save to save the report.

  3. In the dialog, select Spreadsheet (XLSX) to save the report as a .xlsx file.

To export general ledger data from SAP:

  1. Log in to SAP.

  2. In the SAP menu bar, enter FAGLL03 as the transaction code and then select Execute.

  3. Enter the Company code for the data you want to export.

  4. In the Line Item Selection section, select All Items and enter the ending date of the financial year for the data you want to export.

  5. Select Execute.

  6. From the toolbar, select the Change Layout () icon.

  7. In the Change Layout dialog, ensure the following fields are selected and then select Copy ().:

    • G/L Account

    • Amount in Local Currency

    • Text

    • Document Number

    • Document Type

    • Document Date

    • Posting Date

    • Posting Key

    • Local Currency

    • General Ledger Currency

  8. From the toolbar, select the Subtotal () icon.

  9. In the Define Sort Order dialog, ensure the Subtotal () boxes are unchecked for all fields and then select Copy ().

  10. You can save the customized layout to use it again later:

    1. Select the Save Layout () icon from the toolbar.

    2. In the Layout: Save as dialog, rename the layout, add a description and then select Save.

  11. To export the report, from the top menu, select List | Export | Spreadsheet.

  12. In the Export As dialog, enter a name for the file and ensure the file format is Spreadsheet (.xlsx).

  13. Select the Export to button to export your data.

Import your data

After you export your data, you can then import it into your engagement file.

To import the data into your engagement:

  1. Navigate to the Data page. The Data page opens to the Datasets tab.

  • If you’ve previously imported data, select Add data.

  • If your engagement is consolidated and has internal subsidiary entities, select which entity you want to import data for from the list.

  1. Under IMPORT FROM DATA CONNECTOR, select SAP (File) from the list of data sources.

    If the option is not visible, you can use the search box to find it.

  2. At the Upload stage of the import wizard, select or drag-drop the general ledger and trial balance data files you exported, then select Next.

  3. At the Configure Data stage of the import wizard:

    • Reporting period is automatically filled with the reporting period of the data.

    • Year start Date is automatically filled with the start date of the data.

    • Year end Date is automatically filled with the end date of the data.

  4. In Select dataset type, choose the type of datasets you are importing.

    Note that only trial balance and general ledger datasets are currently available for import from SAP S4 HANA.

  5. If you selected Trial balance, select the Import the last 5 years of TB to import up to five years of trial balance data if it is available in the file you exported.

  6. Select Next.

  7. At the Confirmation stage of the import wizard, review the information and select Import.

To view the imported data, click View data next to the dataset you imported.

A new page appears displaying the data.

By default, all the columns in your dataset display. Click Columns to deselect those you want to hide.

Click Filters to filter the data displayed in the dataset.

For more information, refer to View the client's general and subledger data.
