Understand the Trial balance dataset

When importing a trial balance from a CSV or Excel file, you must map the fields, so the trial balance data is displayed properly in the engagement. This topic describes the fields within the Trial balance dataset. For more information on importing trial balance data from CSV files, see Import the client's data from a CSV or Excel file.

Field Usage
Account number (account_id)

The identifier for the account.

Data type: String

Required: Yes
Account name (account_main_description)

The name or description for the account.

Examples: Cash; Property, plant and equipment; Sales

Data type: String

Required: Yes
Financial group number (financial_group_number)

The number for the financial group the account is assigned to in the grouping structure. This information is not provided by the client and is used to automatically map accounts to financial groups.

Accounts currently assigned to groups may be reassigned when you import this data field.

Data type: String

Required: No

Current year balance (current_year)

The current year balance.

Data type: Money

Required: No
Prior year balance (prior_year)

The prior year balance.

Data type: Money

Required: No

 
Prior year 2 balance (prior_year_two)

The balance for prior year 2.

Data type: Money

Required: No

 
Prior year 3 balance (prior_year_three)

The balance for prior year 3.

Note that if your engagement is externally consolidated, you will see both a Parent balance - PY 3 and a Consolidated balance - PY 3 field. If you assign one of those fields, you must also assign the other.

Data type: Money

Required: No
