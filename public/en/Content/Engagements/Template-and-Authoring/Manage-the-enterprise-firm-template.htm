Manage the enterprise firm template

As an enterprise firm author you typically need to perform the following tasks:

  • Create a template – Create your enterprise's customized version of the product template.

  • Edit an existing template – Modify or update your template to reflect changes in standards or methodology or your users’ needs.

  • Revert to the product template – Reset your template to the product template.

  • Update your template – Update your template with the latest changes from a new version of the product template.

This section provides instructions for performing these tasks.

Create a draft template

If you don't have an enterprise firm template or a draft, then your starting point is to create a draft.

The following describes three common scenarios for creating a draft.

Do not customize the template

If you don't want to customize a template, you can set your enterprise firm template to use the enterprise template. The enterprise template contains the default content for your product.

The advantage of using the enterprise template is that you don't need to manually update your template whenever there are new updates.

To use the default template:

  1. Select Use Default.

  • This option will not be visible if you already have an active template.

  • The current template is archived and you now use the enterprise template.

If you do not see the Use Default option, it means there is still an active draft. To return to the default:

  1. Select Delete Draft .

  2. Select Use Default.

Customize the template

If you want to customize the enterprise template to meet specific enterprise requirements:

  1. Select Create Draft. A new window will open with a new draft.

  2. Edit the draft. To learn more, see a list of customizable features under Template setup: Authoring index.

  3. Return to the original window.

  4. In Manage Template, select Publish As [version number]. Your published draft becomes the new minor version of the enterprise firm template. The published draft is added to the archive, and a record of it is displayed in the archived templates list.

Revert or reuse a previous template version

Note: If there is an active draft, you must delete it before proceeding.

You can create a new draft from any archived template, whether to restore a prior version or create a new draft. Previous versions are stored in the archive. Archived templates can only be opened if there isn’t an in-progress draft.

To revert to a previous template version:

  1. Go to the archived templates list.

  2. Open an archived template to create a draft based on that template. A new window will open with a new draft.

  3. Edit the draft. To learn more, see a list of customizable features under Template setup: Authoring index.

  4. Return to the original window.

  5. In Manage Template, select Publish As [version number]. Your published draft becomes the new minor version of the enterprise firm template. The published draft is added to the archive, and a record of it is displayed in the archived templates list.

The published template is added to the archive and a record of it displays in the archived templates list.

Edit a draft template

You can edit the draft until you publish it. This situation only applies if you have already created a draft.

To edit the template:

  1. Select Open Draft. A new window will open with a new draft.

  • Your current template will be the one displayed.

  • The template opens in draft mode. You can now make changes as needed.

  1. Return to the original window.

  2. Select Publish As [version number] to have the changes applied to newly added engagements.

Delete a draft

You can delete a draft if you're not satisfied with your current draft. However, a template cannot be deleted once published. Older versions of a template are archived.

To delete a draft:

  1. Select Delete Draft. You will return to the options to use the default template or to create a draft.

Update the enterprise firm template after a version release

Minor version release

Your enterprise author typically releases a minor version of the product template on an ad hoc basis, that include minor fixes and changes. These updates affect all enterprises. Enterprise authors will then prompt the enterprise firm authors to accept these updates to the enterprise firm template.

Scenario 1: You do not use a customized template

If you do not use a customized template, no action is required. The new minor version will be applied automatically, and all new engagements will use the latest enterprise template version.

Your enterprise author will notify you of the release. Once released, you should communicate the update to your users.

Scenario 2: You use a customized template

In this scenario, you have a customized enterprise firm template that will need to be manually updated. This will ensure that you retain your enterprise customization after updating.

When your enterprise author notifies you of the release, once you accept the update and create a new draft, your existing enterprise firm customizations will automatically carry over to the latest minor version.

To apply your customizations to a minor version release:

Note: If a draft exists, you must publish or delete it before proceeding. An Update button will become available once either is done.

  1. Click Update to receive the latest minor version of the enterprise template.

  • A new draft is created with the latest product updates and your saved template changes combined.

  1. Publish this new draft to make this new minor version of the enterprise firm template available to your users.

Major version release

Caseware typically releases a new major version of the product template annually or when a significant number of updates are required. These updates affect all enterprises. Enterprise authors will then prompt the enterprise firm authors to accept these updates to the enterprise firm template.

Scenario 1: You do not use a customized template

If you do not use a customized template, no action is required. The new major version will be applied automatically, and all new engagements will use the latest enterprise template version.

Scenario 2: You use a customized template

In this scenario, you have a customized enterprise firm template that will need to be manually updated when a new major version of the enterprise template is released to retain your enterprise firm customization after updating. This involves the following three major tasks:

  • Migrate enterprise firm customizations from the previous product template.

  • Review and test the migrated content within a specific timeframe provided by the enterprise author.

  • Publish a new version of the enterprise firm template for use by staff. The new version will include the changes made by the major version release and your enterprise firm’s template customizations.

To apply your customizations to a major version release

Note: If a draft exists, you must publish or delete it before proceeding. A Migrate from previous button will appear once either is done. This button appears only if a new major version becomes available, and there is no draft in progress.

  1. Click Migrate from previous.

  1. Click Open Draft and apply your enterprise firm customizations to the new major version.

  2. Once changes and reviews of the template are complete, select Publish As [version number] to release a new version of the template. You will be required to publish again to test your new version.

  3. Review and test your template:

    • Create a new draft of the enterprise template.

    • Open the new draft to review the contents of the new major version and enterprise firm customizations applied from a previous version.

  4. In Manage Template, select Publish As [version number]. Your published draft becomes the new minor version of the enterprise firm template. The published draft is added to the archive, and a record of it is displayed in the archived templates list.
