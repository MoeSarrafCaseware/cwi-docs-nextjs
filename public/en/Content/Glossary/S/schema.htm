schema

In the context of Caseware Cloud, a structured framework that defines the organization and format of data within the platform. It includes metadata for each column that constitutes a table, specifying the data types, relationships, and constraints. This structured approach ensures that data is consistently organized and can be efficiently queried, analyzed and reported.

For example, in Caseware Cloud, a schema defines the structure of a General ledger dataset, specifying columns such as Account number, Account name and Posting date, along with their respective data types (for example, text, number, date). A schema ensures that all imported data conforms to a consistent format, facilitating accurate financial analysis and reporting.
