View management

We recommend creating custom views to view engagement files and other data to ensure only relevant fields are displayed on the page. The less fields are displayed on the page, the faster the page will load. We recommend adding a maximum of 5 columns per custom view. You can also add filters to custom views to display items based on tags, workflow stages, and other criteria to further reduce the number of items that are displayed on the page. This will improve performance and ensure users only view relevant content.

Create a new view

You can create new views from Settings.

The view is added, and can now be assigned to the desired user types.

Assign views

After you have created a view, you will need to configure what types of users can access it or have it set as a default.
