Collect information from the client

In this section, we'll gather some required information for the engagement.

Inquiries document

An OnPoint AnalyticsAI engagement includes an inquiries document that you can send to your client for completion.

Practice set

To access the inquiries document:

  • Select 0-1 Client Inquiries (Query) from the Documents page.

You can adjust which procedures you want to send by using the show/hide functionality. For example, under File Upload, you can hide Upload Trial Balance if you intend to use a different method for adding a trial balance to the engagement.

You can also add comments to individual procedures in the query document.

For this exercise, we will leave all of the procedures in the 0-1 Client Inquiries (Query) document with their default content.

To send the query:

  1. Select 0-1 Client Inquiries (Query) from the Documents page.
  2. From the top-right of the page, select Send.

    The Open Query dialog displays.

  3. Enter a Query Due Date - use today.
  4. Select an Assignee - use your contact persona.
  5. Enter a message in the Email Body - "Please provide the requested information."
  6. Select OK.

    An email is automatically sent to the assignee, and the status of the document changes to Awaiting response.

    The assignee's email includes a link to view the details of the query.

    When the assignee selects the link, a version of the document appears. The assignee can answer the questions and provide the required files and then select Responded to indicate that the item has been completed.

    Note: The assignee can see any comments that have been entered.

Import the trial balance (Data tab)

You'll need to import a trial balance for your engagement to reconcile your data in the Data Completeness page, and to be able to use Influencers.

You can import a trial balance from:

  • Caseware Working Papers
  • Desktop accounting software
  • An Excel or CSV file
  • Online platforms such as Quickbooks Online and Xero

If you're importing from online accounting software, you have the option to choose to import transaction data as you import the trial balance data.

To populate the trial balance, you'll have to import the financial data to your engagement file. In some cases, you can import this data directly from another accounting software package using the native import utility (for online accounting software) or a desktop utility import we provide.

Practice set

In this example, we'll import a trial balance from a CSV file.

To import a trial balance from a CSV file:

  1. Select 0-2 Import Trial Balance & Online Transactions from the Documents page.
  2. Select Excel or CSV.
    The import dialog displays.
  3. Drag and drop the TrialBalance.csv file into the import area in the grey dashed box.
  4. To map the columns in the import file, choose the field that best matches the data from the drop-down. If you don't want to import a column, select Do not import to exclude it.

  5. Select Import.
    The TRIAL BALANCE- ACCOUNTS page displays with the imported trial balance.

Assign accounts

When you import a trial balance, OnPoint maps as many accounts as possible, but in some cases, you may need to evaluate accounts and assign them manually.

Practice set

At the top of the page, in the Data menu (on the left side of the screen), notice that the Assign accounts icon now includes a number.

This indicates that one account is currently not assigned to a group. Select the icon to open the Assign Accounts page.

On this page, one account is displayed in the Unassigned column.

Select it to move it to the Selected Accounts section. A number of Suggestions display. Select 2.2.1.1 Accounts payable.

Select the Financial Groups category to to view the grouped Trial Balance. Select the Documents tabs.

Import transactions

If you've imported the client's data from a CSV file, you'll need to import the transactions CSV to store and run analyses on your data.

Practice set

  2. Drag and drop the Transactions.csv file to the File Upload area or click and browse to its location.

You can save the layout of the data mapping you've just completed. This allows you to reuse the mapping structure you've initially defined in subsequent reimports of transactions.

To save a DATA LAYOUT:

  1. Select 0-3 Import Transactions from the Data page.
  2. Select SAVE from the Import Transactions | ACME Inc. page.
    This option is only available for mapped transaction data files.
  3. Enter a Data Layout Name in the Save Data Layout dialog, for example, Standard Import Layout.
    The DATA LAYOUT name is updated on the Import Transactions page.
    Now, when you select MAP or REMAP, a drop-down containing available layouts displays.
  4. Close the MAP/REMAP tab.
