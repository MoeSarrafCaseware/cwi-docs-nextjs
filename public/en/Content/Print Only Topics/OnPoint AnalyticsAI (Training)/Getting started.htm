Getting started

You'll need to do some basic set-up tasks before you move on to a client engagement.

Create a new entity

An entity is a workspace that you can use to organize and collaborate on your engagement files from a single location for a client. When your firm first starts to use Cloud, you need to create a new entity for each client.

Note: The instructions below explain the process for adding an individual client entity. You can also import a list of your clients in order to create all of your client entities at once. For more information, refer to Caseware's online help.

Create accounts for your staff members

Once you create entities for each of your clients, grant Cloud access to other staff members in your firm. You can create a new account for each staff member who needs to collaborate on your engagements.

Note: The instructions below describe how to add individual staff members. You can also import a list of your staff members to create all of their user profiles at once. For more information, refer to Caseware's online help.

Practice set

From the Cloud menu, select Staff. This takes you to the Staff page.

Select New, and then ADD | Staff. The Add Staff dialog displays. Similar to creating an entity, there are different property groups where you enter information for new staff members.

General

Enter an email address and a first and last name for the new staff member. The email address will be the username for their account, and they will log into Cloud with it.

The other fields are optional.

Password

There are three initial password setting options:

  • Set by User - the staff member will set their own password when they first log in. Cloud sends an email with instructions on how to log in to the portal.
  • Auto Generated - a random password is chosen by Cloud. Cloud does not send an email to the staff member, so you will need to provide them with the login page URL, their user ID, and their password.
  • Set - you choose a password for the staff member. Again, Cloud does not send an email, so you will need to provide them with their login information.

In this training, we will use the default option: Set by User.

Groups

You can assign the staff member's account to one or more staff groups. If you create groups in Cloud, you can specify permissions and access levels for members of that group. If you add any additional accounts as group members, they automatically inherit the group permissions and access levels.

The All Staff group is available out of the box in Caseware Cloud. All staff members are automatically added to this group by default. In this training, we will use the All Staff group.

System-Wide Roles

When you grant one or more system-wide roles to a staff member, they get a set of access rights that apply to all of the content in your Cloud firm. For example, if you grant the system-wide Viewer role to a staff member, they can see all of the entities in your firm and can view all files and engagements in those entities. System-wide roles are the easiest way to set the access rights for staff members in your firm.

Grant this staff member the system-wide Editor role. This role allows them to view all the firm content and make changes while working on that content.

Apps

You can grant your staff member access to Cloud apps. With this access, they can view engagements for an app and create new engagements or files for it.

Note that by default, the All Staff group grants its members the ability to view and create engagements for all the available Cloud apps. If you need to restrict access to some apps, remove some of the permissions on the global group. This enables you to select which staff member should work with which apps.

Standard Hours

This setting is if your firm uses the Caseware Cloud Time app to handle time and expense entries. Leave this section set to Organization standard hours.

Once you have entered all of the information for your staff member's new account, select Save. They will receive a welcome email from Cloud notifying them that their account has been created and instructing them on how to log in.

Create an account for your client

One of the major benefits of OnPoint AnalyticsAI is that you can collaborate directly with your clients on engagements. You need to set up a contact account so your clients can access their engagement file.

Practice set

From the Cloud menu, select Contacts. This takes you to the Contacts page.

Select New, then ADD | Contacts. The Add Contact dialog displays.

General

Enter a first and last name and an email address for your client contact. In the Entity field, select the client entity that you created earlier. Once you have selected the client entity, select the Contact - Entity Collaborator role to set the access level for this account. This role allows your client to create discussions, tasks, and file requests on Cloud. It also allows them to collaborate on engagements for OnPoint AnalyticsAI.

Password

The same password options for staff accounts apply here. Because the email address is yours, choose Set and uncheck User must change password on next sign in.

Groups

Contact groups work the same way as staff groups. The contact group available out of the box is All Contacts and by default, your client is automatically a member of that group.

Apps

Just like the All Staff group, the All Contacts group grants its members the ability to view and create engagements for all the available Cloud apps. Check the permissions and make sure that your contact can view OnPoint AnalyticsAI engagements.

Once you finish setting up your client contact's new account, select Save. They will receive a welcome email from Cloud notifying them that their account has been created and instructing them on how to log in.
