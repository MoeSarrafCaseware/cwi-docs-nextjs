Send a PBC document

To request information from your client, you need to send a PBC Requests document to your client. Questions in the document can be document requests or some inquiries related to the engagement.

In this training, we are requesting a document from the client.

Practice set

Go to the Documents page and open the 2-920 PBC Document Requests document under Engagement and Acceptance and Planning | PBC Requests.

You can review the questions included in the query document before you send it. If there are any questions you do not want to send to the client, you can select the Visibility icon () beside the question and select Hide.

Once you're done, select SEND. The Open Query dialog displays.

Enter a Query Due Date to specify a date to which the client response falls due. At Assignees, you can choose to assign the document to all your client contacts or to select one or more contacts.

Select the Search Contacts field and add the client contact that you've created earlier. Enter the following email message, then select OK:

Once your Query is sent, the client contact receives an email notification when they log in to Cloud, so they can directly access the Query document and respond.

Once your client uploads the requested file and selects Mark as responded, you immediately see the response in the query document. You also get a notification in Cloud.

Open the query to start reviewing responses. Notice how the time of the response and the name of the contact that has responded to each query question displays in the query document.

Once you review and confirm the information received from the client, select the Accept button at the bottom of the query question.

You can also return a response back to the contact if the information or documents provided are insufficient, incorrect or need clarification. Select the Return button at the bottom of the query question.

Finally, select COMPLETE at the top to mark the query as completed.

At the Documents page, the trial balance file that the client uploaded is automatically added in the Permanent File phase under Financial Reporting Document(s).

Import the trial balance

Instructor Note: Make sure the students are using an engagement file not a firm template.

You can import the trial balance data from:

  • An online accounting software - you can import from Xero, QuickBooks Online or Exact Online.

  • A third-party desktop accounting software - you need to use either the Caseware Cloud Import Utility tool or the Caseware Transformer tool to import from a third-party desktop accounting software. The tool that you'll need to use depends on the desktop accounting software you're importing from. To learn more, visit docs.caseware.com.

  • An excel or a CSV file - if the accounting software package you're using is not supported by any of our import utilities, you can export your client data to a CSV (comma-separated value) file and then import the file. Note that the file can be either an Excel or a CSV file.

Practice set

Now that you have the client's trial balance file, you can import the trial balance information. From the top menu, open the Data page then select the Import tab.

Choose Excel or CSV Import. A confirmation message pops up notifying you that once you import the trial balance you cannot change the reporting period. Select Proceed.

A new dialog displays prompting you to select a file by either dragging and dropping it from your desktop, by browsing it or by selecting a file in the engagement.

When you choose Select file from engagement, you will be able to select the Excel file that you added to the engagement file from the client procedure. Select the file from this dialog, and the Column Mapping dialog displays next.

Select how you want to import each column in the spreadsheet as. For our sample file, import the columns as follows:

Source column Import as
Account Number Account Number
Description Account Name
Current Year CY
Prior Year PY
Prior Year Two PY 2

Once you're done, select the Import.

The contents of the trial balance file are now imported into the engagement file. The net income is displayed at the top corner of the window, and if the trial balance is out of balance, warnings are displayed at the top of the relevant columns.

You can also reimport the trial balance data at any point in time from the Import tab.

Assign accounts

Instructor Note: Make sure the students are using an engagement file not a firm template.

When you import a trial balance, OnPoint maps as many accounts can be, but in some cases, you may need to evaluate accounts and assign them manually.

Practice set

At the top of the page, in the trial balance menu, notice that the Assign accounts icon now includes a number.

This indicates that two accounts are currently not assigned to a group. Select the icon to open the Assign accounts page.

On this page, two accounts are displayed in the Unassigned column.

OnPoint displays a few group suggestions based on the account name "Cash". Select an account to move to the Selected Accounts section. To assign the account to the Cash group, select the green Assign icon that displays beside the suggestion.

On this page, you can also drag and drop accounts to re-assign them to other groups. Drag an account towards the desired destination group and drop it when the group is highlighted in green.

Changes are reflected on the Data page.

Add accounts and groups

Instructor Note: Make sure the students are using an engagement file not a firm template.

You can manually add a custom account or group to the trial balance if needed. New groups require a name and parent group before you can assign accounts. New accounts require a unique number and name before you can enter values.

Practice set

Let's add a group and an account in the trial balance data.

To add a group:

  1. Select the FINANCIAL GROUPS view.

  2. Select Add () | Add group.

  3. At the Group dialog:

    1. Enter a group number at Number.
    2. Enter a group name at Name.
    3. Select the parent group from the existing trial balance groups at Parent Group.

  4. Select Save.

The new group is added to the trial balance listed under the selected parent group. You can now assign accounts to it.

To add an account:

  1. Select Accounts tab.

    2. You can also select FINANCIAL GROUPS to add an account in the Financial grouping.

    Note: You can also select TAX GROUPS and PRESCRIBED GROUPS, if they're enabled in your product.

  2. Select Add () | Add account.

  3. At the Account dialog:

    1. Enter the account number at Number.

    2. Enter the account name at Name.

    3. Select the Parent group drop-down to assign the account to a Financial group.

      Based on your product setup, extra parent group drop-downs can be available. For example, Parent group - Tax Code, Parent group - Chart of Accounts and Parent group - GIFI.

      Note: If you assign an account to a group that's linked to another group, it's automatically assigned to the linked group as well.

    4. Enter the current year balance at Prelim.

    5. Enter the prior year balance at Previous.

  4. Select Save.

Post adjustments to the trial balance

Instructor Note: Make sure the students are using an engagement file not a firm template.

You can post adjusting entries to accounts or groups and include a description of the entry and an annotation.

Practice set

To post an adjusting entry:

  1. Go to the Data page.

  2. Select Accounts from the left panel.

    3. You can also add adjustments in the FINANCIAL GROUPS view option.

    Note: You can also select TAX GROUPS and PRESCRIBED GROUPS, if they're enabled in your product.

  3. Select Add () | Add adjustment.

  4. Add the details of the adjusting entry in the dialog.

    1. Enter the Adjustment Number.

      2. Note: If this is not the first adjustments entry, the application automatically increments the last entered adjustment number and displays it in the Adjustment Number field. If this is the first adjustment entry, 1 displays in the field as the default adjustment number. You can override the default.

    2. Choose an adjustment Type.

      3. Note: The available adjustment types in the drop-down are based on your product setup.

    3. Enter a description for the adjustment.

    4. At Details, select + Account line to post the adjusting entry to an account, or select + Group line to post it to a group.

      Tip: You can create an account as you add its lines using the Add account button.

      If the engagement is a consolidation engagement and you choose to,

      • add an account line adjustment, a list of the parent and subsidiaries accounts displays in the dialog.
      • add an group line adjustment, a list of the parent entity groups displays in the dialog.

        • Note that if the adjustment type is Eliminating, you can also post the adjustment for a subsidiary group. Select the entity name to display a list of available entities (parent and all subsidiaries). For more information on adjustment types, see Common adjustment types.

    5. Select an account or a group and enter the adjustment amount.

    6. Finalize the entry by selecting + Account line or + Group line to add another account or group to balance the adjustment.

    7. Select Save.

Your adjusting entry is posted to the account or group. The data is immediately updated with the new values.

Annotate the trial balance

Instructor Note: Make sure the students are using an engagement file not a firm template.

While you are reviewing the trial balance, you can annotate accounts.

Practice set

From the trial balance menu, select the Columns drop-down menu.

You can select different options here to customize the columns that the Data page shows. Select Variance to add a new column that displays the variance between the final balance in the current period and the final balance from the prior period.

In this sample trial balance, there is no Previous value for the Bank - Savings account. Let's assume you have noticed this and asked your client about it, who informed you that the cash in the savings account came from a sales transaction that happened very late at the end of the year. In the trial balance page, you can add an annotation to include this information.

Select Add Annotation () at the Bank - Savings line. A new dialog displays.

You can choose a purpose for your annotation, link it to a document or an adjustment and add a note to describe the reason for the annotation. You may also see a separate area where you can choose from several annotations suggested by OnPoint PCR.

For this annotation, select Variance for the purpose, enter "Late sales at end of year" for the note section and select Save.

The trial balance displays an icon for the annotation type and includes the description from the note.

Export the trial balance

You can export trial balance data to a CSV file or a PDF file. You can also export to a CSV ASCII format, which is compatible with Working Papers.

Practice set

Let's export the trial balance data to a CSV.

To export trial balance accounts:

  1. Select Accounts from the left panel at the Data page.

  2. Select Export (), then choose Export to CSV.

The trial balance accounts data is exported to the selected file type and download starts immediately.
