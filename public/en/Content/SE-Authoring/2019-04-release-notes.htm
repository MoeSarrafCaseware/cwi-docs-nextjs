SE 2019-04 release notes

The features below are automatically enabled if the corresponding component or module has been enabled.

The Data page

The Data page is a redesign of the Trial Balance page. The purpose of the redesign is to provide flexible reporting methods, support more data sources and parameters, provide a guided workflow for import and mapping as well as improve performance and usability.

New features to the Data page include:

  • A redesigned groups and accounts view

  • Defining the default trial balance groups for the level of detail required by the product

  • Drill-down to transactional details

  • A left pane to easily switch views

  • Data sorting and filtering within each view

  • Search capabilities

  • Data export (trial balance to CSV)

The following features are available using the previous interface and will be improved in future releases:

  • Data import and account mapping (for trial balance and transactions)

  • Adjustments

  • Accessible predefined reports by saving data filters

Primary authors can enable the Data page in their products by selecting the Enable Data page check box in the product settings.

Recommendation: We suggest that distributors spend significant time testing their product prior to deploying the new UI to clients.

Letters, memos and financial statements

  • To easily and efficiently locate missed placeholders in the document, you can now select the placeholder indicator to navigate the placeholders that haven't been specified.

  • Authors can now add a logo and letterhead placeholder in the product or firm template. The placeholder will automatically populate from the new Logo and Letterhead fields in Collaborate. The letterhead automatically displays in the PDF file when users print the document.

Financials new design

Authoring

Authors can now add guidance information in the Financial Statements document.

  • The Add Guidance menu option is now available for areas, text sections, dynamic tables and notes. Users can turn the guidance information on or off using the lightbulb icon next to the area or text section or table.

  • A new Guidance area is also now available to add general guidance information in the document.

Notes

  • You can now have a 3-level notes list by adding second level (Heading1) and third level (Heading2) notes. If you delete a note, all headings and content within this note are also deleted. Note that third level notes don't show in the Document Map unless you expand their respective second level notes (parent notes) in the document.

  • The notes library now displays notes based on hierarchy (sublevel notes or main notes). You can traverse the hierarchy of the displayed notes by selecting View note.

  • When you add a note from library, a blue indicator now shows next to the newly added note in the Document Map. The blue indicator disappears once you select the note.

Dynamic tables

  • You now have the option to edit note references in a row or column. You can also navigate to the referenced note (in the Notes to the Financial Statements area) to view or edit its content.

  • The Description column can now show account numbers in addition to the account name and total. Note that authors must enable this feature in the product.

  • A new table format is now available to support the International Financial Reporting Standards (IFRS) and the Statement of Changes In Equity (SOCIE). The new format flips the table to have dimensions as rows and trial balance groups as columns.

    • Note that any existing setup and information is lost once the table format is changed. Authors need to redo any existing table setup if they switch to IFRS/SOCIE format.

Visibility settings

In the Financial Statements, you now get a confirmation prompt if you attempt to exclude content that's available due to visibility settings. The prompt shows the details of the response conditions logic and provides a link to the checklist document so you can review and change responses if needed.

Note that currently, this feature is only available for visibility settings based on response conditions.

  • Fixed some issues related to the indentation and truncation of note titles in the Document Map in the new financials design.

  • Fixed an inconsistent icon size issue for annotations in the engagement reference copy (PDF file).

  • Fixed a styling issue in the Table of Contents area in the new financials design.
