SE 2019-10 hot fixes

Here is a summary of the hot fixes that have been deployed for SE 2019-10.

Hotfix 1

Version: 2019-10 Build 465

Here is the list of issues that have been fixed with the deployment of Hotfix 1.

To view the details of the root cause and the fix, select the issue identifier (example: SE-12345).

  • [SE-16229] - Dynamic tables fail to correctly display accounts that are manually added in the Data page.

  • [SE-16218] - In the new risk module, users cannot properly enter an exclude reason for areas in the Assessments tab.

  • [SE-16205] - Visibility settings based on Materiality conditions do not work as expected.

  • [SE-16171] - In the financial statements notes section, collapsed group rows in dynamic tables do not display account balances.

  • In the Data page:

    • [SE-16117] - the product decimal separator settings don't apply to manually added accounts.

    • [SE-15998] - users cannot assign accounts after consolidating entities.

    • [SE-15968] - column headers for prior years display incorrectly after importing prior year data.

  • [SE-15948] - Overridden balances in dynamic tables do not display correctly.

  • [SE-15941] -Eliminating adjustments do not display in financial statements dynamic tables.

  • [SE-15902] - Users cannot select a copy-pasted text placeholder.

  • [SE-15844] - In letters, financial statements and memos, a new line is automatically inserted once users add a firm address text formula (fx).

  • [SE-15658] - Account balances change when a group is expanded and collapsed.

  • [SE-15354] - Typing issues in text fields in the new query layout.

Hotfix 2

Version: 2019-10 Build 466

Here is the list of issues that have been fixed with the deployment of Hotfix 2.

To view the details of the root cause and the fix, select the issue identifier (example: SE-12345).

  • [SE-16376] - Primary authors cannot upload macro-enabled excel files to the product template.

  • In dynamic tables:

    • [SE-16311] [SE-16282] - diagnostic labels are missing in dynamic tables with a custom calculation formula that includes division.

    • [SE-16386] - when users override amounts, balances in other tables across multiple areas do not update until users refresh the page.

    • [SE-16229] - new accounts do not display in the tables until users collapse and expand the parent group for that account.

  • [SE-16413] - Amount Variance column balances do not display in the financial statements. Balances only display in the dynamic table edit mode page.

  • [SE-16373] - Group balances do not display in Analysis tabs.

  • [SE-16425] - The outline around overridden balances displays in the financial statements print file (PDF).

Hotfix 3

Version: 2019-10 Build 467

One issue identified when reviewing the production logs and has been fixed with the deployment of Hotfix 3.

To view the details of the root cause and the fix, select the issue identifier (example: SE-12345).

[SE-16510] - Users get an "Invalid Engagement" error when attempting to open or carry forward certain engagement files. Specifically, files that have not been opened since 2019-04 and include uploaded supporting documents.

Hotfix 4

Version: 2019-10 Build 469

Here is the list of issues that have been fixed with the deployment of Hotfix 4.

To view the details of the root cause and the fix, select the issue identifier (example: SE-12345).

  • [SE-16545] - Balances above the trivial amount defined in the Materiality document are marked as material in the Trial Balance page.

  • [SE-16537] - Digit grouping separators and decimal symbols are missing in the values that display in Analysis charts.

  • [SE-16526] - When the new risk module is enabled in the product, the Risk of material misstatement option in the Risk tab in checklists displays incorrect information.
