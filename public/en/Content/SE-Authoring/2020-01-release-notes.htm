SE 2020-01 release notes

Don't retire your Microsoft Excel skills just yet! Cloud Engagements is bringing in Cloud Connector - a MS Excel Add-in. Featuring graphs and charts in Financial Statements, transactions sampling and more, SE 2020-01 rings in the new year with exciting additions to the platform.

Sample selection

#primaryauthor #enduser

The Data page now features a new tab: Transactions, solely dedicated to display account transactions, as well as support sample selection and report generation for verification testing.

In the Transactions tab, users can:

  • analyze the data by sorting and filtering to create a sampling population.

  • generate a sample of the data using the available methods (Manual, Random, Monetary Unit or Systematic).

  • export the sample to a CSV file or copy it to clipboard for further analysis and documentation of test results. Users can also upload the sample file in the Documents page and reference it from a work-program procedure.

You can enable sampling in your product by selecting the Enable sampling check box in the product settings.

Analysis in Financial Statements

#primaryauthor #firmauthor #enduser

A new content section Analysis is now available in Financial Statements. You can customize the types of graphs and charts, as well as the data they present in the product API. To learn more, see Add an analysis section.

Similar to the Analysis tab in checklists, the Analysis section in Financial Statements displays the selected graphs or charts for a specific group or the overall financial analysis. Users can resize graphs and charts to have them display in a full size across the section for better presentation. To learn more, see Review account analysis in Financial Statements.

Formula builder in dynamic table cells

#primaryauthor #firmauthor #enduser

A formula icon () is now available in dynamic table cells in Financial Statements to support scenarios that call for a need to have calculated values in cells, such as: movement rows in the Statement of Cash Flows or ratio analysis rows in the "Earnings per share" section in the Income Statement.

Note: This option is only available for Input column and row types.

The calculation can include numbers, data sourced from the trial balance, data sourced from the financial statements, form values or a combination of all.

Note: The Forms value option is only available if you have developed forms using SE Builder in your product.

In the document, you can select the cell where you've added a calculation to view its details.

If users override a predefined cell calculation (a calculation that's been added in the product template by authors), a Revert option displays in the formula builder - the Calculation tab - so they can always revert to the predefined calculation, if needed.

If users override a cell value, the Account Details popup displays the original calculated value and formula, as well as presents a Revert option.

Enhanced logic for Show/Hide in table rows

#primaryauthor #firmauthor #enduser

The Show/Hide (/ ) option for dynamic table rows now includes additional options. Selecting the icon opens a drop-down that presents either two or three options, based on the row type.

  • Apply default behavior - the default behavior is applied to the row. Note that the default behavior differs based on the row type:

    • For input rows:

      • Always display in the financial statements.

      Display in the print copy of the document if the row has a balance (the cell is not empty and its value is not zero).

    • For custom group Total or Title rows:

      • Always display in the financial statements.

      Display in the print copy of the document if the group includes more than one account balance (the cells are not empty and their values are not zero).

    • For trial balance rows:

      • Display in the financial statements and in the print copy if the balances for current or prior year exist.

  • Always display this row - the row always displays in the financial statements and in the print copy (PDF).

  • Always hide this row - the row is always hidden in the financial statements and in the print copy (PDF).

    • Note: This option is only available for input rows and groups (trial balance group rows and custom Title and Total rows).

To complement this feature, rows that display in the financial statements but will not print show in gray color in the document so you can easily spot them.

New for authors

Here is what's new for authors in SE 2020-01.

Cloud Connector

#primaryauthor

Cloud Connector is a Microsoft Office Add-in that allows you to link data between your Cloud Engagement product and MS Excel. With that linkage in place, you can create excel files for financial statements, schedules or calculations customized to the needs of your solution.

Cloud Connector provides realtime update of the linked data in the excel file. If any value or information changes in the engagement file, it's automatically updated in the excel file - guaranteeing efficient and effective information transfer.

Users can also use Cloud Connector in excel files to create the Statement of Cash Flows proof in audits.

As shown in the figure below, Cloud Connector displays in excel files as Datalink in the ribbon.

To use Cloud Connector, you need to:

  1. have a local installation of Microsoft Office 365 Desktop or online. Note that MS Office version 2019 or prior is not supported.

  2. create a MS Office 365 Desktop excel file and upload it in your template or engagement file.

  3. open the excel file from the Documents page in your template or engagement file.

  4. get the Datalink Add-in from the Microsoft Add-ins Store.

    5. The Datalink Add-in will be available in the section in the store.

  5. use the Datalink Add-in to insert data from your template or engagement file in the excel file.

To learn more about the functionality, watch the Cloud Connector demo clip from our SE 2020-01 webinar.

Components

#primaryauthor

A new product setting Components is introduced to support a more universal approach for visibility logic across the template content items. In this iteration of the Components setting, trial balance Financial groups are the only supported content items.

You can define Components that are relevant to your product or your market. For example, you can define components based on industry type, entity type or reporting framework. You can also set visibility logic for each component. For example, only display content relevant to a specific industry Agriculture if the entity operates within that industry.

In this release, a new group property Component is now available in the Group Properties dialog so you can associate one or more component with a trial balance group.

If you associate multiple components to a trial balance group and define visibility settings for each one of the components, then:

  • the trial balance group is hidden in the engagement if all the associated components are hidden due to visibility conditions.

  • the trial balance group is visible in the engagement if at least one of the associated components is visible due to visibility conditions.

Groupings

#primaryauthor

The Group ID field in the group properties takes up to 256 characters now to accommodate the need for longer IDs. For example, products with iXBRL enabled can have IDs up to 200 characters long.

Visibility conditions

#primaryauthor

A new option is now available in the template Information () | Export visibility conditions, so you can directly export visibility conditions to a CSV file.

New risk module

#primaryauthor

More customizations are available in Risk | Assessment options in the product settings. You can now customize the labels for inherent risks, control risks and risks of material misstatement for your product.

IFRS tables

#primaryauthor

You can now set dimension rows to include prior year data.

New for end users

Here is what's new for end users in SE 2020-01.

Accounts

Users can now assign new accounts to groups without having to navigate to a different page. The Account dialog now features three group drop-downs (Financial, Tax Code and Chart of Accounts).

Note that if the account is assigned to a group that's linked to another group, it's automatically assigned to the linked group as well. For example, if the account is assigned to Inventory in Financial grouping that's linked to Other inventories for sale in Chart of Accounts, the account is automatically assigned to Other inventories for sale in Chart of Accounts as well.

Adjustments

While posting adjustments, users can now create an account as they add the lines of the adjusting entry.

Queries

In this release, query documents have witnessed a few improvements to retain legacy behavior and improve scannability.

  • To efficiently view and scan questions in queries, a filter now displays in the left side navigation in the document once it's sent to contacts.

The figure below shows how staff can filter query questions by status.

The figure below shows how contacts can filter query questions by status.

  • Similar to the old query layout behavior, when users open a query document, its name now displays in the engagement's main navigation at the top.

New risk module

Various UI enhancements have been incorporated in this release to improve the visual appeal, interaction and overall experience. As for new features,

  • an export icon is now available in the Assessments tab so users can export risk assessments information to a PDF.

  • RMM values and risk information now flow to the Risks and Risk Assessment widgets in the Risk tab in checklists.
