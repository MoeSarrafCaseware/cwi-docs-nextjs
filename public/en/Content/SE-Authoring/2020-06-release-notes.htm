SE 2020-06 release notes

During this unprecedented situation, while adjusting to a new mode of operation, juggling work and family or general uncertainty, the Cloud Engagements team unyieldingly delivers.

SE 2020-06 is packed with features and enhancements; standing taller than its predecessor releases in this year.

Flipped sign indicator

#enduser #primaryauthor #firmauthor

When you change the default sign of an account in dynamic tables, an indicator () now displays in the row.

From end users perspective, balances for rows with flipped sign no longer display in italics to indicate that this row has an opposite sign. The new Flipped Sign indicator () displays in the row instead. To learn more, see Edit table rows in the financial statements.

Improved engagement search

#enduser #primaryauthor #firmauthor

A universal engagement search option is now available so you can:

  • Search visible documents in the engagement

  • Search the content within all visible documents in the engagement

Matching results display as you enter a keyword for search. Note that with the universal search option now in place, the search bar in the Documents pane is no longer available.

Note references in text sections

#enduser #primaryauthor #firmauthor

You can now reference note disclosures in text sections in financial statements. The toolbar in text sections now includes a Note Reference icon (). To learn more, see Add a note reference in a text section.

Sign off on adjustments and trial balance

#enduser #primaryauthor

You can now enable the Sign Off option in the Adjustments page and the Data page so users can sign off on the trial balance and adjusting entries.

Two new settings are now available in the Product Settings dialog so you can enable the signoff option.

  • Enable Adjustments Signoff - Select this check box to enable reviewer/preparer signoff in the Adjustments page in your product.

  • Enable Data Page Signoff - Select this check box to enable reviewer/preparer signoff in the Accounts tab and the group tabs in the Data page in your product.

To learn more, see Enable signoff for trial balance and adjustments.

Subfolders in the Documents page

#enduser #primaryauthor #firmauthor

Subfolders are now available on the Documents page so you can further group content for better presentation and accessibility. To learn more, see Finalize your draft.

Watermark in print settings

#enduser #primaryauthor #firmauthor

You can now add a watermark in the financial statements. The Apply watermark option is now available in the document print settings. Once you enable this option, you can add the content of the watermark in the associated text field.

To complement this feature, the Print () icon in the financial statements now displays two options:

  • Print Preview - Launches a print preview of the PDF output.

    • Note that the print settings () are also accessible in the print preview mode. You can make changes in settings and view them without having to close the print preview page and navigate back to the financial statements document.

    In the print preview page, you can also select the print icon () to print the document or the downward arrow () to download the PDF.

  • Download PDF - Initiates PDF output file download.

    • If the watermark option is enabled, the Download PDF option changes to Download PDF (WMK) to denote that watermark text will be display in the PDF.

New for authors

Here is what's new for authors in SE 2020-06.

Components

#primaryauthor

You can now group components. Component categories are now available in the Product Settings so you can further organize the components in your product for multiple industries, entity types or reporting frameworks.

Note that with component categories now available, you can no longer add standalone components. You must create one category at least for the +Component option to be available in the dialog. To learn more, see Define components.

To complement this feature, the Group Properties dialog now includes a new property Component Category. Once you assign one or more categories to a trial balance group, they are included in the dialog, so you can assign the applicable components.

Note: You can assign components from different categories to the same trial balance group.

To learn more, see Component Categories and Components.

Financial statements

#firmauthor

Firm authors can now view the print settings for the predefined areas and page breaks in financial statements, as well as make changes to it if needed.

Note: Predefined areas and page breaks are those that have been authored in the product template. To learn more, see Manage the firm template.

Data links (formulas)

#primaryauthor #firmauthor

A new setting Medium date is now available in the Product Settings dialog, so you can customize the medium date format for the Engagement Properties data links.

Queries

#firmauthor

A new option Show Author ID is now available in the firm template for predefined query questions.

Instead of previously, having to suppress and re-create existing query questions and responses, the Show Author ID option displays a dialog where firm authors can copy existing author IDs.

Note that:

  • Predefined query questions are questions that have been authored in the product template

  • Firms that do not have a firm template but still want to copy author IDs to integrate with Working Papers must create a draft template so they can copy the author IDs. Once they copy the IDs, they can select Delete Draft to delete the firm template they have just created. To learn more about the firm template workflow, see Manage the firm template

  • This feature is available in the old and new query layout

To learn more about integrating queries with Working Papers, see Integrate Working Papers with Queries.

To watch a video that describes how Queries-Working Papers integration works, See Add Author IDs.

New for end users

Here is what's new for end users in SE 2020-06.

Checklists

Query questions linked to checklist procedures have witnessed a couple of enhancements.

  • If the linked query question is hidden in the query document, its responses no longer display beneath the procedure in the checklist document.

  • Files uploaded by clients now display as links, so users can directly access them.

Data page

A new import option Engagement File is now available in the Import tab.

Note: You must enable the Engagement File import option in your product, otherwise it will not be available to users. To enable the Engagement File import option, add the "com.caseware.import.engagementfile" module to the "imports" array in the product.json file.

Once the option is enabled, users can import a trial balance from another engagement file. Note that:

  • Users can only select one engagement file to import from

  • To import from an engagement file that belongs to a different Cloud Engagements app, users must be granted access to that app and the entity that the file belongs to. To learn more, see Set staff access levels

  • Users cannot map the data between the source and destination engagements. For example, the Current Year data column from the source engagement is automatically imported to the Current Year column in the destination engagement

  • Once the data is imported, the year-end date for the destination engagement is automatically set to match the year-end date of the source engagement

  • Existing annotations and issues in the source data are not imported

  • Importing from a consolidation engagement imports the parent entity data only into the destination engagement

  • If the source and destination groupings don't match or partially match, unassigned accounts will populate in the Assign Accounts tab and users must manually assign them. To learn more, see Assign accounts to groups in the Data page

Financial statements

Eliminating adjustments are now included in the Account Details popup in consolidation engagements.

Queries

SE 2020-06 brings in carry forward support in the new query layout and more security measures for file exchange.

  • Contacts can no longer upload suspicious files in File Uploads responses (such as .com, .bat, .exe, .cmd, .sh, .dll, .so, .o, .deb, .msi, .rpm, .ko, .sys, .php).

  • Carry forward is now available for the queries new layout.

XBRL review

Note: This feature is only available for distributor testing only. It will not be deployed to partner or production environments.

Text sections are now included in XBRL review in financial statements. Once users start the review process, a dashed green border marks text sections and the fact details display in the XBRL REVIEW pane so they can review and validate the data.

Note that data links (fx) are underlined in green. If users select a data link, its fact details display in the XBRL REVIEW pane.

XBRL review has also witnessed functionality enhancements related to:

  • Navigating to other documents after users start the XBRL review

    If users navigate to a document that is not part of the selected taxonomy, the XBRL review processes automatically stops. Note that if users restart the review process, they pick up where they left off. That is, the last document they have been reviewing opens with the last selected fact s displayed in the XBRL REVIEW pane.

  • Changing the selected taxonomy after users start the XBRL review

    If users select a different taxonomy, the first document in that taxonomy opens. The first fact configured in that document displays by default in the XBRL REVIEW pane.

  • Disabling actions in the document users are currently reviewing

    Once users start the XBRL review process, the only action they can perform in the document is to create or edit issues (). All other actions are disabled.

Fixes in SE 2020-06

The following listing features a brief description of bugs that have been resolved in SE 2020-06.

Checklists

The following bugs have been resolved.

  • Adding data links (fx) in a table in the procedure description field does not function as expected.

  • Visibility icons are missing for procedure and procedure groups.

Financial statements

The following bugs have been resolved.

  • Dynamic tables do not display account balances that are less than 1.

  • When there is a large number of excluded content in the document, selecting the Add icon () does not display the full list of excluded content.

  • Account details popup in dynamic tables displays incorrect information.

  • Text areas that include tables in financial statements notes always display left-aligned in the PDF print of the document.

  • Print settings for margins in areas are overridden by the document print settings for margins.

  • Alignment issues for negative numbers that are set to display between parenthesis.

  • A styling issue in analysis sections when there is no data to display in a figure.

  • Attempting to print the document with an analysis section times out if users have not selected a graph type.

New risk module

The following bugs have been resolved.

  • Areas added to newly created risks do not display in the Assessment tab unless users refresh the page.

  • Styling issues.

Other fixes

Here is a brief description of general bugs that have been fixed in this release.

  • Firm templates failed to update with the latest product template version.

  • Attempts to create engagements for certain apps fail as the page just keeps loading.

  • Attempting to import trial balance through online accounting platforms does not work as expected.

  • The Save button is disabled when users attempt to change the current year balance for negative values on the Data page.

  • Links to internal pages (such as Transactions Import) added by users in Issues are broken.

  • Incorrect review count is displayed for Not signed off and Optimised out review tools.

  • Attempting to carry forward a consolidated engagement results in an error.

  • Questions set to Hide in the old query layout continue to display when users disable Hidden Items in the document.

  • UI translation issues.

  • Technical issues related to language settings in XBRL.

  • The review marks (green dotted lines) continue to display after users stop the XBRL review process.

  • The Organization type data link (fx) returns incorrect data when custom organization types are defined in the product.
