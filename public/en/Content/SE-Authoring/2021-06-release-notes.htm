SE 2021-06 release notes

Grab a glass of lemonade and settle in for the summer with SE 2021-06! This release introduces exciting new features like the risk library, multiple signoffs, review tools for the new risk module and more.

Multiple signoffs for a role

#enduser#primaryauthor#firmauthor

The Multiple Signoff option is now available for custom signoff roles.

When the Multiple Signoff option for a role is enabled, more than one user can sign off on the role for a document. Signoffs are displayed in chronological order (the most recent signoff displays at the bottom).

To learn more, see Define signoff roles.

Hide sub-level headings for notes in financial statements

#enduser#primaryauthor#firmauthor

To enhance presentation in the financial statements, the Notes Print Settings now include an option to automatically hide sub-level headings for notes in the PDF output if there are no subsequent notes in that group.

For example, if note 1 includes a subnote 1.a and there is no 1.b, 1.c notes, etc., then the note heading of subnote 1.a does not display in the PDF output. Only the content of that note displays.

The option to hide sub-level headings in the PDF output is also available for individual notes in the More actions menu if there are no subsequent notes in that group.

To learn more, see Set up the print options in the financial statements and Hide sub-level headings for notes.

Risk library

#primaryauthor

To support a more efficient workflow for audit engagements, the risk library is now available. This library allows you to create a predefined list of risks. Users can then select the applicable and relevant risks from the library and add them to the engagement.

Note: The risk library is only available for distributors for beta testing. It will not be deployed to partner or production environments. To enable the risk library for your beta site, post a DIST ticket with your request.

Once the risk library is enabled for your site, the Risk Library page is available in your firm settings. To create a risk library, upload an Excel file with the content of your library. You can add multiple risk libraries to accommodate different risk documentation requirements for different products.

When you’re ready, you can publish the library as the major version, then link that version to a product from the Risk | General tab in the product settings.

Note that:

  • The new risks module must be enabled in the product that you want to link a risk library to.

  • You can only link a product to one risk library at a time.

  • You can link the published risk library version to multiple products.

If you need to make changes to a published risk library, you can select to add a new excel file and choose whether it’s a Minor or Patch version of the library. You can add minor versions to use them for planned updates to the library, while patch versions can be added for urgent updates.

Once you publish a risk library version and link it to a product, users can add predefined risks from the library instead of creating them manually.

When users select the Risk from Library option in the Add dialog, they are directed to the risk library where they can then use the filters and search option in the library to find the appropriate risks.

The following filters are available:

  • Type - Filters the displayed risks based on risk type

  • Status - Filters the displayed risks based on whether they have been added to the engagement

  • Direction - Filters the displayed risks based on direction. This filter is only available for assertion level risks.

  • Assertions - Filters the displayed risks based on the associated assertions. This filter is only available for assertion level risks.

In the left panel, users can also select whether to view the displayed risks by Indicators or Categories.

To learn more, see:

  • Create a risk library
  • Add a risk library version
  • Publish a risk library version
  • Associate a risk library to a product

XBRL tagging

#primaryauthor

XBRL tagging capabilities have been expanded to allow authors and end users to add tags for XBRL facts directly in the template or engagement file.

Note: This feature is only available for distributors for beta testing. It will not be deployed to partner or production environments. To view this feature, open a template or an engagement file and make the following change to your URL: https://<server name of your partner site>/<firm name where you perform your authoring>/e/eng/<identifier>/index.jsp?enable-features=customXbrlTagging

When XBRL review mode is enabled, untagged items are highlighted in yellow. You can select the Add XBRL Tag option for a specific item to create a tag for it.

You can then select the Element and Context information for the tag from the available options in the review panel.

Paragraph spacing in financial statements

#primaryauthor

The Use consistent paragraph spacing between content areas setting is now available to allow primary authors to enforce a consistent paragraph spacing between content areas in the printed financial statements.

To learn more, see Customize the financial statement settings.

Grouping structure import

#primaryauthor

The order of the financial groups in a grouping structure is now maintained when you export then import a grouping structure.

New for end users

Here is what's new for end users in SE 2021-06.

Signature log for Simple Electronic Signature

Users can now view and download a signature log in the PDF viewer for documents that were electronically signed by clients in query documents.

To learn more, see Review contact responses and Attach documents to an engagement file.

Review tools for risks

Users can now use review tools to track changes made to risks in the new risks module after reviewer signoff.

If you’ve enabled document level signoff for the risks module, the following review tools are available for users:

  • Added - Marks risks that have been added by staff members.

  • Deleted - Marks risks that have been deleted by staff members.

  • Modified - Marks risks that have been modified by staff members.

If you’ve enabled individual risk signoff, only the Modified review tool is available. Note that when individual risk signoff is enabled, users cannot delete a risk once the reviewer has signed off on it.

Enhanced balance discrepancy check

Balance discrepancy checks in dynamic tables now compare the absolute values for the labeled rows and do not take into account whether the value is negative or positive. This ensures that when balances for credit accounts are compared against debit accounts, the difference in sign between the two accounts is not flagged as a balance discrepancy.
