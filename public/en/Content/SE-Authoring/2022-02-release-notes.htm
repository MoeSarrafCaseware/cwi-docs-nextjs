SE 2022-02 release notes

An exciting new year of SE releases has begun! This release features exciting new changes to risks for both end users and authors, more enhancements for XBRL tagging and more.

Create areas

#enduser #firmauthor

Firm authors and end users now have the ability to create audit areas and assign them to financial groups. They can also change the default audit area assigned to existing financial groups.

The option to create areas is available for firm authors and end users from both the New Risk dialog and the risk assessment report.

When users select the NEW AREA option, the product settings opens to the Areas tab. Users can then add a new area or modify or remove any areas they have already created. Note that the areas added by the primary author cannot be removed or modified.

As part of this enhancement, end users and firm authors can also now assign areas to financial groups in the Data page.

Note that users and firm authors can assign an area to a group even if it is not the same area assigned to the parent group. This allows users to more easily re-assign financial groups to areas when they have divided existing areas into sub-areas.

To learn more, see Add areas and Assign financial groups to areas.

Assign checklists to multiple areas or cycles

#enduser #primaryauthor #firmauthor

You can now assign checklists to multiple areas or cycles from the Checklist Settings.

To learn more, see Review and modify the checklist default settings.

Enhancements to signoffs for risks

#enduser #primaryauthor #firmauthor

To enhance the signoff experience for risks, the following changes have been made:

  • You can now enable signoff for the risk assessment report. Similarly to the risk report, you can select whether end users must sign off the entire report or individually sign off each risk in the report.

  • You can now apply custom signoff schemes to the risk and risk assessment reports. This option is also available to both firm authors and end users.

To learn more, see Enable the new risk module and Review and customize signoff schemes.

UI enhancements for risks

#enduser #primaryauthor #firmauthor

To accompany the enhancements made to risks in this release and improve UI consistency, the following changes have been made:

  • The icons used for risks have been updated.

  • The styling of the risk and risks assessment widgets have been updated to have consistent styling with the risk and risk assessment reports.

  • The risk assessment report now includes a floating header to make it easier to see which assertion corresponds with each rating when users scroll down the page.

  • The assertions filter in the risk report now displays the full name of each assertion.

Document likelihood and magnitude

#enduser #primaryauthor

You can now select whether to enable Likelihood and Magnitude in the risk assessment report. This option allows auditors to document the likelihood and magnitude for risks at the Area or Cycle level to aid their determination of the risk of material misstatement.

If you choose to enable the Likelihood and Magnitude option, you can then set custom labels for Likelihood and Magnitude and customize the scales used for these options.

Once you enable this option, the end user can then define likelihood and magnitude in the risk assessment report.

To learn more, see Enable the new risk module.

Controls testing option for risks

#enduser #primaryauthor

There is now an option in the product settings to enable the Will you be testing controls option for end users.

When this option is enabled, users can select whether they are testing controls for each area or cycle in the risk assessment report.

If users answer that they are not testing controls, the Control risk for each assertion will be automatically set to the highest level risk. Note that this will not overwrite any manual selections made previously.

To learn more, see Enable the new risk module.

Display currency symbol in dynamic tables

#enduser #primaryauthor #firmauthor

To enhance the presentation of the financial statements, you can now set default display options for currency symbols in dynamic tables in the Currency Symbol settings.

You can set the currency symbol to display in table column headers.

The OVERRIDE TABLE CUSTOMIZATION checkbox allows you to override any existing currency symbol settings for individual dynamic tables and apply the default settings instead.

To learn more, see Set default currency symbol options for dynamic tables and Set default currency symbol options for dynamic tables (end user).

Overline and underline description columns

#enduser #primaryauthor #firmauthor

The Table Border settings now includes an option to select whether the underline/overline options selected for dynamic tables apply to description columns.

To learn more, see Set default underline and overline options for dynamic tables and Set default underline and overline options for dynamic tables (end user).

Set default values for engagement glossary terms

#enduser #primaryauthor #firmauthor

You can now set default values for engagement glossary terms to display when no conditions are met.

If the override option is enabled, firm authors and end users will be able to modify the default values.

To learn more, see Define engagement glossary terms.

New for end users

Here is what's new for end users in SE 2022-02.

Duplicate & Unlink option for Cloud Connector

Cloud Connector now has a new Duplicate & Unlink option to allow users to create a copy of their spreadsheet without any formulas.

Note: The availability of this feature is pending Microsoft add-in review.

This allows users to save a snapshot of their spreadsheet where data will not be updated in real-time so they can backup information or send it to their clients.

Disable assertions for areas and cycles

Users can now use the checkbox for assertions to disable assertions for each area and cycle. This allows users to only enable assertions that are relevant for the area or cycle.

Make hidden procedures and documents visible when linking

When users link a risk to a hidden procedure or document, a dialog now displays to inform the user that if they choose to continue, the hidden procedure or document will now become visible.

Import zero balance accounts

Users can now select to include zero balance accounts when importing data from CSV or Excel files.

To learn more. see Import the trial balance from a CSV or Excel file.

Display inconsistent name by default for financial groups on assign accounts page

The Assign Accounts page now displays the inconsistent name by default for financial groups that have either no accounts assigned to them or a zero balance. If there is no inconsistent name for a financial group, the consistent name is displayed by default instead.

Copy and paste documents across engagements

The Copy from Caseware Cloud option now allows users to copy documents from other engagements within the same entity. Users can select which engagement they want to copy documents from and then select the specific document.

To learn more, see Attach documents to an engagement file.

New for authors

Here is what's new for authors in SE 2022-02.

Enhancements to XBRL tagging

#primaryauthor

To continue our expansion of XBRL tagging capabilities, several improvements have been made for both authors and end users.

Note: This feature is only available for distributors for beta testing. It will not be deployed to partner or production environments. To view this feature, open a template or an engagement file and make the following change to your URL: https://<server name of your partner site>/<firm name where you perform your authoring>/e/eng/<identifier>/index.jsp?enable-features=customXbrlTagging

Authors and end users can now search for roles and labels in the XBRL taxonomy when tagging directly from a template or an engagement file.

When you select an item in the taxonomy, information about the item, including the label, description and properties, will be displayed in the right panel of the search dialog.

Additionally, end users and firm authors can now delete tags that they have added. Note that:

  • They cannot delete tags added by primary authors.

  • Firm authors can only delete tags added by firm authors.

  • End users can only delete tags added by end users.

Assign components to content

#primaryauthor

To expand the functionality of components, you can now assign components to:

  • Documents in the new Documents page

  • Sections in financial statements, memos, letters and worksheets

  • Procedures in checklists

  • Query questions

To assign a component to an item, you can select Assign component from the More actions () menu.

End users can view which components are assigned to items and their visibility status in the visibility popover for documents.

To learn more, see:

  • Assign components to a checklist

  • Assign components to a query (new design)

  • Assign components to a letter or memo

  • Assign components to the financial statements
