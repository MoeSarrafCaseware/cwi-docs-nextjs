Enterprise authoring

#enterpriseauthor

SE 2024-09 features the official launch of the enterprise author role.

The enterprise author role enables accountancy enterprises and associations, as well as large firms with multiple offices, to customize product templates provided by primary authors and share those templates with all their member firms. This allows accountancy networks to easily maintain a standard methodology across their member firms, while still giving member firms the flexibility to make further customizations as required for their region.

To manage the enterprise firm template in a central location, accountancy enterprises will have a Cloud site specifically for enterprise authoring in addition to Cloud sites for each of the member firms in the enterprise. Users from the member firms that the enterprise wants to designate as enterprise authors can be added to the enterprise authoring firm site.

In the enterprise authoring firm, enterprise authors can access the template provided by primary authors on the Manage template page in their settings. From this page, they can:

  • Create a draft and make customizations to the template

  • Save the template so they can test it

  • Publish the template and make it available to all the member firms in their enterprise

  • Restore previous versions of the template

Note: Any customizations made by enterprise authors cannot be overridden by new updates made to the product template by primary authors.

Once their work is complete, enterprise authors can then share this template with their member firms, where firm authors can make additional customizations. Note that member firms will always receive their templates from the enterprise authoring firm and not directly from primary authors.

Additional authoring capabilities

In addition to the features already available to firm authors, enterprise authors also have access to customize the following features:

  • Risks
    Enterprise authors have the ability to customize all items on the Assessment, Fields, and Scales tabs of the Risks section in the product settings. Note that while enterprise authors also have access to the General tab, the option to enable the new Risks page and the Library management section is still only available to primary authors.

  • Controls
    If controls are enabled by primary authors, enterprise authors can customize all controls settings. Note that they do not have the ability to disable controls.

  • Assertions
    All items in the Assertions tab of the product settings are available to enterprise authors.

  • Documents page
    Enterprise authors can modify the names, identifiers and visibility settings of primary author-created documents, as well as reorder primary author-created folders and subfolders. They can also add, rename and reorder phases from the product settings dialog.

  • Checklists
    Enterprise authors can modify visibility settings for items within a checklist and can also add annotations for procedures.

Translation for the financial statements

#primaryauthor#enduser

You can now enable automatic machine translation for the financial statements. When you make this feature available in your product, users can translate text sections in the financial statements to any of the content languages available in your product.

To enable this feature, select the Enable translation checkbox in the Financial Statements tab of the product settings and ensure you have more than one content language enabled in your template.

Users can then select the Translation option in the financial statements settings to enable translation for the document. The Translation icon then appears next to text sections in the document.

When users select the translation icon, the Translation dialog opens. From this dialog, they can select which language to translate the content to, view the translation, and select Save to apply it to the text section.

The translated version of the text section displays in the engagement when the user switches the content language of the engagement to the target language. They can then make manual edits to the text if required.

Custom calculations in financial statement text sections

#primaryauthor#firmauthor#networkauthor#enduser

You can now add custom calculations to text sections in the financial statements. Similarly to custom calculations in dynamic tables, you can reference trial balance values and dynamic table cells in your calculations. These calculations will update automatically if the referenced values change.

Cell references within the financial statements

#primaryauthor#firmauthor#networkauthor#enduser

This release introduces cell referencing within the financial statements. You can now reference values from one dynamic table into another. This eliminates the need to manually enter the same value multiple times in different tables in the financial statements.

Custom formulas for the Overall Materiality Table

#primaryauthor#firmauthor#networkauthor#enduser

You can now use custom formulas in the Overall Materiality Table.

Materiality and performance materiality for specific circumstances for areas

#primaryauthor#firmauthor#networkauthor#enduser

In the materiality document, you can now document materiality and performance materiality for specific circumstances for areas.

Authoring features

Here is what's new for authors in SE 2024-09.

Set the ability to add groups from the product settings

#firmauthor#networkauthor

Enterprise authors and firm authors can now determine if end users can add financial groups in engagements. While primary authors still retain the ability to define if this feature is available to end users in the product.json file, enterprise authors and firm authors can now override the primary author’s settings to customize their template.

To determine whether end users can add financial groups, authors can modify the Enable user-defined financial groups checkbox on the Data tab of the product settings.

Note that the Enable user-defined financial groups checkbox is also available to primary authors, so you can control whether end users can add financial groups without having to edit the product.json file.

End-user features

Here is what's new for end users in SE 2024-09.

Version control for the financial statements

Users can now save and compare previous versions of the financial statements. This allows users to keep better track of what changes are made to the document and by whom.

To save a new version, they can select the Version control icon in the Document Map. They can then select New in the Version history panel to save a new version with a title and a description.

To compare the current version of the financial statements with a previous version, they can use the checkboxes to select the previous version and then select Compare. The financial statements then display a comparison highlighting what content was added and deleted since the previous version.

Note that changes made to the following items are not tracked:

  • SE Builder tables

  • Analysis charts

  • Embedded images

  • Embedded PDFs

  • Document settings

Users can also view and delete previous versions from the More actions menu for the version.

Issues for dynamic table rows

Users can now add issues to dynamic table rows. The Issues icon displays when users hover the cursor over the right side of a dynamic table row. They can then select the icon to add an issue.

Note that:

  • Issues added to a child row are displayed at the parent level when the group of rows is collapsed. When the row is expanded, the issues display at the row they were added to.

  • If a row should be hidden due to visibility logic, it will continue to display if an issue is attached to it and it is either outstanding or resolved. The row will be hidden if the issue is cleared. Note that whether an issue is attached to a hidden row or not, that row will not be included when printing the document.

  • If an issue is added to a row and the row is later deleted, the issue is moved to the dynamic table at the table level.

Note that issues cannot be added to page break rows.

Improvement to syncing trial balance

A warning message now displays to users when they attempt to sync their trial balance from another engagement. After they select the Confirm button, this new message displays and informs them that syncing the trial balance is an irreversible action.

Filter adjustments

A new All types dropdown is now available on the Adjustments page to allow users to filter adjustments by type. When users select to export their adjustments, only the filtered adjustments are included in the generated file. This allows users to select only the types of adjustments they need and provide a copy of them to their clients.

Saved views for the Data page

The Save views icon () is now available on the Data page to allow users to save their current view and access previously saved views.

When users save a view, their selections for the following settings are saved:

  • Columns

  • Indicators

  • Filters

  • Variance

  • Split Adjustments

  • Rows per page

Saved views are available to all users in the engagement file.

Improvement to risk scales

#primaryauthor#networkauthor

You can now set seven levels for risk scales instead of the five that were previously available.

  • [SE-47751] - A technical issue that affects the ability to open engagements.

  • [SE-47639] - An error occurs when primary authors attempt to publish a firm template for testing.
