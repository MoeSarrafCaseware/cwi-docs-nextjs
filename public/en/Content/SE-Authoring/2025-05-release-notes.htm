New in SE 2025-05

Welcome to 2025-05! This release introduces improvements to the carry forward feature in financial statements, multi-language support for materiality and summary of misstatement documents, enhancements to Risk functionality, workflow improvements to Documents and Data pages and more.

On this page you can view all the new features included in this release that apply to both authors and end users.

  • Authoring features

  • End-user features

     

Authoring features

Here is what's new for authors in SE 2025-05.

 

Name and categorize inherent risk factors

#primaryauthor#networkauthor

As a primary or enterprise author, you can enable inherent risk factors from your Product settings. This allows you to name and categorize inherent risk factors for selection by your users from the Risk card and/or the Risk dialog box.

The following are category option for inherent risk factors:

  • Complexity

  • Subjectivity

  • Change

  • Uncertainty

  • Susceptibility of misstatement due to management bias

  • Susceptibility of misstatement due to fraud risk factors

  • Other

Category names can be customized. You can also create and customize subcategories under each category. Category options can be selected or deselected as options for your users.

This feature is available only to new engagements.

Risk Assessment per Risk

#primaryauthor#networkauthor

Users can now perform a risk assessment on an individual risk. This includes the same level of assessment that users would apply to the assessment of risk for the entire engagement. The individual assessment returns values beside all the tracking metrics.

This feature supports the user when a greater level of documentation is required to meet the user’s company or industry standards. This feature provides a place to record individual risk assessment levels.

As a primaryor enterprise author, you will need to enable this feature from the Product settings.

This feature is available only to new engagements.

Multilingual capabilities in Materiality and Summary of Misstatements documents

#primaryauthor#firmauthor#networkauthor

The next two feature enhancements are part of a larger initiative to expand multilingual support in Caseware apps.

These enhancements are available only to new engagements.

Multilingual capabilities for Materiality document

As a firm, enterprise or primary author, you now have multilingual capabilities in editable fields in the Materiality form. You can select a default language for your engagements and have the additional flexibility to change the language within editable fields.

Editable fields that include multilingual capabilities include:

  • Document and Sections Guidance

  • Sections Titles

  • Tables Headings

  • Overall Materiality - Benchmarks Input fields in Tables (Custom formulas / Other)

  • Performance Materiality & Trivial - % description

  • Conclusion procedures (including conclusion for Group component allocation)

Multilingual capabilities for Summary of misstatements

Similar to the Materiality form, authors can now access multilingual capabilities in the Summary of misstatements document, specifically the Guidance and Key Data (input) fields.

To learn more about multilingual capabilities in your Caseware apps, refer to What is multi-language support.

 

End-user features

Here is what's new for end users in SE 2025-05.

 

Add selected financial groups, areas and cycles to account tables within the Analysis tab

#enduser

You can now include custom financial groups, areas and/or cycles for analysis within Documents checklists that feature an Analysis tab. You can modify the checklist settings to include custom financial groups, areas and/or cycles using the radio button for selection. You can further specify the selection by clicking the drop-down menu under that category, as illustrated in the below diagram.

This feature is available for new and existing engagements.

Note: this option is not available for either Analysis Documents or for Analysis Charts embedded into the financial statements.

To learn more about analysis in checklists, refer to View account analysis in checklists

Note: the Analysis tab must be enabled in a checklist document to use this feature. The Analysis tab allows you to view various analysis of your client's accounts to help keep track of the steps required for your work program.

For more information on how to enable the Analysistab in a checklist, see Review and modify the checklist default settings.

Filter the documents page by document type

#enduser

Users can now filter the Documents page by document type as well as other specialized search criteria. Being able to filter by document type drastically cuts down the size of the documents page and makes documents easier to find.

Document types include:

  • Forms

  • Checklists

  • Financial Statements

  • Worksheets

  • Queries

  • Letters/memos

  • External files such as

    • PDF

    • Word

    • Excel/CSV

Other filter types include:

  • unsigned reviewer (at least 1 still unsigned reviewer signoff role)

  • unsigned preparer (at least 1 still unsigned preparer signoff)

  • documents with a specific signoff role (signed or not)

This feature is available for new and existing engagements.

Allow end users to assign cycles to financial groups

#enduser

Users can now assign cycles to financial groups from the Data page in their engagements. The ability to assign and see the cycle of a financial group allows users to address why certain groups are showing up in different lead sheets.

This feature is available for new and existing engagements.

To learn more about this feature, refer to Assign financial groups to areas and cycles.

Carry forward improvements to financial statements

#enduser

Users will notice several improvements to the carry forward feature for financial statements. Carry forward is a time-saving utility that allows you to reuse information from a prior year engagement in a current year engagement. Settings, such as currency symbol types, can be carried forward either in bulk or your users can select which changes to carry forward into the current engagement.

This feature is available for new and existing engagements.

All options in the following Document Settings can now be carried forward:

  • Currency symbol

  • Formatting

  • Numbered heading

Note: Each Area has the ability to override the Document Settings for the Area specific numbered heading options. Area setting changes do not carry forward.

To learn more about carry forward, refer to Carry forward an engagement.
