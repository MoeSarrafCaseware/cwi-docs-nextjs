Add Notes and Area titles and headings as reference links to financial statements

#primaryauthor#firmauthor#networkauthor#enduser

Your users can now add reference links to Notes and Area titles and headings in different areas within your financial statement documentation. Reference links can be added to both text areas and dynamic tables. This feature is useful in complex or large reports. Improvements to referencing includes:

  • The addition of Area titles and headings that can be added as references within your user’s document.

  • An added search field within the dialog box for selecting a reference.

  • Expandable and collapsible dropdown lists that can include all subnotes or headings.

This feature is available on both new and existing engagements.

Select and subtotal multiple balances from the Group properties user interface

#primaryauthor

Users can now select and subtotal multiple balances from the balance source in a financial group’s property settings. This feature is useful for creating new groups from existing subgroups and increasing the flexibility in how users view financial data.

End-user features

Automatically carry forward Dynamic Table Border settings

#enduser

Your users can now automatically carry forward Dynamic Table Border settings. Carry forward is a time-saving utility that allows you to reuse information from a prior year engagement in a current year engagement. Settings, such as currency symbol types, can be carried forward either in bulk or your users can select which changes to carry forward into the current engagement.

The Dynamic Table Border settings that carry forward include:

  • Header Rows

  • Total Rows

  • Final Rows

  • Description Column

These settings will be carried forward automatically. The user will not need to accept the changes.

This feature is available for new and existing engagements.

Note: The user can override the Document Settings for the Area specific numbered heading options. Area setting changes do not carry forward.

To learn more about carry forward, refer to Carry forward an engagement.

Risks and Controls copied across engagements

#primaryauthor#firmauthor#networkauthor#enduser

Note: This feature must be enabled in your Product settings.

Users can now copy risks and controls from an engagement, or a component within a group audit, into other engagements within your Caseware application. Enabling this feature will add an icon Copy from external engagement item [] at the top-right corner of your Caseware applications that will allow users to copy data from another engagement into the current engagement. This feature supports consistency and reduces manual effort when preparing a group audit or an engagement.

This feature is available only in new engagements.

After selecting the icon for Copy from external engagement item, a dialog box will open that will allow the user to select the engagement that contains the desired copy. From here, users can

  • Copy risks and controls from more than one engagement.

  • Copy risks and controls from engagements within the same entity

  • Copy risks and controls from engagements from all entities.

  • Copy risks and controls from engagements from all your Caseware applications

  • Copy risks and controls from a component within an engagement.

Once the user confirms a selected engagement, another dialog box will open to allow the user to copy from that engagement’s Risks and Controls. Relationships between items will also be copied over. For example, from within the source engagement, if Risk 1 has a relationship with a Control 2 that will copy over into the current engagement. The user of the current engagement can adjust the data and relationships as needed.

Notes about this feature:

  • Copying risks and controls will not link the source and destination of the data; both can be edited independently.

  • Copying will not override disabled features set within the destination engagement.

 

Checklists copied across engagements as PDFs

In addition to copying Risks and Controls from one engagement to another, users can now copy checklists, as PDFs, in a similar fashion. Checklists that are copied across other engagements are converted into PDFs and can be used as references for populating documents within a current engagement.

This feature is available only in new engagements.

To learn more, refer to Copy risks, controls or checklists from other engagements.

Final rounded balances for Current Year and Prior Year

#enduser

Note: This feature must be enabled in your Product settings.

Your users can now see the Current Year’s final rounded balance and the Prior Year’s final rounded balance as two new columns in the Accounts table under the Data page. These numbers will match what is shown in the Financial Statement. This feature ensures alignment between your data and financial statements.

The rounding conditions (for example, round by thousands vs millions) can be set from the Engagement Settings within the Financial Statements.

This feature is available only in new engagements.

To learn more about rounding numbers, refer to View the client’s trial balance data.
