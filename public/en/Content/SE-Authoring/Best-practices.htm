Best practices

The way you author your financial statements impacts the performance of the document, such as the time to load and display a selected area. It's important that you plan how you will add and set up content.

For the best experience while editing and working with the financial statements, consider the following tips as you set up content.

  • An area with less visibility logic loads and functions faster than an area with many visibility rules.

    Instead of having multiple content items where the same visibility rules are set for each item, group the items in one area and set the visibility logic on that area.

  • An area with less content items loads and functions faster than an area with many items.

    Instead of having too many content items in one area, consider splitting the area to multiple areas and grouping items that share a similar context. For example, items having the same visibility logic.

Example

Content setup - version 1:

When you set up content using the following method, the area loads slowly and functions poorly.

  • Area 1

    • Content 1 (visible when checklist response is review engagement)

    • Content 3 (visible when checklist response is review engagement)

    • Content 4 (always visible)

    • Content 5 (visible when checklist response is compilation engagement)

    • Content 6 (visible when checklist response is compilation engagement)

    • Content 7 (visible when checklist response is compilation engagement)

    • Content 2 (visible when checklist response is review engagement)

Content setup - version 2:

When you set up content using the following method, each area loads much faster and functions better.

  • Area 1 (visible when checklist response is review engagement)

  • Area 2 (always visible)

    • Content 4 (always visible)

  • Area 3 (visible when checklist response is compilation engagement)

    • Content 5 (always visible)
