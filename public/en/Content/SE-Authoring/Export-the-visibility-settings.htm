Export the visibility settings

You can export a .csv file of the visibility conditions you have defined in your solution. This allows you to review the visibility conditions before a release or to verify conditions after updates.

Note: MS Excel CSV format doesn't support Unicode, so certain characters and languages may display incorrectly.

To export visibility conditions:

  1. Go to your authoring site and open the template.

  2. Select Information () | Export visibility conditions.

    3. Download starts automatically.

  3. Open the .csv file using MS Excel.

The following information is available in the file:

Column

Description

Type

The document type (for example, Checklist, Letter, etc.)

Document

The document name (for example, 10 Optimizer)

Level

If it's a top level document then this column is the document identifier and document name

Otherwise, it's the section type or procedure level (for example, Group, Procedure, Text area, Guidance, etc.)

Identifier

If top level document, this column contains the Document name

Otherwise, this column contains the first couple of words of the level to help locate the content that has the conditions

Show/Hide
  • No visibility settings

    • There are no specific conditions on this content. It may inherit from the above level.

  • Normally invisible

    Content is hidden unless a condition is met.

  • Normally visible

    Content is visible unless a condition is met.

Condition Requirement
  • All conditions required

    All conditions must be met for the content to be shown or hidden.

  • Any conditions required

    Any conditions must be met for the content to be shown or hidden.

Condition Override
  • Default show/hide

    The visibility of the content follows the visibility conditions settings (if available).

  • Always hide

    Force the content to be hidden.

  • Always show

    Force the content to show.

Condition Count

The number of available visibility conditions.

Note: Group conditions are counted as a single condition.

Condition Details

The details of the available visibility conditions.

Note: Condition Details is an arbitrary number of columns, where each column contains a single property of a specific condition.
Error codes

Displays Response conditions error codes, where:

  • Deleted- If the procedure in the condition has been deleted

  • No Answer - If the response in the condition does not exist

  • Empty - If the value of the response is no longer available
