Get started with internal XBRL tools

This guide is intended for Developers and Product Managers. It covers three internal tools:

  • The Tagging Merge Tool, which allows you to combine two separate XBRL data files and verify the changes between them.

  • The Taxonomy Checker Tool, which allows you to validate XBRL data against a selected taxonomy. It generates an error report detailing any issues found in the XBRL data. The tool checks both the structure of the data and whether the tags, dimensions and namespaces are relevant to the selected taxonomy.

  • The Taxonomy Namespace Tool, which allows you to generate the namespace map in your XBRL data. This map cannot be exported from an engagement and must be created using this tool. The namespace map is required in your XBRL product script so the system knows which namespaces to use. You only need to use this tool once per product to generate the relevant map. The map can then be merged into your files each time you use the Tagging Merge Tool.

The main section of this guide describes the process of merging tags. Custom Tags are added to an engagement using the Platform Tagging Tool. The Tagging Merge Tool then extracts these Custom Tags and incorporates them into the product template. Once added, these product tags are referred to as System Tags.

Workflow

The following diagram shows the expected workflow within your product, including the tasks required from both Developers and Product Managers.

Note: If your Solutions team is new to Cloud XBRL, please use the UK product script. It helps you stay up to date with new features, reduces development effort and enables your Product Manager to use the Platform Tagging Tool to generate System Tags. Using the UK product script also allows you to directly import the JSON files into your product. If you do not use the script, you will need to create your own solution manually.

Product Manager tasks

Platform Tagging Tool

  • Use the Platform Tagging Tool to generate XBRL data

    Use the Platform Tagging Tool to generate the Custom Tags, numerics and contexts that you want to include as System Tags, numerics and contexts.

  • Export the XBRL data

    Export the tagging data from the engagement you have applied tags to. For instructions, see How do I export XBRL data?

  • Save the exported file

    Save the exported file in a secure location. Be sure to store it in a way that clearly indicates which product it belongs to and the date the data was exported.

Internal XBRL tools

  • Do you have an existing master file?

    This will only be "no" the first time you export data from the product. After that, you will have the latest master file saved.

  • Use the Tagging Merge Tool to merge with the existing master file

    Use the Tagging Merge Tool to combine the new exported file with the existing master file for that product.

  • Save the new file as the latest master file

    Save the file in your designated document area. Keep previous master files for version control and as a backup in case anything goes wrong.

  • Does the file contain the namespace map?

    This step only applies if you haven’t yet used the Taxonomy Namespace Tool for the provided data. Once this is done, you can continue importing data via the Tagging Merge Tool using the Namespace panel.

  • Use the Taxonomy Namespace Tool to add namespaces

    Use the Taxonomy Namespace Tool to create the namespace map. You’ll need the exported file that contains both your existing data and the new namespace map.

  • Do you want to validate the taxonomy data?

    Consider validating the taxonomy data to ensure it's relevant to the specified taxonomy. Perform this step whenever it suits your team. This could be after each update or once tagging is complete.

  • Validate taxonomy data using the Taxonomy Checker Tool

    Use the Taxonomy Checker Tool to verify whether the taxonomy data is relevant for the taxonomy.

  • Get the error log

    Retrieve the error log from the Taxonomy Checker Tool and make the necessary adjustments.

  • Do you have errors to fix in the Platform Tagging Tool?

    If errors are found that can be fixed, return to the beginning of the Platform Tagging Tool workflow and apply the necessary corrections.

Developer tasks

  • Provide the latest master JSON files to Product Developers

  • Do you want to make manual changes to the file?

    As a Developer, you can manually edit the JSON file if the required changes cannot be made through the standard process.

  • Make manual changes (if necessary)

    If you do make manual changes, ensure that you update the master JSON file maintained by your Product Manager so that it remains current.

Product Script

  • Update JSON files in the product script

    Update the product script to ensure its data is current.

How do I get the tool?

The Tagging Merge Tool allows you to manipulate and test exported XBRL data. You can obtain this tool from the SE XBRL team. Please contact them to receive a ZIP file containing the latest version.

You will also need to install Java for the tool to function correctly.

How to install Java

  1. Go to the following link: https://adoptium.net/en-GB/temurin/releases/?version=17&os=any&arch=any

  2. Scroll down and find the download button for Windows.

  3. This will download an installer through your browser.

  4. Run the downloaded file and follow the installation process, using the default settings.

    Note: A system restart may be required for the installation to take effect.

How do I run the tool?

Once you have the ZIP file containing the tool, unzip the folder by right-clicking on it and selecting Extract All, as shown below.

After the folder is unzipped, navigate to the new folder and locate the executable file named xbrl-to-tag-converter. Running this file will launch the tool.

How do I export the XBRL data?

You can extract the XBRL data from the platform by accessing the developer console.

  1. First, you will need to open the engagement from which you want to extract the Custom Tags, contexts and numerics.

  2. Once you're in your engagement, click Start Review in the XBRL menu.

  3. After the review panel appears on the right side of the screen, you can access the developer console. Right-click anywhere on the page and select Inspect.

  4. This will open the developer tools panel.

  5. At the top of the panel, navigate to the Console tab.

  6. In the console, enter the following command: wpw.xbrl.jsonExporter.jsonExport()

  7. After you press Enter, a list of errors will appear for any Custom Tags that are missing an XBRL identifier. If a tag is missing an identifier, it will not be exported.

  8. The tool will then generate a file called customXbrl.json in your Downloads folder. This file will contain all the Custom Tags, contexts and numerics from your engagement.

    Note: This export does not include System Tags, contexts, or numerics.

Tagging Merge Tool

The Tagging Merge Tool allows you to combine two separate XBRL data files and verify the changes between them.

You can input two files in this menu, typically your master JSON file and your newly exported XBRL data. You can review each file individually to view its contents. Press Generate XbrlData.json to begin the merge process.

Merge process

Above is the Merge Files dialog, where you will handle merge conflicts. In the top bar, you can navigate between tags, numerics, contexts and namespaces.

Each section of the Merge Files dialog contains four subsections:

  • Final Contexts

  • Conflicting Contexts

  • Only in First

  • Only in Second

Final Contexts section

The Final Contexts section contains everything that will be included in your final file. Anything not listed here will not be included in the exported file. This section is automatically populated with items that are exact matches in both input files. For example, if a context is identical in both files, it will appear here automatically.

You can click on each entry to view its contents in a detailed panel.

Conflicting Contexts section

The Conflicting Contexts section contains items that exist in both files but differ in some way. Clicking on a row will open the Conflict dialog, where you can compare the two versions and select the one you want to include in the final file.

Once selected, the chosen version will be moved to the Final Contexts section.

Only in First and Only in Second sections

The Only in First and Only in Second sections list items that are only contained in one file. You can click on each row to review the item and choose to add it to the final file by clicking Add. There is also an Add All option at the top of each list, which will move all entries to the Final Contexts section.

After reviewing and resolving all entries, click Export to generate a new master file that contains your changes.

Note: Only items in the Final Contexts section will be included in the export. Make sure that all necessary contexts, numerics, tags and namespaces are included. Items left in the Only in First or Only in Second sections will not be exported.

Review panel

Clicking Review opens the Review panel, which displays the contents of the selected file.

Below you can see the Review panel.

In this panel, you can browse the tags, numerics and contexts. Clicking on any item will show detailed information about it.

The panel also highlights:

  • Unused contexts and numerics: These exist in the file but are not referenced by any tags.

  • Missing contexts and numerics: These are IDs referenced by tags but are not present in the file.

Taxonomy Checker Tool

The Taxonomy Checker Tool allows you to validate XBRL data against a selected taxonomy. It generates an error report detailing any issues found in the XBRL data. The tool checks both the structure of the data and whether the tags, dimensions and namespaces are relevant to the selected taxonomy.

You can use the input field to select the XBRL data file you wish to test. A drop-down menu is provided to choose the taxonomy to test against. If your taxonomy is not available in the drop-down, please refer to the section below on how to add your own taxonomy.

What tests are run?

The tool runs several validation checks to ensure:

  • The data structure is valid

  • There are no blank values

  • All tags, contexts and numerics are valid for the selected taxonomy

Additionally, contexts are checked to ensure they fall under a valid role and primary item for their dimensions.

Taxonomy Namespace Tool

The Taxonomy Namespace Tool allows you to generate the namespace map in your XBRL data. This map cannot be exported from an engagement and must be created using this tool. The namespace map is required in your XBRL product script so the system knows which namespaces to use. You only need to use this tool once per product to generate the relevant map. The map can then be merged into your files each time you use the Tagging Merge Tool.

The tool has two input fields:

  • The first input allows you to select the JSON data file to which you want to add the namespace data.

  • The second input allows you to select the taxonomy from which you want to retrieve the namespace data.

If your taxonomy is not available in the drop-down menu, refer to the section How to add your taxonomy to the drop-down.

Once both selections are made, click the Generate Namespaces button. This will open a dialog displaying the namespaces for the selected taxonomy. You can then click Confirm to add the namespace data.

In the dialog, you can either click Close to cancel or Confirm and Export to generate a new JSON file containing your existing data along with the added namespace map.

How to add your taxonomy to the drop-down

If your taxonomy is not available in the drop-down, you will need to contact the UK team to obtain a ZIP folder containing your taxonomy. Once you have the ZIP file, extract it by right-clicking the file and selecting Extract All, as shown below.

Next, place the extracted folder into the following directory within the tool: xbrl-to-tag-converter-win32-x64\resources\server\taxonomies

After this is done, restart the tool. Your taxonomy will now appear in the drop-down list.
