Import groupings (Trial Balance page)

You can import groupings from CSV or Excel files. For CSV imports, you need to properly prepare the files first. If you have groupings that have been defined in Working Papers, you can export them to an Excel file and import the file.

To import groupings from a CSV file:

  1. Prepare the CSV file
  2. Import groupings

To import groupings from Working Papers:

  1. Export groupings from Working Papers to Excel
  2. Import groupings

Prepare a CSV file for import (Trial Balance page)

The CSV file must include the following five columns, they can have empty values but each column must be available in the file:

  • Map number (known as group number in SE)

  • Account name (known as group name in SE)

  • Postable (accounts can be assigned to the grouping)

  • Type (balance sheet or income statement)

  • Sign (debit or credit)

You can optionally include the following columns:

  • Weight
  • Allow
  • Contrary Name
  • Inconsistent Name
  • Dimension Category
  • Excluded Dimensions
  • Reassign Group for contrary accounts
  • Group Classification
  • Link to a Tax Group
  • Link to a Chart of Accounts Group
  • Cycle
  • Area
  • Components
  • Default group

You can use these columns to help set up automapping (automatic assignment of accounts to groupings) when users import a new trial balance.

The following table describes all the information columns:

Column

Description

Map Number

(mandatory)

Enter a map number for the grouping. Also known as group number in SE.

Can contain alphanumeric characters, decimal points, hyphens and underscores.

Account Name

Enter a name for the grouping. Also known as group name in SE.

Can contain alphanumeric characters, decimal points, hyphens and underscores.

If this field is left blank, the grouping will be named "[Untitled]" by default.

Postable

Indicate whether accounts can be assigned to the grouping.

Can contain a single letter. Expected values:

  • Y - accounts cannot be assigned to the grouping.
  • N - accounts can be assigned to the grouping.

If this field is left blank, you will be able to assign accounts to the grouping by default.

Type

(mandatory)

Indicate whether the grouping belongs to the balance sheet or to the income statement.

Can contain a single letter. Expected values:

  • B - balance sheet
  • I - income statement

Sign

Indicate whether the grouping should be for debit accounts or credit accounts.

Can contain a single letter. Expected values:

  • D - debit
  • C - credit

If this field is left blank, the grouping will be have the debit sign by default.

Group Classification

Enter a classification for the grouping.

Can contain alphanumeric characters, decimal points, hyphens and underscores.

Weight

Enter a weight for the grouping. In cases where multiple groupings have the same classification, then any account with that classification will be assigned to the grouping with the highest weighting number.

Can contain a decimal number from 1 to -1.

If this field is left blank, the grouping will have a weight of 0 by default.

Allow

Indicate whether the grouping has any account balance restrictions.

Can contain a single character. Expected values:

  • + - only accounts with a debit (positive) balance can be assigned.
  • - - only accounts with a credit (negative) balance can be assigned.
  • * - accounts with debit or credit balances can be assigned.

If this field is left blank, you will be able to assign accounts with debit or credit balances by default.
Link to a Tax Group

Enter a group number from the Link to a group in the Tax Code grouping structure drop-down in the Group Properties dialog.

By adding this value, you automatically assign accounts to groups in the Tax grouping structure based on their initial group assignment.

Expected value:Text
Link to a Chart of Accounts Group

Enter a group number from the Link to a group in the Prescribed grouping structure drop-down in the Group Properties dialog.

By adding this value, you automatically assign accounts to groups in the Prescribed grouping structure based on their initial group assignment.

Expected value:Text
Cycle

Enter a cycle from the business cycle list (as defined in the Product Settings). When you assign a group to a business cycle, items in that group will automatically be shown under this cycle in the Risk Assessment document, making it easier to audit by business cycle.

Expected value:Text
Area

Select an area for the group from the area list (as defined in the Product Settings). When you assign a group to an area, items in that group will automatically be shown under this area in the Risk Assessment document, making it easier to audit by area.

Expected value:Text
Components

Components provide a more market-driven, region-driven or product-driven visibility logic for trial balance groups. For example, if the entity operates within the Agriculture industry, then only display trial balance groups that are relevant to agriculture. To learn more, see Components.

Enter one or more component to assign to the group (as defined in the Product Settings).

Expected value:

Text

Use a colon to separate multiple components associated with a group (example: component1;component2;component3).
Contrary Name

If applicable, enter a description for group entries that are opposite to the group's normal sign. For example, "Bank Overdraft" will appear instead of "Bank" when the account balance is a credit.

Expected value:Text
Inconsistent Name

If applicable, enter an alternative description for the group when the current and prior year have opposite signs.

Expected value:Text
Dimension Category Enter a dimension category for the group (select from the list defined in Product Settings). Expected value:Text
Excluded Dimensions

Enter the dimensions that are excluded from the selected category.

Note: This property is only be available if a value for Dimension Category exists.

Expected value:Text

Use a colon to separate multiple excluded dimensions (example: dimension1;dimension2)
Reassign Group for contrary accounts

If applicable, enter a group from the Financials grouping structure for accounts with a contrary balance.

If an account should automatically be assigned to a specific group but it does not match the selected sign (debit or credit) for this group, it is automatically assigned to the reassign target. For example, negative Asset -> positive Liability

Expected value:Text
Default Group

Specify whether or not the group is a default group. To learn more, see Default Group.

Expected value:

  • Y or 1 or TRUE
  • N or 0 or FALSE

Export groupings from Working Papers to Excel

To use the grouping structure you've defined in Working Papers, you need to export it to an Excel file, then convert the file to a .csv.

Open Working Papers and follow the steps below.

To export a grouping from Working papers to an Excel file:

  1. Open the file that you want to export.

  2. On the ribbon, click Engagement | Groupings.

    3. The Groupings worksheet displays.

  3. Right-click a column heading and click Reorder columns.

    4. In the Reorder Table Columns dialog, arrange the Active Columns as: Number > Name > Title > Type > Sign

  4. Click OK.

    5. Ensure that each group contains a unique number, a defined type and sign.

  5. On the ribbon, click Tools | Options.

  6. On the General tab, in the Table List Field Style drop-down menu, select Entry Codes.

  7. Click OK.

  8. On the worksheet, select all the groups by clicking any group number, then pressing the CTRL+A on your keyboard.

  9. Copy the selection by pressing CTRL+C on your keyboard.

  10. Open a new Excel workbook.

  11. Paste the groups by clicking cell A1, then hitting the CTRL+V keys on your keyboard.

  12. Verify that the information is correct in Excel:

    • Check for duplicate group numbers by selecting the whole column and pressing Home | Conditional Formatting | Highlight Cell Rules | Duplicate Values.

    • Check for missing or empty values in the Name, Title, and Type columns.

    • Check the Type column to ensure the classification is correct for each line.

    • Check the Sign column to ensure the sign is correct for each line. If the sign is blank, it will default to Debit.

  13. Save the Excel workbook as a .csv file.

Import groupings from a CSV or an Excel file (Trial Balance page)

Once you have the CSV or Excel file ready, you can start the import process.

To import a grouping from a CSV or Excel:

  1. Go to the Trial Balance tab, then select More actions () | Import Groups.

  2. Drag and drop the .csv file into the pop-up dialog.

  3. Select Yes when prompted to confirm deletion of existing financial groups.

Once the import is complete, review the structure to ensure that the import was successful. You can select the Expand All/Collapse All () to review the imported structure. If necessary, you can also edit a group to add additional properties or change existing ones.
