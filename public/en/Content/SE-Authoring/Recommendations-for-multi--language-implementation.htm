Recommendations for multilingual implementation

Here are the implementation recommendations for multilingual capabilities:

  • To set up translated versions for note disclosures: Create multiple versions of the notes in different languages with visibility settings.

  • To set up translated versions for letters: Create multiple versions of the letter sections in different languages with visibility settings.

  • To set up translated query questions and checklist procedures: Create multiple versions of the procedures and query questions in different languages using dynamic text.

For detailed instructions, see Use- case: A template with French and English language support.

Note that:

  • Multilingual capabilities feature does not automatically translate content. You must provide the translations for the languages that you want your SE product to support.

  • If you delete a language, the translated text that you have added in the template is not deleted.

  • On carry forward, the selected content language is carried over to the next period's file.

  • Firm authors cannot add or delete languages but they can make all other changes.

  • Firm author changes for content language are not overridden when they update the firm template with the latest change from the product template.

    • However, there is one exception:

    If firm or enterprise authors make a language active in the firm template, then you, as the primary author, change it to inactive in the product template, your settings override what's in the firm or enterprise template once firm or enterprise authors update it - and the language becomes inactive.

Add or modify available languages for an engagement

As a primary author, you can set which languages and how many languages are available for documentation within an engagement. As an enterprise or firm author, you select which languages to include based on the available languages set by the primary author.

  1. From the product template, click the [] Information button at the top of the screen.

  2. Click the Settings option to open Product Settings.

  3. Click Content language under the Format section

  4. Click the Add Language button to add a new language for selection.

  5. In the table below the Add language button, you can select or deselect languages for availability in your engagements.

  6. You can also set one language as your default language.

Multilingual capabilities for Materiality form

As a firm, enterprise or primary author, you have multilingual capabilities in editable fields in the Materiality form. You can select a default language for your engagements and have the additional flexibility to change the language within editable fields.

Editable fields that include multilingual capabilities include:

  • Document and Sections Guidance

  • Sections Titles

  • Tables Headings

  • Overall Materiality - Benchmarks Input fields in Tables (Custom formulas / Other)

  • Performance Materiality & Trivial - % description

  • Conclusion procedures (including conclusion for Group component allocation)

To switch languages in an engagement

  1. Click the top-left corner of an engagement, to open the Entity dialog box.

  2. Click the drop-down menu under Content Language

  3. Select the language you wish to use,

  4. Open the Materiality form under the Documents page

  5. Edit or add content in the language that you selected in the editable fields.

  6. Click the checkmark beside the field to save the content.

  7. Repeat if you require the same content in other languages.
