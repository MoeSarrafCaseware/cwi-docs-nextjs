What's new in XBRL - February 2025

The following lists describe the improvements and fixed issues in the February 2025 XBRL updates.

SE Client

  • [SE-53318] - As part of our ongoing performance and maintenance project, XBRL Review mode now uses the hypercube service (getDefinitionLinkbase endpoint) in the XBRL Context Server microservice, rather than the XBRL Server microservice.

  • [SE-53268] - Fixed an issue that prevented the direct export of iXBRL files to the SE Document Manager or Collaborate Files list.

  • [SE-53237] - Fixed an issue in the Add tag dialog, where after selecting a tag, the Associated Role field would not populate.

  • [SE-53127] - Fixed an issue when tagging an engagement with multiple taxonomies, where an incorrect set of tags was listed for the selected taxonomy.

  • [SE-53087] - Made technical improvements to the creation and export of custom numerics.

  • [SE-52982] [SE-53186] - Made technical improvements to the creation and export of custom contexts.

  • [SE-52821] - Aligned the Search button and field, and fixed the boundary box around the concept list in the Add tag dialog.

  • [SE-52818] - Updated the description in the iXBRL Export dialog to provide more accurate guidance on where the generated iXBRL files are saved.

  • [SE-52563] - Repositioned the Close button in various XBRL panels to ensure consistency.

  • [SE-52373] - Fixed an issue where the Context selector would appear empty after selecting a formula in the Content dropdown.

  • [SE-52351] - Fixed an issue where the Edit Dimensions dialog would fail to open after selecting Context editor | New dimensional context.

  • [SE-52343] - Added a Visibility settings dropdown to the Add tag dialog that allows the various visibility settings to be managed from a single location.

  • [SE-52342] - Changed the color of the selected context line in the Add tag dialog to improve visibility.

  • [SE-52341] - Replaced the Clear button with a low-profile icon in the Add tag dialog.

  • [SE-52340] - Rotated the chevron icons in the Add tag dialog to a more logical direction.

  • [SE-52338] - Combined the New context and New dimensional context options into a single Context editor dropdown menu.

  • [SE-52337] - Changed the color of the confirmation button when deleting tags to better indicate the severity of the action.

  • [SE-52336] - Added a warning icon to the XBRL Review panel to better indicate value incompatibilities.

  • [SE-52335] - Changed the color of various warning icons for consistency.

  • [SE-52333] - Changed the color of the confirmation button when deleting contexts to better indicate the severity of the action.

  • [SE-52332] - Updated the font style and size in various input fields for consistency.

  • [SE-52331] - Tagged all interactive elements for use with Pendo.

  • [SE-51466] - All hypercube information is now obtained via a single endpoint.

  • [SE-48879] - Added the ability to resize the Add tag dialog to logical dimensions.

  • [SE-42972] - Fixed an issue where the Continue to generate iXBRL button could be pressed repeatedly, locking the UI in an infinite loop of windows.

Microservices

XBRL Context Server (Ver 2.0.1)

  • [SE-52517] - Migrated hypercube and taxonomy JSONs to all taxonomies supported by the microservice.
