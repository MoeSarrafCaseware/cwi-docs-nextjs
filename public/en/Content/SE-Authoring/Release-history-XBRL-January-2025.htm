What’s new in XBRL - January 2025

The following lists describe the improvements and fixed issues in the January 2025 XBRL updates.

SE Client

Improvements

  • [SE-52334] - In the Context Editor, the Close button has been relabeled to ‘Cancel’ to more accurately reflect its function.

  • [SE-52319] - In the Context Editor, the Copy function now copies over the selected period from the Applicable selector.

  • [SE-52102] - The list of domain segments for a dimension now indicates which domain is the default value.

  • [SE-51418] - Removed the SE Server step from the iXBRL Generation process to improve loading performance.

  • [SE-50130] - The XBRL review dialog now displays a warning when a tag requires a typed dimensional context.

  • [SE-46605] - The Automatic option for decimals in the Numeric Option editor now uses the decimal value specified in Product Settings | Financial Statements.

Fixed issues

  • [SE-52129] - Scaling settings are not correctly applied to engagements during the XBRL export process if a custom tag’s scaling value is set to Document default.

  • [SE-51071] - Empty placeholders do not visually indicate whether they are tagged.

  • [SE-50848] - Some instances of ‘overridden’ are spelled incorrectly, resulting in UI styling issues.

  • [SE-43739] - Note headers are highlighted in an inconsistent manner when selected and when tags are applied.

Microservices

Ver 2.5.1.1

Improvements

  • [SE-52463] - Added cached files from the FT19 taxonomy to the NT19 cache to create a combined taxonomy.

  • [SE-52445] - Upgraded UBPartners to version 2024.2.0.35.

Fixed issues

  • [SE-52684] - iXBRL tags are generated with a ‘null’ decimal value rather than 0, which prevents the microservice from applying a unit of measure to the tags.

Ver 2.4.0.5

Improvements

  • [SE-52461] - Added an endpoint to build hypercubes, hypercube roles, primary items and dimensions.

Fixed issues

  • [SE-52051] - Pages in landscape mode are erroneously removed during iXBRL generation.

  • [SE-51780] - An endpoint attribute disrupts the iXBRL Generation process.

Taxonomies

  • BE - Fin Tax (nbb-cbso-25.0.5)

  • NL - Combined FT19 and NT19

  • ZA - SE IFRS (CIPC)
