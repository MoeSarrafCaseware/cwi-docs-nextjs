What's new in XBRL - July 2025

The following lists describe the improvements and fixed issues in the July 2025 XBRL updates.

SE Client

Enhancements

  • [SE-55954] - Increased unit test code coverage for the add-tag-popover-component in the Add tag dialog to 75%.

  • [SE-56420] - Added a feature flag to the block tagging feature to prevent it from being automatically applied to products.

  • [SE-56389] - Added duration validation styling to paragraphs, formulas, placeholders, tables and titles to enhance the visibility of relevant content.

  • [SE-56497] - Added a guide to explain the different highlight types for each tag in the XBRL Review panel.

Fixed issues

  • [SE-56013] - An incorrect cursor icon is displayed when hovering over an area in the Document Map.

  • [SE-56419] - Custom block tags are removed during the XBRL generation process.

  • [SE-56391] - During an iXBRL export, the Merge duplicate text tags function links duplicate tags using the continuedAt attribute instead of merging them into a single tag.

  • [SE-56042] - When tagging from the taxonomy, the taxonomy panel does not open at the position of the most recently tagged item.

  • [SE-56446] - An unknown XBRL type is displayed in the Add tag dialog when adding a concept.

  • [SE-56662] - The Delete icon in the XBRL Review panel shifts position when selected for the first time.

  • [SE-56570] - The most recently used context option does not appear at the top of the Context drop-down.

XBRL Server (Ver 2.5.9.5, DK/UK)

Enhancements

  • [SE-56307] - Non-numeric tags can now be merged into iXBRL exports using the ContinueAt attribute and ix:continuation nodes.

Fixed issues

  • [SE-56696] - Bracket-wrapped content within ix:hidden non-fraction facts does not display the negative symbol (-) and is therefore considered invalid.

  • [SE-56605] - Footers in iXBRL exports are capped at 60% width, preventing centered text from appearing in the correct position.

  • [SE-55522] - Certain sections in generated iXBRL exports do not retain the intended formatting.

  • [SE-56498] - Bracketed (negative) table cell numerics do not display their brackets in iXBRL exports.

  • [SE-56288] - Table row styling in iXBRL exports does not match the print preview.

  • [SE-56202] - Certain paragraphs in generated iXBRL exports appear duplicated.
