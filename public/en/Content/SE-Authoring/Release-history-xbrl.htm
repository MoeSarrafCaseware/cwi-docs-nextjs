Introducing the XBRL tool

The XBRL tool allows primary authors to:

  • Predefine system tags for your product’s financial statement documents (No action is required from users to apply system tags to their engagements)

  • Define mandatory tags to ensure reports contain tags required by regulatory bodies

End users can also use the tool to review their XBRL tags and make adjustments if the system tags don’t meet their requirements.

To access the XBRL tool, you can select the XBRL icon in the top menu to:

  • Launch the review process to apply custom tags or review system tags

  • Launch the export process to validate and export an iXBRL report

XBRL review panel

When you launch the XBRL review panel, it displays on the right side of the page in the financial statements.

From the review panel, you can:

  • Apply custom tags to content on financial statements

  • Tag the following content:

    • Figures in tables

    • Paragraphs

    • Formulas

    • Placeholders

    • Block tags

    • Footnote tags

    • Note titles

  • Create and edit necessary contexts and numeric options, including dimensional and typed dimensional contexts

  • Attach contexts and numeric options to your tags

  • Mark up to five contexts as favorites to easily access frequently used contexts

  • Edit system tags where necessary

XBRL exports

The export tool enables the production of XBRL reports. When successfully generated, these are added to the engagement’s Documents page with the date and time of the export. The export tool features:

  • A check to ensure there are no duplicate facts

  • An intuitive diagnostic tool to identify and correct errors while exporting reports, including the option to export diagnostics to CSV

Additional key features

The XBRL tool also includes the additional features:

  • While iXBRL review is active, tagged content is highlighted with colors that indicate the tagging status

  • Dynamic warnings during the selection process for tags, contexts and numerics to ensure you don’t attach inappropriate contexts or numerics for the tag

  • When you create contexts and numerics, the tool does not allow you to create duplicate items, or items with invalid dimension selections

  • Automatic GUID generation for custom tags created by end users if the content doesn’t have a GUID defined by the primary author

    Note: The GUIDs generated by this process are unique to the engagement. This function cannot be used by primary authors to create GUIDs for system tags.

  • The review panel always shows the native value of monetary tags, so you can easily select the correct monetary numeric. For example, share capital figures are presented on balance sheets as a positive amount, however they are negative within the trial balance. This situation requires different numeric selections to normal and this is clearly highlighted by the tool.
