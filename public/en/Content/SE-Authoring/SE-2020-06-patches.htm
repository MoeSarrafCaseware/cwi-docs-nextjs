SE 2020-06 patches

Here is a summary of the patches that have been deployed for SE 2020-06.

SE 2020-06-U2

Version: 2020-06 Build 524

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U2.

  • [SE-21533] - Percent variance columns in dynamic tables do not evaluate correctly when decimals are used as a digit separator. The specified percentage is multiplied by 100.

  • [SE-21473] - Documents do not print as expected for page sizes other than Letter. The page layout in the generated PDF does not match the layout in the print preview.

  • [SE-20227] - Data links (fx) referencing contact procedure responses do not evaluate correctly.

  • [SE-21410] - Performance issues when opening engagements for products with a large number of SE builder forms.

SE 2020-06-U3

Version: 2020-06 Build 526

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U3.

  • [SE-21674] - An error occurs when users attempt to print the financial statements after they import transactions to the engagement.

  • [SE-21405] - Data links (fx) referencing contact procedure responses display middle names out of order.

SE 2020-06-U4

Version: 2020-06 Build 527

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U4.

  • When users print financial statements,

    • [SE-21750] - An error occurs if the document contains analysis graphs that were rolled forward from a previous year's file prior to the SE 2020-06 release. The financial statements do not print.

    • [SE-21783] - The Statement of Cash Flows, defined in statement.js, does not display as expected in the generated PDF.

    • [SE-21446] - Analysis graphs do not display as expected in the generated PDF.

  • [SE-21718] - If users add a Custom group row to a dynamic table, close edit mode and then reopen it, the Trial balance data and Input row types are no longer available to be added within the custom group.

  • [SE-21652] - In IFRS dynamic tables, when users refresh or navigate away from the page,

    • Data in dimensions rows is removed

    • Dimension rows that are set to display data for second-level dimensions are changed to display data for the corresponding primary dimension.

  • [SE-21659] - Users are redirected to the Documents page when they select a transaction to view its details if the transaction Entry ID includes a forward slash (/).

SE 2020-06-U5

Version: 2020-06 Build 528

One issue has been fixed with the deployment of SE 2020-06-U5.

[SE-21817] - Dynamic tables do not display as expected in financial statements or worksheet documents and an error occurs when users attempt to print them.

SE 2020-06-U6

Version: 2020-06 Build 530

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U6.

  • [SE-21837] - Dynamic tables that use consolidated data and include input rows as part of custom groups do not display as expected in the financial statements.

  • [SE-21801] - Analysis charts do not display as expected. The vertical axis does not scale to display all values.

  • [SE-21671] - When users print financial statements with analysis sections that contain charts that are larger than a page, the charts do not scale to fit the page.

SE 2020-06-U7

Version: 2020-06 Build 531

One issue has been fixed with the deployment of SE 2020-06-U7.

[SE-21767] - An error occurs when users attempt to use the Copy From Another Product option in the product template to copy financial statements that include Consolidation visibility conditions.

SE 2020-06-U8

Version: 2020-06 Build 534

SE 2020-06-U8 addresses enhancements that mitigate the risk of pervasive outages when:

  • Users attempt to download a reference copy of an engagement with over 600 documents

  • Print errors occur. Load balancing of the print services has been enhanced to ensure effectiveness.

SE 2020-06-U9

Version: 2020-06 Build 538

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U9.

  • When users print the financial statements, the generated PDF does not display as expected. The following list summarizes the PDF output issues:

    • [SE-21947] - Styling issues related to paragraph spacing, in text sections, justification in dynamic table cells and page margins.

    • [SE-22012] - Visibility icons () display in the PDF for lists with visibility logic set up.

    • [SE-22032] - In engagement files based on templates that use the old Statement of Cash Flows, horizontal lines in the Statement of Cash Flows table do not display as expected.

    • [SE-21832] - Data links (fx) referencing procedure responses are doubled.

  • [SE-21926] - Primary authors experience performance issues when editing dynamic tables in financial statements.

  • [SE-22073] - In dynamic tables, if primary authors reference hidden notes, a comma displays before the note number in the note reference column for engagement users.

  • [SE-21972] - An error occurs when users attempt to export iXBRL and XBRL files that include dynamic tables with data links (fx).

  • [SE-21940] - In dynamic table cells, custom calculations added in columns where the Sum of Transactions is selected as the source of data do not work properly. The calculated values are incorrect.

  • [SE-21864] - Diagnostic labels do not function as expected in financial statements. Accounts with disagreeing balances are not flagged for engagement users. The balance discrepancy indicator () does not display and the Balance Check review tools does not flag the discrepancy.

SE 2020-06-U10

Version: 2020-06 Build 544

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U10.

  • [SE-22283] - In financial statements and SE Builder forms, the page size automatically reverts to Letter once the PDF output is produced, overriding the existing page size.

  • [SE-22136] - In financial statements, if a page break is the first visible section in an area, inserts a blank page in the document.

  • [SE-22091] - In checklists, content in procedure responses that are presented in a table format do not display in the PDF output or the reference copy. Empty tables show in the PDF.

  • [SE-21955] - In engagements with large trial balance data, attempts to download a reference copy may fail and an empty zip file is generated.

  • [SE-22196] - When viewing PDF documents within the engagement file, search and scroll functions may not work as expected.

SE 2020-06-U11

Version: 2020-06 Build 549

Here is the list of issues that have been fixed with the deployment of SE 2020-06-U11.

  • [SE-22433] - Styling issues related to justification in dynamic tables and spacing when users export an iXBRL file.

  • [SE-22337] - The placeholders indicator in financial statements displays an incorrect number of remaining placeholders.

  • [SE-22369] - Carry forward does not function as expected in large financial statements. Dynamic tables in notes are not carried forward.

  • [SE-22253]- In dynamic tables, table cells with negative amounts are indented while cells with positive amounts are not.

  • [SE-22270] - When users print the financial statements, note numbers and corresponding note titles display on separate pages.

  • [SE-22333] - When users print the financial statements, dynamic tables and text sections are not aligned.

  • [SE-22505] - An error occurs when users attempt to use Caseware Cloud Connector.
