Integrate single sign-on

Cloud supports single sign-on (SSO) with Entra ID (formerly Azure AD), Active Directory Federation Services (ADFS) and Okta. Using SSO with Cloud reduces the number of passwords staff members are required to memorize and helps administrators manage account issues (such as password resets) from a single location.

Note that,

  • The authorization protocol we use for SSO integration is OpenID Connect.

  • Time Desktop is not compatible with SSO. Once SSO is enabled, SSO users will no longer be able to log in to Time Desktop.

To get started, contact Caseware Support to request that single sign-on is enabled for your firm.
