Centralized client management (early access)

The new Entities app is designed to improve administrative efficiency and create a seamless user experience.

Through centralized client management, Entity Admins can manage client information across multiple business units from a central location. They don't have to navigate between different business units to perform day-to-day client management tasks, such as updating information, sharing access with staff, tagging clients, etc.

Seamless navigation

Navigating through multiple business units is now made seamless by displaying all entities on one page. See below for details on how you can use the various new capabilities.

Number Description
1

Tabs

Each tab represents a specific business unit, and clients within each business unit will be listed in these tabs. The name of each tab indicates the business unit names. Users will see all business units where they have entities assigned to them.
2

Count of entities

The number within brackets reflects the number of entities to which the user has access. If the user has an Admin role, the number will show the total number of entities in the business unit.

When users perform a search or use a filter, this number will update to show the number of matching results in each business unit.
3

Scrolling and Sorting

Users can now use the scroll bar to scan the entire list of entities within a business unit. They no longer need to click through different pages.

The list of entities in each tab can be sorted by columns.
4

List menu

In a firm with many business units, Admins who have access to all of them can use the List menu to quickly access a different business unit, without having to scroll through the page.
5

Navigation buttons

Users can also use the navigation buttons to the top right of the page to navigate between different business units.
6

Search

The advanced search functionality allows users to search for entities across all business units to which they have access. Search results are displayed in their respective business units and the count of entities would be dynamically updated to show the accurate number of matching results.
7

Filters

The new filter UI also includes a filter for business units. Users can filter the view to specific business units if needed.

The filters have also been enhanced to better suit users' needs.

As a result, for fields that can have multiple values (e.g. Tags), users can filter by more than one value and the results will include entities matching any of the selected values.

Note that all currently available filters have been retained.

Entity actions

Actions on entities (Tag, Share, Move, Delete, etc.) can be performed by selecting single or multiple entities. Users can find the appropriate menu for both scenarios as shown in the screenshots below.

Entity imports

The new import feature is designed to support business units. Cloud Admins can now import a list of entities across multiple business units with a single import file. The feature allows users to specify a business unit for each entity in the file.

How can users access the new pages?

The early access phase of the new Entities app is only available to Cloud Admins. They can find the new app in the Cloud menu, among other applications. Once the early access phase is complete, this will replace the current Entities app.

Future improvements

Bulk actions ('Select all' entities option) - We currently support single and multi-select options for entities. In a future release, we will consider including the option to select ALL entities within a business unit and perform actions in bulk.

View management across business units - In the current iteration, any custom views created within a business unit, are available only within that business unit (tab). For example, if a user had created a custom view in BU1, then that view will be available for selection only on that business unit. In a future release, we will provide a more consistent user experience for creating, selecting and working with custom views across business units.
