Subledger details described

When importing general ledger transactions from a CSV or Excel file, you must map the fields, so the transaction data display properly in the engagement.

This topic describes the fields within the SUBLEDGER DETAILS category that appear in the General ledger dataset.

Note that some fields are mandatory for specific data analytics tests to function properly. To learn more see, Mandatory fields for data analytics.

Identifier Type (identifier_type)

Código identificador de los datos dentro del libro auxiliar.

Requisitos

  • Less than 2,500 unique values. These values must be tagged to one of:
    • CU - Customers
    • Gastos pagados por anticipado
    • SU - Suppliers
    • EM-Empleados
    • Activos de inversión
    • OT - Other
    • Activos de capital (activos fijos) (CA)
    • IN - Inventory
    • Compromisos de deuda

Identifier Code (identifier_code)

Código identificador de los datos dentro del libro auxiliar.
Ejemplo: número de cliente, número de proveedor, número de empleado.
Ejemplos: "C123", "V6589", "6597".

Requisitos

If the identifier type is tagged, this must also be tagged.

Identifier Description (identifier_description)

Nombre asociado con el código identificador.
Ejemplos: "ABC Customer Ltd.", "DEF Purchase Inc.", "Sally Jones".

Identifier Active (identifier_active)

Indica si el código identificador está activo o inactivo.

Identifier Street (identifier_street)

Identifier's street - line 1.
Ejemplos: "123 Anywhere Street", "Anywhere 26".

Calle 2 del identificador

Detalle de dirección de la dirección del identificador: línea 2.
Ejemplos: "Suite 1100", "Unit 002B".

Identifier City (identifier_city)

Ciudad donde se ubica el identificador.
Ejemplos: "Toronto", "Los Ángeles".

Estado o provincia del identificador

Estado o provincia donde se ubica el identificador.
Ejemplos: "Ontario", "California".

Identificador de código postal

Código postal donde se ubica el identificador.
Examples: "94704", "M5A 0L6".

Identifier Country (identifier_country)

País (código) donde se ubica el identificador.

Document Number (document_number)

Identificador asociado con el documento de origen. This is the document that supports the transaction entered into the ERP.
Ejemplos: "PO4569" para el pedido de compra, "7956" para el cheque.

Número del documento al que aplica

El número de documento asociado con el documento de origen utilizado para compensar valores debe y haber. Usado para calcular saldo abierto cuando el pago, adelanto, nota de crédito/débito no se ha aplicado por completo, permite el cálculo del elemento abierto.
Ejemplos: "INV-001", "121296".
