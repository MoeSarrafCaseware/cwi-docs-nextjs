Cuentas por pagar - pagos efectuados

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Accounts payable—payments made subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Campo Uso
Id. de transacción
(ap_payment_made_transaction_id)

Una referencia cruzada para el registro del libro mayor. Puede incluir varios niveles diferentes para identificar la transacción. Los ejemplos incluyen centros de costes como empresa, división, región, grupo y sucursal/departamento.

Data type: Text

Required: No

Primary key: The value of this field is used to form the unique identifier of this entry.
Número de pago
(ap_payment_made_payment_id)

El Id. único para cada pago efectuado. Este Id. puede que tenga que crearse mediante la concatenación de campos (por ejemplo, proveedor y número de pago) para identificar cada transacción de una manera única.

Data type: Text

Required: Yes

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>

Fecha de pago
(ap_payment_made_payment_date)

La fecha en la se efectuó el pago o se aplicó el crédito.

Valor devuelto: Fecha

Required: No

Año de pago
(ap_payment_made_fiscal_year)

Ejercicio fiscal en el que transcurre la fecha de pago

Data type: Text

Required: No

Clave primaria
Período de pago
(ap_payment_made_payment_period)

Período fiscal en el que transcurre la fecha de pago

Data type: Text

Required: No

Clave primaria
Tipo de importe (Debe / Haber)
(ap_payment_made_amount_credit_debit_indicator)

Indica si el importe es debe o haber.

Data type: Text

Required: No

Possible values:
C=credit;
D=debit
Tipo de pago
(ap_payment_made_payment_type)

El valor, código o indicador del método por el cual la transacción de cargo o abono se anula o asigna a deuda por el proveedor. Los ejemplos incluyen cheques, transferencias bancarias, efectivo, etc.

Data type: Text

Required: No
Importe de pago
(ap_payment_made_payment_amount)

Importe monetario de la transacción registrado en la divisa funcional o divisa de grupo.

Data type: Money

Required: No
Importe del pago del informe
(ap_payment_made_payment_amount_reporting)

Importe monetario de la transacción registrado en la divisa de informe.

Data type: Money

Required: No
Divisa local del pago
(ap_payment_made_payment_amount_local)

El importe monetario de la transacción en la divisa local.

Data type: Money

Required: No
Estado de registro
(ap_payment_made_posting_status)

Indica si la transacción se ha registrado en el libro mayor:

Data type: Text

Required: No

Possible values: Posted or Not Posted
Número de lote
(ap_payment_made_batch_id)

A batch identifier generated by the source system.

Data type: Text

Required: No
Descripción de pago
(ap_payment_made_description)

Una descripción del pago.

Data type: Text

Required: No
Id. de origen
(ap_payment_made_source_id)

Origen del registro. El código del origen de la entrada del diario, tal como diario de ventas, diario de recibos de efectivo, diario general, diario de nóminas, entrada manual del contable, hoja de cálculo, etc.

Data type: Text

Required: No
Supplier ID (ap_payment_made_supplier_account_id)

Identificador del proveedor al que se realiza el pago o cuyos créditos se han aplicado.

Data type: Text

Required: No
Nombre de proveedor
(ap_payment_made_supplier_account_name)

El nombre del proveedor.

Data type: Text

Required: No
Creado por
(ap_payment_made_entered_by)

El Id. de usuario de la persona que creó el registro.

Data type: Text

Required: No
Fecha de creación
(ap_payment_made_entered_date)

La fecha en la que la transacción se creó en el sistema. Se conoce también como fecha de creación.

Valor devuelto: Fecha

Required: No
Hora de creación
(ap_payment_made_entered_time)

La hora en la que esta transacción se creó en el sistema.

Data type: Date/time

Required: No

Data type: Text

La fecha en la que se aprobó el asiento.

(ap_payment_made_approved_by_time)

La hora en la que se aprobó el asiento.

Data type: Date/time

Required: No
Última modificación realizada por:
(ap_payment_made_last_modified_by)

El Id. de usuario de la última persona que modificó este asiento.

Data type: Text

Required: No
Fecha de la última modificación
(ap_payment_made_last_modified_date)

La fecha en la que se realizó la última modificación del asiento.

Valor devuelto: Fecha

Required: No
Hora de la última modificación
(ap_payment_made_last_modified_time)

La hora en la que se realizó la última modificación del asiento.

Data type: Date/time

Required: No
Número de factura
(ap_payment_made_invoice_number)

Un número de identificación de una transacción generada de forma interna o externa. Los ejemplos incluyen número de cheque, número de transferencia bancaria o Id. del documento original.

Data type: Text

Required: No

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Fecha de la factura
(ap_payment_made_invoice_date)

La fecha de una transacción generada de forma externa. Los ejemplos incluyen la fecha del cheque o la fecha de la transferencia bancaria.

Valor devuelto: Fecha

Required: No
Código de unidad de negocio
(ap_payment_made_business_unit_code)

Se usa para identificar la unidad de negocio, región, sucursal, etc., al nivel en el que se estén auditando los estados financieros.

Data type: Text

Required: No
Número de cuenta debe del libro mayor
(ap_payment_made_gl_debit_account_number)

El número de cuenta del libro mayor en el que se ha publicado el lado debe de la transacción.

Data type: Text

Required: No
Número de cuenta haber del libro mayor
(ap_payment_made_gl_credit_account_number)

El número de cuenta del libro mayor en el que se ha publicado el lado haber de la transacción.

Data type: Text

Required: No
Detalle de línea: Id. de cuenta
(ap_payment_made_line_account_id)

Código de cuenta del libro mayor de la cuenta de ingresos del libro mayor. Esto puede contener un Id. de subcuenta, así como numerosos niveles diferentes para identificar la cuenta, incluidos centros de costes, tales como empresa, división, región, grupo y sucursal/departamento.

Data type: Text

Required: No
Detalle de línea: descripción de la cuenta
(ap_payment_made_line_account_description)

A description of the account whose identifier appears in Line Detail - Account ID.

Data type: Text

Required: No
Divisa del pago
(ap_payment_made_payment_amount_currency)

La divisa funcional o divisa de grupo relacionada con el importe del pago.

Data type: Text

Required: No
Divisa del pago de informe
(ap_payment_made_payment_amount_reporting_currency)

La divisa de informe relacionada con el importe del pago para informes no consolidados.

Data type: Text

Required: No
Divisa local del pago
(ap_payment_made_payment_amount_local_currency)

La divisa para requisitos de informes locales.

Data type: Text

Required: No
Id. de asiento
(ap_payment_made_journal_id)

El identificador único de un asiento contable.

Data type: Text

Required: No
