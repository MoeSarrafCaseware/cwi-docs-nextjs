Cuentas por cobrar - saldos abiertos

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Accounts receivable—open balances subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Campo Uso
Transaction ID (ar_open_balance_transaction_id)

El Id. único de la transacción de cuentas por cobrar abiertas incluye la factura y el efectivo recibidos.

Data type: Text

Required: No

Primary key: The value of this field is used to form the unique identifier of this entry.
Invoice Type (ar_open_balance_invoice_type)

El código, valor o indicador del método por el que el importe debe o haber de la transacción se anula o asigna a deuda por el cliente. Examples include check, wire transfer, cash, credit memo, invoice, interest-only invoice, and so on.

Data type: Text

Required: No
Fecha de vencimiento de la factura

La fecha en la que vence el pago del cliente. No todas las transacciones tendrán una fecha de vencimiento (por ejemplo, las notas de crédito). El vencimiento de una cuenta por cobrar suele calcularse en base a esta fecha.

Valor devuelto: Fecha

Required: No
Balance Amount (ar_open_balance_balance_amount)

El importe monetario del saldo restante registrado en la divisa funcional o divisa de grupo.

Data type: Money

Required: No
Importe del saldo del informe

El importe monetario del saldo registrado en la divisa de informe.

Data type: Money

Required: No
Importe del saldo local

El importe monetario del saldo en la divisa local.

Data type: Money

Required: No
Original Amount (ar_open_balance_original_balance_amount)

El importe monetario del saldo original (importe de la factura original) registrado en la divisa funcional o divisa de grupo.

Data type: Money

Required: No
Importe del saldo original local

El importe monetario en la divisa local (importe de la factura original).

Data type: Money

Required: No
Informe del importe del saldo original

El importe monetario registrado en el informe de la divisa (importe de la factura original).

Data type: Money

Required: No
Grouping Code (ar_open_balance_grouping_code)

Un mecanismo de agrupación para artículos relacionados en un lote o grupo de facturas, como el grupo de facturas que se encuentra en ciertos sistemas de ERP.

Data type: Text

Required: No
Tipo de importe (Debe / Haber)

Indica si el importe es debe o haber.

Data type: Text

Required: No

Possible values: C=credit; D=debit.
Project ID (ar_open_balance_project_id)

El identificador único del proyecto, del que derivan las cuentas por cobrar.

Data type: Text

Required: No
Fiscal Year (ar_open_balance_fiscal_year)

El ejercicio fiscal en el que transcurre la Fecha de la transacción.

Data type: Text

Required: No
Accounting Period (ar_open_balance_accounting_period)

El período contable en el que transcurre la Fecha de la transacción.

Data type: Text

Required: No
Description (ar_open_balance_remark)

Una descripción de texto en formato libre.

Data type: Text

Required: No
Id. de cuenta de cliente

El Id. del cliente del que se espera el pago o al que se han aplicado los créditos no utilizados.

Data type: Text

Required: No

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Nombre de cuenta del cliente

El nombre del cliente.

Data type: Text

Required: No

Primary key: Third alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Invoice ID (ar_open_balance_invoice_number)

El Id. único para cada factura.

Data type: Text

Required: Yes

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Invoice Date (ar_open_balance_invoice_date)

La fecha de la transacción, independientemente de la fecha en la que se creó la factura. La fecha a partir de la cual se calcula la fecha de vencimiento basada en las condiciones de la factura.

Valor devuelto: Fecha

Required: No

Primary key: Fourth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>
Código de unidad de negocio

Se usa para identificar la unidad de negocio, región, sucursal, etc., al nivel en el que se estén auditando los estados financieros.

Data type: Text

Required: No
Other Reference Number (ar_open_balance_reference_number)

Un número de identificación de una transacción generada de forma interna o externa. Los ejemplos incluyen número de cheque, número de transferencia bancaria o Id. del documento original.

Data type: Text

Required: No
Other Reference Date (ar_open_balance_reference_date)

La fecha de una transacción generada de forma externa. Los ejemplos incluyen la fecha del cheque o la fecha de la transferencia bancaria.

Valor devuelto: Fecha

Required: No
Id. del contrato de ventas

El Id. único del contrato de ventas, del que derivan las cuentas por cobrar.

Data type: Text

Required: No
Entry ID (ar_open_balance_journal_id)

El identificador único del asiento contable. Por lo general, el sistema lo genera de forma automática.

Data type: Text

Required: No
Balance Currency (ar_open_balance_balance_amount_currency)

La divisa funcional o divisa de grupo relacionada con el importe del saldo.

Data type: Text

Required: No
Divisa del saldo del informe

La divisa de informe relacionada con el importe del informe de saldo para informes no consolidados.

Data type: Text

Required: No
Divisa del saldo local

La divisa para requisitos de informes locales.

Data type: Text

Required: No
Amount Currency (ar_open_balance_amount_currency)

La divisa funcional o divisa de grupo relacionada con el importe.

Data type: Text

Required: No
Divisa de informe del importe

La divisa de informe relacionada con el importe de informes no consolidados.

Data type: Text

Required: No
Importe de la divisa local

La divisa para requisitos de informes locales.

Data type: Text

Required: No
