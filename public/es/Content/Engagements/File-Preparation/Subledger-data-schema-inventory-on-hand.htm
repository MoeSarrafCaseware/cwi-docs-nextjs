Inventory on hand described

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Inventory on hand subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Campo Uso
Id. de transacción
(inventory_on_hand_transaction_id)

El Id. de transacción asociado con el Estándar de datos de auditoría (ADS) del libro auxiliar de Compra al pago o Pedido al cobro.

Data type: Text

Required: No

Primary key: The value of this field is used to form the unique identifier of this entry.
Id. de producto de inventario
(inventory_on_hand_inventory_product_id)

El Id. del producto es un código que identifica de forma única un artículo que se incluye en el inventario.

Data type: Text

Required: Yes

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
Número de serie
(inventory_on_hand_serial_number)

El número asociado con un producto individual que se usa para realizar el seguimiento y gestión. Por ejemplo, una empresa compra 10 equipos informáticos, y cada uno tiene asignado un número de serie individual.

Data type: Text

Required: No

Primary key: Sixth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
Número de lote
(inventory_on_hand_lot_number)

El número asociado con un producto que se usa para realizar el seguimiento y gestión del lote.

Data type: Text

Required: No

Primary key: Fifth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
Id. de ubicación
(inventory_on_hand_location_id)		 El Id. único de una ubicación específica dentro de una división o sitio en el que el inventario se recibe, mantiene o desde el que se envía. Data type: Text Required: No
Inventario mantenido por proveedores
(inventory_on_hand_inventory_held_for_vendors)

True, si el inventario está presente físicamente y lo han mantenido los proveedores; en caso contrario, false.

Data type: Enum

Required: No

Possible values: true or false.
Inventario mantenido por clientes
(inventory_on_hand_inventory_held_for_customers)

True, si el inventario está presente físicamente y lo han mantenido los clientes; en caso contrario, false.

Data type: Enum

Required: No

Possible values: true or false.
Inventario mantenido por otros
(inventory_on_hand_inventory_held_for_others)		 True, si el inventario está presente físicamente y lo han mantenido otros además de los clientes o proveedores; en caso contrario, false. Data type: Enum Required: No Possible values: true or false.
Inventario obsoleto o dañado
(inventory_on_hand_obsolete_or_damaged_inventory)

Este estado de inventario identifica cantidades de inventario con poco movimiento, dañadas u obsoletas.

Data type: Text

Required: No

Example values: slow moving, damaged, or obsolete
Id. de celda
(inventory_on_hand_bin_id)

El Id. opcional de una ubicación subordinada de inventario. Un código utilizado en un sistema de gestión de inventario.

Data type: Text

Required: No

Primary key: Fourth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>.
Ubicación de celda
(inventory_on_hand_bin_location)		 La descripción de la ubicación subordinada de inventario identificada en el Id. de celda. Data type: Text Required: No
Cantidad del inventario
(inventory_on_hand_quantity)

El recuento de inventario cuya unidad de medida se especifica en la UdM de almacenamiento de inventario.

Data type: Decimal

Required: No
Método de costes de inventario
(inventory_on_hand_inventory_cost_method)

La descripción de la práctica de costes, como LIFO, FIFO, promedio, estándar e identificación específica.

Data type: Text

Required: No
Coste de inventario
(inventory_on_hand_inventory_cost)

El coste por unidad usando el método encontrado en el campo Método de inventario.

Data type: Money

Required: No
Sistema de cantidad disponible
(inventory_on_hand_quantity_on_hand_system)

La cantidad disponible en el último recuento físico por sistema, o la cantidad calculada disponible.

Data type: Decimal

Required: No
Fecha inicial del inventario
(inventory_on_hand_inventory_as_of_date)

La fecha en la que comienza a hacer seguimiento de la cantidad disponible de un artículo del inventario.

Valor devuelto: Fecha

Required: No
Código de UdM de coste de inventario
(inventory_on_hand_inventory_costing_uom_code)

El código de la unidad de medida del coste del inventario.

Data type: Text

Required: No
Código de unidad de negocio
(inventory_on_hand_business_unit_code)

Se usa para identificar la unidad de negocio, región, sucursal, etc., al nivel en el que se estén auditando los estados financieros.

Data type: Text

Required: No

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
UdM de compra de inventario
(inventory_on_hand_inventory_purchasing_uom)

La unidad de medida principal utilizada para expresar cantidades de bienes comprados por la organización.

Data type: Text

Required: No
UdM de almacenamiento de inventario
(inventory_on_hand_inventory_stocking_uom)

La unidad de medida que se utiliza para realizar un recuento de la cantidades de artículos mantenidos en el inventario.

Data type: Text

Required: No
UdM de venta de inventario
(inventory_on_hand_inventory_selling_uom)

La unidad de medida principal en la que se expresan las cantidades de ventas de inventario.

Data type: Text

Required: No
Coste estándar
(inventory_on_hand_standard_cost)

Si se ha mantenido, el coste estándar por unidad.

Data type: Money

Required: No
Coste promedio
(inventory_on_hand_average_cost)

Si se ha mantenido, el coste promedio por unidad.

Data type: Money

Required: No
Coste específico
(inventory_on_hand_specific_cost)

Si se ha mantenido, el coste específico por unidad.

Data type: Money

Required: No
Recuento físico de cantidad disponible
(inventory_on_hand_quantity_on_hand_physical_count)

La cantidad disponible en el último recuento físico por inventario.

Data type: Decimal

Required: No
Fecha de último recuento
(inventory_on_hand_last_count_date)

La fecha del último recuento físico.

Valor devuelto: Fecha

Required: No
Precio de lista del inventario
(inventory_on_hand_inventory_list_price)

La lista o precio minorista sugerido

Data type: Money

Required: No
Compra de inventario hasta la fecha
(inventory_on_hand_inventory_purchase_year_to_date)

La cantidad comprada en esta ubicación hasta la fecha.

Data type: Decimal

Required: No
Venta de inventario hasta la fecha
(inventory_on_hand_inventory_sold_year_to_date)

La cantidad vendida en esta ubicación hasta la fecha.

Data type: Decimal

Required: No
Divisa de la ubicación
(inventory_on_hand_location_currency)

La divisa local.

Data type: Text

Required: No
