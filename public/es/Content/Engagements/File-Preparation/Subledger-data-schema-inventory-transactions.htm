Inventory transactions described

When importing subledger transactions from a CSV or Excel file you must assign fields in the file to Caseware fields in the subledger dataset type you select on import, so that the transaction data displays correctly in the engagement. This topic details the fields in the Inventory transactions subledger.

For more information on the subledger dataset descriptions, see Understand subledger data.

For more information on importing transaction data from CSV files, see Import the client’s transactions from a CSV.

Campo Uso
Id. de transacción
(inventory_transaction_transaction_id)

El Id. de transacción asociado con el Estándar de datos de auditoría (ADS) del libro auxiliar de Compra al pago o Pedido al cobro.

Data type: Text

Required: No

Primary key: The value of this field is used to form the unique identifier of this entry.
Fecha de transacción
(inventory_transaction_transaction_date)

La fecha de actividad, por documento de transacción asociada si corresponde.

Valor devuelto: Fecha

Required: No

Primary key: Third alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Id. de producto de inventario
(inventory_transaction_product_id)

El Id. del producto es un código que identifica de forma única un artículo que se incluye en el inventario.

Data type: Text

Required: Yes

Primary key: First alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Número de serie
(inventory_transaction_serial_number)

Si está en serie, el número de serie se captura aquí. Aunque el número de serie es el término común, el valor no se limita a valores numéricos. Si bien no es habitual que se controle el lote o serie de un artículo (el número de serie identifica de forma ambigua el artículo), no es necesariamente mutuamente excluyente.

Data type: Text

Required: No

Primary key: Fourth alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Número de lote
(inventory_transaction_lot_number)

Si representa un lote, el número de lote se captura aquí. Aunque el número de lote es el término común, el valor no se limita a valores numéricos.

Data type: Text

Required: No Primary key: Fifth alternative. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Tipo de transacción
(inventory_transaction_transaction_type)

Los códigos que describen las acciones específicas del inventario principal que tienen lugar en las transacciones de inventario relacionadas con el movimiento de bienes, el comercio, ajustes de cantidades de los eventos comerciales internos y ajustes de costes, debido a daños o factores del mercado.

Data type: Enum

Required: No

Example values: Goods Received (GR), Goods Shipped (GS), Inventory Transfers (IT), Inventory Adjustments (IA), Other (OT)

Primary key: Second alternate. If Transaction ID is not present in the data, the value of this field is concatenated with the values of the other alternate fields to form the unique identifier of the entry, for example <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>.
Sistema de tipo de transacción
(inventory_transaction_transaction_type_system)

Un código de transacción local para el sistema de software de contabilidad subyacente.

Data type: Text

Required: No
Hora de la transacción
(inventory_transaction_transaction_time)

La hora en la que transcurrió la actividad, por documento de transacción asociada si corresponde.

Data type: Date/time

Required: No
Cantidad de la transacción
(inventory_transaction_transaction_quantity)

La cantidad o artículo afectado expresado en la unidad de almacenamiento de ubicación.

Data type: Decimal

Required: No
Ajuste de coste de transacción
(inventory_transaction_transaction_cost_adjustment)

El aumento o (disminución) por unidad en el coste de almacenamiento.

Data type: Decimal

Required: No
Descripción de transacción
(inventory_transaction_transaction_description)

Una descripción de la transacción.

Data type: Text

Required: No
Id. de pedido asociado a la transacción
(inventory_transaction_transaction_associated_order_id)

El pedido del cliente, pedido de compra (proveedor) u otro documento asociado con la transacción. Se debe especificar si el número encontrado en el Id. de línea de pedido de la transacción asociada es de un Estándar de datos de auditoría (ADS) del libro auxiliar de Compra al pago o Pedido al cobro al utilizar el texto "P2P" u "O2C" respectivamente.

Data type: Text

Required: No
Id. de línea de pedido asociado a la transacción
(inventory_transaction_transaction_associated_order_line_id)

El número de artículo de línea de un pedido del cliente, pedido de compra (proveedor), para diferenciar varios artículos en un único pedido de diferentes cantidades. Se debe especificar si el número encontrado en el Id. de línea de pedido de la transacción asociada es de un Estándar de datos de auditoría (ADS) del libro auxiliar de Compra al pago o Pedido al cobro al utilizar el texto "P2P" u "O2C" respectivamente.

Data type: Text

Required: No
Id. de documento asociado a la transacción
(inventory_transaction_transaction_associated_document_id)

La lista de selección, aviso de envío u otro documento creado después/asociado con el movimiento. Se debe especificar si el número encontrado en el Id. de línea de pedido de la transacción asociada es de un ADS del libro auxiliar de Adquisición para pagar o Pedido al cobro al utilizar el texto «P2P» u «O2C» respectivamente.

Data type: Text

Required: No
Id. de línea de documento asociado a la transacción
(inventory_transaction_transaction_associated_document_line_id)

El número de línea de un documento que no sea un pedido del cliente ni un pedido de venta de "proveedor". El documento debe especificar si el número encontrado en el Id. de partida de la transacción asociada es de un ADS del libro auxiliar de Adquisición para pagar o Pedido al cobro al utilizar el texto "P2P" u "O2C" respectivamente.

Data type: Text

Required: No
Inventario desde la ubicación
(inventory_transaction_inventory_from_location)

The storage location from where the inventory originated.

Data type: Text

Required: No
Inventario hacia la ubicación
(inventory_transaction_inventory_to_location)

La ubicación del almacenamiento a la que se va enviar el inventario.

Data type: Text

Required: No
Fuente externa o sistema de inventario
(inventory_transaction_inventory_system_or_external_source)

Utilizado para indicar la información de asientos asociada con la transacción iniciada desde otro módulo de ADS (por ejemplo, P2P u O2C), o iniciado desde dentro del inventario. Si se inicia desde dentro del inventario, los siguientes campos se usan para capturar la información del usuario y del libro mayor.

Data type: Text

Required: No
Número de cuenta debe de línea del libro mayor de inventario
(inventory_transaction_gl_line_debit_account_number)

El número de cuenta del libro mayor en el que se ha publicado el lado debe de la transacción.

Data type: Text

Required: No
Número de cuenta haber de línea del libro mayor
(inventory_transaction_gl_line_credit_account_number)

El número de cuenta del libro mayor en el que se ha publicado el lado haber de la transacción.

Data type: Text

Required: No
Id. de proyecto
(inventory_transaction_project_id)

El identificador único del proyecto. Por lo general, el sistema lo genera de forma automática.

Data type: Text

Required: No
Id. del proveedor
(inventory_transaction_supplier_account_id)

El Id. único del proveedor. Por lo general, el sistema lo genera de forma automática.

Data type: Text

Required: No
Id. de cuenta de cliente
(inventory_transaction_customer_account_id)

El identificador único del cliente receptor. Suele generarlo automáticamente el sistema.

Data type: Text

Required: No
Código de organización de inventario
(inventory_transaction_inventory_organization_code)

El código de la organización del inventario. Organización de inventario se refiere a la unidad organizativa, física o virtual, en la que los saldos y la transacción de inventario pueden seguirse y monitorizarse. Proporciona información de inventario a módulos como compra y ventas. La forma más sencilla de organización de inventario es el almacén.

Data type: Text

Required: No
Código de UdM del existencias de inventario
(inventory_transaction_inventory_stocking_uom_code)

The standardized measurement unit used to count stock items and to express them as specific quantities.

Data type: Text

Required: No
Código de UdM de coste de inventario
(inventory_transaction_inventory_costing_uom_code)

El código de la unidad de medida del coste del inventario.

Data type: Text

Required: No
Inventario requerido por
(inventory_transaction_inventory_required_by)

La persona u organización que solicita el inventario.

Data type: Text

Required: No
Código de organización de coste
(inventory_transaction_cost_organization_code)

El código de la organización de coste, que se refiere al nivel o unidad organizativo en el que se calculará el coste de inventario de salida. El cálculo puede realizarse a nivel corporativo, a nivel de organización de existencias o en las ubicaciones particulares en las que se almacena el inventario.

Data type: Text

Required: No
Coste de inventario
(inventory_transaction_inventory_cost)

El coste por unidad que utiliza el método encontrado en Método de costes de inventario, en Inventario disponible.

Data type: Text

Required: No
Código de unidad de negocio
(inventory_transaction_business_unit_code)

Se usa para identificar la unidad de negocio, región, sucursal, etc., al nivel en el que se estén auditando los estados financieros.

Data type: Text

Required: No
Creado por
(inventory_transaction_entered_by)

El Id. de usuario de la persona que creó el registro.

Data type: Text

Required: No
Fecha de creación
(inventory_transaction_entered_date)

La fecha en la que la transacción se creó en el sistema. Se conoce también como fecha de creación. Esto debería ser una fecha generada por el sistema (en lugar de una fecha que haya introducido el usuario), cuando sea posible. Esta fecha no tiene que corresponderse necesariamente con la fecha de la propia transacción.

Valor devuelto: Fecha

Required: No
Hora de creación
(inventory_transaction_entered_time)

La hora en la que esta transacción se creó en el sistema. ISO 8601, que representa el tiempo en un formato de 24 horas (hhmm) (por ejemplo, 1:00 PM = 1300).

Data type: Date/time

Required: No
