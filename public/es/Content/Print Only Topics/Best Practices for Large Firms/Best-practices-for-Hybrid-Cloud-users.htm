Best practices for Hybrid Cloud users

In order to optimize Cloud site performance, we recommend Hybrid Cloud users follow the guidelines detailed below.

Use Cloud solely for engagement files

We recommend that Hybrid Cloud users use their Cloud site solely for engagements and minimize their use of other features such as Activities, Cloud Time, and Files. The Files app in particular can cause performance issues for users who are working on engagements if a large quantity of files are stored on your Cloud site.

Disable Time tracking for engagement files

Time tracking automatically tracks the time spent by staff members in engagement files. We recommend that Hybrid Cloud users disable this feature in Working Papers to optimize Cloud performance.

Note: Disabling Time tracking is only supported with Working Papers 2020: Update 1 or later. Users with earlier versions of Working Papers will not be able to disable Time tracking.

To disable Time tracking:

  1. Navigate to the Caseware directory.

    The default location for this directory is the Program Files (x86) directory.

  2. Open the CV.ver file using Notepad or a similar program.

  3. Add the following line to the file: DisableTimeEntries=1

  4. Select File > Save or press CTRL + S to save the file.

Set timeout settings

We also recommend adding the following settings to the CV.ver file to further improve performance.

To set timeout settings:

  1. Navigate to the Caseware directory.

    The default location for this directory is the Program Files (x86) directory.

  2. Open the CV.ver file using Notepad or a similar program.

  3. Add the following lines to the file:

    MasterAvailabilityTimeout=30000

    LongerMasterAvailabilityTimeout=40000

    SSSTimeout=50000

  4. Select File > Save or press CTRL + S to save the file.
