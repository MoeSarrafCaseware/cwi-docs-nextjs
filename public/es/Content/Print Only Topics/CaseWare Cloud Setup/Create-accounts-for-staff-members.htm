Create accounts for staff members

Once you create entities for each of your clients, grant Cloud access to other staff members in your firm. You can create a new account for each staff member who needs to collaborate on your engagements. After you create accounts for your staff members, you can assign them security roles to determine the level of access they have to your firm. The following table briefly describes the roles you can assign your staff members. For more details, see Built-in roles.

Funciones Ambito: Descripción
Admin System-wide

Assign the Admin role to Staff responsible for managing the entire site. Admins can access all content and perform any action. Admins must still be assigned a license to use apps that require one.
Configuración Admin System-wide

Assign the Settings Admin role to staff responsible for Firm settings and template authoring.
Personal Admin System-wide

Assign the Staff Admin role to staff responsible for managing staff. Staff Admins can add, view, edit, and delete all staff in the system.
Entidades Admin System-wide

Assign the Entities Admin role to staff responsible for adding entities and contacts. Entities Admins can add, view, edit, and delete all entities and contacts in the system, and all content within entities.
Propietario System-wide, Entities and content within them Owners can view, edit, delete, and grant others roles on content where they have this role.
Editor System-wide, Entities and content within them Editors can view and edit content where they have this role.
Visor System-wide, Entities and content within them Viewers can view content where they have this role.
Entity Access System-wide, Entities and content within them

Granting the Entity Access role to a user enables the assignment of a security role on content and grants access to the entity summary and details.

Practice set

Note: To add accounts for staff members, you must have either the Admin or Staff Admin role.

  1. Create an Excel file containing your staff member information with email, first-name and last-name columns.

  2. Guarde el archivo como Ejercicio 8E.

  3. From the Cloud menu (), select Staff.

  4. Select New and, under Import, select Staff.

  5. In the first two checkboxes of the Import Staff dialog, choose how the import will handle existing users.

    Note: Select the second checkbox only if you are importing a complete staff or contacts list.

  6. Choose whether or not your text or CSV file contains a header row and what type of delimiter is used.

  7. Select Import from CSV and browse to your Staff List.csv directory file then select Next

  8. Map the property that matches each column.

  9. Select Import.

A confirmation dialog appears showing the number of users successfully added or modified.

After you import a user list, Cloud will automatically send them an email with a link to their login page. A record of this email can be found in Settings | Email | Email Log.

On the login screen, they will be asked to enter their email address and to enter and confirm a password.
