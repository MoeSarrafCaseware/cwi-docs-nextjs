Recopilar información del cliente

Now that you have created the engagement, you will need to collect some information from your client. This is done using queries.

Complete the client queries documents

On the Documents page, in the Planning phase, you can select different queries depending on the information you require from your client.

Por ejemplo:

  • 1-55 Client queries - Engagement letter - This letter must be signed and returned by your client.

  • 2-03 Client queries - Information request - You can request a copy of the trial balance, payroll and other documents.

  • 2-03-1 Client queries - Understanding the business - For example, articles of incorporation, corporate bylaws.

  • 2-30-2 Client queries - General questions - For example, financial statement areas details, changes and clarification.

  • 2-30-1 Client queries - Records and documents - For example, bank reconciliations, bank statements.

Within a query document, if there are questions sets that are not relevant for your client or engagement, you can hide them to make the document more manageable. If you require additional information not listed in the preset queries, you can create your own query using the Client queries - Engagement letters, documents and records document.

Practice set 1 - Modify the query document

For this sample engagement, we only need the signed engagement letter.

Open the 1-55 Client queries - Engagement letter query. By default, the question summary and engagement letter are already included.

Depending on your request, you have the option to modify the description field. You can also select the More actions icon () to make additional changes. In this example, we will not make any changes.

Note: If you need to hide a question set, select the Visibility icon () beside the question, then Hide.

Practice set 2 - Request the documents from the client

Select the SEND button on the top right corner to display the Open Query dialog.

Choose a Query Due Date, and enter the following custom email message under Email Body:

Please provide the requested document. Thank you.

Assign the task to all contacts at the client firm or select an individual user, then select OK to send the query.

The status of the query changes to Awaiting response.

When your contact sends you the required documents, the query status changes to Response submitted. Select MARK RESPONDED.

Once all questions are marked as responded, a notification message displays featuring the MARK RESPONDED button so you can easily proceed to the next step in the workflow.

You can either select MARK RESPONDED or open the query from the Documents page. Preview the uploaded file before accepting the procedure and open the attached file by selecting it.

Once you have confirmed that the client's file contains the information that you need, select Accept to close the question set. Then, complete the query by selecting COMPLETE.

Go back to the Documents page. You'll see that the query is marked as complete, and the signed engagement letter has been added to the document list.

Import data using Cloud Import Utility

Before you import your client data, ensure you complete the prerequisite steps for your accounting software. See Prerequisites for importing from desktop accounting software for more information.

After you complete the prerequisite steps for your accounting software, you can import your client data using the Cloud Import Utility.

To import data using the Cloud Import Utility:

  1. Download the Cloud Import Utility.

  2. Open the downloaded file, then select Next.

    3. The Import From Other Data Source page displays.

  3. In the Import From and Version fields, specify the accounting software you're exporting from.

    The content of the page changes depending on your selection.

    Note: Refer to Supported third party accounting packages for the list of supported packages. If you find outdated packages in the list, report the issue to Caseware for investigation.

  4. Browse to the Import Path of the client file you want to import.

    If your accounting software requires you to specify a client file, you will see the Select Client ID File drop-down. If there is more than one file, be sure to select the correct one here.

  5. Select Next. The New File Wizard Complete page displays.

    This shows the location of a temporary file that will be used to generate the final .zip file with the imported data.

  6. Select Finish to perform the conversion.

    After the data is generated, the Save As dialog displays.

  7. Navigate to the desired location to save the .zip file and select Save.

    Note: If you're importing data into IDEA using the Cloud Import Utility, do not proceed to Step 8. Your next step is to import the zip file into IDEA. Consulte la Ayuda de IDEA para obtener más información.

  8. Upload the generated output file in your Data page.

    1. Go to the Import tab.

    2. Software de contabilidad de escritorio

    3. Browse or drag and drop the .zip file you've saved in Step 8.

      Tip: If transactions are available in the data, you can select the Import available transactional data checkbox to import them as well.

Practice set 3 - Excel or CSV

Instructor Note: Make sure the students are using an engagement file not a firm template.

To import client data from Excel/CSV file:

  1. Select Excel or CSV from the available options.

  2. If planning balances are enabled in your product and your engagement has an annual reporting period, choose whether you are importing a Final Trial Balance or a Planning Trial Balance and then select NEXT.

    If you select Planning Trial Balance, enter a date for the balance before selecting NEXT.

  3. In the import dialog, use one of the following methods to import the trial balance:

    • Drag and drop an Excel or a CSV file into the upload area in the gray dashed box.
    • Buscar el archivo en su equipo.
    • Select the file from the engagement if you have already uploaded it.

  4. Map the columns in the import file by choosing the field that best matches the data from the Import as drop-down menu. If you don't want to import a column, select Do not import to exclude it.

  5. Select Include zero-balance accounts if you want to include zero balance accounts in the import.

  6. Select Import.

    7. Your imported client data populates in the Accounts and group views in the Data page.

Practice set 5 - Annotate the trial balance

Instructor Note: Make sure the students are using an engagement file not a firm template.

While you are reviewing the trial balance, you can annotate accounts.

From the Data page, select the Columnsdrop-down and select the Variance check box. A Variance column displays in the Accounts view page. It shows the variance between the final balance in the current period and the final balance from the prior period.

Notice that in this example, there is no Previous value for the Bank - Savings account. You ask the client about this, and they inform you that the cash in the savings account came from sales that happened very late at the end of the year. You can add an annotation to include this information directly on the trial balance.

Select the Add Annotation icon () on the Bank - Savings line. A new dialog displays.

Under Purpose, select Variance, enter Late sales at end of year in the Note section and Save the annotation. The trial balance displays an icon for the annotation type and includes the description from the note.

Firmar el balance de comprobación y los ajustes

Puedes firmar:

  • Ajustes en la página Ajustes

  • El balance de sumas y saldos en las pestañas Cuentas y Grupos de la página Datos

Tenga en cuenta que si firma el balance de sumas y saldos, la información de firma (datos del firmante y fecha y hora) sólo estará disponible en la exportación en PDF de la página. Los resultados de la exportación CSV no incluyen información de firma.

Seleccione Firmar para firmar como preparador o revisor.

Para eliminar una firma, seleccione Eliminar.
