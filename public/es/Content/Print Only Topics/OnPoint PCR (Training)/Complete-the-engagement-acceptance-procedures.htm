Complete the engagement acceptance procedures

At this point, you have completed all the steps for the initial setup of your OnPoint PCR engagement:

  • created the engagement file

  • reviewed the engagement properties

You are now ready to begin working on the engagement file. Select the Documents page from the menu on top. This page is the starting point for most of the engagement work.

Engagement documents are grouped by five main categories:

  • Acceptance and Planning

  • Trabajo de campo

  • Financial Statements and Wrap-up

  • Permanent File

  • Exportación impuestos

Note: When you start a new engagement, there are only a few categories and documents available in the page. As you complete procedures and sign off documents, other documents populate in the Documents page.

Each category has a progress indicator that shows the percentage of signed off documents in that section.

In the Permanent File section, you can add supporting documents for the engagement. These documents will be carried forward when you create the next period's engagement for this client.

Complete the engagement letter

The first two checklist documents that you have completed outline the basic engagement information. A draft version of your engagement letter has automatically been generated based on your responses to the checklist documents.

At the top of the letter, you will see a guidance section. This contains information about the contents of the letter below, and includes instructions for finalizing the draft. Guidance is displayed by default, but it is not included when you print the letter. You can hide the guidance section by selecting More actions () |Guidance Off.

The engagement letter draft uses red text as placeholder areas where you have to enter information specific to your engagement. Review the letter and update any placeholder sections.

For example, the first text area contains firm information. The firm name and address are populated automatically based on values from your firm profile on Cloud, but there is red text here indicating that you should insert the firm letterhead.

Practice set

To edit a text area in the letter, select the Edit icon () at the top-right of the document.

The text area displays with a blue background, and you can select it to change the content.

There are drop-downs embedded in parts of the letter. Like the red text, the menus are placeholders for information that you need to add to the letter.

For example, in the second text area, you find the drop-down menu Today's Date.

The menu displays a calendar where you can select a date. Once you pick a date, it shows in the specified place in the letter. If you wish to change the selected date, select it again to display the calendar and pick another date.

If you go to the Your Responsibilities section and select an occurrence of the pronoun us, you'll find that the Data Link popup displays showing the Engagement Glossary tab. This denotes that this term is defined in the engagement glossary.

The engagement glossary contains terms, which are words and phrases that are used throughout the product deliverables. When a term is changed in the Glossary, it's changed every where it's used in the deliverables.

An engagement glossary entry can have more than one output form based on how the author has set it up.

In this letter, the term us is used because the following conditions are met in the 1-110 Engagement Acceptance and Continuance checklist:

  • You answered Yes to Is your firm operated as a sole proprietorship?

  • You answered Plural (We) to Would you prefer your firm to be referred to using singular or plural?

If you answered differently, the term me would have been used in the letter.

Send the engagement letter to the client

Now that the draft of your engagement letter is ready, you need the client's signature in order to proceed with the engagement. You can send your draft directly to your client in OnPoint PCR using a Query.

A Query is a document that you can use to communicate directly with the client. You can use queries to:

  • request information - send questions to the client.

  • request documents - ask the client to upload one or more files.

  • send deliverables - send documents to the client to sign and return.

If your client has a contact account in Cloud, they can sign in and directly respond to queries.

Add questions sets to a query

Question sets can help you organize your inquiries and requests. Use question sets if you want to group related query questions in the document. Question sets are mandatory in the Query document. You can't add a standalone question in the document, it must be part of a question set.

To add a question set, select Add () from the left navigation panel. A new question set is added in the document with a default title New Group. You can select the default title to change it; the title can be a data link. You can remove a question set by hovering over it and selecting the () icon or selecting it then selecting More actions () | Delete.

Note: If you delete a question set, all questions in the set are deleted as well.

In the left navigation panel, you can expand or collapse question sets.

Client contacts receive an email notifying them that there is an item in Cloud that requires their attention. When they log in, they can find the query under My Work.

Clients can download a copy of the PDF letter, review it, add their signature and upload the signed document back to the same query. They can also add comments to the query, which are visible in the engagement file to your firm members.

Once they have responded to all of the requests in the query, they can mark the query as Responded. This automatically updates the document status to Responded and the engagement file status to Queries Ready. Your client contacts no longer see the query in their Overview page, but they still see it in the Activities page.

Accept the signed engagement letter

When you review the query, you can see that the client has responded to the procedure, and you can either accept or decline the response. To accept the query, change the query status at the top of the document to Complete.

When you accept the query, the signed letter is attached to the file, replacing the placeholder in the Documents page. The query is now locked and can no longer be edited.
