Application Security Tooling

CodeQL

CodeQL is a static analysis tool developed by GitHub. It treats code as data and allows querying data flow patternshttps://docs.github.com/en/code-security/code-scanning/automatically-scanning-your-code-for-vulnerabilities-and-errors/about-code-scanning-with-codeql to detect the most common flaws based on the Common Weakness Enumeration, as well as some framework-specific vulnerabilities. We use the Security Extended query pack, and scans happen on every Pull Request.

Due to some technical limitations specific to Working Papers, CodeQL is used to scan the Server component (Smart Sync), but the desktop application currently is Scanned with Semgrep, while the aforementioned technical limitations are addressed by GitHub.

Semgrep

Semgrep is a fast, open-source static analysis tool for finding bugs, detecting vulnerabilities in third-party dependencies, and enforcing code standardshttps://semgrep.dev/. We use default presets, and scans happen on every Pull Request. Semgrep is used to scan both the desktop and server components.

Kics

Kics is an open-source solution for static code analysis of Infrastructure as Codehttps://kics.io/. We use default presets, and the scan happens on every Pull Request.

Dependabot

Dependabot is a tool that scans for known security vulnerabilities in third-party dependencies in Github. The development teams’ workflow includes triaging these findings on pull requests, while the Application Security team assists with interpretation and explanations as required for clarity.

Secret scanning

GitHub Advanced Security includes a Secrets scanner that prevents secrets from being introduced into our code base. We have Secrets Scanning enabled for all products with enforcement, so code cannot be approved if it contains secrets unless a developer explicitly whitelists that instance for a specific requirement. The Application Security team reviews all whitelisted secrets to make sure there are no impacts to confidentiality or sensitive information.

Veracode Static Analysis - Pipeline scan (deprecated in December 2022)

Veracode was previously used as a SAST scanner for this application. No new analysis has been performed on the application since December 2022, but the historical results are included in the report. Our methodology, process and tooling described with GitHub Advanced Security has been in place for Caseware Cloud since November 2021.
