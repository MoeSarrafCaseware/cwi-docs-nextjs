SE 2019-02 release notes

The features below are automatically enabled if the corresponding component or module has been enabled.

Creación

  • We've introduced the Use Default button in the firm author template management section which resets the firm template to the primary author template.

  • Authors can now set up input text placeholders for end users to fill out in letters, memos and financial statements.

  • To clarify the available letter case options for Engagement Glossary terms, we've added letter case examples.

Listas de verificación

Based on user feedback, groups and subgroups no longer automatically collapse once all procedures within a group or a subgroup are signed off.

Consolidación

To avoid confusion when creating a consolidation engagement, we've added a column showing the file name in the Consolidation Hierarchy search. You can now easily view and select the consolidation file.

Tablas dinámicas

  • Decimals are now supported in financial statements1.

  • You can now add input rows from the table directly without having to go to edit mode1.

  • You now view all visible groups (that have accounts) expanded if the Show available accounts setting is selected by the primary author.

Financials new design

  • The term Hidden and Hide which represents non-visible content has been revised. It is now replaced by Excluded and Exclude, a term more relevant to how users work with content visibility in financials.

  • In the Notes Library, the table short name now displays for notes that include dynamic tables. Note that if the table doesn't have a short name, the term Dynamic Table displays.

  • A new icon for adding sublevel notes is now available in note areas.

  • Firm authors can now view the content that's been hidden by the primary author.

  • To support the transition to the new financials design, we've enhanced the way you can move sections in the old financials design. You can now move a section to the end of a Grouping Area. For example, if you have a text section in grouping area A, you can now move it to the end of grouping area B (previously, you could only move to the top of grouping area B).

Balance de comprobación

Once the user imports the trial balance, it is now possible to automatically map accounts from any grouping structure (for example, Chart of Accounts) to Financial grouping2.

Fix

  • Fixed an authoring issue where the settings menu for the Risk Assessment report didn't display properly.

  • Fixed an issue where pressing Ctrl+Z (undo) in Checklist documents Guidance areas didn't work as expected.

  • Fixed various incorrect translation issues.

  • Implemented various fixes for dynamic tables related to:

    • incorrect title displaying for custom calculation columns.

    • erroneous XBRL export behavior.

  • Removed the Comment field in Query documents for authors. Query comments are obsolete for template authors as they're intended to help accountants or auditors communicate with clients throughout the information gathering process.

  • Fixed an issue where prior year adjustments didn't show in the trial balance reference copy PDF document.

  • Fixed an authoring issue where inserting multiple dynamic texts automatically added a new line after each dynamic text output.

  • Fixed an issue where some uploaded PDFs had blank pages.

  • Fixed an issue related to Carry Forward, where it wasn't possible to hide files on the Documents page unless they belong to a folder.

  • Fixed an issue where materiality conditions in visibility settings weren't evaluated properly.

1 Setup required by authors.

2 Distributors must provide a script for account mapping.
