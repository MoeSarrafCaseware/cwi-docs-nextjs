2019-07 release notes

The features below are automatically enabled if the corresponding component or module has been enabled.

Redesign of Query documents

Staff-contact collaboration is switching to a new and improved user interface. The new query documents design is now available.

The Query document redesign focused on enhancing style and visual appeal, improving usability, optimizing the layout and organization of components, as well as using clear and simple terms.

Here is a quick glance at the main features.

  • Term changes.

    • Query - previously used to refer to the document and the questions in the document. In the new design, the term Query only refers to the document. While Query Question now refers to the questions in the document.

    • Question Set - replaced the old term Group.

  • Question cards enclosing the question title and body, the response and comments.

    • With question cards, users can easily spot and identify items belonging to the same question.

  • A side navigation pane, where users can view and scroll through all the available query questions grouped by question set.

    • The navigation pane also features a single select and drag action. You can now easily rearrange question sets and query questions.

  • A simpler method for adding query questions.

    • You now select the desired response type as you add the query question in the document. You no longer need to access the settings dialog to add or modify a response type.

To enable the new query design in your product, go to Product Settings | Queries and select Enable new query layout.

Note: We strongly advise that you thoroughly test the existing queries in your product after enabling the new design and verify everything works as expected, before you enable the new design in the production environment.

Personalizar terminología:

You can now customize terms in your product. The Product Settings dialog now includes a new section Terminology, where you can customize terms.

In this release, you can customize the Preliminary and Final terms that are used by default in the Trial Balance page or the Data page (whichever is enabled in your product) and the Analysis tab in checklists.

Ajustes

Users can now export adjustments to a CSV file, a PDF, or to Working Papers (CSV ASCII format).

Estados financieros

In this release, we've added a placeholder indicator in the financial statements document and new features to support reconciliation. We've also enhanced rounding relations and included some feature enhancements for dynamic tables, trial balance group properties and notes.

Dimensiones

To support markets that need to include the Statement of Changes in Equity in their financial statements, you can now assign trial balance groups to specific dimensions within a category.

In the Group Properties dialog, a button is now available next to the Breakdown by dimension property, where you can exclude certain dimensions from the selected category.

Tablas dinámicas

** You can now:

  • View the group total row once you add a custom group in the table edit mode. Previously, the group total row didn't show in the table edit mode, causing users to accidentally add the group rows outside of the custom group.

  • Include data links in the column and row titles. The formula icon () is now available in the table edit mode.

Notas

You can now change the default title of the section (Notes to the Financial Statements).

Text and field placeholders

Users can now easily locate and replace placeholders in the financial statements. Similar to letters and memos, a placeholder indicator is now available in the document. It shows the number of empty placeholders and navigates users to the next empty placeholder in the document.

Herramientas de revisión

We've optimized the look and feel for Review Tools to improve usability.

The filter icon is no longer available. Once users select the Review Tools icon (), check boxes for the tools display so they can filter the view. They can also enable or disable the Review Tools using the available toggle.

Redondeo

** You can now select the financial group that you'd like the rounding differences to be applied to. Once selected, all the rounding differences are added to the group total. In the account details popup, an additional line displays showing the rounding details.

In carry forward, the currently selected group in prior years is retained. While, the currently selected group in the most recent prior year is rolled forward to the current year.

Por ejemplo:

if you've carried forward data for 2018, 2017 and 2016, where,

Cash was selected to hold the rounding differences in 2018

Liabilities was selected to hold rounding differences in 2016 and 2017

then,

Cash becomes the selected group for rounding differences in 2019

Cash remains the selected group for rounding differences in 2018

Liabilities remains the selected group for rounding differences in 2016, 2017

Conciliación

We've added new features to support reconciliation and help users easily spot and identify balance discrepancies to produce reliable, accurate, and high-quality financial statements.

  • In the dynamic table edit mode, you can now label the accounts that you want the application to compare their balances.

    • For example, you can label two records in two different tables in the document with the label 'Cash' to have their balances compared. Note that the comparison happens in real time.

  • Users can now easily spot records that don't agree with each other.

    • An icon displays next to each account balance with identified discrepancies. Clicking the icon displays a pop-up that shows the discrepancies details (account names and balances).

Note that if users attempt to sign off or print the financial statements document without reconciling the identified record discrepancies, they get a warning message.

With this feature, a new review tool Balance Check is also available. It marks all rows with identified balance discrepancies, and all rows where balances agree.(The warning on sign-off/print and Balance Check review tool features are no longer included in 2019-07, and will be available in future releases)

Note: This feature is only available for tables which follow the standard 3-column format (Description, CY, PY). It is not available in IFRS/SOCIE tables, tables displaying a break-down of dimensions or entities, tables displaying both parent and consolidated columns, or tables which display year-to-date and current period as separate columns.

Overwriting a calculated balance

Users can now override account balances sourced from the trial balance in dynamic tables. Changes are automatically reflected in all tables in the document, as well as variance calculations (if available).

For overridden balances, the account details popup shows the overridden balance and includes an additional line showing the original balance.

Note:This feature is not available in IFRS/SOCIE tables or tables displaying a break-down of dimensions or entities

Letters, memos and financial statements

Users are now prompted if they attempt to print or bulk print letters, memos, or financial statements document with empty placeholders.

The prompt notifies users of the number of placeholders that haven't been replaced.

Vaya a la página Datos .

We've enhanced styling and reviewed terms in the Data page to improve user experience.

We have also included features from the old design (the Trial balance page), as well as new features:

  • Support for Tax and Prescribed Accounts (Chart of Accounts) grouping structures is now available, with a designated view for each grouping.

  • A navigation path now displays at the top as users navigate through groups, accounts and transactions so they can easily navigate back and forth.

  • The default view of the page is now based on the user's progress in the engagement.

    • If the user hasn't imported the financial data, then the Data page opens defaulted to the Import view.

    • If the user imported the financial data and assigned accounts (if need be), the Data page opens defaulted to the Accounts view. Users are also redirected to the Accounts view upon successful import of data.

    • If the user has been working in the Groups view before leaving the Data page, it opens defaulted to the Groups view.

  • Users can now:

    • Export trial balance data to Working Papers (CSV ASCII format) or a PDF.

    • Import and view prior year data for up to 5 years.

  • The link to transactions on accounts is now only active if users have successfully imported the transactions data.

  • In consolidation engagements, a separate view for the parent entity data is now available. For each grouping, users can now select Consolidated to view the consolidated data or Parent Entity to view the parent entity data only.

  • You can now import the following group properties using a CSV file.

    • Contrary Name (Text)

    • Inconsistent Name (Text)

    • Dimension Category (one of the dimension categories defined in Product Settings)

    • Reassign Group for contrary accounts (one of the groups in the Financial Groups structure)

    • Group Classification (one of the automap keywords)

    • Preference (Number)

    • Restriction (one of the values of the Account Restriction property)

    • Link to a Tax Group (Number)

    • Link to a Chart of Accounts Group (Number)

    • Cycle (Text)

    • Area (Text)

  • We've enhanced the search functionality for account assignment to improve performance. Matching results now display after users enter at least 3 characters in the search field.

The Documents page

You can now set the Detailed View to be the default view in the Documents page using the product's API. To learn more, see the detailedDocumentsView property in SDK documentation.
