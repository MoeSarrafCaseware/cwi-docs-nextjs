Simple Electronic Signature in queries

#enduser #primaryauthor #firmauthor

To support scenarios where clients are required to sign documents or accept certain terms during the engagement, a new response type Simple Electronic Signature is now available in queries.

To include the Simple Electronic Signature response type in your product template, select the check box in the Product Settings.

Simple Electronic Signature provides a paperless, secure and more efficient method for signing documents. Once you send the query to clients, they can sign the document or agree to terms with a single tap or button click.

Once clients accept the document, a signature log is generated and available for download. The log includes the following information:

  • Query question details
  • Signer name and email
  • Signer location (if the clients have chosen to enable location access)
  • Signer IP Address
  • Signature date
  • Título
  • Document MD5 hash

To learn more, see Enable the new Queries layout and Add a query template (new design).

Documents pane moved to the right

The Documents pane is now available at the right side of the page instead of the left side. Moving the pane is a strategic decision to complement:

  • The global documents search functionality on the top right corner of the page

  • The redesign of the Documents page. To learn more, see New design for the Documents page.

You can open and close the Documents pane using the blue documents icon () on the top right or the arrow on the bottom ().

Signoff options for risks

#enduser #primaryauthor

A new setting Signoff Option is available for the new risk module to allow you to determine how users sign off the risk report.

Two options are available for signoff:

  • Sign off the entire report - One signoff button is available in the report for users to sign off the entire report.

  • Sign off each risk in the report - A signoff button is available in each risk card in the report. Users sign off each risk individually.

You can select both check boxes to apply both options in the report.

To learn more, see Enable the new risk module.

Increased file upload size limit

#enduser #primaryauthor #firmauthor

You can now upload up to 500M files to the engagement. Note that this size limit is per file.

Rearranging documents in bulk print

#enduser #primaryauthor #firmauthor

You can now drag-drop documents in the bulk print set to rearrange them, so they print in the order you desire.

Note that you can only rearrange documents in groups that you define. That is,

  • Firm authors cannot rearrange documents in groups predefined by primary authors

  • Users cannot rearrange documents in groups predefined by the firm authors or primary authors

To learn more, see Set up bulk print.

New design for the Documents page

#primaryauthor

To support large engagement teams that perform large and complex audit engagements, a redesign of the Documents page is available in SE 2020-09 to cater for extending the signoff functionality - beyond the existing one preparer and one reviewer.

The extended signoff functionality will launch in a future release, however, changes have been made to the user interface of the Documents page to better present and display the signoff information.

Note: The new Documents page design is only available for distributors for beta testing. It will not be deployed to partner or production environments. To view the new design, make the following change to your URL:https://<server name of your partner site>/<firm name where you perform your authoring>/e/eng/<identifier>/index.jsp#/se-documents

The new design features a left navigation pane, where phases and their progress indicators display. If you hover over a phase, a visual presentation of its details pops up. Folders and subfolders along with their content are enclosed within faint borders so you can easily tell which items belong to which folder or subfolder.

Smart Copy-Paste

#primaryauthor

Conflicts within visibility settings, dynamic text, formulas and engagement glossary terms in financial statements are now flagged.

If there are missing or inconsistent data, the Broken Links pane becomes available and displays a list of the identified conflicts grouped by document name. You can access the Broken Links pane by selecting the broken link icon ().

To view a conflict, select it to launch a dialog showing the conflict details and presenting you with options to resolve it.

If you want to resolve the conflict later, select CANCEL to close the dialog. To resolve a conflict, you can either:

  • Choose alternate values from the drop-downs

    • That is, if an engagement glossary term copied over from a source document does not exist in your cloud app, select a term from your app's glossary to insert it in the document instead.

  • Select Delete () to remove the item where the conflict resides

    • That is, if an engagement glossary term copied over from a source document does not exist in your cloud app, remove it from the document.

To learn more, see Resolve conflicts in financial statements.

New for end users

Here is what's new for end users in SE 2020-09.

Listas de verificación

If signoff is enabled for procedures and conclusion, a warning message now displays if users attempt to sign off on the document while,

  • One or more visible procedures have not been signed off

  • The Conclusion section has not been signed off

  • Visible procedures in the Conclusion section have not been signed off

Risks module

Risk assessments can now be carried forward.

Aprobación

The logic for preparer and reviewer signoff has been enhanced. Users can now only remove their own signoffs.

A user that has been granted one or more administrator security roles can remove signoffs by other users.

Administrator security roles in Caseware Cloud are:

  • Admin

  • Configuración Admin

  • Personal Admin

  • Entidades Admin

To learn more, see Assign system wide security roles.

Fix

The following listing features a brief description of bugs that have been resolved in SE 2020-09.

Listas de verificación

  • Styling issues in the Analysis tab in checklist documents.

  • Firm authors can no longer modify authoritative references in procedures that have been authored by primary authors.

  • When setting up the Analysis section, users and authors can no longer select a figure that has not been properly defined in the API. An error message displays if the selected figure does not have a unique ID.

Página de datos

  • Incorrect current and prior years display in the engagement once users import data from the accounting software BGL360. For example, data imported for the financial year 2020 shows 2019 as current year and 2018 as prior year in the engagement.

  • The Transactions tab is not available in the Data page after a successful data import from a desktop accounting software. Users need to refresh the page to display the Transactions tab.

  • In locked down engagements, the UI elements in the Data page are not in a read-only state.

Estados financieros

  • Users are unable to print financial statements document in an engagement with large trial balance data.
  • Balance discrepancies in dynamic tables are not flagged for rows with calculated values. Disagreeing balances are not flagged with the stop sign icon () .
  • The logic for balance discrepancy detection in dynamic tables does not include a group Total row if one or more row within this group is hidden.
  • In dynamic tables, the total variance percentage for grouped input rows does not display in the financial statements document. It only displays in the table edit mode.

Risks module

  • Signoffs in risk assessment cards are removed once users expand a card.

  • Query templates no longer display in the audit response and audit reference drop-downs.
