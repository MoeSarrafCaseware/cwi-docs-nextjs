SE 2020-12 release notes

Here we go again…

SE 2020-12 officially launches the new Documents page layout. The second iteration of the documents page redesign incorporates user experience enhancements and extended signoffs. With that, multi-language support and a load of features are also in motion.

New Documents page official launch

#enduser #primaryauthor #firmauthor

SE 2020-12 witnesses the official launch of the new Documents page layout.

A new setting is now available in the product settings Features | Documents, so you can enable or disable the new Documents page layout. To learn more, see Enable the new Documents page layout.

If you enable the new Documents page:

  • The Second Reviewer button is no longer available in your product.

  • The More actions () | Sign-off option is no longer available in your product.

  • The Signoffs tab automatically displays in the product settings dialog, so you can configure custom signoffs in your SE app.

Note: If you make changes in the Signoffs tab (modify the defaults or add schemes/roles), you can no longer disable the new Documents page layout in your product.

You can quickly switch to the old Documents page layout, replace the following part of your URL (highlighted below) with /legacy-documents.

Carry Forward is also supported on the new Documents page.

Roles and signoff schemes

#enduser #primaryauthor #firmauthor

Let's move beyond the simple one preparer and a maximum of two reviewer signoffs in SE products. You can now create custom roles and sign-off schemes to track the required preparer and reviewer workflows on a document.

Roles and sign-off schemes also cater for:

  • Documentation and audit trail requirements (at the market level or firm level)

  • Large firms that have large engagement teams where extended review levels are needed to track progress and validate the work

A new tab Signoffs is now available in the product settings dialog. To create a custom signoff, you must set up roles and schemes:

  • Roles, where you define the preparer role labels and the reviewer role labels. You can add as many roles as needed.

    • For example, you can define Jr. Preparer and Sr. Preparer as preparer roles and Reviewer, Partner, ECQR as reviewer roles. To learn more, seeDefine signoff roles.

  • Schemes, where you set up the applicable combination of preparer roles and reviewer roles for a specific document or a group of documents. You can create as many schemes as needed.

    • For example, you can define a scheme called Requires EQCR and include one preparer role and three reviewer roles (Reviewer, Partner and EQCR). To learn more, see Set up signoff schemes.

Note: The Signoffs tab is only available if you enable the new Documents page layout.

You can then apply schemes to documents by selecting More actions () | Edit. A new option Sign-off Scheme is now available in the Document Properties dialog. It shows a grid with brief details of the available schemes in the product. To learn more, see Review and customize signoff schemes.

Note: The Sign-Off Scheme option is also available for firm authors and end users. To learn more, see How can staff and firm authors review and customize signoff schemes.

Multi-language support for deliverables and content

#enduser #primaryauthor #firmauthor

Multi-language support aims at facilitating SE product offerings in regions and markets where there is a requirement to present deliverables and content in multiple languages.

A new tab Content Language is now available in the product settings, so you can:

  • Agregar idioma

  • Specify the default language (the language that will be used once users create an engagement)

  • Deactivate languages (that are yet to be introduced to your market for instance)

  • Delete languages that are no longer required

The Content Language tab is also available in the firm template, so firm authors can:

  • Cambie el tipo predeterminado.

  • Deactivate languages that are not applicable in their firms

Once authors set up multi-language, the Content language drop-down is available showing the preset default language. Authors and end users can select the desired language for content and deliverables.

A new visibility condition Language is also available to support the multi-language functionality. Authors can set up the Language condition where applicable in the content and deliverables to display the translated text - following the selected content language.

On the Data page, a new option Bulk change group names is now available to allow you to add the translated group names.

To learn more, see What is multi-language support, Recommendations, and Use-case: A product with French and English language support.

Data links (fx) in group names and question set names

#enduser #primaryauthor #firmauthor

Group names in checklists and question set names in queries can now include data links (fx).

Underline and overline options in input rows

#enduser #primaryauthor #firmauthor

To support specific market requirements for financial statements presentation, underline and overline options are now available in dynamic tables input rows.

Previously, underline and overline options were only available for total rows.

Flipped accounts in data links (fx)

#enduser #primaryauthor #firmauthor

Flipped accounts are now supported in data links (fx) referencing trial balance data. A new check box Display opposite sign is now available in the Trial Balance tab in the Data Link dialog.

If the referenced account is flipped, then selecting the check box displays the account in the negative sign format.

New for authors

Here is what's new for authors in SE 2020-12.

Riesgos

#primaryauthor #firmauthor

You can now choose whether or not to include a text field in risks analysis widgets for users to add comments or notes. A new check box Include the comments field in risk analysis widgets is now available in the Risks | General tab in the product settings.

Previously, this text field was always available in analysis widgets.

New for end users

Here is what's new for end users in SE 2020-12.

Página de datos

User selections for data filtering in group tabs are now retained.

If users select options in the Column and Indicators filters then close the engagement, the selected options are retained when they reopen the engagement.

Consultas

Uploading a manually signed document is now available In Simple Electronic Signature response types. Clients can now choose whether to:

  • Accept electronically - Simply select the Accept button to sign

  • Upload signed document - Print the attached document, then sign and upload

  • Copy signed document from Cloud - Select the previously printed, signed and uploaded document from the firm's Cloud site
