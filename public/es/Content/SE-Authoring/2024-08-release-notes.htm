Planificación central

#enduser#primaryauthor#firmauthor

The SE 2024-08 release introduces the central planning feature. Central planning allows users to sync risks, controls and selected checklists across engagements so the work doesn’t need to be duplicated in multiple engagements. To enable central planning in your product from the product settings, select Enable Central Planning on the Central Planning tab. You can then select which checklist documents in your template are synced. All risks and controls are synced by default.

Firm authors are also able to modify which checklists are included when syncing to a central planning engagement. This allows firm authors to customize central planning to suit their firm and needs, as well as include firm author-created checklists.

Once central planning is enabled in your product, users can then select Set as Central Planning Engagement in their engagement settings to designate engagements as central planning engagements.

Note that this action is irreversible. Once an engagement is set as a central planning engagement, the checkbox cannot be deselected.

To create an engagement that is synced to the central planning engagement, users can navigate to the central planning engagement in their cloud site and use the Create Engagement from Source option. Risks, controls and selected checklists in this new engagement will then automatically be synced from the central planning engagement.

In both central planning engagements and destination engagement that are synced to the central planning engagement, the Central planning () icon displays in the top menu bar. Note that the icon also displays next to synced checklists.

Users can select the Central planning () icon in the top menu bar to open the Central Planning Hub. If the user is in the central planning engagement, they can view a list of the destination engagements and whether they are up to date. If they’re in a destination engagement, this page informs them if any changes have been made in the source engagement and allows them to sync those changes.

If a user opens a synced checklist in a destination engagement, a message displays informing them that the document has been completed in the central planning engagement with a link to the checklist in that engagement. Risks and controls will continue to display in the destination engagements but in a read-only state.

To learn more, see Enable central planning. To view end user and firm author documentation about central planning, see Set up central planning and Manage synced checklists for central planning.

Ajustes recalculados

Users now have the ability to create calculated adjustments. When they add an account or group line to an adjustment, they now have the option to select fx in the Amount field and create a calculation to determine the adjustment amount.

Similarly to custom calculations in dynamic tables, users can reference trial balance data in their calculation. If the value of the referenced trial balance account or group is changed, a Recalculate button appears for the adjustment on the Adjustments page. Users can select this button to recalculate and update the adjustment amount.

If more than one adjustment requires recalculation, users can also select the Recalculate all button at the top of the page to recalculate all adjustments at once.

To learn more, see Post adjustments to the trial balance.
