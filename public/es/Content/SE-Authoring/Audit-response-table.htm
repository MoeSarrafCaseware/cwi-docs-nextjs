Audit Response Table

Note: a new enhanced version of Audit Response Table was released. To learn more, see Dynamic Audit Response Table.

The Audit Response Table is an SE Builder custom form that can be added to checklists to assist users in documenting the appropriate nature and extent of audit procedures to address each relevant assertion in order to meet compliance requirements. The Audit Response Table includes the following key features:

  • Information about the assessed risks at the assertion level for each area flows directly from the risk assessment report.

  • Users can document the extent of audit procedures for each assertion and each group of procedures.

  • Embedded warnings inform users if they are performing excessive (Overauditing) or insufficient (Underauditing) procedures or if any changes were made to the assessed risks or to the audit plan after sign-off.

Once the Audit Response Table has been integrated with your solution, you can add it to your checklist from the More actions () menu.

The Audit Response Table also allows you to set visibility settings on procedures based on responses to the Will you be testing controls options available for each area in the risk assessment report.

Note: In order to add the Audit Response Table to individual checklists, you must first add it to the Documents page. You can hide the form after you add it to the Documents page but cannot remove it.

To learn more about the Audit Response Table and how it integrates with SE checklists, see the Audit Response Table Demo recording.

To learn more about how to request access to the Audit Response Table custom form and how to implement the Audit Response Table in your solution, see Get started with the Audit Response Table.

Dynamic Audit Response Table

The Dynamic Audit Response Table is a custom SE Builder form that you can add to checklists. The table assists you in documenting the appropriate nature and extent of audit procedures required to address each relevant assertion in order to meet compliance requirements.

Note: The Dynamic Audit Response Table is an enhanced version of the previous Audit Response Table released in December 2022.

The Dynamic Audit Response Table:

  • Seamlessly integrates into your solutions without extensive development effort. For more information on how to integrate the Dynamic Audit Response Table into your solutions, see Get started with the Dynamic Audit Response Table.
  • Dynamically reads the information about the risk assessment scales and assertions defined in the product settings by the template authors.
  • Includes a configuration settings form to define the required information prior to embedding it into the audit checklists.

Once the Dynamic Audit Response Table has been integrated into your solution, select Add () on your Documents page and add the following documents:

  • Dynamic Audit Response Table - the main repeater SE Builder form that should be added to the Documents page prior to embedding it into the audit checklists. You can hide this form after you add it to the Documents page, but you cannot remove it.
  • Audit Response Table Settings - the configuration settings form required to define the applicable information based on your app specifics prior to embedding the Audit Response Table into the audit checklists.

Required Primary Author changes

Audit Response Table Settings

Assessed risks at financial statement level

If applicable to your solution, insert the procedure ID where the assessed risks at the financial statement level are documented. For instructions on how to identify the procedure ID, see Identify the procedure ID.

Note: If the assessed risks at the financial statement level are not documented in your app, leave this field blank.

Specify both the URL and the name of the document where the assessed risks at the financial statement level are documented to create the reference within the DART table.

Summary of proposed audit responses

Document the groups or subgroups of procedures that are included in the audit program checklists that you want to display in the Dynamic Audit Response Table. The order you enter the information will display in the Dynamic Audit Response Table in the same order.

Nota:

  • The Dynamic Audit Response Table can display up to a maximum of three groups or subgroups of procedures.
  • Ensure the procedure group or subgroup name inserted exactly matches the name of the group or subgroup as it was authored in the audit programs checklists.

Embed the Dynamic Audit Response Table into the audit checklists

Once the Author Settings - Dynamic Audit Response Table form is complete, you can now embed the form into the audit checklists.

For more information on how to request access to the Dynamic Audit Response Table custom form and how to integrate it into your solution, see Get started with the Dynamic Audit Response Table.
