Dimensions Allocation Worksheets

Dimensions allocation worksheets have been developed to assist users in completing Nonprofit Organizations engagements. In other jurisdictions, these may be referred to as NFPs (Not-for-Profit entities) or Charities.

These forms are intended to help users allocate financial groupings balances across dimensions in situations where the Trial Balance does not include those details at the account level.

Once the Dimensions allocation worksheets have been integrated with your solution, you can add the forms by selecting Add () menu on your Documents page. There are two SE Builder forms:

  • Dimension allocation worksheets summary - A repeater type document that can be duplicated as many times as required. One instance of this form is intended to be used to allocate the balances across one Dimension category at one time.

  • Dimension allocation worksheet - A repeater SE Builder form which will include all of the allocation worksheets instances generated from the Dimension allocation worksheets summary documents. You must add this worksheet to your Documents page but you can always hide it by using the visibility settings.

To learn more about how to get access to the Dimensions allocation worksheets custom forms from the Global Components library, see Get started with Dimensions allocation worksheets.

These worksheets seamlessly integrate with SE platform functionality around Dimensions, without requiring any additional code changes or Primary Author configurations for the use cases they are intended to solve.

The concepts used to develop these worksheets can also be used for other types of allocations with some changes. If you are planning to make significant changes to the content or functionality of the SE Builder forms, such as adding, removing or modifying functionality, you will need code ownership. Refer to Request access to the source code for the Dimensions allocation worksheets.

Note: If you take code ownership, you must maintain the SE Builder forms going forward, which includes manually applying any future changes, updates and bug fixes that are deployed for the original form source.
