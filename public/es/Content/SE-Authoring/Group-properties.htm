Propiedades de grupo

The following listing describes each property in the Group Properties dialog.

Número

Enter a unique mapping number. If you have already imported groups, the mapping number must follow the existing mapping mask structure.

Nombre coherente:

Enter a description for the group to be displayed if the balance is consistent with the group's normal sign. This description populates in the Preliminary column in the financial statements.

Nombre de contrapartida

If applicable, enter a description for the group to be displayed if the balance is opposite to the group's normal sign. This description populates in the Preliminary column in the financial statements.

For example, "Bank Overdraft" will appear instead of "Bank" when the balance is a credit.

Nombre inconsistente

If applicable, enter an alternative description for the group to be displayed when the current and prior year have opposite signs. This description populates in the Preliminary column in the financial statements.

Note: This name takes precedence over the consistent name and contrary name regardless of the group balance in the Trial Balance.

Signo

Select the applicable sign (debit or credit).

Tipo

Select the appropriate statement type for this group (Balance Sheet, Income Statement of Comprehensive Income).

Fórmula

If applicable, enter an AOCI formula to calculate the values for this group. Once your user imports the trial balance, this group will display an amount derived from the balances in other groups.

Sample AOCI formula:

Tagbal("1.234",perspec({yearOffset}),{type},{code})

, where 1.234 is the OCI group in the Income Statement section

Desglose por dimensión

If applicable, select dimension categories for the group from the Breakdown by dimension drop-down (values in the list are defined in Product Settings). This enables users to filter the trial balance so that they can view balance information by dimension.

Incluir dimensiones

This button is only available if a dimension category is selected in Breakdown by dimension. It allows you to exclude specific dimensions from the selected category.

The button displays the available dimensions in the selected category. You can exclude a specific dimension by clearing the check box next to it.

Cycle and Area

Select a business cycle and area for the group Area/Cycle drop-downs (values in the drop-downs are defined in the Product Settings). When you assign a group to a business cycle, items in that group will automatically be shown under this cycle in the Risk Assessment document, making it easier to audit by business cycle.

Categorías de componentes

Component categories organize the components in your product for multiple industries, entity types or reporting frameworks. For example, you can group components related to industry type in a category and components related to reporting frameworks in another category. Para más información, consulte Añadir zonas.

Select one or more categories to assign to the group from the Component Categories drop-down (the categories in the drop-down are defined in the Product Settings). Once you assign a category to a group, it's included in the Group Properties dialog and you can assign the applicable components.

Componentes

Components help you set up a market-driven, region-driven or product-driven visibility logic for trial balance groups. For example, if the entity operates within the Agriculture industry, then display trial balance groups that are relevant to agriculture. Para más información, consulte Añadir zonas.

Before you can assign components, you must select one or more component categories. Once you select a category for the group, it's included in the dialog. Select one or more components to assign to the group from the category drop-down (components in the category drop-down are defined in the Product Settings).

If you associate multiple components to a trial balance group and define visibility settings for each one of the components, then the visibility logic varies based on whether the assigned components belong to the same category or different categories:

  • If the assigned components belong to the same category, the trial balance group is visible if at least one component is visible - otherwise the group is hidden

  • If the assigned components belong to different categories, the trial balance group is visible if at least one component from each category is visible - otherwise, the group is hidden

One exception to the above rules is if users add an account, select the Show Hidden check box (in the Assign Accounts tab) and assign the new account to a hidden group. In this case, the group and its parent groups become visible.

For example, let's assume that the group Operating expenses is hidden due to component visibility settings. The user creates an account Other expenses and assigns it to Operating expenses. In this case, Operating expenses and its parent groups automatically become visible.

Estructuras de grupos vinculados

Select groups in other grouping structures to link the financial group to it. This enables you to automatically assign accounts to groups in other grouping structures based on their initial group assignment.

Permitir asignación de cuenta

Select this check box to allow accounts to be assigned to this group.

Reasignar objetivo para cuentas de contrapartida

If applicable, select a group ("reassign target") for accounts with a contrary balance. If an account would be automatically assigned to this group but it does not match the selected sign (debit or credit) for this group, it will be automatically assigned to the reassign target.

For example, negative Asset --> positive Liability

Clasificación

Type or select a classification from a master chart of accounts. This master list of classifications allows direct automapping from an accounting package by matching the classification applied to the group to the account-type in the accounting package.

Ajustar preferencia de grupo

Weight is used to automap accounts to a group based on a scale. For example, groups with a positive weight will be preferred over groups without a weight assigned.

Restricciones al saldo de cuenta

Specify the types of accounts that can be assigned based on their balance:

  • Cuentas con cualquier saldo que asignar
  • Solo se pueden asignar cuentas con saldo (positivo) de débito
  • Solo se pueden asignar cuentas con saldo (negativo) de crédito

Grupo primario

Select the higher level (next level up) group in the structure.

Grupo predeterminado

This property allows you to mark the group as one of your Default Groups or preferred groups. If the Default Groups Only option is enabled in the Groups View (Financials or Tax, or Prescribed), then only the default groups display in the page.

For example, if you have a large grouping structure, you can mark the most common groups that you work with (your preferred groups) by selecting the Default group check box. At the Groups View, select the Default Groups Only option to only display the marked groups (the ones that have the Default Group check box selected) in the page.
