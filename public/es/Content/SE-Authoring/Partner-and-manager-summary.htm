Resumen de socios y gerentes

Partner and manager summary (PMS) is a document designed to provide managers and partners with a high-level view of the key information that drives audit decisions for an engagement. PMS can be customized based on the solution specific settings.

Once the Partner and manager summary (PMS) is integrated with your solution, select Add () on your Documents page and add the following documents:

  • Partner and manager summary - The main form which includes all functionality, including primary author configuration settings and all the widgets.

  • PA - Partner and manager summary worksheet - An SE Builder form that is required to support the functionality of the main form. You must add this worksheet to your Documents page but you can always hide it by using the visibility settings.

  • GC - Author mode visibility form - An SE Builder form that drives the visibility settings on the content of the form based on the type of author that is accessing it. You must add this worksheet to your Documents page but you can always hide it by using the visibility settings.

    Note: The GC - Author mode visibility form is a form that is used by other Global Components forms. If it was previously added to your solution, there are no further actions required at this step.

Configuración

Note: Primary author configurations required.

  • The Settings table allows primary authors to customize the content of the Partner and manager summary (PMS) based on the solution it is integrated into.

  • The Form tabs column includes a list of tabs names that form the structure of the PMS document. Use this column to modify the tabs names based on your solution specific requirements.

  • In the Widget column, there is a list of available widgets for each of the tabs. Choose which widgets to include based on your solution specific requirements.

  • Engagement phases allows primary authors to align SE Engagement document phases with PMS tabs. You can select one or multiple phases that correspond to any particular tab. This information is used throughout the rest of the PMS functionality to properly filter the correct information for each of the applicable widgets. To find out more about Engagement phases, see Define the engagement phases.

  • Folders allows primary authors to further filter which folders align with a particular tab and be used throughout the rest of the PMS functionality to properly filter the correct information for each of the applicable widgets. By default, all folders associated with an Engagement phase will be selected. Note that if the Engagement phase selection is changed, then the Folders selection should be revised.

Nota:

  • This form does not support adding multiple tabs through the primary author settings.

  • Widgets included are designated to individual tabs and cannot be moved across tabs.

  • Each widget is exclusively designed for this form and cannot be embedded into other areas of the SE engagement.

If you are planning to make significant content or functionality changes in SE Builder forms, such as adding, removing or modifying functionality, you will need code ownership.

Note: Assuming code ownership means you will be responsible for maintaining the SE builder forms going forward. This includes manually applying any future changes, updates and bug fixes that are deployed for the original form source.

For more information, see Request access to the Partner and manager summary.

Below we will detail how each widget works and how it should be configured, where applicable.

Planificación

Información financiera

Note: Primary author configurations required.

The Financial information widget includes Analysis graph information selected to be presented in the PMS. To learn more about analysis modules, see Setting Up the Analysis Module.

As the primary author, specify the following:

  • The URL of the Analysis document that you want to include in the PMS document.

  • Both the URL and the name of the Analysis document to create the reference within the PMS document.

Note: The Financial information widget supports the addition of one Analysis document and will display all the information, such as graphs, included within that document. Depending on the extensiveness of your Analysis documents, you may want to create a separate Analysis document with smaller subsets of graphs to be included in the PMS Financial information widget.

Overall engagement information

Note: Primary author configurations required.

The Overall engagement information widget lists out the key responses from the engagement based on the information defined in the table below:

  • Procedure ID - Insert the procedure ID of the information you want to be presented. For instructions on how to identify the procedure ID, refer to Identify the procedure ID.

  • Procedure name - Provide the description that you want to be presented in the table. If no description is provided, the default text of the procedure will be used.

  • Yes / No response - The checkbox allows selecting Yes or No as a response based on the procedure type. It will reflect Yes for procedures responded to as Narrative or Procedure and No for those responded to as N/A. This check box is specific to certain types of procedures that entail Narrative, Procedure or N/A as inline responses. For other procedure types, the checkbox should remain unchecked.

  • Edit severity color - Select the pencil icon to define the color of the responses based on their severity. For example: High - Red / Medium - Orange / Low - Green.

Documents requiring specific role sets

Note: No primary author required.

The Documents requiring specific role sets widget lists out all the documents from the engagement phases selected for the Planning tab. These documents have a different signoff scheme from the default one as specified in the Product → Settings → Signoffs → Schemes.

To learn more about Signoffs Schemes, see Set up signoff schemes.

Engagement team

Note: Primary author configurations required.

The Engagement team widget displays documented team information specific to your engagement.

Note that this widget supports the following cases:

Case 1: Where the Engagement team information is documented within a single procedure in a SE checklist, allowing for multiple rows, it will display information about Position and Name. Answer SE Checklist if this scenario applies and fill in the following information:

  • Procedure ID - Insert the procedure ID where the engagement team information is documented. For instructions on how to identify the procedure ID, refer to Identify the procedure ID.

  • Read procedure information - Click on the button to retrieve information on how responses are defined for the procedure.

  • Position/Name - Specify which of the procedure responses represent the Position or Name.

Case 2: Where the Engagement team information is documented in a different SE Builder document/table, respond with SE Builder table if this scenario applies, and follow the instructions listed below:

Risk assessment and response

Riesgos

Note: Primary author configurations required, if applicable.

The Risks widget displays the information about Risks documented in the engagement file:

  • Financial statement level risk: If your app supports documenting the Financial statement level risk:

    • Insert the procedure ID where the engagement team information is documented. For instructions on how to identify the procedure ID, refer to Identify the procedure ID.

    • Document the URL and the name of the checklist containing this information to create the reference on the PMS document tab.

  • Automatically displays the total number of risks documented in the engagement file

  • Automatically displays the number of significant risks identified in the engagement file

  • Automatically displays the number of fraud risks identified in the engagement file

The Risk report reference included on the widget automatically directs the end user to the Risks assessment tab in the engagement file.

Controles

Note: No primary author required.

The Controls widget displays the information about Controls documented in the engagement file:

  • Automatically displays the total number of controls documented in the engagement file.

  • Automatically displays the number of key controls documented in the engagement file.

The Control report reference included on the widget automatically directs the end user to the Controls report tab in the engagement file.

Documents with custom conclusion

Note: No primary author required.

The Documents with custom conclusion widget displays the audit programs checklists where the default conclusion additional text area was blank by default at the engagement creation and was modified by the end user at the engagement level.

Note: This widget supports displaying information only in the following scenario:

  • The audit program checklist includes the Conclusion section as defined in the checklist default settings

  • Returns the name of the checklist with a modified conclusion if:

    • The additional text area from the conclusion section was blank by default and was modified by the end user at the engagement level.

    • The audit program checklist is visible on the Documents page.

Finalizada

Problemas

Note: No primary author required.

The Issues widget automatically displays the information about Issues documented in the engagement file, grouping them by:

  • Pendiente

  • Resuelto

  • Tildado

The Issues page reference included on the widget automatically directs the end user to the Summary of issues report in the engagement file.

Ajustes

Note: No primary author required.

Adjustments widget automatically displays the information about Adjustments posted in the engagement file:

  • The total number of adjustments recorded in the engagement file for the current year, excluding any Tax Adjustments posted.

  • The total number of proposed (Unrecorded - Factual / Projected / Judgmental) adjustments recorded in the engagement file for the current year.

  • The total number of actual (Normal and Reclassifying) adjustments recorded in the engagement file for the current year.

  • The total number of eliminating adjustments recorded in the engagement file for the current year if there are any and the engagement is set as a Consolidation.

Document the URL and the name of the Summary of Misstatements document to create the reference on the PMS document tab, if this applies for your app.

The Adjustments page reference included on the widget automatically directs the end user to the Adjustments tab in the engagement file.

Documents with overridden visibility

Note: No primary author required.

The Documents with overridden visibility widget displays all the documents from the engagement which have had their default visibility settings, originally set by primary author or firm author, modified by the end-user at the engagement level, by either:

  • Overriding the document visibility settings to display a document intended to be hidden.

  • Overriding the document visibility settings to hide a document intended to be shown.

Note that this widget displays information about any document that has had its visibility settings modified directly on the Documents page (Forced Hidden) and not through the visibility settings dialog. This also includes instances when this change was done by the primary author or firm author.
