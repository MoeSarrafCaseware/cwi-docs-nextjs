Importación de archivo Excel

This release has added a new processing step to enable file validation during tagging.

Entry number error message

Now, when mapping a column in a transactions file to ENTRY NUMBER, LINE NUMBER or ENTRY ID, an error displays if any blank cells are detected.

Click on the warning to show the blank cells and their line numbers. With this information, you can correct the file using a tool of your choice, or contact your client with this information and ask for a corrected transactions file.

An error also displays if there are blank rows.

You will not be able to import files with errors -- the Import button on the Import Transactions page will be inactive.

Prueba mensaje de error.

When mapping a column in a transactions file to AMOUNT, an error displays if any non-numeric properties are present.

Click on the warning to view the affected data.

You will not be able to import files with errors -- the Import button on the Import Transactions page will be inactive.

Entradas duplicadas

If duplicate transactions are detected in CSV files, either in the same file, or a subsequent file in an engagement, the duplicate transactions will be ignored on import.

Attempting to import duplicate transactions will no longer corrupt the transactions database. Additionally, if an import fails, users will be able to upload and import another file. Prior to this release this action would also result in a corrupted datbase.

Note: Transaction duplicates are based on the entry number property, not the entire transaction.
