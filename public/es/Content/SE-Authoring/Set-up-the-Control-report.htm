Set up the Control report

Note: This topic is not applicable for Cloud apps that have the new risk module enabled.

You can enable the Control report to allow users to add controls that they've defined to mitigate the identified risks throughout the engagement workflow. You can enable a control report to support large and complex audits (if available in your product).

To enable the Control report in your product, specify the com.caseware.ca.general.control-report in the form property in the product.json file. To learn more, see Enable or disable features in your product.

Once you enable the report in your product, you can add it in your draft and set it up.

To add the control report, select Add () at the documents page, then choose Control Report.

To set up the Control report:

  1. Add guidance information.

    2. Select the text field to start adding content. The formatting toolbar displays. Use the toolbar to make the following changes:

  1. If you want to add default controls in the report, select +Control.

    1. Enter a name and a description for the control.

    2. Select the Cycle and Area associated with the control.

  2. Select a control to view its details.

  3. Select Edit () to modify the control details if needed.

    1. Associate it to a financial group at FINANCIAL GROUPS.

    2. Add a document reference at REFERENCES.

    3. Associate it to a risk by selecting one from the drop-down list at RISKS.

    4. Associate it to a cycle or area by selecting one from the drop-down lists at CYCLES or AREAS.

    5. Once you're done, select Save.
