Test custom XBRL tags

Note: This feature is only available for distributors for beta testing. To view this feature, open an engagement file and make the following change to your URL: https://<server name of your partner site>/<firm name where you perform your authoring>/e/eng/<identifier>/index.jsp?enable-features=customXbrlTagging;testTaxonomy

You can test tagging XBRL facts directly in an engagement file to ensure the options available to end users when creating custom tags for XBRL facts are correct.

You can also tag XBRL facts directly in the product template, however, we recommend you continue to use scripts to tag the default content in your product. This ensures that the tags you create are considered system tags which optimizes performance when users review XBRL facts and export XBRL instance documents.

To tag an XBRL fact:

  1. Seleccione el icono XBRL ().

  2. En la sección REVISIÓN XBRL , seleccione la taxonomía que desea revisar.

  3. Seleccione INICIAR REVISIÓN.

    El documento de estados financieros se abre mostrando la primera área que contiene hechos XBRL. Tagged facts are marked by a green underline and untagged facts are highlighted in yellow.

  4. Select an untagged fact then select Add XBRL Tag.

  5. Seleccionar una etiqueta de la lista.

    You can also search for a specific tag using the search box.

    When you select an item in the taxonomy, information about the item, including the label, description and properties, will be displayed in the right panel of the search dialog.

  6. Seleccione SELECT.

  7. In the XBRL review panel, select the Context information for the tag from the available options in the drop-down.

    The drop-down options are sourced from the information in your XBRL taxonomy.
