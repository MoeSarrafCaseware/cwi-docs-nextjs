Use-case: A product with French and English language support

This topic describes a use-case for adding French language support in a product template. You can use this use-case as a reference when setting up multiple languages in your template.

Note: You must repeat the steps listed in this use-case for each active language in your template.

Assumptions:

  • English content and deliverables have been added and authored in the template.

  • You've added French in the template and marked it as an active language.

Prerequisite: Switch the content language of the template to French.

Step 1: Add the French language in the product settings

Add the French language and make other changes if needed, such as changing the default language from English to French.

For detailed instructions, see Set up multiple content languages.

Step 2: Rename assertions, phases, cycles, areas, dimensions and components

To start, select the Information icon () at the top menu, then choose Settings. The Product Settings dialog opens.

To rename assertions, phases, cycles, areas, dimensions and components:

  1. Go to the appropriate tab.

  2. Rename each item in the tab:

    • In the Assertions tab, enter the French abbreviation and full name of each assertion.

    • In the Phases, Cycles and Areas tabs, enter the French name of each phase, cycle and area.

      Note: The Cycles and Areas tabs are only available in your product if they have been enabled in your product.json file. To learn more, see Enable or disable features in your product.

    • In the Dimensions tab, enter the French name of each dimension category and dimension

    • In the Components tab, enter the French name of each component category, component name and component ID.
