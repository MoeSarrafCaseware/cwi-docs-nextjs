What’s new in XBRL - December 2024

The following lists describe the improvements and fixed issues in the December 2024 XBRL updates.

Cliente

Improvements

  • [SE-49037] - Added the ability to include the engagement name in the file name of the generated XBRL report.

  • [SE-49039] - In the event of an error, the XBRL review and generation panels now close rather than lock into a loading animation.

  • [SE-49591] [SE-49592] - Added some technical improvements related to testing.

  • [SE-49987] - Added the ability for users to view results based on partial search terms in the taxonomy viewer.

  • [SE-50431] - Added all missing currencies to the Currency selector.

  • [SE-50536] - Improved the rate at which fact validations are applied to custom tags.

  • [SE-40559] - XBRL numeric units now support multiple units of measure, as well as the ability to combine units of measure.

  • [SE-49599] - When expanding the context selector in the XBRL Review pane, the list of contexts now automatically scrolls to the position of the selected context.

  • [SE-50931] - Formulas are now represented by their expected value in the Content selector, rather than ‘Formula #’, unless there is no value in the formula.

Fixed issues

  • [SE-43739] - Note headers are highlighted in an inconsistent manner when applying tags.

  • [SE-48157] - The XBRL review panel does not close after navigating away from XBRL documents.

  • [SE-48635] - The Delete icon is displayed as an option for system tags when users are reviewing XBRL facts.

  • [SE-49159] - When users are editing system contexts in the context editor, the Period and Applicable fields do not display.

  • [SE-49415] - The name of the Search field in the context editor is not capitalized.

  • [SE-49424] - The XBRL review panel contains spacing issues.

  • [SE-49428] - System tags are not highlighted in the correct color during XBRL review.

  • [SE-49439] - The XBRL review panel does not load if there is an invalid taxonomy namespace.

  • [SE-49594] - The taxonomy search results are not reset upon selecting Clear.

  • [SE-49595] - The taxonomy viewer fails to expand concept drop-downs on the first attempt.

  • [SE-49623] - The taxonomy viewer allows users to select headings in the taxonomy as tags.

  • [SE-49905] - The context editor contains text alignment issues.

  • [SE-50036] - The cursor style does not change when hovering over a custom tag.

  • [SE-50283] - If the ID of a context starts with a numeral, it is incorrectly filtered as an unused context.

  • [SE-50284] - If the context editor fails to load a context, the XBRL review panel freezes.

  • [SE-50376] - An unknown error is displayed after selecting Start Review in the XBRL review panel.

  • [SE-50486] - A function prevents the XBRL generation process from initiating.

  • [SE-50813] - Custom numeric units that are immediately applied to a tag are lost after an XBRL export.

  • [SE-51066] - When tagging formulas or placeholders inside a paragraph, the option to add a unit of measure is closed upon selecting a formula, as if it is not required. The unit of measure must be added retroactively.

Microservices

  • XBRL Context Server - Ver 1.2

    • Includes a security upgrade that affects Tagging tool users. Presently, only UK users are on production.

  • XBRL Context Server - Ver 1.2.1

    • Includes a technical fix to aid error logging.

  • XBRL Server - Ver 2.4.8.10

    • [SE-40333] - Added a new endpoint that lists all relevant namespaces within a taxonomy.

    • [SE-46875] - Merged NL NT18 and FT18 into a single taxonomy and added cached files so that it can be loaded on request.

    • [SE-49277] - Fixed an issue where nested concepts would appear duplicated in the ROI102 taxonomy.

    • [SE-49656] - Fixed an issue where nested concepts would not appear under the relevant headings in the UK102 taxonomy.

    • [SE-50238] - Added support for advanced units of measure (complex numerics) for XBRL generation.

    • [SE-50483] - Added an option to include review functionality when exporting iXBRL documents.

    • [SE-50637] - Fixed an issue where PDF rendering would break during the iXBRL generation process.

    • [SE-50901] - Increased the server resources to improve performance in the XBRL platform.
