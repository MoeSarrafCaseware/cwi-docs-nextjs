Workflow (how to finalize the draft?)

Before you finalize and distribute your draft, here are a few things to consider...

  • XBRL (eXtensible Business Reporting Language) identifiers are available throughout the application to support markets that need to submit their financial statements in XBRL format.

  • If engagement consolidation is relevant in your solution, note that you need to decide whether you want the consolidated data to be displayed in the financial statements document or not. For more information, see Specify whether or not to use consolidated data and Set up the column data.

  • If interim reporting is relevant in your solution, you need to:

    • enable interim reporting properties in the product API, see Engagement properties for more information.

    • set the column types in table to be Current Period, see Set up the column data.

    También podrá:

    • set up visibility so that specific documents are available in the template based on the current period. See Reporting period.

    • set up dynamic text and engagement glossary terms so that specific terms are available in the narrative (in letters or memos or financial statements) based on the current period. See Add data links.

  • If carry forward is relevant in your solution and you've configured input rows or columns in dynamic tables for current year (or current period) and prior year (or prior period) data, the application does not automatically fix the values for years (or periods) in input rows or columns after the carry forward action.

  • You must add a note to the users specifying the input rows or columns that they need to fix the year values for manually after they perform a carry forward.

The following steps summarize the recommended workflow for finalizing your draft. You can use this workflow for guidance.

  1. Add a file placeholder, a link or upload a file

  2. Organize the items in your template

  3. Set visibility
