Comparaisons

L’analyse de comparaison peut être utilisée pour comparer deux ensembles de renseignements ou plus afin de vérifier que l'information est la même ou différente selon vos choix. Par exemple, vous pouvez utiliser ce test pour confirmer que l'utilisateur responsable de la saisie des détails de la transaction diffère de l'utilisateur qui autorise ces transactions. Ce test peut être utile pour valider la cohérence entre les rapports de réception et les factures correspondantes.

Ce test analytique peut être utilisé pour repérer :

  • Les possibles tentatives de manipulation des états financiers

  • Les domaines de l'information financière dont l'exactitude ou la précision est moindre

  • Des problèmes systémiques dans le traitement des transactions par un service

  • Des paramètres du système mal configurés pouvant nécessiter une amélioration et une évaluation plus poussée

  • La faiblesse des contrôles internes

  • Une insuffisance dans les activités de surveillance et de détection

  • Une formation insuffisante

  • Un risque d'anomalie significative due à la fraude

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

  • Champs de référence – Champs uniques utilisés pour créer un identifiant de transaction unique, comme le champ ID de l’écriture pour l’ensemble de données du grand livre général. Ces colonnes ne font pas partie du résultat, mais elles sont utilisées pour trouver les transactions qui font partie du résultat. Ce champ est déjà défini dans le test et ne peut pas être modifié.

  • Champs principaux – Deux ou plusieurs champs de n'importe quel type doivent être marqués aux fins de comparaison. Les champs marqués doivent être du même type.

Paramètres

Le paramètre suivant doit être défini pour exécuter ce test :

Type de comparaison – Sélectionnez la manière dont les données des champs sélectionnés doivent être comparées les unes aux autres. Voici vos options :

  • Égal

  • Pas égal

  • Supérieur à

  • Supérieur ou égal à

  • Inférieur à

  • Inférieur ou égal à

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Comparaison de champs de caractères – Compare deux champs de caractères et les répertorie selon qu'ils sont égaux ou non.

  • Comparaison de champs de date – Compare deux champs de date et les répertorie selon qu'ils sont égaux ou non.

  • Comparaison de champs numériques – Compare deux champs numériques et les répertorie selon qu'ils sont égaux ou non.

Spécifications techniques

Lorsque vous exécutez le test analytique Comparaisons, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.

  2. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est valable seulement si des champs spécifiques ont été sélectionnés. Si toutes les colonnes sont accessibles, elle n'est pas nécessaire.

  3. Vérifiez qu’au moins deux champs ont été sélectionnés et que tous les champs sélectionnés sont du même type.

  4. Obtenez le type de comparaison à partir des paramètres saisis. Les options sont les suivantes :

    • Égal (==)

    • Pas égal (!=)

    • Supérieur ou égal à (>=)

    • Supérieur à (>)

    • Inférieur ou égal à (<=)

    • Inférieur à (<)

  5. Effectuez l'analyse de base en comparant les champs sélectionnés avec le type de comparaison.

    • Lorsque l'on compare deux champs, extraire tous les enregistrements pour lesquels la comparaison renvoie true.

    • Lors de la comparaison de trois champs ou plus, extraire tous les enregistrements pour lesquels la comparaison entre au moins deux des champs est true.

  6. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.

  7. En option, les filtres de l'étape 1 peuvent être placés sur les données une fois l'analyse de base terminée, plutôt qu'avant.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.
