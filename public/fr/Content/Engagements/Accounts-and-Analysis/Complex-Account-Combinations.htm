Combinaisons de comptes complexes

Ce test analytique Combinaisons de comptes complexes repère toutes les écritures qui contiennent un nombre de comptes supérieur à ce qui a été précisé. Ce test peut être utilisé pour repérer des transactions commerciales complexes, non courantes ou inhabituelles qui peuvent nécessiter un examen approfondi.

Champs utilisés pour l'analyse

Ce test utilise deux champs pour effectuer l'analyse :

  • Les champs d'identifiants contenant des nombres ou des caractères qui indiquent la manière dont les combinaisons de comptes sont regroupées. Par exemple, le champ Écriture de journal peut être utilisé pour regrouper les comptes par écriture de journal.

  • Les champs de compte contenant des chiffres ou des caractères utilisés pour calculer le nombre de combinaisons par identifiant.

Paramètres

Un paramètre doit être défini pour que ce test puisse être exécuté :

Nombre de combinaisons de comptes – Indiquez le nombre minimum de comptes qui permettraient d'indiquer que le groupe de comptes est risqué ou inhabituel. Ce nombre doit être une valeur entière. Par exemple, si vous saisissez 11, le test repère tous les regroupements comportant un minimum de 11 comptes.

Spécifications techniques

Lorsque vous exécutez le test analytique Combinaisons de comptes complexes, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  2. Vérifiez que les champs d’analyse de base nécessaires ont été sélectionnés. Il s'agit des champs « identifiant » et « compte ».

  3. Vérifiez que le nombre de combinaisons a été saisi. Il doit s'agir d'un nombre entier positif. La valeur par défaut est 11.

  4. Totalisez le fichier source par colonnes d’identifiants et par colonnes de comptes. Assurez-vous que le total est totalisé par le nombre d'enregistrements.

  5. Totalisez l'étape 5 par colonnes d'identifiants et totalisez le nombre d'enregistrements pour chaque colonne d'identifiants.

  6. Extrayez tous les champs d’identifiants dont le nombre d'enregistrements est supérieur ou égal au paramètre Nombre de combinaisons.

  7. Extrayez toutes les transactions sources présentant la même combinaison de champs d'identifiants que celle obtenue à l'étape précédente.

  8. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.
