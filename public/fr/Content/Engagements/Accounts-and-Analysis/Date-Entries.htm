Écritures de date

L'analyse des écritures de date permet de repérer les transactions comportant certaines valeurs de date. Voir la section Configurations de test pour obtenir une liste complète des types de valeurs de date qui peuvent être repérées.

Ce test analytique peut être utilisé pour repérer :

  • Les possibles tentatives de manipulation des états financiers

  • Les domaines de l'information financière dont l'exactitude ou la précision est moindre

  • Les transactions qui contiennent des estimations significatives

  • Des paramètres du système mal configurés pouvant nécessiter une amélioration et une évaluation plus poussée

  • Comptabilisation incorrecte de transactions dans la bonne période intermédiaire

  • La faiblesse des contrôles internes

  • Une insuffisance dans les activités de surveillance et de détection

  • Une formation insuffisante

  • Des écarts dans la plage des dates prévues peuvent indiquer des problèmes de démarcation

  • Un long délai entre les dates saisies et les dates du report ou le fait que la date saisie soit antérieure à la date du report qui peut indiquer une rétrodatation ou une postdatation

  • Un risque d'anomalie significative due à la fraude

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

  • Champs de référence – Champs uniques utilisés pour créer un identifiant de transaction unique, comme le champ ID de l’écriture pour l’ensemble de données du grand livre général. Ces colonnes ne font pas partie du résultat, mais elles sont utilisées pour trouver les transactions qui font partie du résultat. Ce champ est déjà défini dans le test et ne peut pas être modifié.

  • Champs principaux – Un ou deux champs de date.

Paramètres

Les paramètres suivants doivent être définis pour exécuter ce test :

  • Date – Sélectionnez une ou plusieurs dates auxquelles l'analyse sera effectuée.

  • Période saisie – Si votre configuration de test recherche des plages de dates, saisissez le nombre de périodes qui seront utilisées. Par exemple, si vous sélectionnez 2 et choisissez Semaines comme type de période et Entre comme type d'intervalle, le test recherchera les dates situées deux semaines avant et après la date sélectionnée.

  • Sélectionner le type de période – Sélectionnez le type de période, par exemple jours, semaines ou mois.

  • Type de plage – Pour certaines configurations de ce test, vous devez indiquer si vous recherchez des transactions à des dates antérieures, postérieures à la plage de dates spécifiée ou entre les dates.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Transactions autour d'une certaine date – Dresse la liste des transactions autour d'une certaine date.

  • Transactions tombant dans une plage de dates – Dresse la liste des transactions dans une plage de dates que l’analyse permet d'extraire entre deux dates spécifiées.

  • Transactions avec deux dates dans une certaine plage de dates – Dresse la liste des transactions avec deux dates tombant dans une certaine plage de dates.

  • Transactions dont deux dates se situent autour d'une certaine date – Dresse la liste des transactions dont les deux dates se situent dans une certaine plage.

  • Transactions à une certaine date – Dresse la liste des transactions reportées certains jours de l'année.

  • Transactions reportées un certain jour de la semaine – Dresse la liste des transactions comptabilisées certains jours de la semaine.

  • Transactions comptabilisées un certain jour de l'année – Dresse la liste des transactions comptabilisées certains jours de l'année.

Spécifications techniques

Lorsque vous exécutez le test analytique Écritures de date, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première. Notez qu’il n’est pas encore possible de définir des filtres et que ce le sera dans les versions ultérieures du test.

  2. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  3. Obtenez la configuration du test à effectuer :

    • Recherchez les transactions autour d'une certaine date.

    • Recherchez les transactions dans une certaine plage de dates.

    • Transactions ayant deux dates dans une certaine plage de dates.

    • Transactions dans lesquelles deux dates se trouvent autour d'une certaine date.

    • Transactions reportées à une certaine date.

    • Transactions reportées certains jours de la semaine

    • Transactions reportées certains jours de l'année

  4. Pour le test des transactions autour d'une certaine date :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez que l'utilisateur a sélectionné une date.

    3. Vérifiez que l'utilisateur a saisi le nombre de périodes.

    4. Vérifiez que l'utilisateur a sélectionné si le nombre de périodes correspond à des jours, à des semaines ou à des mois.

    5. Vérifiez que l'utilisateur a sélectionné l'un des éléments suivants :

      1. Supérieur à (>=) – Indique que la valeur du champ de date est supérieure (ultérieure) à la date sélectionnée.

      2. Inférieur à (<=) – Indique que la valeur du champ de date est inférieure (antérieure) à la date sélectionnée.

      3. Entre – Indique que la valeur du champ de date est comprise entre les dates sélectionnées.

    6. Extrayez toutes les transactions dans lesquelles la colonne de date sélectionnée est comprise dans la période définie par l'utilisateur. Voici quelques exemples :

      1. L'utilisateur sélectionne une semaine avec la date du 31 décembre 2023. Le test extrait toutes les transactions effectuées entre le 31 décembre 2023 et le 7 janvier 2024.

      2. L'utilisateur sélectionne Entre, puis toutes les transactions entre le 23 décembre 2023 et le 7 janvier 2024 seront sélectionnées.

  5. Pour le test Rechercher les transactions à l'intérieur d'une plage de dates :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez que l'utilisateur a saisi une date inférieure et une date supérieure.

    3. Extrayez toutes les transactions dont le champ de date se situe entre les dates inférieure et supérieure, y compris les transactions qui se produisent aux dates inférieure et supérieure.

  6. Pour le test Transactions ayant deux dates dans une certaine plage de dates :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez la sélection d'un deuxième champ de date différent du premier champ de date.

    3. Vérifiez que l'utilisateur a saisi le nombre de périodes.

    4. Vérifiez que l'utilisateur a sélectionné si le nombre de périodes correspond à des jours, à des semaines ou à des mois.

    5. Vérifiez que l'utilisateur a sélectionné l'un des éléments suivants :

      1. >=

      2. <=

      3. Entre

    6. Extrayez les transactions sur la base des éléments suivants :

      1. >= – Extraire les transactions dont la date 2 est postérieure à la date 1 sur la base de la période sélectionnée.

      2. <= – Extraire les transactions pour lesquelles la date 2 est antérieure à la date 1 sur la base de la période sélectionnée.

      3. Entre – Extraire les transactions dans lesquelles la date 2 est antérieure ou postérieure à la date 1 sur la base de la période sélectionnée.

  7. Pour le test Transactions dans lesquelles deux dates se trouvent autour d'une certaine date :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez la sélection d'un deuxième champ de date différent du premier champ de date.

    3. Vérifiez que l'utilisateur a entré une date.

    4. Vérifiez que l'utilisateur a sélectionné le champ de date.

    5. Extrayez les transactions dans lesquelles le champ de date sélectionné par l'utilisateur est antérieur à la date saisie par l'utilisateur, et le champ de la date non sélectionnée est postérieur à la date saisie par l'utilisateur.

  8. Pour le test Transactions reportées certains jours de la semaine :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez qu'un ou plusieurs jours de la semaine ont été sélectionnés.

    3. Extrayez les transactions qui tombent les jours de la semaine sélectionnés par l'utilisateur pour le champ de date sélectionné.

  9. Pour le test Transactions reportées certains jours de l'année :

    1. Vérifiez qu'un champ date a été sélectionné.

    2. Vérifiez qu'un ou plusieurs jours de l’année ont été sélectionnés.

    3. Extrayez les transactions qui tombent les jours de l’année sélectionnés par l'utilisateur pour le champ de date sélectionné.
