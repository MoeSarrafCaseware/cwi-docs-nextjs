Doublons

Le test analytique Doublons détecte les transactions ou les enregistrements en double ou potentiellement frauduleux dans les états financiers ou les documents comptables d'une entreprise. Il permet de déceler des erreurs, des exceptions ou des activités frauduleuses telles que la double comptabilisation des produits, des charges ou des actifs.

Ce test analytique peut être utilisé pour repérer :

  • Les possibles tentatives de manipulation des états financiers

  • Les domaines de l'information financière dont l'exactitude ou la précision est moindre

  • Les transactions fictives

  • Des problèmes systémiques dans le traitement des transactions par un service

  • Des paramètres du système mal configurés pouvant nécessiter une amélioration et une évaluation plus poussée

  • L’échec du processus de validation des données

  • La faiblesse des contrôles internes

  • Une insuffisance dans les activités de surveillance et de détection

  • Une formation insuffisante

  • Un risque d'anomalie significative due à la fraude

  • Les transactions reportées en double par erreur

  • Processus redondants ou manque de coordination entre les services

  • Problèmes avec les rapprochements

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

  • Champs de référence – Champs uniques utilisés pour créer un identifiant de transaction unique, comme le champ ID de l’écriture pour l’ensemble de données du grand livre général. Ces colonnes ne font pas partie du résultat, mais elles sont utilisées pour trouver les transactions qui font partie du résultat. Ce champ est déjà défini dans le test et ne peut pas être modifié.

  • Champs en double – Un ou plusieurs champs qui seront comparés pour rechercher les doublons.

  • Champs différents – Un ou plusieurs champs qui doivent être différents pour rechercher les écritures en double. Ces champs sont facultatifs et sont utilisés seulement pour l'analyse lorsque vous recherchez des doublons avec un champ différent. Par exemple, si vous recherchez le même paiement effectué à des dates différentes, le champ du paiement sera le champ dupliqué, et le champ de la date sera le champ qui doit être différent.

Paramètres

Il n'y a pas de paramètres définis par l'utilisateur pour ce test.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Écritures en double – Par ID de compte, type d'identifiant, numéro d'écriture, date de la saisie, saisi par et montant

  • Écritures en double – Par type de document d'origine et numéro de document d'origine

  • Écritures en double – Par type de document et numéro de document

  • Écritures en double – Par champ Saisi par l'utilisateur différent

  • Montant en double par identifiant

  • Date et montant de la saisie en double par type de document

  • Écritures en double – Par description et montant

  • JE différents : Même numéro de compte, montant et date de saisie

  • Écritures en double – Par numéro de compte, montant, numéro de document et date de saisie

  • Date d'entrée différente : Same Account Number, Amount, Document Number

  • Numéros de facture en double

  • Dates de facture fournisseur en double

  • Numéros de chèques en double

  • Montants de facture fournisseur en double

  • Transactions en double

  • Transaction en double avec une date de facture différente

  • Transaction en double avec date saisie différente

Spécifications techniques

Lorsque vous exécutez le test analytique Doublons, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première. Notez qu’il n’est pas encore possible de définir des filtres et que ce le sera dans les versions ultérieures du test.

  2. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  3. Vérifiez qu'au moins un champ a été sélectionné pour effectuer l'analyse des doublons.

  4. Vérifiez qu'un champ différent facultatif a été sélectionné.

  5. Vérifiez qu'un champ de plage de dates a été sélectionné.

  6. Test de base :

    • Seuls les champs en double sont marqués – Effectuez l'analyse en recherchant des correspondances dans la base de données et exportez toutes les correspondances dans le fichier de résultats.

    • Champ différent sélectionné – Effectuez l'analyse en recherchant les correspondances dans lesquelles le champ sélectionné comme différent ne correspond pas. Exportez toutes les correspondances dans le fichier de résultats.

  7. Ajoutez une colonne supplémentaire qui regroupe les doublons. Par exemple, la première série de doublons sera regroupée sous le numéro 1 dans cette colonne, et la deuxième série sera regroupée sous le numéro 2.
