Trous

Cette analyse des trous repère les transactions manquantes dans une certaine séquence de transactions.

Ce test analytique peut être utilisé pour repérer :

  • Des paramètres du système mal configurés pouvant nécessiter une amélioration et une évaluation plus poussée

  • La faiblesse des contrôles internes

  • Une insuffisance dans les activités de surveillance et de détection

  • Une formation insuffisante

  • Les écritures de journal à la clôture de l'exercice sont souvent reportées au moyen d'une séquence de numérotation spéciale; l'auditeur peut déterminer cette séquence et générer une liste aux fins d'examen

  • Les transactions manquantes susceptibles d'être annulées

  • Un risque d'anomalie significative due à la fraude

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

Champ principal – Le champ numérique, de date ou de caractères qui sera utilisé comme champ principal pour l'analyse des trous.

Paramètres

Les paramètres suivants doivent être définis pour exécuter ce test :

  • Plage inférieure – Sélectionnez le point de départ de l'analyse des trous. Il s'agit généralement de la première transaction, mais il se peut que ce ne soit pas le cas s'il manque des transactions au début.

  • Plage supérieure – Sélectionnez le point final de l'analyse des trous. Il s'agit généralement de la dernière transaction, mais il se peut que ce ne soit pas le cas s'il manque des transactions à la fin.

  • Incrément – Saisissez l'incrément attendu pour le champ analysé. En général, l'incrément est de un.

  • Exclure les jours fériés – Sélectionnez s'il faut inclure ou non les jours fériés dans les tests où le champ analysé est un champ de date.

  • Sélectionner les jours fériés – Si vous avez choisi d'exclure les jours fériés, indiquez les dates qui sont des jours fériés.

  • Exclure les week-ends – Sélectionnez s'il faut inclure ou non les week-ends dans les tests où le champ analysé est un champ de date.

  • Masque – Si le champ analysé est un champ de caractères, entrez un masque pour indiquer quelle partie contient la partie numérique à utiliser pour la détection des trous. Par exemple, si vous utilisez un champ de facture au format ABC123456, le masque sera CCCNNNNNN, N représentant la partie numérique du champ. Seule la partie numérique est analysée pour les trous.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Trous dans les dates – Recherchez les dates pour lesquelles il n'y a pas de transactions alors qu'elles sont attendues sur la base des paramètres sélectionnés.

  • Trous dans les numéros de documents – Recherchez les trous dans les listes numériques, comme les numéros de chèques manquants.

Spécifications techniques

Lorsque vous exécutez le test analytique des trous, il faut réaliser les étapes suivantes pour exécuter le test :

  1. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  2. Vérifiez qu'un champ a été sélectionné pour l'analyse de base. Ces champs doivent être des champs numériques, de date ou de caractères. Ce test analytique ne fonctionne pas pour les champs d’heure.

  3. Classez la base de données par ordre croissant en fonction du champ Trou.

  4. L'analyse des trous sera effectuée différemment selon le type de champ :

    • Numérique – Permet d'obtenir les points de départ et d'arrivée (plage de trou) ainsi que l'incrément. Les valeurs par défaut doivent être la plus petite et la plus grande, et la valeur par défaut de l'incrément est 1. L’utilisateur peut modifier cette valeur. Créez une liste de tous les trous qui se trouvent dans la plage et dans l’incrément sélectionnés.

      Remarque : Remarque : Si la colonne contient des valeurs non entières, elles sont arrondies à la décimale la plus proche.

    • Date – Obtenir les dates les plus anciennes et les plus récentes saisies par l'utilisateur. L'incrément par défaut est d'un jour. Ce test permet également à l'utilisateur d'exclure les week-ends et les jours fériés de la détection des trous.

    • Caractère – Obtenir le masque pour indiquer quelles parties de la valeur du champ sont les parties numériques qui seront soumises à la détection des trous.

  5. Il y a deux ensembles de résultats. Le premier ensemble énumère le trou de début et de fin et le nombre d'éléments manquants, et le deuxième ensemble énumère les éléments manquants.
