Joindre

Ce test analytique combine deux ensembles de données distincts en un seul ensemble de données unifié. Ce test est utile lorsque vous devez combiner deux ensembles de données, car l'un d'eux contient des données principales qui enrichissent les renseignements que vous pouvez tirer de l'autre ensemble de données transactionnelles. Par exemple, vous pouvez utiliser ce test pour identifier les fournisseurs n'ayant effectué aucune transaction.

Notez que les deux ensembles de données joints sont appelés ensemble de données A (le fichier primaire) et ensemble de données B (le fichier secondaire).

Ce test prend en charge les jonctions suivantes :

  • Jonction interne (conserve seulement les lignes qui ont des correspondances dans les deux ensembles de données A et B)

  • Jonction externe (conserve toutes les lignes qui ont des correspondances dans l'ensemble de données A ou B, ou dans les deux)

  • Jonction externe gauche (conserve toutes les lignes de l'ensemble de données A et les lignes correspondantes de l'ensemble de données B)

  • Jonction externe droite (conserve toutes les lignes de l'ensemble de données B et les lignes correspondantes de A)

  • Enreg. sans concordance dans le fichier secondaire

  • Enreg. sans concordance dans le fichier primaire

Ce test analytique peut être utilisé pour :

  • Vérifier que toutes les transactions des fournisseurs sont correctement enregistrées. Un fichier de données maîtres des fournisseurs permet de s'assurer que tous les fournisseurs sont consignés, ce qui réduit le risque de manquer des transactions.

  • Confirmer que les renseignements sur le fournisseur sont exacts. Permet de s'assurer que les paiements sont versés à des fournisseurs légitimes et que les erreurs dans les renseignements sur les fournisseurs sont réduites au minimum.

  • Il est essentiel aux fins de la conformité de s'assurer que le fichier de données maîtres du fournisseur est tenu à jour conformément aux exigences réglementaires et aux politiques de l'entreprise. Les auditeurs vérifient la conformité aux lois, aux règlements et aux politiques internes pertinents.

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

Colonnes principales :

  • Fichier primaire :

    • Un ou plusieurs champs qui seront utilisés pour la correspondance

    • Un ou plusieurs champs à inclure dans le résultat de la jonction.

  • Fichier secondaire :

    • Un ou plusieurs champs qui seront utilisés pour la correspondance.

    • Un ou plusieurs champs à inclure dans le résultat de la jonction.

Pour les deux champs de jonction, il faut également indiquer si la jonction doit être effectuée dans un ordre croissant ou décroissant. Les champs des fichiers primaire et secondaire doivent correspondre, car ils seront utilisés comme clé pour la jonction.

Paramètres

Un paramètre doit être défini pour que le test puisse être exécuté :

Type de jonction :

  • Jonction interne (conserve seulement les lignes qui ont des correspondances dans les deux ensembles de données A et B)

  • Jonction externe (conserve toutes les lignes qui ont des correspondances dans l'ensemble de données A ou B, ou dans les deux)

  • Jonction externe gauche (conserve toutes les lignes de l'ensemble de données A et les lignes correspondantes de l'ensemble de données B)

  • Jonction externe droite (conserve toutes les lignes de l'ensemble de données B et les lignes correspondantes de A)

  • Enreg. sans concordance dans le fichier secondaire

  • Enreg. sans concordance dans le fichier primaire

Configurations de test

La seule configuration possible pour ce test est celle des transactions sans compte fournisseur.

Spécifications techniques

Lorsque vous exécutez le test analytique Doublons, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Demandez s'il s'agit d'un résultat définitif ou d'un résultat intermédiaire.

  2. Obtenez le type de jonction effectuée parmi les types suivants :

    • Jonction interne (conserve seulement les lignes qui ont des correspondances dans les deux ensembles de données A et B)

    • Jonction externe (conserve toutes les lignes qui ont des correspondances dans l'ensemble de données A ou B, ou dans les deux)

    • Jonction externe gauche (conserve toutes les lignes de l'ensemble de données A et les lignes correspondantes de l'ensemble de données B)

    • Jonction externe droite (conserve toutes les lignes de l'ensemble de données B et les lignes correspondantes de A)

    • Enreg. sans concordance dans le fichier secondaire

    • Enreg. sans concordance dans le fichier primaire

  3. Obtenez le fichier primaire :

    1. Obtenez les champs correspondants

    2. Obtenez la liste des champs qui seront inclus dans le résultat

  4. Obtenez le fichier secondaire :

    1. Obtenez les champs correspondants

    2. Obtenez la liste des champs qui seront inclus dans le résultat

  5. Demandez si les correspondances sont dans l'ordre croissant ou décroissant ou un mélange.

  6. Effectuez la jonction appropriée en fonction des sélections de l'utilisateur.
