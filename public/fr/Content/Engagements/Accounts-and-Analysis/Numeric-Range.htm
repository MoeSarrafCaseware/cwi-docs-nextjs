Plage numérique

Le test analytique Plage numérique repère les transactions à l'intérieur d'une fourchette de valeurs définie. Son objectif principal est de signaler les transactions de grande valeur, mais vous pouvez ajuster la configuration pour détecter les transactions qui se situent en dessous d'un seuil défini ou dans des fourchettes de valeurs précises. Vous pouvez également choisir d'utiliser des seuils en pourcentage au lieu de seuils numériques. Par exemple, vous pouvez configurer le test afin de repérer le pourcentage de transactions les plus élevées.

Ce test analytique peut être utilisé pour repérer :

  • Les transactions qui contiennent des estimations significatives

  • Les transactions complexes, irrégulières ou inhabituelles qui peuvent justifier une enquête approfondie

  • Les transactions importantes

  • Les transactions manquantes susceptibles d'être annulées

  • Les problèmes d'évaluation

  • Les transactions importantes au sein de la population

  • Les transactions de faible valeur ou de valeur nulle au sein de la population

  • Les transactions ou les soldes de valeur nulle

  • Une fourchette de valeurs, par exemple juste en dessous d'un point de démarcation

  • Les comptes débiteurs avec opérations de crédit ou les comptes créditeurs avec opérations de débit

  • Les clients ayant des limites de crédit élevées

  • Les soldes de comptes clients dépassant un certain montant

  • Les soldes des comptes fournisseurs dépassant un certain montant

Champs utilisés pour l'analyse

Cette analyse utilise un champ numérique qui contient des montants ou des quantités. Vous pouvez sélectionner le champ à utiliser lorsque vous définissez les paramètres du test.

Paramètres

De nombreux paramètres doivent être définis pour exécuter le test :

  • Sélectionner les champs – Sélectionnez dans la liste un ou plusieurs champs numériques à utiliser pour effectuer le test.

  • Numérique ou pourcentage – Sélectionnez s'il faut effectuer le test en utilisant la valeur numérique des champs ou des valeurs en pourcentage. L'option Numérique est sélectionnée par défaut. L'option choisie ici détermine les autres paramètres de ce test.

Si vous sélectionnez Numérique, effectuez des sélections pour les paramètres suivants :

  • Seuil de valeur élevée – Saisissez une valeur dans ce champ pour extraire toutes les transactions dont les montants sont supérieurs à cette valeur.

  • Seuil de faible valeur – Saisissez une valeur dans ce champ pour extraire toutes les transactions dont les montants sont inférieurs à cette valeur.

  • Plages de valeurs – Saisissez une ou plusieurs plages de valeurs pour extraire toutes les transactions dont les montants sont compris dans ces plages.

  • Valeurs exactes – Saisissez une valeur exacte pour extraire toutes les transactions ayant cette valeur. Par exemple, vous pouvez utiliser ce paramètre pour extraire toutes les transactions de valeur zéro.

Si vous sélectionnez Pourcentage, effectuez des sélections pour les paramètres suivants :

  • Seuil de pourcentage élevé – Saisissez un pourcentage dans ce champ pour extraire toutes les transactions dont le pourcentage est supérieur à cette valeur.

  • Seuil de faible pourcentage – Saisissez un pourcentage dans ce champ pour extraire toutes les transactions dont le pourcentage est inférieur à cette valeur.

  • Plages de valeurs – Saisissez une ou plusieurs plages de pourcentages pour extraire toutes les transactions dont les pourcentages sont compris dans ces plages.

  • Valeurs exactes – Saisissez un pourcentage exact pour extraire toutes les transactions correspondant à ce pourcentage.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Plage de montants – Relève les transactions dans une plage définie pour les champs de montants sur la base d'une valeur numérique ou d'un pourcentage.

  • Plage de remise – Relève les transactions dans une plage définie pour les champs de remise sur la base d'une valeur numérique ou d'un pourcentage.

  • Plage de quantité – Relève les transactions dans une plage définie pour les champs de quantité sur la base d'une valeur numérique ou d'un pourcentage.

  • Plage de jours d'échéance – Relève les transactions dans une plage définie pour les champs de jours d'échéance sur la base d'une valeur numérique ou d'un pourcentage.

  • Plage numérique – Relève les transactions dans une plage définie sur la base d'une valeur numérique ou d'un pourcentage.

  • Valeur monétaire élevée – Relève les transactions dont le champ montant est supérieur ou égal à la valeur numérique définie.

Spécifications techniques

Lorsque vous exécutez le test analytique Plage numérique, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.

  2. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  3. Vérifiez que le champ sélectionné pour effectuer le test est numérique.

  4. En fonction des paramètres sélectionnés par l'utilisateur, déterminez le type d'extraction du montant :

    • Égal à la valeur

    • Supérieur à la valeur

    • Supérieur ou égal à la valeur

    • Inférieur à la valeur

    • Inférieur ou égal à la valeur

    • Pas égal à la valeur

    • Une fourchette de valeurs

  5. Demandez à l'utilisateur si l'analyse est basée sur un pourcentage ou une valeur.

    • Si une valeur est sélectionnée, elle est demandée à l'utilisateur. Si l'option est l'extraction par plage, deux valeurs seront nécessaires.

    • Si un pourcentage est sélectionné, le pourcentage est demandé à l'utilisateur, et la valeur qu'il représente est calculée. Cette valeur varie en fonction du type d'extraction, par exemple 10 % pour des valeurs élevées donneront une valeur différente de 10 % pour des valeurs faibles. Si l'option est l'extraction par plage, deux valeurs seront nécessaires.

  6. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.
