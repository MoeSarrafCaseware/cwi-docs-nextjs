Motif numérique récurrent

Le test analytique des motifs numériques récurrents est conçu pour identifier des motifs spécifiques définis par l'utilisateur qui font partie d'une valeur. Ces motifs peuvent se trouver au début, à la fin ou intégrés dans la valeur. Vous pouvez choisir de rechercher une combinaison de chiffres spécifique ou un certain nombre de chiffres récurrents dans les valeurs.

Notez que ce test ne prend pas en compte les valeurs décimales. Les valeurs 1000 et 1000.10 seraient considérées comme identiques. Vous avez également la possibilité d'ignorer les valeurs arrondies si vous utilisez le test pour rechercher des chiffres récurrents.

Ce test analytique peut être utilisé pour repérer :

  • Transactions potentiellement non soutenues

  • Un risque d'anomalie significative due à la fraude

  • Valeurs avec la combinaison 999 qui peuvent indiquer des valeurs juste en dessous d'une valeur seuil

  • Valeurs récurrentes pouvant indiquer des numéros fictifs

Champs utilisés pour l'analyse

Ce test utilise un ou plusieurs champs numériques pour détecter les montants arrondis ou les valeurs de quantité arrondies. Vous pouvez définir les champs à utiliser en modifiant le paramètre Sélectionner les champs.

Paramètres

Un paramètre doit être défini pour que le test puisse être exécuté :

  • Sélectionner les champs – Sélectionnez dans la liste un ou plusieurs champs numériques à utiliser pour effectuer le test.

  • Sélectionnez la combinaison de chiffres ou le nombre de chiffres récurrents que vous souhaitez rechercher. Notez que le test ne prend pas en compte les chiffres au-delà de la décimale. Par exemple, les valeurs 1 000 et 1 000,10 sont considérées comme identiques.

  • If you select Digit combination, enter values for the following parameters:

    • Combinaison de chiffres - Saisissez la combinaison de chiffres à rechercher dans les valeurs. La valeur par défaut est 999.

    • Emplacement de la combinaison de chiffres - Choisir de rechercher la combinaison de chiffres au début, à la fin ou à l'intérieur des valeurs. Si vous choisissez de le rechercher au début des valeurs, vous pouvez également indiquer la longueur totale des valeurs à rechercher. Par exemple, vous pouvez rechercher les valeurs qui commencent par 123 et qui ont une longueur totale de 4 chiffres.

  • If you select Recurring digits, enter values for the following parameters:

    • Nombre de chiffres récurrents - Saisissez le nombre de chiffres récurrents à rechercher. La valeur par défaut est 3.

    • Emplacement des chiffres récurrents - Permet de rechercher les chiffres récurrents au début ou à la fin des valeurs.

    • Ignore rounded amounts - Select whether to ignore rounded amount values.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Montants ayant un motif numérique récurrent

  • Quantité avec motif numérique récurrent

  • Numérique avec motif récurrent - Tous les champs

Spécifications techniques

Lorsque vous exécutez le test analytique Plage numérique, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est valable seulement si des champs spécifiques ont été sélectionnés. Si toutes les colonnes sont accessibles, elle n'est pas nécessaire.

  2. Vérifiez qu'un champ a été sélectionné pour l'analyse de base.

  3. Obtenir les paramètres nécessaires pour l'analyse de base :

    • Motif des chiffres

      1. Obtenir le modèle de chiffres

      2. Obtenir l'emplacement du motif du chiffre.

        • Si le motif des chiffres se trouve au début, il faut obtenir s'il existe un paramètre facultatif pour la longueur totale de la valeur :

        • Si la longueur est de 0, toutes les valeurs sont incluses.

        • La valeur de la longueur doit être supérieure ou égale au motif. Par exemple, si le motif est 999, la longueur minimale est de 3.

      3. Si un utilisateur choisit 6 comme longueur et que le motif de départ est 999, 999000 sera sélectionné, mais pas 99900, car il ne comporte que cinq caractères.

      4. Extraire toutes les valeurs qui répondent aux paramètres sélectionnés. Les valeurs décimales sont ignorées pour les motifs de chiffres à la fin de la valeur. Les valeurs décimales ne sont pas arrondies.

    • Chiffres récurrents

      1. Obtenir le nombre de chiffres récurrents.

      2. Permet de savoir si les chiffres récurrents se trouvent au début ou à la fin de la valeur numérique. Si elles se trouvent à la fin, les valeurs décimales ne sont pas prises en compte. Les valeurs décimales ne sont pas arrondies.

      3. Obtenir si le modèle de chiffres récurrents doit ignorer les montants arrondis.

      4. Extraire toutes les valeurs qui répondent aux paramètres sélectionnés.
