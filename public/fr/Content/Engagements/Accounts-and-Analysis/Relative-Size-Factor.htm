Facteur de la taille relative

Le test d’analyse du facteur de la taille relative repère les exceptions lorsque le montant le plus important d'un sous-ensemble dans une clé donnée se trouve en dehors de la norme de ce sous-ensemble. Une clé est un champ spécifique ou une combinaison de champs utilisé pour regrouper les données en sous-ensembles à des fins d'analyse.

Ce test analytique peut être utilisé pour repérer :

  • Observations aberrantes ou modèles inhabituels

  • Transactions qui s'écartent sensiblement de la norme

Champs utilisés pour l'analyse

Les champs ci-dessous sont utilisés pour cette analyse :

  • Champs de référence – Champs uniques utilisés pour créer un identifiant de transaction unique, comme le champ ID de l’écriture pour l’ensemble de données du grand livre général. Ces colonnes ne font pas partie du résultat, mais elles sont utilisées pour trouver les transactions qui font partie du résultat. Ce champ est déjà défini dans le test et ne peut pas être modifié.

  • Champ numérique – Le champ utilisé pour calculer le facteur de taille relative (FTR).

  • N’importe quel champ – Un ou plusieurs champs à utiliser pour créer les sous-ensembles à partir desquels le FTR est généré.

Paramètres

Les paramètres suivants doivent être définis pour exécuter ce test :

  • FTR – Saisissez le multiple entre le montant le plus élevé et le deuxième montant le plus élevé du sous-ensemble. Par exemple, si le FTR est fixé à deux, le montant le plus élevé devra être deux fois plus important que le deuxième pour être sélectionné.

  • Ignorer les petits montants – Sélectionnez s’il faut exclure ou non les petits montants.

  • Petit montant – Si vous avez choisi d'ignorer les petits montants, entrez la valeur qui serait considérée comme la valeur minimale à inclure dans le test. Par exemple, si vous entrez 100 $, toutes les transactions inférieures à 100 $ seront exclues de l'analyse.

  • Valeurs positives ou négatives – Sélectionnez si le test doit être effectué sur des valeurs positives ou négatives.

Configurations de test

La seule configuration disponible pour ce test est le facteur de taille relative. Ce test analytique repère les exceptions lorsque le montant le plus important d'un sous-ensemble dans une clé donnée se trouve en dehors de la norme pour ce sous-ensemble.

Spécifications techniques

Remarque : Remarque : Ces spécifications techniques sont basées sur la sélection de nombres positifs. Pour les nombres négatifs, le plus petit nombre est considéré comme le plus grand pour les étapes suivantes.

Lorsque vous exécutez le test analytique des mots-clés, il faut réaliser les étapes suivantes :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.

  2. Vérifiez qu'un champ de montant a été sélectionné.

  3. Vérifiez un ou plusieurs champs (à l'exception du montant) qui seront utilisés pour la création des sous-ensembles.

  4. Totalisez le fichier sur la base des champs qui ont été sélectionnés pour le sous-ensemble, ces champs doivent être dans l'ordre de sélection, la totalisation doit inclure le nombre d'enregistrements par sous-ensemble. Lors de la sélection, l'analyse porte seulement sur des valeurs négatives ou positives en fonction de l’indicateur du type de valeurs sur lesquelles l'analyse doit être effectuée.

  5. Extrayez tous les enregistrements dont le nombre est supérieur à 1 (exclut tous les sous-ensembles dans lesquels il n'y a qu'une seule transaction; pour cette analyse, vous avez besoin de deux transactions ou plus par sous-ensemble).

  6. Reliez le fichier créé à l'étape 6 au fichier d'origine par les champs du sous-ensemble sélectionnés. Le fichier original doit également être filtré sur le champ du montant en fonction de l'indicateur positif ou négatif. Si l'indicateur est positif, le filtre doit être montant > 0, et si l'indicateur est négatif, il doit être montant < 0. Cette étape consiste à créer un fichier de toutes les transactions qui feront partie de l'analyse FTR.

  7. Extrayez les deux premières transitions pour chaque sous-ensemble de l'étape 7. Inclure seulement le ou les champs du sous-ensemble, le champ du montant et la clé de l'enregistrement.

  8. À partir de l'étape 8, extrayez l'enregistrement le plus important en termes de montant pour chaque sous-ensemble.

  9. Renommez le champ de montant de l'étape 8 en Montant le plus élevé.

  10. À partir de l'étape 7, extrayez toutes les transactions à l'exception de celles de l'étape 6 (enregistrements supérieurs). Cela permet d'obtenir une liste de transactions qui exclut la transaction la plus élevée.

  11. À partir de l’étape 11, extrayez la transaction la plus importante par sous-ensemble unique, ce qui donne la deuxième transaction la plus importante pour les sous-ensembles.

  12. À partir de l'étape 11, renommez le champ montant en Deuxième montant le plus élevé.

  13. À partir de l'étape 9, calculer la valeur moyenne pour chaque clé unique. Cette moyenne exclura le montant le plus élevé du sous-ensemble unique extrait à l'étape 9.

  14. Renommez le champ de la valeur moyenne de l'étape 14 en Moyenne X la plus élevée.

  15. Combinez les transactions de l'étape 9 (valeurs les plus élevées) et de l'étape 12 (deuxièmes valeurs les plus élevées).

  16. Combinez les transactions de l'étape 16 (valeur la plus élevée et deuxième valeur la plus élevée) avec celles de l'étape 15 (moyenne X la plus élevée). Le résultat devrait maintenant contenir les colonnes du sous-ensemble, la colonne Montant le plus élevé, la colonne Deuxième montant le plus élevé, la colonne de la Moyenne X la plus élevée et la colonne Décompte.

  17. Créez la colonne Facteur de taille relative sur la base du montant le plus élevé ou du deuxième montant le plus élevé, en utilisant généralement quatre décimales pour l'affichage.

  18. Extrayez toutes les transactions qui sont égales ou supérieures au paramètre du FTR. Si le paramètre prévoit que les petits montants doivent être ignorés, il faut également exclure toutes les transactions dont la valeur du montant le plus élevé est inférieure à la valeur du petit montant incluse dans les paramètres.

  19. Extrayez les transactions liées au FTR.

  20. Joignez le fichier source original au test du fichier FTR. La jonction sera uniquement mise en correspondance, et la correspondance sera basée sur les champs sélectionnés pour le sous-ensemble ainsi que sur le champ du montant correspondant au champ du montant le plus élevé.
