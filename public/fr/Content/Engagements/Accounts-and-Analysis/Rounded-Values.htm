Valeurs arrondies

Ce test analytique Valeurs arrondies repère les valeurs numériques qui peuvent être arrondies, comme 10, 500 ou 9 000. Dans les données comptables, la présence de montants arrondis est généralement rare et peut indiquer une manipulation potentielle des données.

Ce test analytique peut être utilisé pour repérer :

  • Les possibles tentatives de manipulation des états financiers

  • Diminution de l'exactitude et de la précision de domaines de l'information financière

  • Les transactions qui contiennent des estimations significatives

  • Un risque d'anomalie significative due à la fraude

Remarque : Remarque : Certaines industries ou entreprises peuvent avoir de nombreuses transactions arrondies en raison de la nature de leurs opérations.

Champs utilisés pour l'analyse

Ce test utilise un ou plusieurs champs numériques pour détecter les montants arrondis ou les valeurs de quantité arrondies. Vous pouvez définir les champs à utiliser en modifiant le paramètre Sélectionner les champs.

Paramètres

Trois paramètres doivent être définis pour que le test puisse être exécuté :

  • Sélectionner les champs – Sélectionnez dans la liste un ou plusieurs champs numériques à utiliser pour effectuer le test.

  • Valeur d'arrondi – Saisissez une valeur entière arrondie, telle que 10, 100 ou 1 000, à utiliser pour rechercher des valeurs arrondies. Par exemple, si vous saisissez 10, le test recherchera toutes les valeurs qui sont des multiples de 10, comme 10, 20 et 30. Ce test n'est pas limité aux multiples de 10. Vous pouvez également saisir 5, 50, 500 ou toute autre valeur entière. La valeur par défaut est 100.

  • Ignorer les décimales – Sélectionnez s'il faut ignorer ou non les valeurs décimales. Ce champ détermine si les valeurs décimales sont considérées comme des valeurs arrondies. Par exemple, si vous choisissez d'ignorer les décimales, la valeur 100,99 est considérée comme une valeur arrondie, mais si vous choisissez de ne pas ignorer les décimales, il ne s'agit pas d'une valeur arrondie. La valeur par défaut est d'ignorer les décimales.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Montants arrondis – Dresse la liste des lignes de transaction dont le montant est arrondi.

  • Quantité arrondie – Dresse la liste des lignes de transaction dont la quantité est arrondie.

  • Valeurs arrondies – Dresse la liste des lignes de transaction dont la valeur est arrondie.

Spécifications techniques

Lorsque vous exécutez le test analytique des montants arrondis, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Vérifiez que le ou les champs analytiques principaux sont numériques, si ce n'est pas le cas, quittez le test.

  2. Déterminez si les décimales seront incluses dans l'analyse.

  3. Obtenez de l'utilisateur la valeur numérique qui sera utilisée pour l'arrondi, effectuez le test et extrayez toutes les transactions qui sont des valeurs arrondies de la valeur numérique pour le résultat. Veillez à intégrer si l'utilisateur a inclus ou exclu les décimales. Si plusieurs colonnes ont été sélectionnées et qu'une colonne échoue, la ligne échouera au test et sera placée dans les résultats.

  4. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.
