Combinaisons de comptes inhabituelles

Le test analytique des combinaisons de comptes inhabituelles permet d'identifier les transactions qui présentent des combinaisons inhabituelles par rapport au reste de la population. L'utilisateur peut décider du pourcentage de la population qui serait considéré comme inhabituel. The default is 3%.

Ce test analytique peut être utilisé pour repérer :

  • Les possibles tentatives de manipulation des états financiers

  • La baisse de l'exactitude et de la précision des rapports financiers

  • Transactions potentiellement non soutenues

  • Une insuffisance dans les activités de surveillance et de détection

  • Une formation insuffisante

  • Les transactions complexes, irrégulières ou inhabituelles qui peuvent justifier une enquête approfondie

  • Interprétation erronée des méthodes comptables en raison d'un manque de formation ou d'erreurs de système entraînant des classifications incorrectes dans les états financiers

  • Les paramètres du système qui ne sont pas correctement configurés

  • Transactions peu fréquentes

  • Un risque d'anomalie significative due à la fraude

Champs utilisés pour l'analyse

Les champs ci-dessous sont utilisés pour cette analyse :

Champs principaux –

  • Un ou plusieurs champs qui déterminent comment le groupe définira les combinaisons. Par exemple, vous pouvez utiliser le champ Journal.

  • Un ou plusieurs champs qui déterminent les regroupements. Il s'agit généralement du champ Numéro de compte , mais d'autres champs peuvent également être utilisés si nécessaire.

  • Champ Montant Un seul champ de montant peut être sélectionné pour le regroupement.

Paramètres

Un paramètre doit être défini pour que ce test puisse être exécuté :

Pourcentage pour les combinaisons inhabituelles - Saisir un pourcentage jusqu'à deux décimales à utiliser pour déterminer ce qui est considéré comme une combinaison inhabituelle. Ce pourcentage doit être supérieur à 0 % et peut atteindre un maximum de 50 %. La valeur par défaut est 11.

Spécifications techniques

Lorsque vous exécutez le test analytique Combinaisons de comptes complexes, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.

  2. Validez que les colonnes de référence nécessaires ont été sélectionnées, à savoir qu'une colonne de montant a été sélectionnée, une ou plusieurs colonnes qui composent la colonne de regroupement telle qu'un numéro de compte et une ou plusieurs colonnes qui composent l'ID telle que le numéro d'écriture du journal.

  3. Créez une colonne NEW_ID qui concatène la ou les colonnes qui composent l'ID.

  4. Créez une colonne NEW_GROUP qui concatène la ou les colonnes qui composent le groupement.

  5. Triez les données par NEW_ID et NEW_GROUP.

  6. Remplacer la colonne NEW_GROUP par des valeurs numériques croissantes. Pour ce faire, la colonne NEW_GROUP est regroupée pour obtenir des valeurs uniques, puis un numéro unique est attribué à partir de zéro. Cette étape est réalisée au cas où la longueur de la colonne de NEW_GROUP serait trop importante.

  7. Extraire les combinaisons de crédits par ID. Pour ce faire, il faut extraire tous les montants négatifs (crédit). Les combinaisons de crédits sont ajoutées à une colonne CREDIT_COMBINATIONS. Par exemple, si NEW_GROUP_NUMERIC a les groupes 1 et 2, la colonne CREDIT_COMBINATIONS indiquera 1, 2.

  8. Extraire les combinaisons de débit par ID. Pour ce faire, il faut extraire tous les montants positifs (débit). Les combinaisons de débit sont ajoutées à une colonne DEBIT_COMBINATIONS. Par exemple, si NEW_GROUP_NUMERIC a les groupes 1 et 2, la colonne DEBIT_COMBINATIONS indiquera 1, 2.

  9. Regrouper les NEW_ID pour obtenir des valeurs uniques.

  10. Pour chaque NEW_ID, ajoutez les DEBIT_COMBINATIONS et les CREDIT_COMBINATIONS. Cette étape permet d'obtenir une combinaison unique pour chaque identifiant.

  11. Remplacer les NaN par des valeurs vides.

  12. Regroupez les DEBIT_COMBINATIONS et les CREDIT_COMBINATIONS afin d'obtenir le nombre de chaque combinaison.

  13. Ajouter un pourcentage basé sur l'effectif / l'effectif total.

  14. Extract all rows that are equal to or less than the unusual_combinations_percentage value.

  15. Fusionner l'ensemble de données précédent par identifiant pour obtenir les identifiants dont la valeur est inférieure au pourcentage de combinaisons inhabituelles.

  16. Fusionner l'ensemble de données précédent avec l'ensemble de données original pour obtenir toutes les transactions qui appartiennent aux identifiants marqués.

  17. Supprimer toutes les colonnes créées afin que le résultat ne contienne que les colonnes de l'enregistrement d'origine.

  18. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.
