Éléments inhabituels – Comparaison avec la population

Le test analytique Éléments inhabituels – Comparaison avec la population permet de repérer les transactions considérées comme inhabituelles par rapport à un groupe de transactions sélectionnées.

Par exemple, ce test peut être utilisé pour trouver les transactions enregistrées par les utilisateurs qui enregistrent peu de transactions par rapport aux autres. Les utilisateurs seraient comparés à la population pour obtenir un pourcentage de transactions, et tout utilisateur en dessous d'un certain seuil apparaîtrait dans le résultat.

Vous pouvez également utiliser ce test pour analyser les champs de date. La population de transactions est totalisée par date, et les dates présentant un faible nombre de transactions sont signalées dans le résultat.

Ce test analytique peut être utilisé pour repérer :

  • Les transactions fictives ou non justifiées

  • Des problèmes systémiques dans le traitement des transactions par un service

  • Des paramètres du système mal configurés pouvant nécessiter une amélioration et une évaluation plus poussée

  • Les possibles tentatives de manipulation des états financiers

  • Les domaines de l'information financière où la précision et l'exactitude sont moindres

  • La déficience des contrôles internes qui peut avoir d'autres conséquences dont il faut tenir compte

  • Une insuffisance dans les activités de surveillance et de détection

  • Les transactions effectuées en dehors des jours ouvrables normaux

  • Un risque d'anomalie significative due à la fraude

Champs utilisés pour l'analyse

Les champs suivants sont obligatoires pour cette analyse :

  • Champs de référence – Champs uniques utilisés pour créer un identifiant de transaction unique, comme le champ ID de l’écriture pour l’ensemble de données du grand livre général. Ces colonnes ne font pas partie du résultat, mais elles sont utilisées pour trouver les transactions qui font partie du résultat. Ce champ est déjà défini dans le test et ne peut pas être modifié.

  • Champs inhabituels – Les champs dont les valeurs sont examinées pour déterminer si elles sont inhabituelles. Il peut s'agir de champs numériques, de dates, d'heures ou de caractères contenant un certain type de schéma récurrent. Par exemple, un champ numérique contenant des identifiants d'utilisateurs pourrait être utilisé pour relever les transactions enregistrées par des utilisateurs qui enregistrent rarement des transactions. Les champs contenant des montants ou des quantités ne sont pas les mieux adaptés à ce test, car ils ne présentent généralement pas de schéma récurrent.

Paramètres

Les paramètres suivants doivent être définis pour ce test :

  • Valeur en pourcentage – Saisissez la valeur en pourcentage que le test utilisera pour déterminer quelles valeurs doivent être considérées comme inhabituelles. Par exemple, si vous saisissez 3 %, tout élément de la colonne qui représente 3 % ou moins de la population est considéré comme inhabituel et inclus dans les résultats du test.

  • Type de pourcentage – Si le pourcentage doit être inférieur, inférieur ou égal, égal, supérieur ou supérieur ou égal à la valeur du pourcentage.

Configurations de test

Remarque : Remarque : Les configurations proposées varient en fonction du produit que vous utilisez.

Les configurations suivantes sont possibles pour ce test :

  • Éléments inhabituels par rapport à la population

  • Éléments de date inhabituels par rapport à la population

Spécifications techniques

Lorsque vous exécutez le test analytique Éléments inhabituels – Comparaison avec la population, il faut réaliser les étapes suivantes pour l’exécution du test :

  1. Si nécessaire, placez des filtres sur les données afin qu'un sous-ensemble soit utilisé pour l'analyse. Si aucun filtre n'est placé, l'analyse sera effectuée sur l'ensemble du fichier de données. Cette étape peut également être réalisée à la dernière étape, et non à la première.

    Remarque : Remarque : Les filtres ne sont pas encore accessibles et seront inclus dans une version ultérieure.

  2. Vérifiez que les champs de référence nécessaires ont été sélectionnés. Si les champs n'ont pas été sélectionnés, créez un champ de référence unique. Cette étape est réalisée seulement si des champs spécifiques ont été sélectionnés. Si tous les champs sont accessibles, cette étape n'est pas nécessaire.

  3. Obtenez les colonnes pour créer le sous-ensemble sur lequel l'analyse sera effectuée. Ce sous-ensemble peut être basé sur une ou plusieurs colonnes, celles-ci pouvant être de n'importe quel type et dans n'importe quel ordre.

  4. Obtenez le pourcentage pour comparaison.

    • Si l'utilisateur a sélectionné l'une des options suivantes, un seul pourcentage est nécessaire :

      • Inférieur à (<)

      • Inférieur ou égal à (<=)

      • Égal à (==)

      • Supérieur ou égal à (>=)

      • Supérieur à (>)

    • Si l'utilisateur sélectionne une plage, il doit y avoir un pourcentage de valeur inférieure et un pourcentage de valeur supérieure. La plage peut être basée sur les éléments suivants :

      • Supérieur à (>)

      • Supérieur ou égal à (>=)

      • Inférieur ou égal à (<=)

      • Inférieur à (<)

  5. Totalisez les colonnes des sous-ensembles afin d'obtenir le nombre d'enregistrements par sous-ensemble unique.

  6. Calculez le pourcentage de chaque sous-ensemble sur la base du nombre total d'enregistrements dans la population.

  7. Extrayez tous les pourcentages des sous-ensembles qui se situent dans la comparaison de pourcentages sélectionnée à l'étape 4.

  8. Extrayez les transactions (lignes) qui se rapportent aux sous-ensembles extraits à l'étape 7.

  9. Extrayez les champs de résultats sélectionnés par l'utilisateur. Tous les champs sont extraits par défaut.

    Remarque : Remarque : La capacité à choisir des champs de résultats n’est pas encore proposée et sera incluse dans une version ultérieure.
