Préparer un fichier CSV ou Excel pour importer des transactions ou des données de comptabilité auxiliaire

Vous pouvez importer vos transactions ou d'autres données de comptabilité auxiliaire dans un engagement Cloud à partir de fichiers CSV ou Excel correctement formatés.

Pour plus d'informations sur l'importation de transactions ou d'autres données de comptabilité auxiliaire à partir d'un fichier CSV ou Excel dans les missions Cloud, voir Importer les transactions et autres données de comptabilité auxiliaire du client à partir d'un fichier CSV ou Excel.

Exigences relatives au fichier CSV

Pour garantir la réussite de l'importation :

  • Le fichier doit être un fichier .csv avec un encodage UTF-8 .

  • Si le fichier CSV comprend des en-têtes de colonne sans données (seules les valeurs d'en-tête existent), les fichiers sont rejetés et le téléchargement échoue.

  • La taille du fichier doit être inférieure à 5 Go.

  • La première ligne du CSV (la ligne d'en-tête) est exclue de l'importation. Les transactions doivent commencer à la deuxième ligne.

  • S'il y a une valeur entre guillemets (") :

    • La valeur doit comporter deux guillemets (un guillemet ouvrant et un guillemet fermant). Si une valeur comporte un guillemet, ajoutez-en un autre ou supprimez le guillemet.

    • La valeur doit être échappée en l'entourant d'une paire supplémentaire de guillemets ou de barres obliques inverses (\).

    • Les valeurs composées de plus d'un mot - dont un ou deux ont des guillemets - doivent être entièrement placées entre guillemets.

      • Exemples :

      Dans les données ci-dessous, la valeur Definitely "real" cash consiste en un mot entre guillemets :

      accountId,accountDescription,amount

      123,Définitivement de l'argent "réel",1000000

      Vous devez le modifier pour qu'il ressemble à ceci :

      accountId,accountDescription,amount

      123, "Définitivement de la "vraie" monnaie",1000000

      Ou comme ceci :

      accountId,accountDescription,amount

      123, "Définitivement \"real\"cash",1000000

      Dans les données ci-dessous, la valeur "cash" est entre guillemets :

      accountId,accountDescription,amount

      123, "cash",1000000

      Vous devez le modifier pour qu'il ressemble à ceci :

      accountId,accountDescription,amount

      123,""cash"",1000000

      Ou comme ceci :

      accountId,accountDescription,amount

      123,\"real\",1000000

Exigences relatives au fichier Excel

Pour garantir une importation réussie :

  • Le format du fichier doit être .xlsx.

  • Le fichier ne doit pas contenir de cellules fusionnées.

  • Les données doivent exister sur la première feuille, car seule la première feuille est incluse dans l'importation.

  • Les données doivent commencer à la deuxième ligne, car la première ligne (la ligne d'en-tête) est exclue de l'importation.

  • Au moins une colonne doit contenir des données après la première ligne.

  • Il ne doit pas y avoir de lignes vides.

Champs obligatoires pour l'importation de transactions

Pour importer des transactions en vue de les visualiser ou d'exécuter une section d'échantillons, vous devez vous assurer que les champs suivants sont inclus dans les données d'importation :

  • Date de publication
  • Identifiant du compte
  • Montant
