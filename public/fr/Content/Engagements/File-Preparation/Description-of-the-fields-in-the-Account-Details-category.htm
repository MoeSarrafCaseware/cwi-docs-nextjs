Détails du compte décrits

Lorsque vous importez des transactions du grand livre à partir d'un fichier CSV ou Excel, vous devez mettre en correspondance les champs afin que les données des transactions s'affichent correctement dans la mission.

Cette rubrique décrit les champs de la catégorie DÉTAILS DU COMPTE qui apparaissent dans le jeu de données du grand livre.

Notez que certains champs sont obligatoires pour que certains tests d'analyse de données fonctionnent correctement. Pour en savoir plus, voir Champs obligatoires pour l'analyse des données.

Identifiant du compte(account_id)

Un identifiant ou le compte dans lequel l'entrée a été enregistrée.
Exemples "1001" ou "Compte bancaire 1".

Utilisation

  • Doit être étiqueté pour que toutes les caractéristiques liées à la transaction fonctionnent.
  • Associer la balance de vérification aux informations relatives aux transactions.
  • L'analyse de la balance de vérification permet d'obtenir des informations sur les transactions.
  • Dans tous les tests d'AnalyticsAI - Tous les tests requièrent un identifiant de compte.

Exigences

  • Moins de 5 000 valeurs uniques.
  • Correspondance avec les comptes de la balance de vérification importés.

Description du compte principal(account_main_description)

Description de l'identifiant du groupe de comptes dans lequel l'entrée a été enregistrée.
Exemples : " Trésorerie ", " Immobilisations corporelles ".

Utilisation

Description visuelle des comptes.

Saisi par(entered_by)

Identifiant de la personne qui crée l'entrée.

Utilisation

Dans ces tests AnalyticsAI :

  • Saisie et approbation par le même utilisateur
  • Utilisateurs inhabituels

Date de saisie(entered_date)

La date et l'heure réelles de l'enregistrement (validation) dans le grand livre - fournies automatiquement par le système - ne correspondent pas nécessairement à la date à laquelle l'enregistrement a été effectué dans le grand livre.

Utilisation

Dans les moments inhabituels, le test AnalyticsAI.

Exigences

Le format de la date doit être correctement défini :

Personne responsable de l'entrée(entry_responsible_person)

Identifiant de la personne qui approuve l'entrée.

Utilisation

Dans le cadre du test AnalyticsAI, le même utilisateur saisit et approuve l'analyse.
