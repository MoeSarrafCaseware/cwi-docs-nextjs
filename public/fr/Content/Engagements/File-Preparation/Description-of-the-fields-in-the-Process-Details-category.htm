Détails du processus décrits

Lorsque vous importez des transactions du grand livre à partir d'un fichier CSV ou Excel, vous devez mettre en correspondance les champs afin que les transactions s'affichent correctement dans la mission.

Cette rubrique décrit les champs de la catégorie DÉTAILS DU PROCESSUS qui apparaissent dans le jeu de données du grand livre.

Notez que certains champs sont obligatoires pour que certains tests d'analyse de données fonctionnent correctement. Pour en savoir plus, voir Champs obligatoires pour l'analyse des données.

Numéro du document d'origine(originating_document_number)

Numéro d'identification du document original qui lance le flux du processus de gestion. Il s'agit du document qui est le précurseur de cette transaction enregistrée.
Exemples : "PO-001", "132134".

Exigences

  • Moins de 2 500 valeurs uniques.
  • Les données de la colonne applicable dans le fichier d'importation de la transaction (le fichier CSV) doivent être affectées aux valeurs de numéro de document d'origine ci-dessous :
    • GJ - Journal général
    • AR - Ajustements des comptes clients (crédits, retours, autres ajustements)
    • PC - Contrats d'achat
    • CD - Décaissements
    • AP - Ajustements des comptes fournisseurs (crédits, retours, autres ajustements)
    • PR - Demandes d'achat
    • CR - Encaissements
    • FA - Immobilisations
    • PO - Bons de commande
    • SJ - Journal des ventes (liste des factures de vente)
    • IA - Actifs d'investissement
    • GS - Marchandises expédiées
    • PJ - Journal des achats (liste des factures d'achat)
    • SC - Contrats de vente
    • GR - Marchandises reçues
    • PL - Journal des salaires
    • SO - Commandes clients
    • OT - Autres
