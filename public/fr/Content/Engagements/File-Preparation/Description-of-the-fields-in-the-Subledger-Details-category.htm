Détails du grand livre auxiliaire décrits

Lorsque vous importez des transactions du grand livre à partir d'un fichier CSV ou Excel, vous devez mettre en correspondance les champs afin que les données de la transaction s'affichent correctement dans la mission.

Cette rubrique décrit les rubriques de la catégorie DÉTAILS DU SOUS-JACENT qui apparaissent dans le jeu de données du grand livre.

Notez que certains champs sont obligatoires pour que certains tests d'analyse de données fonctionnent correctement. Pour en savoir plus, voir Champs obligatoires pour l'analyse des données.

Type d'identifiant(identifier_type)

Type d'identification des données dans le livre auxiliaire.

Exigences

  • Moins de 2 500 valeurs uniques. Ces valeurs doivent être associées à l'une d'entre elles :
    • CU - Clients
    • PE - Dépenses payées d'avance
    • SU - Fournisseurs
    • EM - Employés
    • IA - Actifs d'investissement
    • OT - Autres
    • CA - Immobilisations (actifs fixes)
    • IN - Inventaire
    • DL - Passif de la dette

Code d'identification(code_identifiant)

Code d'identification des données dans le livre auxiliaire.
Exemple : numéro de client, numéro de fournisseur, numéro de salarié.
Exemples : "C123", "V6589", "6597".

Exigences

Si le type d'identifiant est balisé, il doit l'être également.

Description de l'identifiant(identifier_description)

Nom associé au code d'identification.
Exemples : "ABC Customer Ltd.", "DEF Purchase Inc.", "Sally Jones".

Identifiant actif(identifier_active)

Indique si le code d'identification est actif ou inactif.

Rue de l'identifiant(identifier_street)

Rue de l'identifiant - ligne 1.
Exemples : "123 Anywhere Street", "Anywhere 26".

Identifiant Rue 2(identifier_address_street2)

Détail de l'adresse de l'identifiant - ligne 2.
Exemples : "Suite 1100", "Unité 002B"

Ville de l'identifiant(identifier_city)

Ville où se trouve l'identifiant.
Exemples : "Toronto", "Los Angeles".

État ou province de l'identifiant(identifier_state_ou_province)

État ou province où se trouve l'identifiant.
Exemples : "Ontario", "Californie".

Code postal de l'identifiant(identifier_zip_ou_postal_code)

Code postal où se trouve l'identifiant.
Exemples : "94704", "M5A 0L6".

Pays de l'identifiant(identifier_country)

Pays (code) où se trouve l'identifiant.

Numéro du document(document_number)

Identifiant associé au document source. Il s'agit du document qui justifie la transaction saisie dans le PGI.
Exemples : "PO4569" pour le bon de commande, "7956" pour le chèque.

Application du document au numéro(document_Apply_To_Number)

Numéro de document associé au document source utilisé pour compenser les débits et les crédits. Utilisé pour calculer le solde ouvert lorsque le paiement, l'avance, la note de crédit/débit n'ont pas été entièrement appliqués, ce qui permet de calculer l'élément ouvert.
Exemples : "INV-001", "121296".
