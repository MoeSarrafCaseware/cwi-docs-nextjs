Détails de la transaction décrits

Lorsque vous importez des transactions du grand livre à partir d'un fichier CSV ou Excel, vous devez mettre en correspondance les champs afin que les données des transactions s'affichent correctement dans la mission. En outre, certains champs sont cohérents d'une ligne à l'autre dans une entrée. Lors d'une importation, ces champs sont stockés séparément et sont remplis avec les valeurs de la première ligne de l'entrée. Pour plus d'informations, voir Comment les données du grand livre sont stockées lors de l'importation.

Cette rubrique décrit les champs de la catégorie DÉTAILS DE TRANSACTION qui apparaissent dans le jeu de données du grand livre.

Notez que certains champs sont obligatoires pour que certains tests d'analyse de données fonctionnent correctement. Pour en savoir plus, voir Champs obligatoires pour l'analyse des données.

Numéro d'entrée(entry_number)

Identifiant séquentiel unique pour chaque écriture au journal.

Utilisation

  • Regroupe les lignes de transaction dans les transactions (si l'ID d'entrée n'est pas marqué).
  • Identifiant visuel pour les transactions.
  • Si le numéro et l'identifiant d'entrée ne sont pas marqués, chaque transaction se voit attribuer un identifiant unique aléatoire.
  • Dans le test Sequence Gaps AnalyticsAI.

Exigences

  • Si l'ID d'entrée n'est pas balisé, il ne doit pas y avoir de cellules de numéro d'entrée vides.

  • Si l'ID d'entrée est attribué, seule la première valeur du numéro d'entrée pour chaque ID d'entrée unique sera importée.

Montant(montant)

Montant de la ligne de transaction. Cette valeur peut être positive pour un débit et négative pour un crédit.

Utilisation

  • Rapprochement/complétude des données.
  • Dans ces tests AnalyticsAI :
    • Entrées en double
    • Se termine en 999
    • Montants élevés
    • Montants arrondis
    • 2 chiffres de Benford
    • Montants nuls

Exigences

  • Les valeurs positives et négatives peuvent être incluses dans l'un ou l'autre :
    • Une colonne avec des valeurs négatives formatées dans un formatage négatif standard. En voici quelques exemples :
      • -4000.00
      • (4000,00)
      • 4000,00-
    • Séparer les colonnes "débit" et "crédit".
  • Les virgules ou les points sont interprétés comme des séparateurs décimaux.

Commentaire détaillé(detail_comment)

Description de l'entrée de ligne.

Utilisation

Dans ces tests AnalyticsAI :

  • Descriptions manquantes
  • Mots clés spécifiques

Date d'enregistrement(posting_date)

Date d'entrée en vigueur - l'entrée de données a été comptabilisée (validée) dans le grand livre, quelle que soit la date de création.

Utilisation

  • Associe les transactions à une date et à une période fiscale.
  • Dans ces tests AnalyticsAI :
    • Entrées en double
    • Jours insolites

Exigences

Pour être affichées dans la liste des transactions, les transactions doivent se situer dans la fourchette d'exercices définie dans l'engagement.

Statut du détachement(posting_status)

Indique si l'entrée a été comptabilisée ou non.

Utilisation

  • Les opérations marquées comme non comptabilisées ne sont pas utilisées pour calculer la somme des opérations dans le cadre des contrôles d'exhaustivité des données.
  • Les transactions dont le statut de comptabilisation est défini sur comptabilisé sont utilisées dans les tests AnalyticsAI.

Pour en savoir plus, voir les résultats des tests d'AnalyticsAI.

Exigences

  • Moins de 2 500 valeurs uniques. Ces valeurs doivent être associées à l'une d'entre elles :

    • Affiché
    • Non-affiché

Type de document(document_type)

Champ énuméré décrivant le document source original, avec chèque, débit-mémo, crédit-mémo, frais financiers, facture, commande-client, commande-fournisseur, paiement-autre, rappel, tegata, bon, expédition, réception, ajustement manuel, autre.

Utilisation

  • Filtrage des transactions en fonction du type de journal.
  • Dans le test AnalyticsAI des utilisateurs inhabituels.

Exigences

  • Moins de 2 500 valeurs uniques.
  • Les données de la colonne applicable dans le fichier d'importation de la transaction (le fichier CSV) doivent être affectées aux valeurs de type de document ci-dessous :
    • GJ - Entrées du grand livre général
    • CD - Décaissement de trésorerie
    • CR - Encaissements
    • SJ - Factures de vente
    • PJ - Factures d'achat
    • PL - Entrées dans le grand livre des salaires
    • AR - Ajustements des comptes clients (crédits, retours, autres ajustements)
    • AP - Ajustements des comptes fournisseurs (crédits, retours, autres ajustements)
    • FA - Ajustements de l'actif immobilisé
    • IN - Ajustements des stocks
    • SC - Contrats de vente
    • SO - Commandes clients
    • PC - Achats Contrats
    • PR - Demandes d'achat
    • PO - Bons de commande
    • GS - Marchandises expédiées
    • GR - Marchandises reçues
    • OT - Autres
