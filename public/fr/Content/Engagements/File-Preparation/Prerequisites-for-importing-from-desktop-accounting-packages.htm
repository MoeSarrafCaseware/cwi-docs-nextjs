Conditions préalables à l'importation à partir d'un logiciel de comptabilité de bureau

Vous pouvez importer des données clients à partir de la plupart des logiciels comptables tiers pour les utiliser avec une variété d'applications Caseware Cloud ou IDEA. Avant de procéder à l'importation, assurez-vous d'avoir effectué les étapes préalables pour votre logiciel. Pour en savoir plus sur l'importation de données clients, voir Importer des données à partir d'un logiciel de comptabilité de bureau.

Remarque : Si votre logiciel de comptabilité ne figure pas sur cette page, il n'y a pas de conditions préalables. Vous pouvez suivre les instructions de la section Importer des données à partir d'un logiciel de comptabilité de bureau pour importer les données de vos clients.

Accès aux dimensions de la comptabilité

  • Installez Microsoft Windows SQL Server localement sur la machine qui effectue l'importation.

  • Fichier d'importation : filename.bak

    (Obtenu en lançant Accounting Dimensions et en effectuant une opération de sauvegarde de l'ensemble des données)

  • Conservez une copie de sauvegarde de votre base de données à l'aide de l'outil Access Dimensions Administrator (ADA) fourni avec le progiciel.

  • Si l'importation est exécutée sur la même machine que celle où le fichier Access Accounting Dimensions original est actif, assurez-vous que le nom du fichier de la base de données de sauvegarde est différent du nom de la base de données active et que le fichier de la base de données se trouve dans un répertoire différent de celui de la base de données active. Sinon, la base de données originale doit être supprimée du serveur SQL avant l'importation.
