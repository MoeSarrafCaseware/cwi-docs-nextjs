Stock disponible décrit

Lorsque vous importez des transactions de comptabilité auxiliaire à partir d'un fichier CSV ou Excel, vous devez affecter les champs du fichier aux champs Caseware dans le type d'ensemble de données de comptabilité auxiliaire que vous sélectionnez lors de l'importation, afin que les données de la transaction s'affichent correctement dans la mission. Cette rubrique détaille les champs du livre auxiliaire Inventaire disponible.

Pour plus d'informations sur les descriptions des jeux de données du livre auxiliaire, voir Comprendre les données du livre auxiliaire.

Pour plus d'informations sur l'importation de données de transaction à partir de fichiers CSV, voir Importer les transactions du client à partir d'un fichier CSV.

Champ d'application Utilisation
ID de la transaction
(inventory_on_hand_transaction_id)

L'identifiant de la transaction associé aux normes de données d'audit du grand livre auxiliaire (ADS) Procure to Pay ou Order to Cash.

Type de données : Texte

Il s'agit d'une obligation : Non

Clé primaire : La valeur de ce champ est utilisée pour former l'identifiant unique de cette entrée.
ID du produit en stock
(inventory_on_hand_inventory_product_id)

L'identifiant du produit est un code qui identifie de manière unique un article que vous avez en stock.

Type de données : Texte

Il s'agit d'une obligation : Oui

Clé primaire : Premier suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
Numéro de série
(numéro de série de l'inventaire_en_main)

Le numéro associé à un produit individuel qui est utilisé pour le suivi et la gestion. Par exemple, une entreprise achète 10 ordinateurs, chaque ordinateur se voyant attribuer un numéro de série individuel.

Type de données : Texte

Il s'agit d'une obligation : Non

Clé primaire : Sixième suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
Numéro de lot
(numéro de lot de l'inventaire_en_main)

Numéro associé à un produit, utilisé pour le suivi et la gestion du lot.

Type de données : Texte

Il s'agit d'une obligation : Non

Clé primaire : Cinquième suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <FirstAlternate>-<SecondAlternate>-<ThirdAlternate>-<FourthAlternate>-<FifthAlternate>-<SixthAlternate>.
ID du lieu
(inventory_on_hand_location_id)		 L'identifiant unique d'un emplacement spécifique au sein d'une division ou d'un site où les stocks sont reçus, conservés et distribués. Type de données : Texte requis : Non
Stocks détenus pour les fournisseurs
(inventaire_en_main_inventaire_détenu_pour_les_fournisseurs)

Vrai, si l'inventaire est physiquement présent et qu'il est détenu pour les vendeurs ; faux, dans le cas contraire.

Type de données : Enum

Il s'agit d'une obligation : Non

Valeurs possibles : true ou false.

(inventaire_en_main_inventaire_détenu_pour_les_clients)

Il s'agit d'une obligation : Non

Valeurs possibles : true ou false.

(inventaire_en_main_inventaire_détenu_pour_les_autres)		 Vrai, si le stock est physiquement présent et s'il est détenu pour d'autres personnes que des clients ou des fournisseurs ; faux, dans le cas contraire. Type de données : Enum Requis : Non Valeurs possibles : vrai ou faux.

Exemples de valeurs : lent, endommagé ou obsolète
