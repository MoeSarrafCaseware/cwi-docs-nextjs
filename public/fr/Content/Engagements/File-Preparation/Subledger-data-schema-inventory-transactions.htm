Mouvements de stock décrits

Lorsque vous importez des transactions de comptabilité auxiliaire à partir d'un fichier CSV ou Excel, vous devez affecter les champs du fichier aux champs Caseware dans le type d'ensemble de données de comptabilité auxiliaire que vous sélectionnez lors de l'importation, afin que les données de la transaction s'affichent correctement dans la mission. Cette rubrique détaille les champs du livre auxiliaire des transactions d'inventaire.

Pour plus d'informations sur les descriptions des jeux de données du livre auxiliaire, voir Comprendre les données du livre auxiliaire.

Pour plus d'informations sur l'importation de données de transaction à partir de fichiers CSV, voir Importer les transactions du client à partir d'un fichier CSV.

Champ d'application Utilisation
ID de la transaction
(inventory_transaction_transaction_id)

L'identifiant de la transaction associé aux normes de données d'audit du grand livre auxiliaire (ADS) Procure to Pay ou Order to Cash.

Type de données : Texte

Il s'agit d'une obligation : Non

Clé primaire : La valeur de ce champ est utilisée pour former l'identifiant unique de cette entrée.
Date de la transaction
(date_transaction_inventaire)

La date de l'activité, par document de transaction associé, le cas échéant.

Type de données : Date

Il s'agit d'une obligation : Non

Clé primaire : Troisième suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <PremierAlternatif>-<DeuxièmeAlternatif>-<TroisièmeAlternatif>.
ID du produit en stock
(inventory_transaction_product_id)

L'identifiant du produit est un code qui identifie de manière unique un article que vous avez en stock.

Type de données : Texte

Il s'agit d'une obligation : Oui

Clé primaire : Premier suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <PremierAlternatif>-<DeuxièmeAlternatif>-<TroisièmeAlternatif>.
Numéro de série
(numéro de série de l'opération d'inventaire)

S'il est sérialisé, le numéro de série est saisi ici. Bien que le numéro de série soit le terme courant, la valeur n'est pas limitée à des valeurs numériques. S'il est rare qu'un article soit à la fois contrôlé par un numéro de série et par un lot (le numéro de série identifie l'article sans ambiguïté), ces deux types de contrôle ne s'excluent pas nécessairement l'un l'autre.

Type de données : Texte

Il s'agit d'une obligation : Non

Clé primaire : Quatrième suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <PremierAlternatif>-<DeuxièmeAlternatif>-<TroisièmeAlternatif>.
Numéro de lot
(numéro de lot de l'opération d'inventaire)

S'il s'agit d'un lot, le numéro de lot est saisi ici. Bien que le numéro de lot soit le terme courant, la valeur n'est pas limitée à des valeurs numériques.

Type de données : Texte

Il s'agit d'une obligation : Pas de clé primaire : Cinquième solution. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <PremierAlternatif>-<DeuxièmeAlternatif>-<TroisièmeAlternatif>.
Type de transaction
(type_transaction_inventaire)

Les codes qui décrivent les principales actions spécifiques à l'inventaire qui ont lieu dans les transactions d'inventaire liées au mouvement des marchandises, au commerce, aux ajustements de quantité résultant d'événements commerciaux internes et aux ajustements de coût, dus à des facteurs de marché ou à des dommages.

Type de données : Enum

Il s'agit d'une obligation : Non

Exemples de valeurs : Entrées de marchandises (GR), sorties de marchandises (GS), transferts de stocks (IT), ajustements de stocks (IA), autres (OT)

Clé primaire : Deuxième suppléant. Si l' identifiant de transaction n'est pas présent dans les données, la valeur de ce champ est concaténée avec les valeurs des autres champs alternatifs pour former l'identifiant unique de l'entrée, par exemple <PremierAlternatif>-<DeuxièmeAlternatif>-<TroisièmeAlternatif>.

(inventory_transaction_transaction_associated_document_id)

Il s'agit d'une obligation : Non

(inventory_transaction_transaction_associated_document_line_id)
