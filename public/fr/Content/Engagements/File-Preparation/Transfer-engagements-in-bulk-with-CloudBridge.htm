Transférer des engagements en masse avec CloudBridge

Utilisez CloudBridge pour transférer les données de plusieurs Working Papers vers Cloud.

Remarque : Ces instructions ne s'appliquent qu'à CloudBridge (toutes les régions, en vrac). Pour les instructions de transfert individuel, voir Transfert d'engagements individuels avec CloudBridge.

Obtenir CloudBridge (Toutes les régions, en vrac)

Contactez votre représentant du support Caseware pour demander l'accès à l'utilitaire.

Cette version de CloudBridge est :

  • Destiné aux applications de toutes les régions.

  • Utilisé pour transférer les données de plusieurs Working Papers vers le Cloud.

  • Autonome ; il n'a pas besoin d'être ajouté à d'autres documents de travail.

  • Convient à une migration vers l'informatique en nuage à l'échelle de l'entreprise.

Meilleures pratiques

Avant de commencer, nous vous recommandons d'appliquer les bonnes pratiques suivantes :

  • Organisez les fichiers sur votre disque local ou réseau (par exemple, par ordre alphabétique, par priorité, par ordre de téléchargement, etc.)

  • Pour les utilisateurs hybrides, téléchargez les copies de synchronisation depuis le Cloud en utilisant l' outil Caseware Batch

  • Créer (ou importer) toutes les entités nécessaires

En outre, nous vous recommandons

  • Fermez toutes les autres applications pendant l'exécution de CloudBridge.

  • Exécutez Cloudbridge pendant la nuit ou en dehors des heures de travail.

  • Limitez chaque téléchargement à 500 engagements à la fois.

Configurer CloudBridge

Après avoir installé CloudBridge, vous devez vous connecter au site Cloud de destination où les données d'engagement seront transférées.

Pour configurer CloudBridge :

  • Faites un clic avec le bouton droit de la souris sur le fichier de commandes et choisissez Exécuter en tant qu'administrateur. Ouverture de CloudBridge.

  • Dans la boîte de dialogue Modifier les serveurs, saisissez le nom et l'URL du site Cloud de destination où les données d'engagement seront transférées. Vous pouvez ajouter d'autres sites Cloud en sélectionnant Ajouter un serveur, mais notez que les données ne peuvent être transférées que vers une seule application dans un seul site Cloud à la fois. Lorsque vous avez terminé, sélectionnez Terminé.

  • Dans le menu déroulant Serveur, sélectionnez le site Cloud concerné et cliquez sur Connecter.

  • L'invite de connexion s'affiche. Saisissez vos informations d'identification et sélectionnez Sign-in pour vous connecter au site Cloud.

Transfert des données d'engagement

Une fois que vous avez connecté CloudBridge à votre site Cloud de destination, vous êtes prêt à commencer le transfert des données d'engagement.

Remarque : Les comptes dont le solde est nul ne sont pas automatiquement importés. Si vous avez besoin de comptes à solde nul dans un engagement Cloud de destination, vous devez les ajouter et les affecter manuellement.

Transférer les données relatives à l'engagement :

  1. Dans le menu principal de CloudBridge, sélectionnez Nouveau téléchargement.

  2. A l'étape Général :

    1. Saisissez un nom pour le formulaire.

    2. Sélectionnez l'application dans laquelle vous travaillerez avec les données d'engagement transférées. Notez que si vous devez utiliser les données dans une autre application, vous devez effectuer un transfert supplémentaire et séparé.

    3. Sélectionnez Next.

  3. Dans l'étape Paramètres du fichier source :

    1. Indiquez le répertoire des Working Papers sources contenant les données que vous souhaitez transférer.

    2. (Facultatif) Filtrer les missions des Working Papers sources en saisissant une date de début et une date de fin. Seules les données relatives aux engagements à l'intérieur de la plage seront transférées.

    3. Sélectionnez les points de données à transférer. Les options comprennent les ajustements, le grand livre, la balance des comptes (plusieurs années), les documents/dossiers externes et les risques et contrôles.

    4. Si vous sélectionnez Balance de vérification (plusieurs années), vous avez la possibilité d'utiliser un fichier de correspondance personnalisé. Pour plus d'informations, voir Créer une mission de consolidation .

    5. Sélectionnez les fichiers externes à transférer. Les options comprennent Word, Excel, PowerPoint, PDF, CSV, texte, JPG, PNG ou tous les fichiers externes.

    6. Sélectionnez Next.

  4. A l'étape des paramètres du fichier cloud :

    1. Sélectionnez une méthode de dénomination pour les nouveaux engagements créés à l'aide des données transférées. Les options comprennent le nom du fichier Working Papers, le nom d'exploitation et l'entité Cloud.

    2. Sélectionnez un format de date de fin d'année à inclure dans le nom des nouveaux engagements.

    3. Choisissez d'inclure la date de fin d'année ou de la placer au début (préfixe) ou à la fin (suffixe) des nouveaux noms d'engagement.

    4. Sélectionnez Next.

  5. Dans l'étape Fichiers à télécharger :

    1. Sélectionnez l'entité Cloud dans laquelle vous souhaitez transférer les données (pour chaque fichier source).

    2. Sélectionnez l'engagement Cloud dans lequel vous souhaitez transférer les données (pour chaque fichier source). Si aucun engagement dans le nuage n'est sélectionné, un nouvel engagement sera pris dans l'entité sélectionnée.

    3. Sélectionnez Next.

  6. A l'étape Upload settings (Paramètres de téléchargement) :

    1. Indiquez la durée maximale du transfert. Les options comprennent 1, 2, 3 et 4 heures. Une fois la durée écoulée, CloudBridge se fermera, même si le transfert est incomplet.

    2. Spécifiez le délai d'attente pour chaque engagement de source. Les options comprennent 5, 10, 15 et 20 minutes. Une fois la durée écoulée, le transfert passe à l'engagement source suivant, même si le précédent est incomplet.

    3. Sélectionnez Next.

  7. À l'étape du résumé :

    1. Passez en revue tous les paramètres que vous avez sélectionnés. Si vous devez apporter d'autres modifications, sélectionnez Précédent.

    2. Sélectionnez Télécharger les transactions.

Le transfert des données d'engagement commence. Vous verrez l'état d'avancement de votre transfert et recevrez un rapport final une fois celui-ci terminé. Pour effectuer un autre transfert, recommencez cette procédure.

Pour créer un fichier de remappage personnalisé :

Si votre entreprise utilise une structure de mappage personnalisée, vous devez créer votre propre fichier de remappage (.ini) en fonction de votre structure de mappage personnalisée.

Pour créer un fichier de remappage personnalisé :

  1. Copier les composants.

    1. Ouvrez votre fichier Working Papers.

    2. Nous vous recommandons de créer une copie de sauvegarde de votre fichier avant de continuer(Fichier | Sauvegarde).

    3. Sur le ruban, cliquez sur Fichier | Copier les composants. L'assistant de copie des composants s'affiche.

    4. Cliquez sur Copier dans ce fichier.

    5. Sur la page Sélectionner une source, cliquez sur l'onglet Fichier , puis sur Parcourir...

    6. Naviguez jusqu'à C:\NProgram Files (x86)\NCaseware\NScripts\Ncloudbridge\N et ouvrez le dossier de l'application, puis sélectionnez le fichier .ac (par exemple Caseware ReviewComp\NCaseware ReviewComp.ac). Cliquez sur Suivant.

    7. Dans la page Copier les composants, sélectionnez Groupements/Mappage.

    8. Sélectionnez Mapping et cliquez sur Options.. La boîte de dialogue Options de copie de mappage s'affiche.

  2. Numéros de remap.

    1. Cliquez sur Remap. La boîte de dialogue Remap s'affiche.

    2. Faites glisser un numéro de mappage de la liste Numéros de mappage de fichiers de destination et déposez-le dans le numéro de mappage approprié de la liste Numéros de mappage de fichiers de source .

    3. Répétez l'opération pour attribuer d'autres numéros de mappage.

    4. Une fois que vous avez terminé, cliquez sur Appliquer pour enregistrer vos paramètres, puis sur OK.

  3. Générer le fichier .ini .

    1. Dans la boîte de dialogue Options de copie de mappage , cliquez sur Annuler.

    2. Dans l' assistant de copie des composants, cliquez sur Annuler. Un fichier de remappage portant le nom copy.ini est créé dans le dossier de votre fichier Working Papers ( C:\NProgram Files (x86)\NCaseware\NData).

Vous pouvez maintenant utiliser ce fichier de remappage dans Cloudbridge pour transférer les données de vos Working Papers vers votre engagement.

Pour créer un fichier de remappage personnalisé :

  1. Renommez le fichier de remappage de manière à ce qu'il suive la convention d'appellation <Product Name>.ini, où <Product Name> correspond au nom du produit dans l'étape générale.

    Par exemple :

    • Caseware ReviewComp.ini

    • OnPoint Audit.ini

    • US Review.ini

  2. Déplacez le fichier de remappage vers C:ÚsersääPublicäCloudBridge.

  3. Dans l'étape Paramètres du fichier source, sélectionnez Balance de vérification (plusieurs années) | Utiliser un fichier de mappage personnalisé.

Remarque : Si CloudBridge signale "No mapping file found", vérifiez que le nom et l'emplacement du fichier de remappage sont corrects.
