Transférer des engagements individuels avec CloudBridge

Utiliser CloudBridge pour transférer les données d'un engagement individuel de Working Papers vers Cloud. Si vous n'avez pas installé CloudBridge, voir Obtenir CloudBridge.

Configurer CloudBridge

Avant de pouvoir transférer des données d'un engagement Working Papers vers Cloud, vous devez configurer CloudBridge en ajoutant le document CloudBridge (CLBG) au gestionnaire de documents dans Working Papers.

Ajouter le document CLBG aux documents de travail :

  1. Assurez-vous que votre copie des documents de travail est intégrée à votre entreprise en nuage. Pour en savoir plus, visitez Intégrer les documents de travail avec Caseware Cloud.

  2. Assurez-vous que vous avez installé CloudBridge.

  3. Ouvrez votre fichier client dans Working Papers.

  4. Dans l'onglet Document , cliquez sur À partir de la bibliothèque | CloudBridge. La boîte de dialogue Bibliothèque de documents s'affiche.

  5. Cliquez sur CLBG et cliquez sur Ajouter.

Le document CLBG s'affiche dans le gestionnaire de documents.

Transfert des données d'engagement

After you add the CLBG document to a Working Papers engagement, you can transfer your trial balance and general ledger tax data to your Cloud engagement (adjustments posted to groups are not supported, only account balances).

Remarque :

Vous pouvez également inclure les données financières de votre client dans le transfert. Pour ce faire, vous devez remplir les conditions préalables suivantes :

  1. Assurez-vous que Working Papers est intégré à votre site Cloud.

  2. Reportez votre dossier d'engagement des documents de travail.

  3. (Facultatif) Si vous transférez des données à partir d'une version non licenciée de Working Papers, copiez et collez cw:program/scripts/cloudbridge/index.html dans le champ URL de la barre d'outils Working Papers, puis appuyez sur la touche Entrée de votre clavier pour lancer CloudBridge.

  4. Complétez la procédure de transfert ci-dessous.

Notez que les comptes dont le solde est nul ne seront pas transférés automatiquement. Si vous souhaitez inclure des comptes à solde nul, vous devez les ajouter manuellement et les affecter aux groupes désignés.

Transférer les données relatives à l'engagement dans le nuage :

  1. Dans le gestionnaire de documents, double-cliquez sur CLBG. Le document CLBG s'affiche.

  2. Dans la section Étape 1, choisissez une entité et un engagement appropriés.

  3. Dans la section Étape 2, sélectionnez les données de balance de vérification appropriées à exporter.

  4. Dans la section Étape 2, cochez la case Utiliser un fichier de remappage . Le fichier de remappage est utilisé pour transférer la structure de mappage des comptes actuels dans les Working Papers vers votre engagement Cloud.

    1. Canada Si votre engagement utilise le mappage par défaut des documents de travail ou a été créé avec le modèle PEG, utilisez le fichier de remappage situé dans le dossier où vous avez installé CloudBridge(C:\NProgram Files (x86)\NCaseware\NScripts\Ncloudbridge). Si votre mission utilise une structure de mappage personnalisée propre à votre entreprise, créez un fichier de remappage personnalisé , puis ajoutez-le.

    2. ÉTATS-UNIS : Si votre engagement utilise le mappage par défaut des documents de travail ou a été créé avec le modèle PEG, utilisez le fichier de remappage situé dans le dossier où vous avez installé CloudBridge(C:\NProgram Files (x86)\NCaseware\NScripts\Ncloudbridge). Si votre mission utilise une structure de mappage personnalisée propre à votre entreprise, vous devrez modifier manuellement le fichier de remappage par défaut à l'aide d'un éditeur de texte (par exemple, le Bloc-notes).

  5. Sélectionnez Choisir un fichier pour rechercher et ajouter le fichier de remappage.

  6. Canada (Facultatif) Si vous avez d'autres écritures de journal relatives aux dons, aux frais de repas et de représentation ou à la publicité étrangère, la section Étape 3 est disponible sur la page. La sectionÉtape 3 vous permet de transférer les données GL à la feuille de calcul fiscale appropriée dans l'engagement Caseware ReviewComp. Dans cet exemple, nous ajouterons les dons aux clients à la feuille de calcul Donations.

  7. Une fois que vous avez terminé, sélectionnez Transférer les données.

Créer un fichier de remappage personnalisé (.ini)

Si votre entreprise utilise une structure de mappage personnalisée, vous devez créer votre propre fichier de remappage (.ini) en fonction de votre structure de mappage personnalisée.

Pour créer un fichier de remappage personnalisé :

  1. Copier les composants.

    1. Ouvrez votre fichier Working Papers.

    2. Nous vous recommandons de créer une copie de sauvegarde de votre fichier avant de continuer(Fichier | Sauvegarde).

    3. Sur le ruban, cliquez sur Fichier | Copier les composants. L'assistant de copie des composants s'affiche.

    4. Cliquez sur Copier dans ce fichier.

    5. Sur la page Sélectionner une source, cliquez sur l'onglet Fichier , puis sur Parcourir...

    6. Naviguez jusqu'à C:\NProgram Files (x86)\NCaseware\Scripts\Ncloudbridge\NCaseware ReviewComp et sélectionnez Caseware ReviewComp.ac, puis cliquez sur Suivant.

    7. Dans la page Copier les composants, sélectionnez Groupements/Mappage.

    8. Sélectionnez Mapping et cliquez sur Options.. La boîte de dialogue Options de copie de mappage s'affiche.

  2. Numéros de remap.

    1. Cliquez sur Remap. La boîte de dialogue Remap s'affiche.

    2. Faites glisser un numéro de mappage de la liste Numéros de mappage de fichiers de destination et déposez-le dans le numéro de mappage approprié de la liste Numéros de mappage de fichiers de source .

    3. Répétez l'opération pour attribuer d'autres numéros de mappage.

    4. Une fois que vous avez terminé, cliquez sur Appliquer pour enregistrer vos paramètres, puis sur OK.

  3. Générer le fichier .ini .

    1. Dans la boîte de dialogue Options de copie de mappage , cliquez sur Annuler.

    2. Dans l' assistant de copie des composants, cliquez sur Annuler.

Un fichier de remappage portant le nom copy.ini est créé dans le dossier de votre fichier Working Papers ( C:\NProgram Files (x86)\NCaseware\NData).

Vous pouvez maintenant utiliser ce fichier de remappage dans Cloudbridge pour transférer les données de vos Working Papers vers votre engagement.
