Comprendre les ensembles de données du grand livre auxiliaire

Voir Description des jeux de données de comptabilité auxiliaire pour les types de jeux de données de comptabilité auxiliaire que vous pouvez importer dans Caseware Cloud. Chaque description d'ensemble de données énumère les champs obligatoires et facultatifs qui composent les transactions pour ce type d'ensemble de données de comptabilité auxiliaire, ainsi que l'objectif de chaque champ. Le diagramme ci-dessous explique les éléments de base des descriptions des ensembles de données du grand livre auxiliaire.

Pour plus d'informations sur l'importation de données de transaction à partir de fichiers CSV, voir Importer les transactions du client à partir d'un fichier CSV.

Anatomie de la description d'un jeu de données du grand livre auxiliaire

 

Élément Description
1 Nom de l'utilisateur. Le nom du champ Caseware tel qu'il apparaît dans les menus déroulants de la fenêtre d'importation des données, où vous mettez en correspondance les champs de vos données entrantes avec les champs Caseware.
2 Description du champ. Elle détaille l'objectif du champ et, le cas échéant, des exemples de valeurs possibles.
3 Nom du champ interne. Le nom interne du champ Caseware, que vous devez connaître si vous manipulez des données importées pour des carnets analytiques ou d'autres travaux de développement de tiers où vous manipulez les données brutes importées.
4 Clé primaire. Si un champ est une clé primaire, sa valeur est utilisée pour identifier de manière unique l'entrée de données ou la ligne de données.
5

Type de données. Les types de valeurs qu'un champ peut avoir et les opérations qui peuvent être effectuées.

  • Texte - Toute valeur alphanumérique.

  • Date-Date. Notez que si vous mappez un champ contenant la date et l'heure, seule la partie date est importée.

  • Données/heure - Date et heure.

  • Enum - Une liste de valeurs alphanumériques peut être spécifiée.

  • Argent - Les valeurs doivent être numériques et représenter des montants d'argent.
6 Il s'agit d'une obligation. Indique si le champ est obligatoire pour pouvoir importer une transaction de ce type dans Caseware Cloud.
7 Clé primaire alternée. Si le champ désigné comme clé primaire est absent des données que vous importez, la valeur de la clé primaire alternative, ou de l'une des clés primaires alternatives, est utilisée pour créer un code d'identification unique pour l'entrée de données en concaténant leurs valeurs : <PremièreAlternative>-<DeuxièmeAlternative>-<TroisièmeAlternative>

 

Description des ensembles de données de Subledger

Ensemble de données Subledger Description
Comptes fournisseurs – Factures

L'ensemble de données Comptes fournisseurs-factures contient toutes les factures reçues au cours de la période examinée.
Comptes fournisseurs – Paiements effectués

L'ensemble de données Comptes fournisseurs - Paiements effectués contient toutes les transactions de paiement (chèque, virement, espèces, etc.) effectuées au cours de la période.
Comptes fournisseurs – Fournisseurs

Le jeu de données Comptes fournisseurs contient tous les détails relatifs au fournisseur, tels que les coordonnées, l'adresse, la limite de crédit, etc.
Comptes fournisseurs – Soldes non utilisés

Le jeu de données Comptes ouverts-payables contient des détails concernant toutes les transactions ouvertes, non payées ou non résolues à la clôture des activités à une date donnée.
Comptes clients – Factures

L'ensemble de données Comptes débiteurs-factures contient toutes les factures générées au cours de la période examinée.
Comptes clients – Encaissements

L'ensemble de données Comptes débiteurs-espèces reçues contient toutes les transactions de paiement (chèque, virement, espèces, etc.) reçues au cours de la période.
Comptes clients – Clients L'ensemble de données Accounts receivable-customers contient tous les détails relatifs au client, tels que ses coordonnées, son adresse, sa limite de crédit, etc.
Comptes clients – Soldes non utilisés Le jeu de données Comptes clients - soldes ouverts contient des détails concernant toutes les transactions clients ouvertes, non payées ou non résolues à la clôture des activités à une date donnée.
Mouvements de stock Capture toutes les transactions affectant les comptes d'inventaire pendant la période spécifiée (par exemple, les réceptions, les expéditions, les transferts, les retours et les ajustements).
Stock disponible Saisit les quantités d'articles en stock, par emplacement et par montant, à la date spécifiée.
