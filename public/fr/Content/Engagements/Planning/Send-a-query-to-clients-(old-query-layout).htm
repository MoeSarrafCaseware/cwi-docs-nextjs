Envoi d'une requête aux clients (ancienne présentation de la requête)

Lorsque votre mission nécessite des informations supplémentaires de la part du client, vous pouvez demander ces informations en créant une requête à partir d'un fichier de mission. Une demande peut être attribuée à un contact client, qui reçoit alors une notification par courrier électronique l'invitant à y répondre.

Vous pouvez créer des demandes pour différentes choses :

  • Demande d'information - soumettre des questions au client.

  • Demande de documents - demander au client de télécharger un ou plusieurs fichiers.

  • Envoyer les produits à livrer - envoyer les fichiers au client.

Créer un document de requête

Lorsque vos tâches de mission requièrent des informations de la part de vos clients, vous pouvez créer une requête pour demander des informations à votre client.

Pour ajouter une demande de client au dossier d'engagement:

  1. Dans le menu supérieur, sélectionnez Documents.

  2. Sélectionnez l'icône Ajouter un document (), puis sélectionnez Requête.

  3. Sélectionnez un type de requête et saisissez un nom.

    4. Remarque: Les types de requêtes disponibles varient en fonction de votre produit Smart Engagement.

  4. Sélectionnez Créer une requête.

Une requête vierge s'ouvre. Vous commencez à ajouter du contenu si nécessaire. Si vous avez créé votre requête à partir d'un modèle, il se peut que vous n'ayez que des modifications mineures à apporter.

Remarque: Si vous avez créé une requête vierge ou si vous apportez des modifications importantes à votre document, consultez la section Modifier les questions dans la demande du client pour plus d'informations.

Envoyer une requête

Une fois que vous avez examiné la demande du client et apporté toutes les modifications nécessaires, vous pouvez envoyer le document à vos clients.

Pour envoyer une demande de client:

  1. Ouvrir une demande de client.

  2. Sélectionnez SEND.

  3. La boîte de dialogue Ouvrir une requête s'affiche.

  4. Sélectionnez une date d'échéance pour la réponse du client dans le champ Date d'échéance de la requête.

  5. Sélectionnez un contact client dans la liste déroulante Attributaires .

    6. Vous pouvez sélectionner plusieurs contacts ou cocher la case Assigner à tous les contacts pour envoyer la demande à tous les contacts du client.

  6. Ajoutez un message dans la zone de texte Corps de l'e-mail qui sera inclus dans l'e-mail de notification.

  7. Sélectionnez OK pour envoyer la demande du client.

Un courriel de notification est envoyé aux contacts désignés. Vos contacts clients peuvent se connecter à Cloud et consulter la demande du client dans la section Mon travail de la page d'aperçu.

Voir les requêtes

Pour suivre vos progrès et ceux de votre client, vous pouvez vous rendre sur le tableau de bord des requêtes. Le tableau de bord présente une liste des demandes des clients visibles dans le dossier d'engagement. Il se compose des colonnes suivantes :

  • Query -Titre de la demande du client.

  • Statut -statut de la demande du client. Il peut s'agir de l'un des éléments suivants :

    		•  Vous n'avez pas encore envoyé la requête au client.
    Vous avez rouvert la demande pour y apporter des modifications ou demander des informations complémentaires ou des clarifications.
    Le client a répondu à certaines questions dans la demande.
    Vous avez envoyé la demande mais le client n'a pas encore répondu.
    Le client a répondu à toutes les questions posées dans la demande.
    Vous avez annulé la demande du client.

  • Progression du client -suivi de la progression du client

    Il indique le nombre de requêtes répondues par rapport au nombre total de requêtes dans la demande.

  • Date d'échéance - date d'échéance de la réponse du client

Pour consulter les demandes des clients dans votre dossier de mission :

  1. Ouvrez le dossier d'engagement.

  2. Sélectionnez Requêtes dans le menu supérieur.

Une liste des demandes de clients disponibles dans le fichier d'engagement s'affiche sur la page. Sélectionnez une colonne pour trier la liste par statut, avancement, date d'échéance ou par ordre alphabétique.

Remarque : Une demande de client cachée n'apparaît pas dans la liste.
