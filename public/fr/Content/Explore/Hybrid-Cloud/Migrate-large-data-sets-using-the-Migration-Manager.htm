Migration de grands ensembles de données (plus de 50 fichiers)

Le gestionnaire de migration est un outil administratif qui vous aide à migrer simplement et en toute sécurité de grandes quantités de données vers le cloud. Les organisations qui souhaitent utiliser le gestionnaire de migration doivent contacter le support Caseware avec les informations suivantes :

  • Date(s) de migration prévue(s)

  • Taille des données du client actif

  • Taille des données des clients inactifs

  • Nombre de fichiers à migrer (année en cours et année précédente)

  • Nombre d'utilisateurs

En outre, votre organisation doit confirmer qu'elle n'essaiera pas de migrer des fichiers qui ont été associés à une autre organisation.

Prérequis :

  • Votre organisation doit utiliser les Working Papers 2019 ou une version ultérieure.

  • Votre organisation doit disposer de Working Papers dotés de la technologie SmartSync . Si vous ne disposez pas de SmartSync, consultez la section Différences entre SmartSync et non-SmartSync, et contactez votre distributeur local pour plus d'informations.

  • Votre organisation doit avoir un site Caseware Cloud. Si vous n'avez pas de site Cloud, vous pouvez demander une invitation.

  • Vous devez disposer d'un navigateur internet qui prend en charge TLS 1.2 ou une version ultérieure. Pour plus d'informations, voir Désactiver l'accès à Caseware Cloud pour les navigateurs plus anciens et non supportés.

  • Vous devez installer le Gestionnaire de migration sur l'ordinateur ou le serveur contenant les données à migrer.

  • Un administrateur doit faire fonctionner le Gestionnaire de migration au nom de l'organisation (des privilèges administratifs sont requis).

  • Vous devez connecter le gestionnaire de migration à votre site cloud pour accéder à toutes les options de migration. Dans le menu principal, cliquez sur Connexion et entrez l'URL de votre site Cloud et vos informations d'identification d'administrateur. Il n'est pas nécessaire d'inclure le protocole web(http:// ou https://)dans l'URL.

    Remarque : Si le site Cloud de votre organisation est configuré comme faisant partie d'une unité commerciale, vous devez connecter le gestionnaire de migration à l'URL d'une unité commerciale individuelle, plutôt qu'à l'URL du site Cloud de premier niveau (où les membres du personnel se connectent généralement). Au cours du processus de migration, les données migreront vers l'unité commerciale individuelle et ne seront accessibles qu'aux utilisateurs de l'unité commerciale. Vous pouvez ensuite répéter le processus avec les URL d'autres unités commerciales individuelles.

Recommandations :

  • Validez l'état de vos fichiers avant la migration. Pour plus d'informations, voir Valider les fichiers d'engagement .

  • Dans la mesure du possible, veillez à ce que les propriétés d'engagement de vos fichiers contiennent autant d'informations que possible pour faciliter le processus de cartographie automatique. Pour plus d'informations, voir Télécharger des entités dans le nuage .

  • Compresser tous les fichiers parents avant la migration pour réduire les temps de migration. Veillez à inclure tous les sous-dossiers lors de la compression.

  • Sauvegarder tous les fichiers parents avant la migration. Les fichiers originaux sur votre serveur seront migrés vers le Cloud.

  • N'exécutez qu'une seule instance du Gestionnaire de migration à la fois. Les instances multiples n'accélèrent pas le temps de traitement et peuvent avoir un impact négatif sur les performances du nuage.

  • Redémarrer les serveurs SmartSync auto-hébergés après une migration pour effacer la liste des fichiers d'engagement. Si vous migrez vos données par étapes, redémarrez le serveur après chaque étape.

Pour intégrer des fichiers d'engagement provenant d'un serveur SmartSync :

Les cabinets qui cherchent à tirer parti des capacités de gestion de Caseware Sherlock, tout en continuant à utiliser un serveur SmartSync sur site, peuvent intégrer leurs fichiers d'engagement dans le Cloud directement via le gestionnaire de migration. Cette fonction permet aux entreprises d'intégrer des fichiers historiques, d'extraire des données historiques et d'utiliser ces données dans Caseware Sherlock.

Notes :

  • Il n'est pas nécessaire de migrer vos fichiers d'engagement pour ce processus.
  • Pour vous assurer qu'aucune modification n'est apportée aux fichiers historiques originaux, nous vous recommandons de faire des copies de tous les fichiers parents que vous avez l'intention d'intégrer et d'effectuer les processus de validation, de mappage et d'intégration sur les copies, plutôt que sur les fichiers parents originaux. Veiller à ce que toutes les copies de synchronisation des enfants restent fermées pendant toute la durée de l'intégration. Après l'intégration, supprimez les copies du fichier parent avant que le personnel n'ouvre les copies de synchronisation enfant.

Pour intégrer des fichiers d'engagement provenant d'un serveur SmartSync :

  1. Cliquez avec le bouton droit de la souris sur l'exécutable du Gestionnaire de migration et cliquez sur Exécuter en tant qu'administrateur.
  2. Dans le menu principal, cliquez sur Intégrer les fichiers.
  3. Cliquez sur l'icône de document et recherchez votre fichier de mappage (CWCloudMigrationMng.csv). Sélectionnez le fichier et cliquez sur Ouvrir.
  4. Si vous souhaitez intégrer une série de fichiers (par exemple, les fichiers 1 à 50) plutôt que tous les fichiers à la fois, indiquez le premier et le dernier numéro de la série dans les champs correspondants.
  5. Cliquez sur INTEGRER.
  6. Si vous intégrez des fichiers provenant d'une version antérieure de Working Papers, une boîte de dialogue s'affiche pour vous permettre de convertir ou d'ignorer ces fichiers. Les fichiers convertis ne seront plus accessibles dans la version précédente des Working Papers, tandis que les fichiers ignorés ne seront ni convertis ni intégrés dans Cloud.

Les fichiers sont intégrés à Cloud et les résultats de l'intégration s'affichent. Pour obtenir des informations plus détaillées, consultez le fichier journal automatisé en cliquant sur le lien fourni. Cliquez sur FERMER pour revenir au menu principal.

Important : Avant que votre personnel n'ouvre les copies de synchronisation enfant des fichiers, vous devez supprimer les copies du fichier parent.
