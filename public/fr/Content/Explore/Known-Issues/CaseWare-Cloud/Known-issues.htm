Problèmes connus dans Caseware Cloud

Cette rubrique décrit les problèmes connus lors de l'utilisation de Caseware Cloud.

Courriels de notification d'activité pour les utilisateurs SSO

Les liens vers les activités dans les courriels de notification ne fonctionnent pas comme prévu pour les utilisateurs SSO. Si vous n'êtes pas connecté lorsque vous sélectionnez le lien, vous serez redirigé vers un lien brisé une fois connecté et vous ne pourrez pas voir l'activité. Pour accéder à l'activité, vous pouvez sélectionner le lien dans l'e-mail après vous être connecté à Cloud.

Les notifications de demande de fichier ne prennent pas en charge le texte enrichi

L'e-mail de notification de demande de fichier envoyé aux contacts ne prend pas en charge le texte enrichi. Cela signifie que les sauts de ligne et autres mises en forme, telles que le gras ou l'italique, que vous appliquez au texte de la description de l'avis ne s'afficheront pas dans l'e-mail.

En outre, si vous saisissez du texte à la fois dans le champ de description de l'avis et dans le champ Message supplémentaire , le contenu des deux champs s'affichera sous la forme d'un seul paragraphe. Nous vous recommandons de ne remplir qu'un seul de ces champs lorsque vous envoyez une notification de demande de fichier à des contacts.

Saisie manuelle de la date

Lorsque vous saisissez manuellement des dates dans des champs de date, tels que les champs de date de début et de fin dans la boîte de dialogue de création de mission et le champ de date d'incorporation dans la boîte de dialogue de création d'entité, le champ revient automatiquement à la date antérieure à celle qui a été sélectionnée. Notez que ce problème ne se produit que si votre heure locale est réglée sur UTC ou sur un fuseau horaire UTC+. Nous vous recommandons de sélectionner la date à l'aide du sélecteur de date du calendrier pour éviter ce problème.

Les notifications de chat ne s'affichent pas si le navigateur est inactif

Si vous êtes connecté à votre site Cloud, mais que votre navigateur est inactif ou que votre ordinateur est en mode veille, les notifications du chat ne s'afficheront pas et vous risquez de manquer les nouveaux messages des autres utilisateurs. Nous vous recommandons de fermer la page si vous savez que vous serez absent de votre ordinateur ou d'actualiser la page pour mettre à jour le contenu.

Lier des fichiers Google existants à partir d'un lecteur partagé

Lorsque l'on tente de créer un lien vers un fichier Google existant qui est stocké sur un lecteur partagé, Cloud renvoie une erreur indiquant que le "fichier n'existe pas dans Google Drive". Ce problème est dû à une incompatibilité entre les paramètres de sécurité du lecteur partagé et le système de sécurité de Cloud. Pour contourner le problème, vous pouvez déplacer le fichier Google vers un disque non partagé, tel que votre "My Drive", et créer un lien vers le fichier à partir de ce disque.

Intégration de Xero à plusieurs locataires

Si vous accédez à votre compte Xero via un environnement multi-tenant, vous ne pouvez pas l'intégrer à Caseware Cloud. Actuellement, l'intégration de Xero n'est possible que dans un environnement à locataire unique.

erreur "User Name must be unique" lors de la création de nouveaux contacts

Si vous tentez de créer un nouveau contact sans fournir d'adresse électronique, l'erreur "Le nom d'utilisateur doit être unique" s'affiche. Un problème similaire se produit dans les fichiers Time Desktop intégrés dans le cloud : si l'adresse électronique d'un contact existant fait défaut, l'erreur "User Name must be unique" s'affiche lors de la création d'un nouveau contact. Pour contourner temporairement le problème, vous pouvez saisir des adresses électroniques fictives pour éviter l'erreur dans Cloud et Time Desktop.
