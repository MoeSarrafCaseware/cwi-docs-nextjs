Création de requêtes

Lesmembres du personnel peuvent créer des requêtes dans l'engagement et modifier les requêtes qu'ils ont créées.

Lesauteurs d'entreprises peuvent créer des requêtes dans le modèle d'entreprise ou modifier des requêtes qu'ils ont créées.

Notez que :

  • Le personnel ne peut pas supprimer les requêtes fournies avec l'application Cloud ni modifier le contenu existant de ces requêtes. Toutefois, ils peuvent y inclure des contenus supplémentaires.

  • Les auteurs de l'entreprise ne peuvent pas supprimer les requêtes qui accompagnent leur Cloud App. Toutefois, ils peuvent les supprimer dans le modèle d'entreprise afin qu'ils n'apparaissent pas dans la mission.

  • Les auteurs de l'entreprise ne peuvent pas modifier le contenu des requêtes fournies avec leur application Cloud, mais ils peuvent y ajouter du contenu supplémentaire.

Que vous soyez un membre du personnel ou un auteur du cabinet, vous pouvez ajouter une requête vierge en utilisant l'icône Ajouter () sur la page Documents ou créer une requête à partir des modèles disponibles dans votre application Cloud.

Ajouter des indications à une requête

Chaque document relatif à une demande de client comporte une section "Orientations" dans sa partie supérieure. Vous pouvez utiliser ces sections d'orientation pour aider votre contact client, en lui donnant des instructions sur la manière de remplir la demande du client ou en fournissant un contexte pour les questions que vous posez.

Le texte d'orientation est ajouté à votre document.

Ajouter des groupes à une requête de client

Dans les documents de demande de client et de liste de contrôle, tout le contenu du document est contenu dans un ou plusieurs groupes.

Le groupe sera nommé par défaut Nouveau groupe . Vous pouvez modifier le nom en le sélectionnant et en saisissant un nouveau texte.

Ajouter des questions à votre requête de client

Une fois que votre document a un groupe, vous pouvez ajouter une ou plusieurs requêtes. Une requête est une question posée au client ou une tâche à accomplir dans le cadre de la mission. Chaque requête comporte deux parties :

  • Description - la question ou la demande du client.

  • Réponse - le champ où le client peut saisir des informations ou fournir un fichier.

Sélection + requête.

Une requête vide est ajoutée à votre document. La description de cette requête est Nouvelle requête, et la réponse par défaut est Téléchargement de fichiers.

Texte dynamique

Un texte dynamique est un texte conditionnel. Il s'agit d'un champ dans lequel un texte différent s'affiche en fonction d'une condition ou d'un ensemble de conditions.
Valeur des formulaires

Insérer la valeur d'un champ de formulaire spécifique. Lorsque vous sélectionnez un formulaire et un champ, la valeur du champ sélectionné est insérée dans le champ de texte.

Remarque : Cette option n'est disponible dans la boîte de dialogue Liaison de données que si des formulaires personnalisés sont disponibles dans votre produit, tels que les formulaires fiscaux.

Lier une question de requête à une procédure de la liste de contrôle

Vous pouvez lier une question à une procédure connexe dans une liste de contrôle afin de compléter le document de la liste de contrôle.

Par exemple, si une liste de contrôle contient une procédure liée à l'activité de votre client pour laquelle vous n'aurez pas de réponse optimale et qu'il est préférable que vous demandiez ces informations au client, vous pouvez lier la question à la procédure correspondante dans le document de la liste de contrôle. Une fois que vous avez recueilli les informations requises auprès du client, vous pouvez ajouter la réponse dans la liste de contrôle. Pour en savoir plus, voir Voir les réponses des clients concernant les procédures de la liste de contrôle.

Remarque : Si la question liée à la requête est masquée dans le document de requête, ses réponses ne s'affichent pas dans le document de la liste de contrôle.

Vous pouvez voir la réponse du client dans la liste de contrôle correspondante après que le personnel l'a acceptée. Si le client télécharge un fichier dans le cadre de la réponse, le fichier s'affiche sous forme de lien afin que vous puissiez y accéder directement à partir de la liste de contrôle.

Ajouter des identifiants d'auteur

Les utilisateurs de Working Papers (WP) peuvent automatiquement extraire les fichiers téléchargés dans les documents de requête et les enregistrer dans des dossiers désignés dans le fichier WP.

Pour prendre en charge cette intégration, vous devez ajouter des identifiants internes dans votre document de requête - connus sous le nom d'ID d'auteur.

Remarque : Vous ne pouvez ajouter un identifiant d'auteur que si vous personnalisez le modèle d'entreprise. Pour accéder au modèle d'entreprise, vous devez avoir le rôle d'administrateur des paramètres ou un rôle similaire. Pour plus d'informations, voir Gérer le modèle d'entreprise .

Copier les identifiants d'auteur

Vous pouvez copier les identifiants des auteurs des questions prédéfinies. Les questions prédéfinies sont des questions qui ont été rédigées dans le modèle de produit. Les auteurs de Working Papers (WP) peuvent utiliser l'identifiant de l'auteur pour configurer un emplacement dans WP afin de recevoir des fichiers à partir d'un document de requête.

Remarque : Vous ne pouvez copier un identifiant d'auteur que si vous personnalisez le modèle d'entreprise. Pour accéder au modèle d'entreprise, vous devez avoir le rôle d' administrateur des paramètres ou un rôle similaire. Pour plus d'informations, voir Gérer le modèle d'entreprise .
