Pour commencer

Avant de commencer à travailler sur des missions EBP OnPoint, vous devez configurer le nouveau portail Caseware Cloud pour votre entreprise. Pour les besoins de cette formation, vous devez suivre trois étapes :

  • créer des espaces de travail dans Caseware Cloud pour chacun de vos clients
  • créer des comptes pour les membres de votre personnel
  • créer des comptes pour vos clients

Cette section vous guide tout au long des étapes ci-dessus. Une fois que vous les avez complétés, vous êtes prêt à créer un engagement EBP OnPoint.

Créer une nouvelle entité interne

Une entité est un espace de travail que vous pouvez utiliser pour organiser et collaborer sur vos dossiers de mission à partir d'un emplacement unique pour un client. Lorsque votre entreprise commence à utiliser le cloud, vous devez créer une nouvelle entité pour chaque client.

Remarque: Les instructions ci-dessous expliquent la procédure d'ajout d'une entité client individuelle. Vous pouvez également importer une liste de vos clients afin de créer toutes vos entités clients en une seule fois. Pour plus d'informations, consultez l'aide en ligne de Caseware.

Créez des comptes pour les membres de votre personnel

Une fois que vous avez créé des entités pour chacun de vos clients, accordez l'accès au nuage à d'autres membres du personnel de votre entreprise. Vous pouvez créer un nouveau compte pour chaque membre du personnel qui doit collaborer à vos missions.

Remarque: Les instructions ci-dessous décrivent comment ajouter des membres individuels du personnel. Vous pouvez également importer une liste des membres de votre personnel afin de créer tous leurs profils d'utilisateur en une seule fois. Pour plus d'informations, consultez l'aide en ligne de Caseware.

Kit d'entraînement

Dans le menu Nuage, sélectionnez Personnel. Vous accédez ainsi à la fonctionnalité AnalyticsAI.

Select New, and then ADD | Staff. La boîte de dialogue Importation de personnel s'affiche. Comme pour la création d'une entité, il existe différents groupes de propriétés dans lesquels vous pouvez saisir diverses informations sur le compte.

Général

Saisissez une adresse électronique ainsi qu'un nom et un prénom pour le nouveau membre du personnel. L'adresse électronique sera le nom d'utilisateur de leur compte et ils se connecteront à Cloud avec cette adresse.

Les autres champs sont facultatifs.

Mot de passe

Il existe trois options de réglage du mot de passe initial :

  • Défini par l'utilisateur - le membre du personnel définira son propre mot de passe lors de sa première connexion. Le nuage envoie un courriel contenant des instructions sur la manière de se connecter au portail.
  • Généré automatiquement - un mot de passe aléatoire est choisi par le nuage. Cloud n'envoie pas d'e-mail au membre du personnel, vous devrez donc lui fournir l'URL de la page de connexion, son identifiant et son mot de passe.
  • Définir - vous choisissez un mot de passe pour le membre du personnel. Là encore, Cloud n'envoie pas d'e-mail, vous devrez donc leur fournir leurs informations de connexion.

Dans cette formation, nous utiliserons l'option par défaut : Défini par l'utilisateur.

Groupes

Vous pouvez affecter le compte du membre du personnel à un ou plusieurs groupes de personnel. Si vous créez des groupes dans Cloud, vous pouvez spécifier des autorisations et des niveaux d'accès pour les membres de ce groupe. Si vous ajoutez d'autres comptes en tant que membres du groupe, ils héritent automatiquement des autorisations et des niveaux d'accès du groupe.

Le groupe " All Staff" est disponible d'emblée dans Caseware Cloud. Tous les membres du personnel sont automatiquement ajoutés à ce groupe par défaut. Dans cette formation, nous utiliserons le groupe All Staff.

Rôles à l'échelle du système

Lorsque vous attribuez un ou plusieurs rôles systémiques à un membre du personnel, celui-ci obtient un ensemble de droits d'accès qui s'appliquent à l'ensemble du contenu de votre entreprise en nuage. Par exemple, si vous accordez à un membre du personnel le rôle de visualiseur à l'échelle du système, il peut voir toutes les entités de votre cabinet et peut consulter tous les dossiers et missions de ces entités. Les rôles à l'échelle du système constituent le moyen le plus simple de définir les droits d'accès des membres du personnel de votre entreprise.

Attribuer à ce membre du personnel le rôle d' éditeur pour l'ensemble du système. Ce rôle leur permet de visualiser l'ensemble du contenu de l'entreprise et d'apporter des modifications tout en travaillant sur ce contenu.

Applications

Vous pouvez accorder à votre collaborateur l'accès aux applications en nuage. Grâce à cet accès, ils peuvent consulter les engagements d'une application et créer de nouveaux engagements ou dossiers pour celle-ci.

Notez que par défaut, le groupe All Staff permet à ses membres de visualiser et de créer des engagements pour toutes les applications Cloud disponibles. Si vous devez restreindre l'accès à certaines applications, supprimez certaines autorisations du groupe global. Cela vous permet de choisir quel membre du personnel doit travailler avec quelle application.

Heures normales

Ce paramètre s'applique si votre entreprise utilise l'application Caseware Cloud Time pour gérer les entrées de temps et de dépenses. Laissez cette section réglée sur les heures standard de l'entreprise.

Une fois que vous avez rempli toutes les informations relatives au nouveau compte de votre collaborateur, sélectionnez Enregistrer. Ils recevront un courriel de bienvenue de Cloud les informant que leur compte a été créé et leur indiquant comment se connecter.

Créer un compte pour votre client

L'un des principaux avantages de OnPoint EBP est que vous pouvez collaborer directement avec vos clients sur les missions. Vous devez créer un compte de contact pour que vos clients puissent accéder à leur dossier d'engagement.

Kit d'entraînement

Dans le menu Cloud, sélectionnez Groupes. Vous accédez ainsi à la fonctionnalité AnalyticsAI.

Sélectionnez Nouveau, puis AJOUTER | Contact. La boîte de dialogue File Upload s'affiche.

Général

Saisissez un prénom, un nom et une adresse électronique pour votre contact client. Dans le champ Entité , sélectionnez l'entité client que vous avez créée précédemment. Après avoir sélectionné l'entité cliente, sélectionnez le rôle Contact - Collaborateur de l'entité pour définir le niveau d'accès à ce compte. Ce rôle permet à votre client de créer des discussions, des tâches et des demandes de fichiers sur le cloud. Il leur permet également de collaborer à des missions pour OnPoint PCR.

Mot de passe

Les mêmes options de mot de passe que pour les comptes du personnel s'appliquent ici. Dans cette formation, nous utiliserons l'option par défaut : Défini par l'utilisateur.

Groupes

Les groupes de contact fonctionnent de la même manière que les groupes de personnel. Le groupe de contacts disponible d'emblée est Tous les contacts et, par défaut, votre client est automatiquement membre de ce groupe.

Applications

Tout comme le groupe " Tout le personnel" , le groupe " Tous les contacts" permet à ses membres d'afficher et de créer des engagements pour toutes les applications Cloud disponibles. Vérifiez les autorisations et assurez-vous que votre contact peut voir les engagements EBP OnPoint.

Une fois que vous avez terminé de configurer le nouveau compte de votre contact client, sélectionnez Enregistrer. Ils recevront un courriel de bienvenue de Cloud les informant que leur compte a été créé et leur indiquant comment se connecter.
