Pour commencer

Avant de commencer à travailler sur des demandes PBC, vous devez configurer le portail Caseware Cloud pour votre organisation. Pour plus d'informations, voir Démarrer avec Caseware Cloud Connector.

Une fois que vous avez configuré votre portail, vous pouvez poursuivre les exercices de ce guide.

Ce chapitre vous guide dans la réalisation de ces étapes de configuration. Une fois les étapes suivantes terminées, vous serez prêt à créer un engagement :

  • Configurer des espaces de travail dans Caseware Cloud pour chacun de vos clients
  • Créez des comptes pour les membres de votre personnel
  • Créer des comptes pour vos clients

Créer une nouvelle entité interne

Une entité est un espace de travail que vous pouvez utiliser pour organiser et collaborer à votre mission à partir d'un seul endroit pour un client. Vous devrez créer une nouvelle entité pour chaque client.

Remarque: Les instructions ci-dessous expliquent la procédure d'ajout d'une entité client individuelle. Vous pouvez également importer une liste de vos clients afin de créer toutes vos entités clients en une seule fois. Pour plus d'informations, voir Créer des entités client.

Kit d'entraînement

Dans le menu Cloud, sélectionnez Entités. Vous accédez ainsi à la fonctionnalité AnalyticsAI.

Sélectionnez Nouveau, puis choisissez Télécharger. La boîte de dialogue Create Entity (Créer une entité).

Pour chacune des sections ci-dessous, saisissez les informations indiquées.

Créez des comptes pour les membres de votre personnel

Une fois que vous aurez créé des entités pour chacun de vos clients, vous serez prêt à accorder l'accès à votre entreprise en nuage à d'autres membres du personnel de votre organisation. Vous pouvez créer un nouveau compte pour chaque membre du personnel qui devra collaborer à vos missions.

Kit d'entraînement

Remarque: Les instructions ci-dessous expliquent la procédure à suivre pour ajouter des membres du personnel. Vous pouvez également importer une liste des membres de votre personnel afin de créer tous leurs profils d'utilisateur en une seule fois. Pour plus d'informations, consultez l'aide en ligne de Caseware.

Dans le menu Nuage, sélectionnez Personnel. Vous accédez ainsi à la fonctionnalité AnalyticsAI.

Select New, and then under Add, select Staff. La boîte de dialogue Importation de personnel s'affiche. Tout comme lors de la création d'une entité, il existe différentes zones dans lesquelles vous pouvez saisir des informations sur le nouveau compte.

Général

Saisissez une adresse électronique ainsi qu'un nom et un prénom pour le nouveau membre du personnel. L'adresse électronique sera le nom d'utilisateur de leur compte et ils se connecteront à Cloud avec cette adresse.

Les champs Prénom, Nom et Email (ou Nom d'utilisateur) sont obligatoires.

Mot de passe

Vous pouvez choisir la manière dont le mot de passe du nouveau compte sera défini :

  • Défini par l'utilisateur - le membre du personnel définira son propre mot de passe lors de sa première connexion. Caseware PBC Requests envoie un e-mail contenant des instructions sur la manière de se connecter au portail.
  • Généré automatiquement - un mot de passe aléatoire est choisi par Caseware PBC Requests. Il n'envoie pas d'e-mail au membre du personnel, vous devrez donc lui fournir l'URL de la page de connexion, son nom d'utilisateur et son mot de passe.
  • Définir - vous choisissez un mot de passe pour le membre du personnel. Caseware PBC Requests n'envoie pas d'e-mail, vous devrez donc lui fournir ses informations de connexion.

Laissez l'option de mot de passe sur Défini par l'utilisateur.

Groupes

Vous pouvez affecter le compte du membre du personnel à un ou plusieurs groupes de personnel. Si vous créez des groupes, vous pouvez spécifier des autorisations et des niveaux d'accès pour les membres de ce groupe, et tous les comptes que vous ajoutez en tant que membres du groupe héritent automatiquement de ces paramètres.

Caseware PBC Requests n'inclut par défaut que le groupe All Staff , et tous les membres du personnel sont automatiquement inclus dans le groupe, vous n'aurez donc rien à faire dans cette section.

Laisser cette page telle quelle.

Applications

Vous pouvez accorder l'accès aux applications à votre collaborateur. Vous pouvez accorder au membre du personnel la possibilité de voir les engagements pour une application et la possibilité de créer de nouveaux engagements ou dossiers pour cette application.

Notez que par défaut, le groupe " Tout le personnel" permet de visualiser et de créer des engagements pour toutes les applications. Cela simplifie l'accès, mais si vous avez besoin de contrôler plus étroitement l'accès à certaines applications, vous pouvez supprimer certaines de ces autorisations du groupe global. Cela vous permet de choisir les membres du personnel qui doivent pouvoir travailler avec certaines applications.

Heures normales

Les paramètres pour les heures standard du personnel ne sont applicables à votre organisation que si vous utilisez l'application CasewareCloud Time pour gérer les entrées de temps et de dépenses. Laissez cette section réglée sur les heures standard de l'organisation.

Une fois que vous avez terminé de saisir toutes les informations relatives au nouveau compte de votre collaborateur, sélectionnez Enregistrer. Ils recevront un courriel de bienvenue les informant que leur compte a été créé et leur indiquant comment se connecter.

Créer des entités clientes

L'un des principaux avantages des demandes PBC est que vous pouvez collaborer directement avec vos clients sur les missions. Pour que vos clients puissent accéder à leur engagement, vous devez leur créer un compte de contact sur Cloud.

Kit d'entraînement

Dans le menu Cloud (), sélectionnez Audit. Vous accédez ainsi à la fonctionnalité AnalyticsAI.

Sélectionnez Nouveau, puis sous Ajouter, sélectionnez Contact. La boîte de dialogue File Upload s'affiche.

Général

Saisissez un prénom, un nom et une adresse électronique pour votre contact client.

Remarque: Pour éviter tout problème avec Google, veillez à utiliser une adresse électronique valide.

Dans le champ Entité , sélectionnez l'entité client que vous avez créée précédemment. Après avoir sélectionné l'entité cliente, sélectionnez le rôle Contact - Collaborateur de l'entité pour définir le niveau d'accès à ce compte. Le rôle de collaborateur de l'entité permet à votre client de créer des discussions, des tâches et des demandes de dossiers, et de collaborer à des missions.

Mot de passe

Les mêmes options de mot de passe que celles dont vous disposiez lors de la création des comptes de votre personnel sont également disponibles ici. Laissez l'option de mot de passe sur Défini par l'utilisateur.

Groupes

Les groupes de contacts fonctionnent de la même manière que les groupes de personnel. Par défaut, le seul groupe de contacts est " Tous les contacts" - votre client fera automatiquement partie de ce groupe.

Laisser cette page telle quelle.

Applications

Le groupe Tous les contacts accorde automatiquement aux contacts du client la possibilité de visualiser les engagements ou les fichiers pour tous les types d'applications dans le nuage. Vérifiez les autorisations et assurez-vous que votre contact peut voir les demandes d'engagement de Caseware PBC.

Une fois que vous avez terminé de saisir toutes les informations relatives au compte de votre contact client, sélectionnez Enregistrer. Ils recevront un courriel de bienvenue les informant que leur compte a été créé et leur indiquant comment se connecter.
