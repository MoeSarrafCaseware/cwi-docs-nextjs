Notes de mise à jour SE 2019-02

Toutes les fonctionnalités ci-dessous sont automatiquement activées si le composant ou le module correspondant a été activé.

Création

  • Nous avons introduit le bouton Utiliser la valeur par défaut dans la section de gestion du modèle de l'auteur de l'entreprise, qui réinitialise le modèle de l'entreprise au modèle de produit par défaut.

  • Les auteurs peuvent désormais définir des espaces réservés pour le texte à saisir par les utilisateurs finaux dans les lettres, les mémos et les états financiers.

  • Pour clarifier les options de majuscules disponibles pour les termes du glossaire d'engagement, nous avons ajouté des exemples de majuscules.

Listes de contrôle

Suite aux commentaires des utilisateurs, les groupes et les sous-groupes ne se réduisent plus automatiquement une fois que toutes les procédures d'un groupe ou d'un sous-groupe ont été signées.

Consolidation

Pour éviter toute confusion lors de la création d'un engagement de consolidation, nous avons ajouté une colonne indiquant le nom du fichier dans la recherche de la hiérarchie de consolidation. Vous pouvez maintenant facilement visualiser et sélectionner le fichier de consolidation.

Tableaux dynamiques

  • Les décimales sont désormais prises en charge dans lesétatsfinanciers*.

  • Vous pouvez désormais ajouter des lignes de saisie directement à partir du tableau sans devoir passer en modeédition*.

  • Vous voyez maintenant tous les groupes visibles (qui ont des comptes) étendus si le paramètre Afficher les comptes disponibles est sélectionné par l'auteur principal.

Nouveau design pour les finances

  • Les termes Hidden et Hide , qui représentent des contenus non visibles, ont été révisés. Il est désormais remplacé par Excluded et Exclude, un terme plus adapté à la façon dont les utilisateurs travaillent avec la visibilité du contenu dans les données financières.

  • Dans la bibliothèque de notes, le nom court du tableau s'affiche désormais pour les notes qui incluent des tableaux dynamiques. Notez que si le tableau n'a pas de nom court, le terme Tableau dynamique s'affiche.

  • Une nouvelle icône permettant d'ajouter des notes de niveau inférieur est désormais disponible dans les zones de notes.

  • Les auteurs du cabinet ne peuvent plus modifier le contenu des états financiers qui a été ajouté par l'auteur principal.

  • Pour faciliter la transition vers le nouveau modèle financier, nous avons amélioré la façon dont vous pouvez déplacer les sections dans l'ancien modèle financier. Vous pouvez maintenant déplacer une section à la fin d'une zone de regroupement. Par exemple, si vous avez une section de texte dans la zone de regroupement A, vous pouvez désormais la déplacer à la fin de la zone de regroupement B (auparavant, vous ne pouviez la déplacer que vers le haut de la zone de regroupement B).

Balance

Une fois que l'utilisateur a importé la balance générale, il est désormais possible de faire correspondre automatiquement les comptes de n'importe quelle structure de regroupement (par exemple, le plan comptable) au regroupement financier2.

Corrections

  • Correction d'un problème de rédaction où le menu des paramètres du rapport d'évaluation des risques ne s'affichait pas correctement.

  • Correction d'un problème qui empêchait d'appuyer sur Ctrl+Z (annuler) dans les zones de guidage des documents de la liste de contrôle.

  • Correction de plusieurs problèmes de traduction incorrecte.

  • Mise en œuvre de divers correctifs pour les tables dynamiques liées à :

    • affichage incorrect du titre des colonnes de calcul personnalisées.

    • un comportement erroné de l'exportation XBRL.

  • Suppression du champ Commentaire dans les documents de requête pour les auteurs. Les commentaires d'interrogation sont obsolètes pour les auteurs de modèles, car ils sont destinés à aider les comptables ou les auditeurs à communiquer avec les clients tout au long du processus de collecte d'informations.

  • Correction d'un problème où les ajustements de l'année précédente n'apparaissaient pas dans le document PDF de la copie de référence de la balance de vérification.

  • Correction d'un problème de rédaction où l'insertion de plusieurs textes dynamiques ajoutait automatiquement une nouvelle ligne après chaque sortie de texte dynamique.

  • Correction d'un problème où certains PDF téléchargés avaient des pages blanches.

  • Correction d'un problème lié au report, où il n'était pas possible de cacher des fichiers sur la page Documents à moins qu'ils n'appartiennent à un dossier.

  • Correction d'un problème où les conditions de matérialité dans les paramètres de visibilité n'étaient pas évaluées correctement.

1 Configuration requise par les auteurs.

2 Les distributeurs doivent fournir un script pour le mappage des comptes.
