Notes de mise à jour SE 2019-07

Toutes les fonctionnalités ci-dessous sont automatiquement activées si le composant ou le module correspondant a été activé.

Refonte des documents d'interrogation

La collaboration entre le personnel et les contacts est passée à une nouvelle interface utilisateur améliorée. La nouvelle conception des documents d'interrogation est maintenant disponible.

La nouvelle interface met l'accent sur l'amélioration du style et de l'attrait visuel, l'amélioration de la convivialité, l'optimisation de la présentation et de l'organisation des composants, ainsi que l'utilisation de termes clairs et simples.

Voici un aperçu des principales caractéristiques.

  • Changements de durée.

    • Requête - anciennement utilisée pour faire référence au document et aux questions qu'il contient. Dans la nouvelle interface, le terme Requête fait uniquement référence au document. Tandis que la question de l'interrogation fait désormais référence aux questions/requêtes/interrogations contenues dans le document.

    • Ensemble de questions - a remplacé le terme Groupe.

  • Cartes de questions contenant le titre et le corps de la question, la réponse et les commentaires.

    • Les cartes de questions permettent de repérer et d'identifier facilement les éléments appartenant à une même question.

  • Un volet de navigation latéral, dans lequel vous pouvez visualiser et faire défiler toutes les questions disponibles, regroupées par ensemble de questions.

    • Le volet de navigation comporte également une action unique de sélection et de déplacement. Vous pouvez désormais facilement réorganiser les ensembles de questions et les questions d'interrogation.

  • Une méthode plus simple pour ajouter des questions d'interrogation.

    • Vous pouvez maintenant sélectionner le type de réponse souhaité lorsque vous ajoutez la question dans le document. Il n'est plus nécessaire d'accéder à la boîte de dialogue des paramètres pour ajouter ou modifier un type de réponse.

Pour activer la nouvelle conception des requêtes dans votre produit, allez dans Paramètres du produit | Requêtes et sélectionnez Activer la nouvelle présentation des requêtes.

Remarque : Nous vous conseillons vivement de tester minutieusement les requêtes existantes dans votre produit après l'activation de la nouvelle conception et de vérifier que tout fonctionne comme prévu, avant d'activer la nouvelle conception dans l'environnement de production.

Terminologie personnalisée

You can now customize terms in your product. La boîte de dialogue Paramètres du produit comprend désormais une nouvelle section Terminologie, dans laquelle vous pouvez personnaliser les termes.

Dans cette version, vous pouvez personnaliser les termes préliminaires et finaux utilisés par défaut dans la page Balance de vérification ou dans la page Données (selon ce qui est activé dans votre produit) et dans l'onglet Analyse des listes de contrôle.

Régularisations

Les utilisateurs peuvent désormais exporter les ajustements vers un fichier CSV, un fichier PDF ou des documents de travail (format CSV ASCII).

États financiers

Dans cette version, nous avons ajouté un indicateur de placeholder dans le document des états financiers et de nouvelles fonctionnalités pour soutenir la réconciliation. Nous avons également amélioré les relations d'arrondi et apporté quelques améliorations aux tableaux dynamiques, aux propriétés des groupes de balances de vérification et aux notes.

Dimensions

Pour aider les marchés qui doivent inclure l'état des variations des capitaux propres dans leurs états financiers, vous pouvez désormais affecter des groupes de balances de vérification à des dimensions spécifiques au sein d'une catégorie.

Dans la boîte de dialogue Propriétés du groupe, un bouton est désormais disponible à côté de la propriété Ventilation par dimension , qui permet d'exclure certaines dimensions de la catégorie sélectionnée.

Tableaux dynamiques

Vous pouvez maintenant le faire :

  • Affichez la ligne du total du groupe une fois que vous avez ajouté un groupe personnalisé dans le mode d'édition du tableau. Auparavant, la ligne du total du groupe ne s'affichait pas dans le mode d'édition du tableau, ce qui amenait les utilisateurs à ajouter accidentellement les lignes du groupe en dehors du groupe personnalisé.

  • Inclure des liens de données dans les titres des colonnes et des lignes. L'icône de formule () est maintenant disponible dans le mode d'édition du tableau.

Notes

Vous pouvez maintenant modifier le titre par défaut de la section(Notes aux états financiers).

Texte et caractères de remplacement des champs

Vous pouvez désormais facilement localiser et remplacer les espaces réservés dans les états financiers. Comme pour les lettres et les mémos, un indicateur d'espace réservé est désormais disponible dans le document. Il indique le nombre d'espaces vides et dirige les utilisateurs vers l'espace vide suivant dans le document.

Outils de révision

Nous avons optimisé la présentation des outils de révision dans les états financiers afin d'en améliorer la convivialité.

L'icône du filtre n'est plus disponible. Lorsque l'utilisateur sélectionne l'icône Outils de révision (), les cases à cocher des outils s'affichent afin qu'il puisse filtrer la vue. Ils peuvent également activer ou désactiver les outils de révision à l'aide du bouton de basculement disponible.

Arrondissement

Vous pouvez maintenant sélectionner le groupe financier auquel vous souhaitez appliquer les différences d'arrondi. Une fois la sélection effectuée, toutes les différences d'arrondi sont ajoutées au total du groupe. Dans la fenêtre contextuelle des détails du compte, une ligne supplémentaire s'affiche pour indiquer les détails de l'arrondi.

En cas de report, le groupe actuellement sélectionné au cours des années précédentes est conservé. En revanche, le groupe actuellement sélectionné dans l'année précédente la plus récente est reporté à l'année en cours.

Exemple :

si vous reportez des données pour 2018, 2017 et 2016, où,

L'argent liquide a été choisi pour conserver les différences d'arrondi en 2018

La rubrique "Passifs" a été sélectionnée pour tenir compte des écarts d'arrondi en 2016 et 2017

ensuite,

L'argent liquide devient le groupe sélectionné pour les différences d'arrondi en 2019

L'argent liquide reste le groupe sélectionné pour les différences d'arrondi en 2018

Les passifs restent le groupe sélectionné pour les différences d'arrondi en 2016, 2017

Rapprochement

Nous avons ajouté de nouvelles fonctionnalités pour faciliter le rapprochement et aider les utilisateurs à repérer et à identifier facilement les écarts de solde afin de produire des états financiers fiables, précis et de grande qualité.

  • Dans le mode d'édition du tableau dynamique, vous pouvez maintenant étiqueter les comptes dont vous voulez que l'application compare les soldes.

    • Par exemple, vous pouvez étiqueter deux enregistrements dans deux tables différentes du document avec l'étiquette "Trésorerie" pour que leurs soldes soient comparés. Notez que la comparaison se fait en temps réel.

  • Les utilisateurs peuvent désormais facilement repérer les enregistrements qui ne concordent pas entre eux.

    • Une icône s'affiche à côté de chaque solde de compte présentant des divergences identifiées. En cliquant sur l'icône, une fenêtre contextuelle s'ouvre et affiche les détails des divergences (noms des comptes et soldes).

Notez que si les utilisateurs tentent de signer ou d'imprimer les états financiers sans réconcilier les divergences d'enregistrement identifiées, ils reçoivent un message d'avertissement.

Cette fonctionnalité s'accompagne d'un nouvel outil d'évaluation, Balance Check. Il marque toutes les lignes où des écarts de solde ont été identifiés, ainsi que toutes les lignes où les soldes concordent.(Les fonctionnalités d'avertissement lors de la signature/impression et de l'outil de vérification du solde ne sont plus incluses dans la version 2019-07, et seront disponibles dans les versions ultérieures)

Remarque : Cette fonction n'est disponible que pour les tableaux qui suivent le format standard à trois colonnes (Description, CY, PY). Elle n'est pas disponible dans les tableaux IFRS/SOCIE, les tableaux affichant une ventilation des dimensions ou des entités, les tableaux affichant les colonnes "mère" et "consolidé", ou les tableaux qui affichent la période écoulée et la période en cours en tant que colonnes distinctes.

Ecraser un solde calculé

Les utilisateurs peuvent désormais remplacer les soldes des comptes provenant de la balance générale dans les tableaux dynamiques. Les modifications sont automatiquement répercutées dans tous les tableaux du document, ainsi que dans les calculs de variance (le cas échéant).

Pour les soldes annulés, la fenêtre contextuelle des détails du compte affiche le solde annulé et comprend une ligne supplémentaire indiquant le solde d'origine.

Note :Cette fonction n'est pas disponible dans les tableaux IFRS/SOCIE ou dans les tableaux affichant une ventilation des dimensions ou des entités

Lettres, mémos et états financiers

Users are now prompted if they attempt to print or bulk print letters, memos, or financial statements document with empty placeholders.

L'invite informe les utilisateurs du nombre de caractères de remplacement qu'ils ont manqués.

La page Données

Nous avons amélioré le style et revu les termes de la page Données afin d'améliorer l'expérience de l'utilisateur.

Nous avons également repris des éléments de l'ancienne version (la page Balance de vérification), ainsi que de nouvelles fonctionnalités :

  • La prise en charge des structures de regroupement des comptes fiscaux et des comptes prescrits (plan comptable) est désormais disponible, avec une vue désignée pour chaque regroupement.

  • Un chemin de navigation s'affiche désormais en haut lorsque les utilisateurs naviguent dans les groupes, les comptes et les transactions, afin qu'ils puissent facilement aller et venir.

  • L'affichage par défaut de la page est désormais basé sur la progression de l'utilisateur dans l'engagement.

    • Si l'utilisateur n'a pas importé les données financières, la page Données s'ouvre par défaut sur la vue Importation .

    • Si l'utilisateur a importé les données financières et attribué des comptes (le cas échéant), la page Données s'ouvre par défaut sur la vue Comptes . Les utilisateurs sont également redirigés vers la vue Comptes lorsque l'importation des données est réussie.

    • Si l'utilisateur a travaillé dans la vue Groupes avant de quitter la page Données, celle-ci s'ouvre par défaut dans la vue Groupes .

  • Les utilisateurs peuvent désormais :

    • Exportation des données de la balance des comptes vers des documents de travail (format CSV ASCII) ou vers un PDF.

    • Importer et consulter les données de l'année précédente sur une période pouvant aller jusqu'à 5 ans.

  • Le lien vers les transactions sur les comptes n'est désormais actif que si les utilisateurs ont importé avec succès les données relatives aux transactions.

  • Dans les missions de consolidation, une vue séparée des données de l'entité mère est désormais disponible. Pour chaque groupe, les utilisateurs peuvent désormais sélectionner Consolidé pour visualiser les données consolidées ou Entité mère pour visualiser uniquement les données de l'entité mère.

  • Vous pouvez désormais importer les propriétés de groupe suivantes à l'aide d'un fichier CSV.

    • Nom du contraire (Texte)

    • Nom incohérent (Texte)

    • Catégorie de dimension (une des catégories de dimension définies dans les paramètres du produit)

    • Réaffecter le groupe pour les comptes contraires (l'un des groupes de la structure des groupes financiers)

    • Classification du groupe (un des mots-clés de l'automap)

    • Préférence (nombre)

    • Restriction (une des valeurs de la propriété Account Restriction)

    • Lien avec un groupe de taxe (numéro)

    • Lien vers un groupe de plans comptables (nombre)

    • Cycle (Texte)

    • Zone (Texte)

  • Nous avons amélioré la fonctionnalité de recherche pour l'imputation afin d'améliorer les performances. Les résultats correspondants s'affichent désormais après que les utilisateurs ont saisi au moins 3 caractères dans le champ de recherche.

La page Documents

Vous pouvez désormais définir la vue détaillée comme vue par défaut dans la page Documents à l'aide de l'API du produit. Pour en savoir plus, voir la propriété detailedDocumentsView dans la documentation SDK.
