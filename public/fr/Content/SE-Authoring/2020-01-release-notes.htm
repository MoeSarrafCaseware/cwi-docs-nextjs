Notes de mise à jour SE 2020-01

N'abandonnez pas tout de suite vos compétences en Microsoft Excel ! Cloud Engagements lance Cloud Connector - un complément MS Excel. Avec des graphiques et des tableaux dans les états financiers, l'échantillonnage des transactions et plus encore, SE 2020-01 marque le début de la nouvelle année avec des ajouts passionnants à la plateforme.

Échantillonnage

#primaryauthor #enduser

La page Données comporte désormais un nouvel onglet : Transactions, uniquement dédié à l'affichage des transactions des comptes, ainsi qu'à la sélection d'échantillons et à la génération de rapports pour les tests de vérification.

In the Transactions tab, users can:

  • analyser les données en les triant et en les filtrant pour créer une population d'échantillonnage.

  • générer un échantillon de données en utilisant les méthodes disponibles (manuelle, aléatoire, unité monétaire ou systématique).

  • exporter l'échantillon vers un fichier CSV ou le copier dans le presse-papiers pour une analyse plus approfondie et la documentation des résultats du test. Les utilisateurs peuvent également télécharger le fichier d'exemple dans la page Documents et y faire référence à partir d'une procédure de programme de travail.

Vous pouvez activer l'échantillonnage dans votre produit en cochant la case Activer l'échantillonnage dans les paramètres du produit.

Analysis in Financial Statements

#Premier auteur #premier auteur #enduser

Une nouvelle section Analyse est désormais disponible dans les états financiers. Vous pouvez personnaliser les types de graphiques et de diagrammes, ainsi que les données qu'ils présentent dans l'API du produit. Pour en savoir plus, voir Ajouter une procédure.

Comme l'onglet Analyse des listes de contrôle, la section Analyse des états financiers affiche les graphiques ou diagrammes sélectionnés pour un groupe spécifique ou l'analyse financière globale. Les utilisateurs peuvent redimensionner les graphiques et les tableaux afin de les afficher en taille réelle dans toute la section pour une meilleure présentation. Pour en savoir plus, voir les graphiques d'analyse dans les états financiers.

Constructeur de calculs dans les cellules des tableaux dynamiques

#Premier auteur #premier auteur #enduser

Une icône de formule () est désormais disponible dans les cellules des tableaux dynamiques des états financiers afin de prendre en charge les scénarios qui nécessitent des valeurs calculées dans les cellules, telles que : les lignes de mouvement dans l'état des flux de trésorerie ou les lignes d'analyse des ratios dans la section "Bénéfice par action" du compte de résultat.

Remarque : Cette option n'est disponible que pour les types de colonnes et de lignes d'entrée.

Le calcul peut inclure des chiffres, des données provenant de la balance de vérification, des données provenant des états financiers ou une combinaison de tous ces éléments.

Remarque : L'option de valeur Formulaires n'est disponible que si vous avez développé des formulaires à l'aide de SE Builder dans votre produit.

Dans le document, vous pouvez sélectionner la cellule dans laquelle vous avez ajouté un calcul pour en afficher les détails.

Si les utilisateurs remplacent un calcul de cellule prédéfini (un calcul qui a été ajouté dans le modèle de produit par les auteurs), une option Revenir s'affiche dans le générateur de formules - l'onglet Calcul - afin qu'ils puissent toujours revenir au calcul prédéfini, si nécessaire.

Si les utilisateurs remplacent la valeur d'une cellule, la fenêtre contextuelle Détails du compte affiche la valeur calculée et la formule d'origine, ainsi qu'une option Revenir.

Logique améliorée pour l'affichage/masquage dans les lignes de tableau

#Premier auteur #premier auteur #enduser

L'option Afficher/Masquer pour les lignes des tableaux dynamiques comprend désormais des options supplémentaires. La sélection de l'icône ouvre une liste déroulante qui présente deux ou trois options, en fonction du type de ligne.

  • Appliquer le comportement par défaut - le comportement par défaut est appliqué à la colonne. Notez que le comportement par défaut diffère en fonction du type de colonne.

    • Pour les lignes d'entrée :

      • Cette ligne doit toujours figurer dans les états financiers.

      Afficher dans la copie d'impression du document si la ligne a un solde (la cellule n'est pas vide et sa valeur n'est pas nulle).

    • Pour les lignes de Total ou de Titre des groupes personnalisés :

      • Cette ligne doit toujours figurer dans les états financiers.

      Affichage dans la copie d'impression du document si le groupe comprend plus d'un solde de compte (les cellules ne sont pas vides et leurs valeurs ne sont pas nulles).

    • Pour les lignes de la balance de vérification :

      • Display in the financial statements and in the print copy if the balances for current or prior year exist.

  • Toujours afficher cette colonne - la colonne est toujours affichée dans les états financiers et dans la copie imprimée (PDF).

  • Toujours masquer cette colonne - la colonne est toujours masquée dans les états financiers et dans la copie imprimée (PDF).

    • Remarque : Cette option n'est disponible que pour les lignes de saisie et les lignes de groupe (lignes de groupe de la balance générale, lignes de groupe personnalisées et lignes de total ).

Pour compléter cette fonctionnalité, les lignes qui apparaissent dans les états financiers mais ne s'impriment pas apparaissent en gris dans le document, ce qui permet de les repérer facilement.

Cloud Connector

#primaryauthor

Cloud Connector is a Microsoft Office Add-in that allows you to link data between your Cloud Engagement product and MS Excel. Grâce à ce lien, vous pouvez créer des fichiers Excel pour les états financiers, les tableaux ou les calculs adaptés aux besoins de votre solution.

Cloud Connector permet une mise à jour en temps réel des données liées dans le fichier Excel. If any value or information changes in the engagement file, it's automatically updated in the excel file - guaranteeing efficient and effective information transfer.

Les utilisateurs peuvent également utiliser Cloud Connector dans des fichiers Excel pour créer la preuve du tableau des flux de trésorerie dans les audits.

Comme le montre la figure ci-dessous, Cloud Connector affiche les fichiers Excel en tant que Datalink dans le ruban.

Pour utiliser Cloud Connector, vous devez :

  1. disposer d'une installation locale de Microsoft Office 365 Desktop ou en ligne. Notez que la version 2019 ou antérieure de MS Office n'est pas prise en charge.

  2. créer un fichier Excel de MS Office 365 Desktop et le télécharger dans votre modèle ou fichier d'engagement.

  3. ouvrez le fichier Excel à partir de la page Documents de votre modèle ou de votre dossier d'engagement.

  4. Obtenez le complément Caseware Cloud Connector dans la boutique Microsoft Add-ins.

    5. Les Datalink Sera disponible dans la section du magasin.

  5. use the Datalink Add-in to insert data from your template or engagement file in the excel file.

Pour en savoir plus sur cette fonctionnalité, regardez le clip de démonstration de Cloud Connector lors de notre webinaire SE 2020-01.

Composants

#primaryauthor

Un nouveau paramètre de produit Composants est introduit pour soutenir une approche plus universelle de la logique de visibilité à travers les éléments de contenu du modèle. Dans cette version du paramètre Composants, les groupes financiers de la balance de vérification sont les seuls éléments de contenu pris en charge.

Vous pouvez définir des composantes pertinentes pour votre produit ou votre marché. Par exemple, vous pouvez définir des composants en fonction du type de secteur, du type d'entité ou du cadre de reporting. Vous pouvez également définir une logique de visibilité pour chaque composant. Par exemple, n'affichez le contenu relatif à un secteur spécifique de l' agriculture que si l'entité opère dans ce secteur.

Dans cette version, une nouvelle propriété de groupe, Composant, est désormais disponible dans la boîte de dialogue Propriétés du groupe afin que vous puissiez associer un ou plusieurs composants à un groupe de balance générale.

Si vous associez plusieurs composants à un groupe de balances de vérification et que vous définissez des paramètres de visibilité pour chacun des composants, alors.. :

  • le groupe de balances de vérification est masqué dans l'engagement si tous les composants associés sont masqués en raison de conditions de visibilité.

  • le groupe de balances de vérification est visible dans la mission si au moins un des composants associés est visible en raison des conditions de visibilité.

Groupe

#primaryauthor

Le champ ID du groupe dans les propriétés du groupe peut désormais contenir jusqu'à 256 caractères pour répondre à la nécessité d'avoir des ID plus longs. Par exemple, les produits pour lesquels iXBRL est activé peuvent avoir des identifiants d'une longueur maximale de 200 caractères.

Conditions de visibilité

Nouveau pour les utilisateurs finaux

Comptes

Vous pouvez désormais assigner de nouveaux comptes à des groupes sans avoir à naviguer sur une autre page. La boîte de dialogue Compte comporte désormais trois listes déroulantes de groupes (Financier, Code fiscal et Plan comptable).

Notez que si le compte est affecté à un groupe lié à un autre groupe, il est automatiquement affecté au groupe lié également. Par exemple, si le compte est affecté à l' inventaire dans le regroupement financier lié aux autres stocks à vendre dans le plan comptable, le compte est automatiquement affecté aux autres stocks à vendre dans le plan comptable également.

Requêtes

  • le bouton Sélectionner la destination est absent des documents de requête dans le modèle de produit et le modèle d'entreprise.

  • le temps de chargement d'un document de requête contenant plus de 100 questions est plus long que prévu.

  • les messages d'erreur traduits ne s'affichent pas correctement dans la page Créer une requête .
