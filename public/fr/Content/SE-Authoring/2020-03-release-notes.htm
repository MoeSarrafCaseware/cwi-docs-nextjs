Notes de mise à jour SE 2020-03

La SE 2020-03 apporte de nouvelles fonctionnalités . Par exemple, la possibilité d'exporter des groupes et leurs structures ou de copier des documents d'un modèle à l'autre. Des ajouts que nous avons qualifiés de "petits". Pourtant, de façon tout à fait inattendue, ce sont eux qui ont suscité la plupart des discussions et des questions lors du webinaire sur la version 2020-03 !

Nouveau pour les auteurs

Voici les nouveautés pour les utilisateurs finaux dans la SE 2020-03.

Contrôles

#primaryauthor

Remarque : Cette fonction n'est disponible qu'avec Caseware Audit. Il ne sera pas déployé dans des environnements de partenariat ou de production.

Une nouvelle section " Contrôles" est désormais disponible dans les paramètres du produit, car nous continuons à développer le nouveau module de risque :

  • Automatisation de la mise en œuvre des contrôles et de la documentation de conception.

  • Tester l'efficacité opérationnelle des contrôles.

Dans l'onglet Général , cochez la case Activer les contrôles dans le module de risque pour configurer les options générales et les champs relatifs à la conformité et à la documentation de l'identification et du test des contrôles.

Dans l'onglet Champs , vous pouvez spécifier les champs qui s'affichent dans le rapport de contrôle et personnaliser les étiquettes.

La figure ci-dessous montre la section Évaluation de l'efficacité opérationnelle dans l'onglet Champs. Remarquez que chaque champ peut être activé ou désactivé dans le rapport. Les étiquettes des champs activés peuvent également être personnalisées.

Remarque : Le rapport sur les contrôles "Out-Of-The-Box" sera disponible dans une prochaine version. Cependant, vous pouvez utiliser SE Builder pour créer un rapport de contrôle personnalisé basé sur les paramètres du produit. Contactez l' équipe produit Cloud Engagements pour plus d'informations et pour obtenir de l'aide.

États financiers

#primaryauthor

Un message d'avertissement s'affiche désormais dans les zones d'états financiers comportant plus de 100 sections de contenu pour vous informer que vous (et les utilisateurs de votre produit) devez naviguer dans la zone à l'aide de la carte du document, et non en faisant défiler les pages.

To learn more, see Best practices for financial statements authoring.

Copier-coller intelligent

#primaryauthor #firmauthor

L'adoption de cette approche vous aide à développer et à diffuser des produits de manière efficace, tout en réduisant la probabilité d'erreurs potentielles.

Pour activer le copier-coller intelligent, attribuez la valeur true à la propriété Importation de documents dans le fichier product.json du modèle dans lequel vous souhaitez copier des documents.

Une fois que vous avez activé le copier-coller intelligent, l'option Copier à partir d'un autre produit devient disponible dans le modèle sur lequel vous travaillez actuellement et vous pouvez copier des documents rédigés dans d'autres modèles de produits.

Dans la version SE 2020-03, la fonction de copier-coller intelligent a une fonctionnalité limitée que nous prévoyons d'étendre dans les prochaines versions.

  • Vous ne pouvez copier que des lettres, des mémos, des feuilles de travail et des étatsfinanciers .

  • La copie de documents d'un modèle à l'autre ne permet pas d'identifier les données manquantes ou incohérentes dans les paramètres de visibilité, le texte dynamique, les formules, les termes du glossaire d'engagement et les graphiques d'analyse.

Remarque : Vous devez identifier et résoudre manuellement les divergences de données pour les paramètres de visibilité, le texte dynamique, les formules, les termes du glossaire d'engagement et les graphiques d'analyse entre les modèles source et de destination après avoir copié des documents.

Pour en savoir plus, voir Qu'est-ce que le copier-coller intelligent, Copier le contenu d'un autre modèle et Limites actuelles.

XBRL

#primaryauthor

Remarque : Cette fonction n'est disponible qu'avec Caseware Audit. Il ne sera pas déployé dans des environnements de partenariat ou de production.

Vous pouvez désormais examiner les faits XBRL dans les états financiers et exporter les documents d'instance pour vérifier qu'ils sont générés comme prévu.

Si vous avez activé XBRL dans votre produit, l'icône XBRL Review and Export est disponible dans votre modèle de produit.

Vous pouvez sélectionner la taxonomie que vous souhaitez réviser ou exporter. Vous pouvez interrompre le processus de révision à tout moment et le reprendre plus tard. Pour plus de détails sur la manière de tester, voir XBRL Review and Export.

Remarque : Dans la SE 2020-03, la révision et l'exportation XBRL ne sont disponibles que pour les zones de tableaux dynamiques des états financiers. Nous prévoyons d'étendre cette fonctionnalité à d'autres documents dans les prochaines versions.

Nouveau pour les utilisateurs finaux

Voici les nouveautés pour les utilisateurs finaux dans la SE 2020-03.

Impression en vrac

Une nouvelle option Paramètres d'impression en masse est désormais disponible. Cette option permet aux utilisateurs de regrouper deux documents ou plus afin de les fusionner et de les imprimer sous la forme d'un seul document PDF.

Vous pouvez configurer plusieurs groupes d'impression en bloc. Le même document peut également être inclus dans plusieurs groupes d'impression en masse.

Une fois qu'un groupe d'impression groupée est configuré, il s'affiche dans l'option d'impression de la page Documents et dans chaque document d'engagement. Notez que lorsque les impressions en masse sont configurées, vous avez toujours la possibilité d'imprimer un document individuellement, si vous le souhaitez.

Page de données

La page Données présente de nouveaux ajouts liés aux onglets de groupe et à la colonne Ajustements .

  • Les options d'exportation sont désormais disponibles dans les onglets des groupes. Vous pouvez sélectionner Exporter () pour exporter les données vers un fichier CSV, PDF ou Working Papers. Vous pouvez également exporter la structure du groupe.

    • Notez que l'exportation prend en compte les sélections existantes sur la page. For example, if users have disabled Parent Groups, then they're not included in the exported file.

  • Les ajustements s'affichent désormais en bleu. Vous pouvez sélectionner un ajustement pour accéder à la page Ajustements , où les écritures d'ajustement correspondantes sont mises en évidence afin que vous puissiez en consulter les détails.

    • La figure ci-dessous montre comment l'écriture d'ajustement correspondante est mise en évidence dans la page Ajustements .

Problèmes

Dans la SE 2020-03, nous avons amélioré la logique de suppression des questions afin d'éviter la perte d'informations et d'améliorer la piste d'audit.

  • Les utilisateurs ne peuvent plus supprimer une question s'ils ne l'ont pas créée.

  • Seul un administrateur peut supprimer les questions créées par d'autres utilisateurs. An Admin user has one or more Cloud security roles:

    • Administrateur

    • Paramètres Admin

    • Entités Admin

    • Administrateur du personnel
