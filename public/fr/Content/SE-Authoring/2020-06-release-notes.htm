Notes de mise à jour SE 2020-06

Dans cette situation sans précédent, alors qu'elle s'adaptait à un nouveau mode de fonctionnement, qu'elle jonglait avec le travail et la famille ou qu'elle était dans l'incertitude générale, l'équipe de Cloud Engagements s'est montrée inflexible.

La SE 2020-06 regorge de fonctionnalités et d'améliorations ; elle est plus grande que les versions précédentes de cette année.

Indicateur de signe inversé

#enduser #premier auteur #firmauthor

Lorsque vous modifiez le signe par défaut d'un compte dans les tableaux dynamiques, un indicateur s'affiche désormais dans la ligne.

Du point de vue de l'utilisateur final, les soldes des lignes dont le signe est inversé ne s'affichent plus en italique pour indiquer que cette ligne a un signe opposé. Le nouvel indicateur de signe inversé () s'affiche dans la ligne à la place. Pour en savoir plus, voir Modifier les lignes des tableaux dans les états financiers.

Amélioration de la recherche d'engagement

#enduser#premier auteur#firmauthor

Une option de recherche universelle d'engagement est désormais disponible pour vous permettre de.. :

  • Recherche de documents visibles dans l'engagement

  • Rechercher le contenu de tous les documents visibles dans la mission

Les résultats correspondants s'affichent lorsque vous saisissez un mot-clé pour la recherche. Notez qu'avec la mise en place de l'option de recherche universelle, la barre de recherche du volet Documents n'est plus disponible.

Noter les références dans les sections du texte

#enduser#premier auteur#firmauthor

Vous pouvez désormais faire référence aux informations fournies par les notes dans les sections de texte des états financiers. La barre d'outils des sections de texte comprend désormais une icône de référence de note. Pour en savoir plus, voir Ajouter une référence de note dans une section de texte.

Signer les ajustements et la balance des comptes

#enduser #primaryauthor

Vous pouvez désormais activer l'option de signature dans la page Ajustements et la page Données afin que les utilisateurs puissent signer la balance de vérification et les écritures d'ajustement.

Deux nouveaux paramètres sont désormais disponibles dans la boîte de dialogue Paramètres du produit pour vous permettre d'activer l'option de signature.

  • Activer la signature des ajustements - Cochez cette case pour activer la signature du réviseur/préparateur dans la page Ajustements de votre produit.

  • Activer la signature de la page de données - Cochez cette case pour activer la signature du réviseur/préparateur dans l'onglet Comptes et les onglets de groupe de la page de données de votre produit.

Pour en savoir plus, voir Approuver la balance et les régularisations.

Sous-dossiers dans la page Documents

#enduser#premier auteur#firmauthor

Des sous-dossiers sont désormais disponibles sur la page Documents afin de regrouper le contenu pour une meilleure présentation et accessibilité. Pour en savoir plus, voir Créer un brouillon.

Filigrane dans les paramètres d'impression

#enduser #premier auteur #firmauthor

Vous pouvez désormais ajouter un filigrane dans les états financiers. L'option Appliquer un filigrane est désormais disponible dans les paramètres d'impression du document. Une fois cette option activée, vous pouvez ajouter le contenu du filigrane dans le champ de texte associé.

Pour compléter cette fonctionnalité, l'icône Imprimer () des états financiers affiche désormais deux options :

  • Aperçu avant impression - Lance un aperçu avant impression de la sortie PDF.

    • Notez que les paramètres d'impression () sont également accessibles en mode aperçu avant impression. Vous pouvez modifier les paramètres et les visualiser sans avoir à fermer la page d'aperçu avant impression et à revenir au document des états financiers.

    Dans la page d'aperçu avant impression, vous pouvez également sélectionner l'icône d'impression () pour imprimer le document ou la flèche vers le bas () pour télécharger le PDF.

  • Télécharger le PDF - Lance le téléchargement du fichier de sortie PDF.

    • Si l'option filigrane est activée, l'option Télécharger PDF devient Télécharger PDF (WMK) pour indiquer que le texte du filigrane sera affiché dans le PDF.

Composants

#primaryauthor

Vous pouvez maintenant regrouper des composants. Les catégories de composants sont désormais disponibles dans les paramètres du produit, ce qui vous permet d'organiser davantage les composants de votre produit pour plusieurs secteurs d'activité, types d'entités ou cadres de reporting.

Notez que les catégories de composants étant désormais disponibles, vous ne pouvez plus ajouter de composants autonomes. Vous devez créer au moins une catégorie pour que l'option +Composant soit disponible dans la boîte de dialogue. Pour en savoir plus, voir Définir la matérialité.

Pour compléter cette fonctionnalité, la boîte de dialogue Propriétés du groupe comprend désormais une nouvelle propriété Catégorie de composant. Une fois que vous avez affecté une ou plusieurs catégories à un groupe de balances de vérification, elles sont incluses dans la boîte de dialogue, ce qui vous permet d'affecter les composants applicables.

Remarque : Vous pouvez affecter des composants de différentes catégories au même groupe de balance générale.

Pour en savoir plus, voir Catégories de composants et Composants.

Requêtes

#firmauthor

Au lieu de devoir supprimer et recréer les questions et réponses existantes, l'option Afficher l'identifiant de l'auteur affiche une boîte de dialogue dans laquelle les auteurs de l'entreprise peuvent copier les identifiants existants.

  • Les entreprises qui n'ont pas de modèle d'entreprise mais qui souhaitent copier les identifiants d'auteur pour les intégrer aux documents de travail doivent créer un projet de modèle afin de pouvoir copier les identifiants d'auteur. Une fois qu'ils ont copié les identifiants, ils peuvent sélectionner Supprimer le projet pour supprimer le modèle d'entreprise qu'ils viennent de créer. To learn more about the firm template workflow, see Manage the firm template

  • Cette fonction est disponible dans l'ancienne et la nouvelle présentation des requêtes

Pour en savoir plus sur l'intégration des requêtes dans les documents de travail, voir Intégrer les documents de travail aux requêtes.

Pour visionner une vidéo décrivant le fonctionnement de l'intégration Queries-Working Papers, voir Ajouter des identifiants d'auteur.

Nouveau pour les utilisateurs finaux

Voici les nouveautés pour les utilisateurs finaux dans la SE 2020-06.

Listes de contrôle

Les questions liées aux procédures de listes de contrôle ont fait l'objet de quelques améliorations.

  • Si la question liée à la requête est masquée dans le document de requête, ses réponses ne s'affichent pas dans le document de la liste de contrôle.

  • Les fichiers téléchargés par les clients s'affichent désormais sous forme de liens, ce qui permet aux utilisateurs d'y accéder directement.

Page de données

Une nouvelle option d'importation Fichier d'engagement est désormais disponible dans l'onglet Importation .

Remarque : Vous devez activer l'option d'importation du fichier d'engagement dans votre produit, sinon elle ne sera pas disponible pour les utilisateurs. Pour activer l'option d'importation du fichier d'engagement , ajoutez le module "com.caseware.import.engagementfile" au tableau "imports" du fichier product.json.

Une fois l'option activée, les utilisateurs peuvent importer une balance de vérification à partir d'un autre fichier d'engagement. Notez que :

  • Les utilisateurs ne peuvent sélectionner qu'un seul fichier d'engagement à importer

  • Pour importer un fichier d'engagement appartenant à une autre application Cloud, vous devez avoir accès à cette application et à l'entité à laquelle le fichier appartient. Pour en savoir plus, voir Définir les niveaux d'accès du personnel

  • Les utilisateurs ne peuvent pas faire correspondre les données entre les engagements de source et de destination. Par exemple, la colonne de données Année en cours de la mission source est automatiquement importée dans la colonne Année en cours de la mission de destination

  • Une fois les données importées, la date de fin d'exercice de la mission de destination est automatiquement fixée à la date de fin d'exercice de la mission source

  • Les annotations et les questions existantes dans les données sources ne sont pas importées

  • L'importation à partir d'une mission de consolidation permet d'importer les données de l'entité mère uniquement dans la mission de destination

  • Si les regroupements de sources et de destinations ne correspondent pas ou partiellement, des comptes non attribués apparaîtront dans l'onglet Attribuer des comptes et vous devrez les attribuer manuellement. Pour en savoir plus, voir Attribuer des comptes aux groupes dans la page Données.

États financiers

Les ajustements d'élimination sont désormais inclus dans la fenêtre contextuelle des détails du compte dans les missions de consolidation.

Requêtes

La SE 2020-06 apporte une aide au report dans la nouvelle présentation des requêtes et davantage de mesures de sécurité pour l'échange de fichiers.

  • Les contacts ne peuvent plus télécharger de fichiers suspects dans les réponses de téléchargement de fichiers (tels que .com, .bat, .exe, .cmd, .sh, .dll, .so, .o, .deb, .msi, .rpm, .ko, .sys, .php).

  • Le report est maintenant disponible pour les requêtes de la nouvelle mise en page.

Revue XBRL

Remarque : This feature is only available for distributor testing only. Il ne sera pas déployé dans des environnements de partenariat ou de production.

Les sections de texte sont désormais incluses dans la révision XBRL des états financiers. Une fois que les utilisateurs ont lancé le processus de révision, une bordure verte en pointillés marque les sections de texte et les détails des faits s'affichent dans le volet XBRL REVIEW afin qu'ils puissent réviser et valider les données.

Les liaisons de données (fx) sont soulignées en vert. Si l'utilisateur sélectionne un lien de données, le détail des faits s'affiche dans le volet XBRL REVIEW .

L'examen de XBRL a également été l'occasion d'améliorer les fonctionnalités dans les domaines suivants :

  • Navigation vers d'autres documents après que les utilisateurs ont commencé la révision XBRL

    Si les utilisateurs naviguent vers un document qui ne fait pas partie de la taxonomie sélectionnée, le processus de révision XBRL s'arrête automatiquement. Notez que si les utilisateurs redémarrent le processus de révision, ils reprennent là où ils s'étaient arrêtés. En d'autres termes, le dernier document qu'ils ont examiné s'ouvre avec les derniers faits sélectionnés affichés dans le volet XBRL REVIEW .

  • Modification de la taxonomie sélectionnée après que les utilisateurs ont commencé la révision XBRL

    Si l'utilisateur sélectionne une autre taxonomie, le premier document de cette taxonomie s'ouvre. Le premier fait configuré dans ce document s'affiche par défaut dans le volet XBRL REVIEW .

  • Désactivation des actions dans le document que les utilisateurs sont en train de consulter

    Une fois que les utilisateurs ont entamé le processus de révision XBRL, la seule action qu'ils peuvent effectuer dans le document est de créer ou de modifier des questions (). Toutes les autres actions sont désactivées.

Corrections dans SE 2020-06

La liste suivante contient une brève description des problèmes qui ont été résolus dans cette version.

Listes de contrôle

The following bugs have been resolved.

  • L'ajout de liaisons de données (fx) dans un tableau dans le champ de description de la procédure ne fonctionne pas comme prévu.

  • Les icônes de visibilité manquent pour les procédures et les groupes de procédures.

États financiers

The following bugs have been resolved.

  • Les tableaux dynamiques n'affichent pas les soldes de comptes inférieurs à 1.

  • Lorsque le document contient un grand nombre de contenus exclus, la sélection de l'icône Ajouter () n'affiche pas la liste complète des contenus exclus.

  • La fenêtre contextuelle des détails du compte dans les tableaux dynamiques affiche des informations incorrectes.

  • Les zones de texte qui comprennent des tableaux dans les notes sur les états financiers s'affichent toujours alignées à gauche dans l'impression PDF du document.

  • Les paramètres d'impression des marges dans les zones sont remplacés par les paramètres d'impression des marges du document.

  • Problèmes d'alignement pour les nombres négatifs qui sont affichés entre parenthèses.

  • Un problème de style dans les sections d'analyse lorsqu'il n'y a pas de données à afficher dans une figure.

  • La tentative d'impression du document avec une section d'analyse échoue si les utilisateurs n'ont pas sélectionné un type de graphique.

Nouveau dialogue sur les risques.

The following bugs have been resolved.

  • Les domaines ajoutés aux risques nouvellement créés ne s'affichent pas dans l'onglet Évaluation à moins que les utilisateurs ne rafraîchissent la page.

  • Problèmes de style.

Autres corrections

Here is a brief description of general bugs that have been fixed in this release.

  • Les modèles d'entreprise n'ont pas été mis à jour avec la dernière version du modèle de produit.

  • Les tentatives de création d'engagements pour certaines applications échouent car la page ne cesse de se charger.

  • La tentative d'importation de la balance des comptes par le biais des plateformes de comptabilité en ligne ne fonctionne pas comme prévu.

  • Le bouton Enregistrer est désactivé lorsque les utilisateurs tentent de modifier le solde de l'année en cours pour des valeurs négatives sur la page Données.

  • Les liens vers les pages internes (telles que l'importation de transactions) ajoutées par les utilisateurs dans les questions sont rompus.

  • Un nombre de révisions incorrect est affiché pour les outils de révision Non signé et Optimisé .

  • La tentative de report d'un engagement consolidé entraîne une erreur.

  • Les questions définies comme étant cachées dans l'ancienne présentation de la requête continuent à s'afficher lorsque les utilisateurs désactivent les éléments cachés dans le document.

  • Problèmes de traduction de l'interface utilisateur.

  • Questions techniques liées aux paramètres linguistiques dans XBRL.

  • Les marques de révision (lignes pointillées vertes) continuent à s'afficher après que les utilisateurs ont interrompu le processus de révision XBRL.

  • Le lien de données du type d' organisation (fx) renvoie des données incorrectes lorsque des types d'organisation personnalisés sont définis dans le produit.
