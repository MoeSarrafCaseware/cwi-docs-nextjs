Signature électronique simple dans les requêtes

#enduser #premier auteur #firmauthor

Pour soutenir les scénarios dans lesquels les clients sont tenus de signer des documents ou d'accepter certaines conditions au cours de la mission, un nouveau type de réponse, Signature électronique simple, est désormais disponible dans les requêtes.

Pour inclure le type de réponse Signature électronique simple dans votre modèle de produit, cochez la case dans les paramètres du produit.

Lasignature électronique simple offre une méthode de signature des documents sans papier, sécurisée et plus efficace. Une fois que vous avez envoyé la demande aux clients, ils peuvent signer le document ou accepter les conditions d'une simple pression ou d'un clic sur un bouton.

Une fois que les clients ont accepté le document, un registre des signatures est généré et peut être téléchargé. Le journal contient les informations suivantes :

  • Détails de la question
  • Nom et courriel du signataire
  • Localisation du signataire (si les clients ont choisi d'activer l'accès à la localisation)
  • Adresse IP du signataire
  • Date de la signature
  • Nom du document
  • Diagramme MD5 du document

Pour en savoir plus, voir Activer le nouveau modèle de requête et Ajouter un modèle de requête (nouveau modèle).

Le volet des documents est déplacé vers la droite

Le volet Documents est désormais disponible sur le côté droit de la page et non plus sur le côté gauche. Le déplacement du volet est une décision stratégique à compléter :

  • La fonctionnalité de recherche globale de documents dans le coin supérieur droit de la page

  • La refonte prochaine de la page Documents. Pour en savoir plus, consultez la rubrique Nouvelle conception de la page Documents.

Vous pouvez ouvrir et fermer le volet Documents en utilisant l'icône bleue de documents () en haut à droite ou la flèche en bas ().

Options de signature pour les risques

#enduser #primaryauthor

Un nouveau paramètre Option de signature est disponible pour le nouveau module de risque afin de vous permettre de déterminer comment les utilisateurs signent le rapport de risque.

Deux options sont disponibles pour la signature :

  • Signature de l'ensemble du rapport - Un bouton de signature est disponible dans le rapport pour permettre aux utilisateurs de signer l'ensemble du rapport.

  • Approuverchaque risque dans le rapport - Un bouton d'approbation est disponible dans chaque fiche de risque du rapport. Les utilisateurs signent chaque risque individuellement.

Vous pouvez cocher les deux cases pour appliquer les deux options dans le rapport.

Pour en savoir plus, voir Activer le nouveau module de risque.

Augmentation de la taille maximale des fichiers téléchargés

#enduser #premier auteur #firmauthor

Vous pouvez désormais télécharger jusqu'à 500 millions de fichiers vers l'engagement. Notez que cette limite de taille s'applique à chaque fichier.

Réorganisation des documents lors de l'impression en masse

#enduser #premier auteur #firmauthor

Vous pouvez désormais faire glisser les documents dans le jeu d'impression en masse pour les réorganiser afin qu'ils s'impriment dans l'ordre que vous souhaitez.

Notez que vous ne pouvez réorganiser les documents que dans les groupes que vous avez définis. C'est-à-dire,

  • Les auteurs principaux ne peuvent pas réorganiser les documents dans des groupes prédéfinis par les auteurs primaires

  • Les utilisateurs ne peuvent pas réorganiser les documents dans des groupes prédéfinis par les auteurs de l'entreprise ou les auteurs principaux

Pour en savoir plus, voir Configuration de l'impression en masse.

Nouveau design pour la page Documents

#primaryauthor

Pour aider les grandes équipes de mission qui réalisent des missions d'audit importantes et complexes, une nouvelle conception de la page Documents est disponible dans le SE 2020-09 pour permettre d'étendre la fonctionnalité de signature - au-delà de l'actuel préparateur et réviseur.

La fonctionnalité étendue de signature sera lancée dans une prochaine version, mais des changements ont été apportés à l'interface utilisateur de la page Documents pour mieux présenter et afficher les informations de signature.

Remarque : La nouvelle conception de la page Documents n'est disponible que pour les distributeurs dans le cadre d'un test bêta. Il ne sera pas déployé dans des environnements de partenariat ou de production. Pour afficher le nouveau design, modifiez votre URL comme suit: https://<server name of your partner site>/<firm name where you perform your authoring>/e/eng/<identifier>/index.jsp#/se-documents

Le nouveau design comporte un volet de navigation à gauche, où s'affichent les phases et leurs indicateurs de progression. Si vous survolez une phase, une présentation visuelle de ses détails apparaît. Les dossiers et les sous-dossiers, ainsi que leur contenu, sont entourés de bordures claires, ce qui permet de distinguer facilement les éléments appartenant à tel ou tel dossier ou sous-dossier.

Copier-coller intelligent

#primaryauthor

Les conflits entre les paramètres de visibilité, le texte dynamique, les formules et les termes du glossaire des engagements dans les états financiers sont désormais signalés.

En cas de données manquantes ou incohérentes, le volet Liens brisés devient disponible et affiche une liste des conflits identifiés, regroupés par nom de document. Vous pouvez accéder au volet Liens brisés en sélectionnant l'icône de lien brisé ().

Pour visualiser un conflit, sélectionnez-le pour ouvrir une boîte de dialogue affichant les détails du conflit et vous proposant des options pour le résoudre.

Si vous souhaitez résoudre le conflit ultérieurement, sélectionnez ANNULER pour fermer la boîte de dialogue. Pour résoudre un conflit, vous pouvez soit

  • Choisissez d'autres valeurs dans les menus déroulants

    • En d'autres termes, si un terme du glossaire d'engagement copié à partir d'un document source n'existe pas dans votre application cloud, sélectionnez un terme dans le glossaire de votre application pour l'insérer à la place dans le document.

  • Sélectionnez Supprimer () pour supprimer l'élément où réside le conflit

    • En d'autres termes, si un terme du glossaire d'engagement copié à partir d'un document source n'existe pas dans votre application en nuage, supprimez-le du document.

Pour en savoir plus, voir Soldes d'annulation dans les états financiers.

Nouveau pour les utilisateurs finaux

Voici les nouveautés pour les utilisateurs finaux dans la SE 2020-09.

Listes de contrôle

Si la signature est activée pour les procédures et les conclusions, un message d'avertissement s'affiche désormais si les utilisateurs tentent de signer le document alors qu'ils sont en train de le faire,

  • Une ou plusieurs procédures visibles n'ont pas été approuvées

  • La section Conclusion n'a pas été signée

  • Les procédures visibles dans la section Conclusion n'ont pas été signées

Module sur les risques

Les évaluations des risques peuvent désormais être reportées.

Approbation

La logique d'approbation du préparateur et du réviseur a été améliorée. Les utilisateurs ne peuvent plus supprimer que leurs propres signatures.

Un utilisateur qui s'est vu attribuer un ou plusieurs rôles de sécurité en tant qu'administrateur peut supprimer les signatures d'autres utilisateurs.

Les rôles de sécurité de l'administrateur dans Caseware Cloud sont les suivants

  • Administrateur

  • Paramètres Admin

  • Administrateur du personnel

  • Entités Admin

Pour en savoir plus, voir Attribuer des rôles de sécurité à l'échelle du système.

Corrections

La liste suivante contient une brève description des problèmes qui ont été résolus dans cette version.

Listes de contrôle

  • Problèmes de style dans l'onglet Analyse des documents de la liste de contrôle.

  • Lesauteurs du cabinet ne peuvent plus modifier les références faisant autorité dans les procédures rédigées par les auteurs principaux.

  • Lors de la configuration de la section Analyse , les utilisateurs et les auteurs ne peuvent plus sélectionner un chiffre qui n'a pas été correctement défini dans l'API. Un message d'erreur s'affiche si la figure sélectionnée n'a pas d'identifiant unique.

Page de données

  • Des erreurs dans l'affichage de l'année en cours et des années précédentes apparaissent dans la mission lorsque les utilisateurs importent des données du logiciel comptable BGL360. Par exemple, les données importées pour l'exercice 2020 indiquent 2019 comme année en cours et 2018 comme année précédente dans l'engagement.

  • L'onglet Transactions n'est pas disponible dans la page Données après une importation réussie de données à partir d'un logiciel de comptabilité de bureau. Les utilisateurs doivent rafraîchir la page pour afficher l'onglet Transactions .

  • Dans les engagements verrouillés, les éléments de l'interface utilisateur de la page de données ne sont pas en lecture seule.

États financiers

  • Les utilisateurs ne sont pas en mesure d'imprimer les documents relatifs aux états financiers dans le cadre d'une mission comportant des données de balance de vérification importantes.
  • Les écarts de solde dans les tableaux dynamiques ne sont pas signalés pour les lignes contenant des valeurs calculées. Les soldes en désaccord ne sont pas signalés par l'icône du panneau d'arrêt () .
  • La logique de détection des écarts de solde dans les tableaux dynamiques n'inclut pas un groupe Total ligne si une ou plusieurs lignes de ce groupe sont masquées.
  • Dans les tableaux dynamiques, le pourcentage d'écart total pour les lignes de saisie groupées ne s'affiche pas dans le document des états financiers. Il ne s'affiche qu'en mode d'édition du tableau.

Module sur les risques

  • Les signatures dans les cartes d'évaluation des risques sont supprimées lorsque les utilisateurs développent une carte.

  • Les modèles de requête ne s'affichent plus dans les menus déroulants de la réponse d'audit et de la référence d'audit.

Autres corrections
