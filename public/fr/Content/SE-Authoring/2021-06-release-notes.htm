Notes de mise à jour SE 2021-06

Prenez un verre de limonade et installez-vous pour l'été avec SE 2021-06 ! Cette version introduit de nouvelles fonctionnalités intéressantes telles que la bibliothèque des risques, les signatures multiples, les outils de révision pour le nouveau module de risque et bien plus encore.

Signatures multiples pour un rôle

#enduser#premier auteur#firmauthor

L'option de signature multiple est désormais disponible pour les rôles de signature personnalisés.

When the Multiple Signoff option for a role is enabled, more than one user can sign off on the role for a document. Les signatures sont affichées dans l'ordre chronologique (la signature la plus récente apparaît en bas).

Pour en savoir plus, voir Définir les rôles d'approbation.

Masquer les intitulés de sous-niveaux pour les notes dans les états financiers

#enduser#premier auteur#firmauthor

Dans le document 6-20 États financiers , les paramètres d'impression des notes comprennent désormais une option permettant de masquer automatiquement les en-têtes de sous-niveaux des notes dans la sortie PDF s'il n'y a pas de notes ultérieures dans ce groupe.

Par exemple, si la note 1 comprend une sous-note 1.a et qu'il n'y a pas de notes 1.b, 1.c, etc., l'en-tête de la sous-note 1.a ne s'affiche pas dans la sortie PDF. Seul le contenu de cette note s'affiche.

L'option de masquer les titres de sous-niveaux dans la sortie PDF est également disponible pour les notes individuelles dans le menu Plus d'actions s'il n'y a pas de notes ultérieures dans ce groupe.

Pour en savoir plus, voir Paramétrer les options d'impression des états financiers et Masquer les intitulés de sous-niveaux des notes.

Bibliothèque des risques

#primaryauthor

La bibliothèque des risques est désormais disponible pour faciliter le déroulement des missions d'audit. Cette bibliothèque vous permet de créer une liste prédéfinie de risques. Vous pouvez sélectionner les risques applicables et pertinents dans la bibliothèque et les ajouter aux missions.

Remarque : La bibliothèque des risques n'est disponible que pour les distributeurs dans le cadre de tests bêta. Il ne sera pas déployé dans des environnements de partenariat ou de production. Pour activer la bibliothèque des risques pour votre site bêta, envoyez un ticket DIST avec votre demande.

Une fois la bibliothèque de risques activée pour votre site, la page Bibliothèque de risques est disponible dans les paramètres de votre entreprise. Pour créer une bibliothèque de risques, téléchargez un fichier Excel contenant le contenu de votre bibliothèque. Vous pouvez ajouter plusieurs bibliothèques de risques afin de répondre aux différentes exigences en matière de documentation des risques pour différents produits.

Lorsque vous êtes prêt, vous pouvez publier la bibliothèque en tant que version majeure, puis lier cette version à un produit à partir de l'onglet Risk | General dans les paramètres du produit.

Notez que :

  • Le module des nouveaux risques doit être activé dans le produit auquel vous souhaitez associer une bibliothèque de risques.

  • Vous ne pouvez lier un produit qu'à une seule bibliothèque de risques à la fois.

  • Vous pouvez lier la version publiée de la bibliothèque des risques à plusieurs produits.

Si vous devez apporter des modifications à une bibliothèque de risques publiée, vous pouvez sélectionner l'ajout d'un nouveau fichier Excel et choisir s'il s'agit d'une version mineure ou d'une version corrective de la bibliothèque. Vous pouvez ajouter des versions mineures afin de les utiliser pour les mises à jour planifiées de la bibliothèque, tandis que des versions correctives peuvent être ajoutées pour les mises à jour urgentes.

Une fois que vous avez publié une version de la bibliothèque de risques et que vous l'avez liée à un produit, les utilisateurs peuvent ajouter des risques prédéfinis à partir de la bibliothèque au lieu de les créer manuellement.

Lorsque les utilisateurs sélectionnent l'option Risque de la bibliothèque dans la boîte de dialogue Ajouter, ils sont dirigés vers la bibliothèque de risques où ils peuvent utiliser les filtres et l'option de recherche de la bibliothèque pour trouver les risques appropriés.

Les types suivants sont disponibles :

  • Type - Filtre les risques affichés en fonction du type de risque

  • Statut - Filtre les risques affichés selon qu'ils ont été ajoutés ou non à la mission

  • Type - Filtre les risques affichés en fonction du type de risque Ce filtre n'est disponible que pour les risques au niveau de l'assertion.

  • Assertions - Filtre les risques affichés en fonction des assertions associées. Ce filtre n'est disponible que pour les risques au niveau de l'assertion.

Dans le panneau de gauche, vous pouvez également choisir de visualiser les risques par indicateurs ou par catégories.

Pour en savoir plus, voir :

  • Créer une bibliothèque de risques
  • Ajouter une version de la bibliothèque des risques
  • Publier une version de la bibliothèque des risques
  • Associer une bibliothèque de risques à un produit

Marquage XBRL

#primaryauthor

Les capacités de marquage XBRL ont été étendues pour permettre aux auteurs et aux utilisateurs finaux d'ajouter des marquages pour les faits XBRL directement dans le modèle ou le fichier d'engagement.

Remarque : Cette fonctionnalité n'est disponible que pour les distributeurs dans le cadre de tests bêta. Il ne sera pas déployé dans des environnements de partenariat ou de production. Pour visualiser cette fonctionnalité, ouvrez un modèle ou un fichier d'engagement et apportez la modification suivante à votre URL : https://<nom du serveur de votre site partenaire>/<nom de l'entreprise où vous rédigez vos textes>/e/eng/<identifiant>/index.jsp?enable-features=customXbrlTagging

Lorsque le mode de révision XBRL est activé, les éléments non marqués sont surlignés en jaune. Vous pouvez sélectionner l'option Ajouter une balise XBRL pour un élément spécifique afin de créer une balise pour celui-ci.

Vous pouvez ensuite sélectionner les informations relatives à l 'élément et au contexte de la balise parmi les options disponibles dans le panneau de révision.

To learn more, see Test custom XBRL tags.

Espacement des paragraphes dans les états financiers

#primaryauthor

Le paramètre Utiliser un espacement cohérent des paragraphes entre les zones de contenu est désormais disponible pour permettre aux auteurs primaires d'appliquer un espacement cohérent des paragraphes entre les zones de contenu dans les états financiers imprimés.

Pour en savoir plus, voir Imprimer les états financiers.

Importation de la structure de regroupement

#primaryauthor

L'ordre des groupes financiers dans une structure de regroupement est désormais conservé lorsque vous exportez puis importez une structure de regroupement.

Journal de signature pour les signatures électroniques simples

Vous pouvez désormais visualiser et télécharger un journal des signatures dans le visualiseur PDF pour les documents qui ont été signés électroniquement par les clients dans les documents de requête.

Pour en savoir plus, voir Examiner les réponses des contacts et Joindre des documents à un dossier de mission.

Examiner les outils pour détecter les risques

Les utilisateurs peuvent désormais utiliser des outils de révision pour suivre les changements apportés aux risques dans le nouveau module de risques après l'approbation du réviseur.

Si vous avez activé la validation au niveau du document pour le module risques, les outils de révision suivants sont à la disposition des utilisateurs :

  • Ajouté - Marque les risques qui ont été ajoutés.

  • Supprimé - Marque les risques qui ont été supprimés.

  • Modifié - Marque les risques qui ont été modifiés.

Si vous avez activé l'approbation individuelle des risques, seul l'outil de révision modifié est disponible. Notez que lorsque l'approbation individuelle des risques est activée, les utilisateurs ne peuvent pas supprimer un risque une fois que l'évaluateur l'a approuvé.

Vérification améliorée des écarts de solde

Les contrôles des écarts d'équilibre dans les tableaux dynamiques comparent désormais les valeurs absolues des lignes étiquetées et ne tiennent pas compte du fait que la valeur est négative ou positive. Ainsi, lorsque les soldes des comptes créditeurs sont comparés à ceux des comptes débiteurs, la différence de signe entre les deux comptes n'est pas signalée comme une anomalie de solde.
