Notes de mise à jour SE 2022-06

L'été est là, tout comme notre dernière série de mises à jour de la SE 2022-06. Cette version introduit des valeurs d'arrondi pour les tableaux dynamiques, des soldes de planification, des améliorations du formatage et de la présentation des états financiers, etc.

Arrondissement dans les états financiers

#enduser#premier auteur#firmauthor

Vous pouvez désormais activer l'arrondi dans les états financiers. L'activation de l'arrondi supprime la nécessité d'arrondir manuellement les valeurs et améliore la lisibilité des états financiers.

Une fois que vous avez activé l'arrondi dans les paramètres du produit, vous pouvez choisir d'arrondir à un, à des milliers, à des millionsou à des milliards.

Vous pouvez aussi :

  • Sélectionner le nombre de décimales à afficher

  • Déterminer s'il faut afficher les informations d'arrondi dans les en-têtes de description et les en-têtes de colonne

  • Personnaliser la terminologie utilisée pour les arrondis

Une fois l'arrondi activé, vous pouvez non seulement afficher les valeurs arrondies dans les tableaux dynamiques, mais aussi utiliser les conditions de mise à l'échelle/arrondi lors de l'ajout de termes de glossaire d'engagement et de texte dynamique, ainsi que lors de l'application de paramètres de visibilité au contenu.

Notez que lorsque l'arrondi est activé, les auteurs du cabinet et les utilisateurs finaux peuvent également modifier les paramètres d'arrondi utilisés dans la mission.

Pour en savoir plus, voir

  • Personnaliser les paramètres des états financiers

  • Conditions de visibilité

  • Conditions relatives aux termes du glossaire et à la logique d'affichage dynamique du texte

  • Personnaliser les arrondis dans les états financiers

Solde provisoire

#enduser#premier auteur#firmauthor

La version SE 2022-06 introduit la fonctionnalité des soldes de planification. La planification des soldes est un nouvel outil qui vous permet d'enregistrer et de suivre les soldes intermédiaires avant la fin de l'année. Cela vous permet de planifier l'audit à l'avance et de vous adapter aux changements au fur et à mesure qu'ils se produisent, tout en remplissant l'obligation de documenter votre travail.

L'auteur principal doit activer les soldes de planification dans les paramètres du produit.

Vous pouvez ensuite personnaliser la terminologie des soldes de planification dans l'onglet Terminologie des paramètres du produit. Notez que vous pouvez désormais personnaliser la terminologie utilisée pour les soldes précédents.

Pour en savoir plus, voir Activer les soldes de planification et Personnaliser la terminologie.

Symboles monétaires dans les tableaux dynamiques

#enduser#premier auteur#firmauthor

Afin d'améliorer encore les options de présentation et de formatage disponibles pour les états financiers, des options supplémentaires sont désormais disponibles pour les symboles de devise.

En plus de l'option existante d'afficher par défaut les symboles monétaires dans les en-têtes de colonne pour tous les tableaux dynamiques, vous pouvez désormais choisir de les afficher par défaut sur la première et la dernière ligne des tableaux dynamiques.

Notez que le symbole de la devise ne s'affiche que sur les lignes qui sont :

  • Inclus dans le PDF généré lorsque vous imprimez ou téléchargez les états financiers.

  • Soit un type de ligne Balance de vérification , soit un type de ligne Entrée.

  • Pas de ligne de titre

Si la première ou la dernière ligne d'un tableau ne remplit pas ces conditions, le symbole de la devise s'affichera sur la prochaine ligne disponible qui remplit ces conditions. Par exemple, si vous avez défini le symbole de la devise pour qu'il s'affiche sur la dernière ligne, mais que la ligne est définie pour ne pas être incluse lors de l'impression, le symbole s'affichera à la place sur la ligne précédente. Si cette ligne ne répond pas non plus aux critères, le symbole s'affiche sur la ligne supérieure.

Vous avez également la possibilité de choisir d'afficher le symbole ou le code de la devise dans les tableaux.

En outre, vous pouvez désormais choisir d'afficher ou de masquer les symboles monétaires dans n'importe quelle ligne de la balance de vérification, de l' entréeou de la dimension dans les tableaux individuels.

Pour en savoir plus, voir Définir les options de devise par défaut pour les tableaux dynamiques et Spécifier l'option de symbole de devise pour les lignes.

Vous pouvez maintenant sélectionner le nom de l'entité dans la fenêtre contextuelle Propriétés de l'engagement pour naviguer rapidement vers cette entité dans votre site Cloud.

Modifier les PDF

Vous pouvez désormais modifier les PDF qui ont été téléchargés dans votre engagement.

Lorsque vous ouvrez un PDF dans votre engagement, vous pouvez sélectionner le bouton Copier l'URL () pour copier l'URL du fichier. Cette URL peut ensuite être utilisée pour ouvrir le fichier dans Adobe Acrobat.

Après avoir modifié le PDF dans Acrobat, vous pouvez enregistrer le fichier dans le même chemin d'accès pour synchroniser les modifications apportées au PDF dans votre engagement.

Pour en savoir plus, voir Modifier un document PDF ajouté.

Corrections

La liste suivante contient une brève description des problèmes qui ont été résolus dans cette version.

  • Dans la page Données , la colonne de l'année précédente n'utilise pas la valeur d'importance relative correcte.

  • Incohérences de l'interface utilisateur sur la page des données .

  • Problèmes de traduction.

  • Dans les missions d'audit, le solde des transactions n'est pas calculé dans les tableaux d'exhaustivité des données si les données de la balance de vérification et les données des transactions ont été importées à partir de deux sources différentes.

  • Dans les entreprises en version bêta, le bouton Utiliser par défaut n'est pas disponible sur la page du modèle d'entreprise si le mode auteur de l'entreprise a été désactivé.

  • Dans les listes de contrôle, lorsque les utilisateurs choisissent d'afficher les éléments cachés, ceux-ci ne sont pas visibles tant que l'utilisateur n'a pas actualisé la page.

  • Les utilisateurs doivent cliquer deux fois dans les zones de texte avant de pouvoir commencer à taper du texte.

  • Dans les produits qui prennent en charge plusieurs langues, lorsque la langue sélectionnée par les utilisateurs lors de la connexion à Cloud est différente de la langue du contenu de l'engagement, les documents affichent le contenu dans les deux langues.

  • Amélioration de l'interface utilisateur de la page du modèle d'entreprise afin de clarifier la fonction du bouton " Mise à jour" . Le bouton indique maintenant Migrer les personnalisations du modèle précédent.

  • Dans les listes de contrôle, le texte dynamique et les termes du glossaire d'engagement qui dépendent des réponses à la procédure ne s'affichent pas tant que l'utilisateur n'a pas actualisé la page.

  • Lorsque les utilisateurs exportent des ajustements au format PDF, les écritures d'ajustement ne s'affichent pas comme prévu.

  • Lorsque les utilisateurs impriment les états financiers, les cellules de la colonne de saisie dans les lignes de la balance de vérification s'affichent en blanc si la ligne de la balance de vérification a des lignes enfants et est réduite.

  • Dans les paramètres d'impression des documents, lorsque les utilisateurs ajustent les marges des documents à une taille non recommandée, le message d'avertissement affiche toujours la taille de la marge en pouces, même si les centimètres sont l'unité de mesure sélectionnée pour l'engagement.
