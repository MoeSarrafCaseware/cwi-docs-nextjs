Notes de mise à jour SE 2023-01

La nouvelle année commence avec la dernière version de SE ! La SE 2023-01 apporte de nouvelles modifications intéressantes, notamment le tout nouveau rapport sur les contrôles et des améliorations significatives en matière de consolidation.

Nouveau rapport sur les contrôles

#enduser#premier auteur#firmauthor

SE 2023-01 features the official launch of the new controls report in the risk module. Avec le nouveau rapport sur les contrôles, il n'est plus nécessaire de créer un rapport personnalisé sur les contrôles à l'aide de SE Builder. Cette fonctionnalité améliore l'expérience de l'utilisateur en matière de documentation des contrôles et renforce la conformité de la plate-forme aux exigences en matière de documentation d'audit et la prise en charge de ces exigences.

Vous pouvez activer les contrôles pour le module de risque dans les paramètres du produit.

Vous pouvez aussi :

  • Activer et personnaliser les signatures pour le rapport de contrôle. Notez qu'une fois la signature activée, les utilisateurs finaux et les auteurs du cabinet peuvent personnaliser les signatures dans le modèle de cabinet et dans les fichiers de mission

  • Personnaliser les champs utilisés dans la boîte de dialogue Créer un nouveau contrôle

  • Déterminer comment les contrôles sont identifiés

  • Ajouter un lien d'aide concernant les contrôles de votre produit

Dans l'onglet Contrôles | Champs , vous pouvez personnaliser les champs disponibles dans le rapport des contrôles et déterminer s'ils sont disponibles pour les utilisateurs dans la boîte de dialogue Créer un nouveau contrôle.

Une fois les contrôles activés, l'onglet Contrôles est accessible aux utilisateurs finaux dans le module de risque et le nom du module devient Risques et contrôles si les risques et les contrôles sont tous deux activés.

To learn more, see Enable controls in the risk module.

Amélioration de la consolidation

#enduser#premier auteur#firmauthor

Un certain nombre d'améliorations ont été apportées à la consolidation des missions pour tenir compte des hiérarchies de consolidation plus complexes et des circonstances dans lesquelles les entités subsidiaires ne nécessitent pas de missions distinctes.

La vue graphique existante pour la consolidation comprend maintenant

  • Une option de recherche vous permet de trouver facilement des entités subsidiaires.

  • Mise à jour des dessins pour chaque carte subsidiaire.

  • La possibilité de développer et de réduire les branches de la hiérarchie.

  • Lorsque des filiales ayant des dossiers de mission distincts sont ajoutées, un lien vers le dossier de mission est disponible dans la fiche de la filiale afin de faciliter la navigation entre les dossiers de mission.

Une nouvelle vue en grille a également été ajoutée.

En plus de ces changements, vous pouvez désormais utiliser l'option Ajouter un CT importé pour créer des entités filiales internes pour votre entité mère et télécharger des données de balance de vérification sans lier un fichier de mission distinct pour la filiale.

Pour soutenir davantage ces améliorations, les mises à jour suivantes ont également été effectuées :

  • Vous pouvez désormais ajouter des ajustements pour les filiales internes.

  • La page Données a été mise à jour pour permettre de filtrer les données de la balance pour des entités spécifiques dans la hiérarchie de consolidation.

  • Lorsque vous ajoutez un calcul à partir de la boîte de dialogue Lien de données, vous pouvez spécifier si les données de balance liées sont celles d'une filiale, consolidées ou de l'entité mère.

  • Vous pouvez désormais créer des formules de lien de données qui déterminent les données à utiliser selon que le document est configuré pour utiliser des données consolidées ou non consolidées.

  • Vous pouvez désormais ajouter une colonne Consolidation totale aux tableaux dynamiques afin d'afficher le solde consolidé total.

Pour en savoir plus, voir Créer un engagement de consolidation et Consulter les données des engagements de consolidation dans la page Données.

Améliorations des dimensions

#enduser#premier auteur#firmauthor

Un certain nombre d'améliorations liées aux dimensions ont été ajoutées dans le cadre de cette version.

Réglages des dimensions

#enduser#premier auteur#firmauthor

Les auteurs de cabinets et les utilisateurs finaux peuvent désormais créer et modifier des dimensions dans l'onglet Dimensions des paramètres du produit. L'onglet Dimensions a également été actualisé afin d'améliorer l'expérience de l'utilisateur.

Vous pouvez aussi maintenant :

  • Ajouter des dimensions de troisième niveau.

  • Faites glisser les dimensions pour les réorganiser au sein d'une catégorie.

  • Affectez des catégories de dimensions aux groupes financiers à partir du menu Autres actions () ou de la boîte de dialogue Propriétés du groupe .

  • Firm authors and end users can also assign dimension categories to financial groups from the Group properties dialog.

    Notez que,

    • Les auteurs de l'entreprise ne peuvent modifier que les catégories de dimensions attribuées aux groupes par d'autres auteurs de l'entreprise et par les utilisateurs finaux.

    • Les utilisateurs finaux ne peuvent modifier que les catégories attribuées par d'autres utilisateurs finaux.

    • Les auteurs du cabinet et les utilisateurs finaux ne peuvent pas modifier les catégories de dimension pour les groupes qui ont hérité des catégories de groupes de niveau supérieur

    Pour en savoir plus, voir Ajouter des dimensions.

Filtre de dimensions pour les groupes financiers

#enduser#premier auteur#firmauthor

Une liste déroulante Dimensions est disponible dans l'onglet Groupes financiers de la page Données pour filtrer les groupes financiers par catégories de dimensions.

Notez que pour les missions consolidées, au lieu d'une liste déroulante, il y a une vue Dimensions séparée pour l'onglet Groupes financiers .

Dimensions des tableaux dynamiques

#enduser#premier auteur#firmauthor

Pour améliorer la possibilité d'utiliser les dimensions dans les tableaux dynamiques, vous pouvez filtrer par dimensions :

  • Lors de l'insertion d'une ligne de balance générale.

  • Lors de l'insertion d'un lien de données de balance de vérification dans le cadre d'un calcul dans les cellules d'un tableau.

Créer de nouveaux documents Word et Excel

#enduser#premier auteur#firmauthor

Vous pouvez désormais créer de nouveaux documents Word et Excel à partir d'un engagement sur la page Documents.

Pour en savoir plus, voir Ajouter de nouveaux documents Word et Excel.

Intégrer des PDF dans les états financiers

#enduser#premier auteur#firmauthor

Vous pouvez désormais intégrer des PDF dans les états financiers. Notez que,

  • Vous pouvez ajouter plusieurs zones de PDF incorporés aux états financiers.

  • Un seul PDF peut être téléchargé dans chaque zone de PDF incorporé .

  • Lorsque vous intégrez un PDF, il est inclus dans la table des matières des états financiers.

Pour en savoir plus, voir Intégrer un PDF dans les états financiers.

Masquer les lignes d'en-tête des tableaux dynamiques

#enduser#premier auteur#firmauthor

Vous pouvez choisir de masquer les lignes d'en-tête pour les tableaux dynamiques individuels. Lorsqu'une ligne d'en-tête est masquée, elle ne s'affiche pas lorsque les utilisateurs consultent les états financiers ou dans le PDF généré lorsque les utilisateurs impriment les états financiers.

Pour en savoir plus, voir Masquer les lignes d'en-tête des tableaux dynamiques.

#primaryauthor

Vous pouvez désormais personnaliser le terme "risque" pour votre produit dans l'onglet Terminologie des paramètres de votre produit. Le terme personnalisé s'affichera partout où le terme "risque" est utilisé.

Nouveau pour les utilisateurs finaux

Voici les nouveautés pour les utilisateurs finaux dans la SE 2023-01.

Impression de documents en masse

Les documents imprimés en masse sont désormais clairement étiquetés lorsque vous copiez des fichiers provenant d'autres missions.

Amélioration de l'aperçu avant impression des états financiers

Lorsque vous sélectionnez l'option Aperçu avant impression , l'aperçu s'ouvre maintenant sur la section des états financiers que vous étiez en train de consulter.

Corrections dans SE 2023-01

La liste suivante contient une brève description des problèmes qui ont été résolus dans cette version.

  • Dans les anciennes requêtes, le dossier cible et le nom du fichier pour les questions de type téléchargement de fichier ne s'affichent pas comme prévu.

  • Lorsque les utilisateurs modifient le nom d'une requête après qu'elle a été envoyée aux contacts, le nom mis à jour ne s'affiche pas sur les contacts.

  • Dans les tableaux dynamiques, les en-têtes des colonnes de calcul personnalisées ne s'affichent pas pour les auteurs principaux.

  • Les utilisateurs peuvent ajouter de nouveaux comptes sans leur attribuer de nom lorsqu'ils ajoutent des comptes à partir de la boîte de dialogue Ajouter un ajustement.

  • Dans les tableaux dynamiques, les références de notes ajoutées aux lignes de commentaires ne sont pas sauvegardées.

  • Dans les missions de consolidation, les icônes d'importance relative ne s'affichent pas pour les colonnes des entités filiales dans la vue consolidée de l'onglet Groupes financiers.

  • Dans les missions de consolidation, les soldes en double s'affichent dans la vue consolidée de l'onglet Groupes financiers.

  • Les détails des conditions ne s'affichent pas pour le texte dynamique avec les conditions de groupe.

  • Lorsque les utilisateurs ajoutent un nouveau compte et lui attribuent un groupe parent avec une catégorie de dimension, le groupe parent n'est pas sauvegardé lors de la première tentative.

  • Dans les missions de consolidation, le document relatif au seuil de signification ne présente que des données consolidées.

  • Une amélioration technique liée aux anciennes requêtes.
