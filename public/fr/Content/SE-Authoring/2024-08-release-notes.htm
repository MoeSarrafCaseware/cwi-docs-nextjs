Notes de mise à jour SE 2024-08

Bienvenue dans la version SE 2024-08 ! Cette version introduit la toute nouvelle fonctionnalité de planification centrale, une mise à jour des paramètres du produit liés à l'Analytics Hub, des ajustements calculés et plus encore.

Sur cette page, vous pouvez voir toutes les nouvelles fonctionnalités incluses dans cette version qui s'appliquent à la fois aux auteurs et aux utilisateurs finaux.

Planification centrale

#enduser#premier auteur#firmauthor

La version SE 2024-08 introduit la fonction de planification centrale. La planification centralisée permet aux utilisateurs de synchroniser les risques, les contrôles et les listes de contrôle sélectionnées entre les missions, de sorte que le travail n'a pas besoin d'être dupliqué dans plusieurs missions. Pour activer la planification centrale dans votre produit à partir des paramètres du produit, sélectionnez Activer la planification centrale dans l'onglet Planification centrale . Vous pouvez ensuite sélectionner les documents de la liste de contrôle de votre modèle qui seront synchronisés. Tous les risques et contrôles sont synchronisés par défaut.

Les auteurs du cabinet peuvent également modifier les listes de contrôle à inclure lors de la synchronisation avec un engagement de planification central. Cela permet aux auteurs de l'entreprise de personnaliser la planification centrale en fonction de leur entreprise et de leurs besoins, et d'inclure des listes de contrôle créées par l'auteur de l'entreprise.

Une fois que la planification centrale est activée dans votre produit, les utilisateurs peuvent alors sélectionner Définir comme engagement de planification centrale dans leurs paramètres d'engagement pour désigner les engagements comme des engagements de planification centrale.

Notez que cette action est irréversible. Une fois qu'une mission est définie comme une mission de planification centrale, la case à cocher ne peut pas être désélectionnée.

Pour créer un engagement synchronisé avec l'engagement de planification centrale, les utilisateurs peuvent naviguer jusqu'à l'engagement de planification centrale dans leur site cloud et utiliser l'option Créer un engagement à partir de la source . Les risques, les contrôles et les listes de contrôle sélectionnés dans ce nouvel engagement seront alors automatiquement synchronisés à partir de l'engagement de planification central.

Dans les missions de planification centrale et les missions de destination synchronisées avec la mission de planification centrale, l'icône Planification centrale () s'affiche dans la barre de menu supérieure. Notez que l'icône s'affiche également à côté des listes de contrôle synchronisées.

Les utilisateurs peuvent sélectionner l'icône Planification centrale () dans la barre de menu supérieure pour ouvrir le Hub de planification centrale. Si l'utilisateur se trouve dans l'engagement de planification centrale, il peut consulter une liste des engagements de destination et voir s'ils sont à jour. S'ils se trouvent dans un engagement de destination, cette page les informe si des modifications ont été apportées dans l'engagement source et leur permet de synchroniser ces modifications.

Si un utilisateur ouvre une liste de contrôle synchronisée dans une mission de destination, un message s'affiche pour l'informer que le document a été complété dans la mission de planification centrale, avec un lien vers la liste de contrôle dans cette mission. Les risques et les contrôles continueront à s'afficher dans les missions de destination, mais en lecture seule.

To learn more, see Enable central planning. Pour consulter la documentation destinée à l'utilisateur final et à l'auteur de la société concernant la planification centrale, voir Configurer la planification centrale et Gérer les listes de contrôle synchronisées pour la planification centrale.

Nouveau pour les utilisateurs finaux

Voici les nouveautés pour les utilisateurs finaux dans la SE 2024-08.

Ajustements calculés

Les utilisateurs ont désormais la possibilité de créer des ajustements calculés. Lorsqu'ils ajoutent une ligne de compte ou de groupe à un ajustement, ils ont désormais la possibilité de sélectionner fx dans le champ Montant et de créer un calcul pour déterminer le montant de l'ajustement.

De même que pour les calculs personnalisés dans les tableaux dynamiques, les utilisateurs peuvent faire référence aux données de la balance générale dans leurs calculs. Si la valeur du compte ou du groupe de la balance de vérification référencée est modifiée, un bouton Recalculer apparaît pour l'ajustement sur la page Ajustements . Les utilisateurs peuvent sélectionner ce bouton pour recalculer et mettre à jour le montant de l'ajustement.

Si plusieurs ajustements doivent être recalculés, les utilisateurs peuvent également sélectionner le bouton Recalculer tout en haut de la page pour recalculer tous les ajustements en même temps.

Pour en savoir plus, voir Comptabiliser les ajustements de la balance de vérification.

Corrections dans SE 2024-08

La liste suivante contient une brève description des problèmes qui ont été résolus dans cette version.

  • Dans les états financiers, les utilisateurs ne peuvent pas ajouter de notes à partir de la bibliothèque de notes.

  • Dans les états financiers, les utilisateurs ne peuvent pas faire défiler le plan du document s'il y a un grand nombre de notes.

  • Lorsque les auteurs principaux suppriment les étiquettes personnalisées pour les évaluations de risques, les noms des évaluations de risques s'affichent en blanc au lieu d'afficher le nom par défaut.

  • Lorsque les utilisateurs ouvrent un fichier Excel lié, l'engagement se bloque.

  • Un problème technique lié à l'impression.

  • L'icône Ajouter est visible dans le menu supérieur pour les auteurs, bien qu'aucun élément pouvant être ajouté à partir de la boîte de dialogue Ajouter ne soit disponible dans le modèle.

  • Dans les listes de contrôle, une erreur se produit lorsque les utilisateurs désélectionnent leur réponse pour une liste de sélection et choisissent immédiatement une autre réponse.

  • Lorsque les utilisateurs sélectionnent l'option Télécharger la copie de référence , certains commentaires de requête ne sont pas inclus dans le PDF généré.

  • Lorsque les utilisateurs importent la balance comptable d'une autre mission, ils peuvent rechercher des missions dans d'autres entités si l'option Synchroniser la balance comptable est activée, bien que la synchronisation de la balance comptable ne soit disponible que pour les missions de la même entité.

  • Une erreur se produit lorsque les utilisateurs tentent d'ajouter une nouvelle zone lors de l'ajout d'un risque.

  • Lorsque les auteurs principaux mettent à jour un modèle de produit afin d'activer les signatures pour les risques et d'attribuer des schémas de signature autres que ceux par défaut, les schémas de signature ne sont pas appliqués correctement dans les engagements reportés si

    • L'engagement de l'année précédente a été créé avant la mise à jour du modèle de produit.

    • L'engagement reporté a été créé après la mise à jour du modèle de produit.

  • Lorsque les utilisateurs tentent d'ajouter une nouvelle section en haut d'une zone d'états financiers, la section est ajoutée au bas de la zone.

  • Amélioration technique concernant les espaces réservés aux contacts.

  • Dans les états financiers, les titres des domaines et des notes ne s'affichent pas.

  • Les utilisateurs ne peuvent pas modifier les champs de texte dans les documents.

  • Dans les paramètres du produit, lorsque les auteurs principaux activent l'arrondi dans les états financiers, les options de paramétrage correspondantes ne s'affichent pas.

  • Dans les modèles de produits, l'icône des paramètres de visibilité des conclusions de la liste de contrôle ne fonctionne pas.

  • Lorsque les utilisateurs importent un solde de planification, la date de la colonne CY - Planification ne s'affiche pas sur la page de données , à moins qu'ils ne rafraîchissent la page.

  • Certains documents n'affichent aucun contenu.
