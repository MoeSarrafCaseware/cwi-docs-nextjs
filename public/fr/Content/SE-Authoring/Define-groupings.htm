Définir les groupes

Les regroupements fournissent un contexte aux comptes que les utilisateurs importent dans le produit. Vous devez définir des regroupements afin d'établir une structure pour les comptes de la balance générale.

Les soldes du document des états financiers sont organisés et affichés en fonction des regroupements que vous avez définis. Si vos utilisateurs peuvent importer une balance de vérification dans le produit, vous devez définir une structure de regroupement.

Les liens de données suivants sont disponibles :

  • Regroupement financier - ressemble à la présentation du bilan et du compte de résultat. Vous pouvez ajouter jusqu'à trois structures de regroupement financier différentes à un produit. Il prend en charge :

    • générer les états financiers et les notes.

    • la réalisation d'une évaluation des risques dans le rapport RMM.

      Remarque : Seule la structure de regroupement financier par défaut peut être utilisée pour effectuer l'évaluation des risques.

    • fournir le niveau de détail approprié pour les graphiques d'analyse, les repères de matérialité, conditions de visibilité et les feuilles de route.

  • Groupement fiscal - Correspond aux codes fiscaux applicables à votre marché. La correspondance automatique entre le groupe financier et le code fiscal correspondant est disponible.

  • Plan comptable - Certains marchés peuvent utiliser un plan comptable standard qui ne ressemble pas nécessairement aux postes des états financiers.

Avant d'ajouter des groupes, vous devez les activer dans les paramètres du produit. Vous pouvez ensuite ajouter manuellement des groupes ou les importer à partir d'un fichier Excel, d'un fichier CSV ou de Working Papers.

Remarque : Pour que les structures de regroupement financier secondaires soient incluses dans le PDF généré lorsque les utilisateurs sélectionnent l'option Télécharger la copie de référence, vous devez ajouter ce qui suit à votre fichier product.json :

Titre : "Groupes financiers secondaires 2", "uri" : "index.jsp#/se-groups/financial-2" },
