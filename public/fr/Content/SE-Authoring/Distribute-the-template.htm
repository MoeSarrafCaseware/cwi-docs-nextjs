Distribuer le modèle

Une fois la rédaction terminée, enregistrez le projet de modèle.

Meilleure pratique : Nous vous recommandons de sauvegarder fréquemment le projet pendant que vous le rédigez. Pour en savoir plus, voir iXBRL.

Pour enregistrer votre projet :

  1. Naviguez jusqu'au menu Cloud, puis sélectionnez Paramètres.

  2. Sélectionnez le nom de votre produit, puis sélectionnez Gérer le modèle.

  3. Sélectionnez Enregistrer sous pour créer un modèle prêt à être distribué et utilisé.

    4. La version du modèle et la date de création s'affichent sur la page sous les rubriques Version du brouillon et Date de publication respectivement.

Meilleure pratique : Nous vous recommandons de tester votre modèle avant de le distribuer.

Pour tester le modèle, créez un fichier d'engagement et parcourez le flux de travail pour vous assurer que tout ce que vous avez configuré fonctionne comme prévu. Par exemple,

  • la logique de visibilité fonctionne comme prévu.

  • l'importation de la balance de vérification est réussie. Les groupes et les comptes sont correctement mappés.

  • le document relatif aux états financiers est correctement alimenté en données. Vous pouvez ajouter avec succès le tableau des flux de trésorerie.

  • les paramètres relatifs aux polices, aux décimales, etc. sont corrects.

  • les espaces réservés, le texte dynamique et les éléments du glossaire s'affichent correctement à l'endroit où vous les avez définis.

Pour distribuer le modèle, vous devez l'exporter puis contacter Third Party Solutions pour qu'il soit déployé et prêt à l'emploi.

To export the template:

  1. Naviguez jusqu'au menu Cloud, puis sélectionnez Paramètres.

  2. Sélectionnez le nom de votre produit, puis sélectionnez Gérer le modèle.

  3. Sélectionnez Exporter pour générer le fichier *.se.

  4. Enregistrer le fichier dans le contrôle de source / bitbucket pour le déployer.

Vous pouvez également envoyer le fichier *.se à Caseware pour un dépannage si nécessaire.

Remarque : Si vous apportez des modifications à un modèle pour un produit SE existant, notez que vos modifications ne s'appliquent que Nouvel engagement Les changements de modèle ne s'appliquent pas aux missions existantes (c'est-à-dire aux missions qui ont été créées à partir de l'ancien modèle de produit).
