Licences et prix

Lorsque vous vendez votre produit SE, vous pouvez facturer vos clients en utilisant l'une des méthodes suivantes :

  • Entreprise (redevance unique)

    Il s'agit d'une redevance unique qu'une entreprise paie pour acheter votre produit SE.

    Remarque : Il s'agit de la méthode de facturation par défaut du produit, aucune configuration n'est nécessaire pour l'activer.

  • Paiement par utilisateur

    • Cela signifie que vous facturez vos clients par membre du personnel. Pour activer la facturation par utilisateur, ajoutez la propriété "requiresPaymentPerUser":true au niveau racine du fichier collaborate.json de votre produit. Pour en savoir plus, consultez la documentation du SDK Smart Engagements.

    Remarque : Pour que la méthode de paiement par utilisateur soit facturée correctement, vous devez supprimer manuellement le produit du groupe Tout le personnel dans le nuage. Pour en savoir plus, voir Activer les applications pour les utilisateurs et les groupes.

  • Paiement par engagement

    Cela signifie que vous facturez vos clients par dossier d'engagement. Pour activer la facturation par engagement, ajoutez la propriété "requiresPaymentPerInstance" : true au niveau racine du fichier collaborate.json de votre produit. Pour en savoir plus, consultez la documentation du SDK Smart Engagements.

    Remarque : Pour que la méthode de paiement à l'engagement soit correctement facturée, assurez-vous que le produit est disponible dans le groupe "Tout le personnel" dans le nuage. Pour en savoir plus, voir Activer les applications pour les utilisateurs et les groupes.

La configuration des propriétés "requiresPaymentPerUser" et "requiresPaymentPerInstance" est effectuée par produit, il ne s'agit pas d'un paramètre universel pour tous les produits.

Avertissement :
- You can only have one billing method enabled in your product at a time.
- Assurez-vous que la méthode de facturation ajoutée dans le fichier collaborate.json correspond à la méthode de facturation de la plateforme de back-office Caseware. Par exemple, si la plateforme de back-office Caseware a configuré votre solution pour facturer les clients par engagement, alors le fichier collaborate.json de votre produit SE doit avoir la propriété "requiresPaymentPerInstance" : true ajoutée au niveau racine pas le "requiresPaymentPerUser":true.
