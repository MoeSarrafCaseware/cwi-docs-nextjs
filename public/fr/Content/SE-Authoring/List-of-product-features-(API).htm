Arrondi automatique

Pour activer l'arrondi automatique dans les états financiers, créez un script appelé financial-rounding-setup.js et incluez-le dans le tableau "js" de votre fichier product.json. Dans ce script, il faut inclure les éléments suivants :

(function() {
     'use strict' ;
     wpw.financials.financialsSetup.roundingErrorGroupNumbers = ['A', 'B'] ;
})();

a" et "B" sont des chaînes représentant les numéros des groupes de la structure dont la somme doit être égale à zéro. Si nécessaire, vous pouvez inclure plus de deux groupes.

Par exemple, si la structure de regroupement contient deux groupes, Actif et Passif, dont la somme doit être égale à zéro, remplacez la chaîne "A" de la fonction ci-dessus par le numéro de groupe de l'Actif et remplacez la chaîne "B" par le numéro de groupe du Passif. Le passif et l'actif sont ensuite additionnés et l'on vérifie qu'il n'y a pas d'erreur d'arrondi.

Notez que les différences d'arrondi sont appliquées par défaut au premier compte non nul du niveau le plus bas de l'enfant qui apparaît dans le premier groupe de la balance générale. Pour plus d'informations, voir Modifier le groupe par défaut pour les écarts d'arrondi.

Contrôle des contacts

Cette propriété contrôle l'accès des contacts à l'engagement. Par défaut, les contacts ne peuvent ni modifier ni consulter un engagement. Pour accorder un accès en vue aux contacts affectés à l'engagement, définissez contactAccess sur READ.

Notez qu'une fois que vous avez accordé aux contacts l'accès aux engagements, vous ne pouvez pas limiter les zones du fichier d'engagement auxquelles ils peuvent accéder. Nous recommandons d'utiliser cette fonction uniquement pour les produits de reporting et non pour les produits d'audit. Nous vous recommandons également d'examiner attentivement la fonctionnalité avant de lancer votre produit afin de vous assurer qu'elle fonctionne comme vous l'attendez.

Activer l'importation de types de données

Permet d'activer les options de type de données pour les données du grand livre, les données des transactions du grand livre auxiliaire et les importations de la balance de vérification à partir de fichiers CSV. Cela permet aux utilisateurs de sélectionner le type de jeu de données qui s'applique à leur importation de données.

Pour activer un type de jeu de données, incluez l'ID de schéma correspondant dans le fichier product.json, comme indiqué dans l'exemple de code ci-dessous. Notez que vous devez ajuster le code pour supprimer les ID de schéma pour tous les types de jeux de données que vous ne souhaitez pas activer pour les utilisateurs. See the descriptions linked below for more information.

"importTaxonomy" : {
					"id": "cwi.taxonomy.transactions", "version" : "0",
					"schemaIds": ["cwi.schema.general_ledger", "cwi.schema.ap_invoices", "cwi.schema.ap_payments_made", "cwi.schema.ap_suppliers", "cwi.schema.ap_open_balances", "cwi.schema.ar_invoices", "cwi.schema.ar_cash_received", "cwi.schema.ar_customers", "cwi.schema.ar_open_balances", "cwi.schema.inventory_transactions", "cwi.schema.inventory_on_hand" ] }

Pour permettre aux utilisateurs d'importer des données qui ne correspondent à aucun type de jeu de données, vous pouvez ajouter "schemaless" : true à la section "feature" du fichier product.json. Par conséquent, les utilisateurs auront la possibilité de sélectionner le type de jeu de données "Autres".

iXBRL

IXBRL est une norme ouverte qui permet aux entreprises de préparer des états financiers dans un format qui fournit les données structurées exigées par les régulateurs et les analystes, tout en permettant aux préparateurs de conserver un contrôle total sur la mise en page et la présentation de leur rapport.

Un fichier d'aide à la conversion et un pointeur vers ce fichier doivent être ajoutés dans l'API. See the XBRL SDK tutorial for the detailed instructions.

Si vous avez mis en place au moins une taxonomie XBRL, le mode de révision XBRL est disponible pour les utilisateurs finaux et activé par défaut. Si vous souhaitez cacher une taxonomie XBRL aux utilisateurs finaux, vous pouvez définir la propriété "hideFromReviewMode" à true dans la XBRLTaxonomyDefinition.

Notez que si toutes les taxonomies sont cachées, le mode de révision XBRL n'est pas disponible pour les utilisateurs finaux.

Pour en savoir plus sur la révision et l'exportation de documents XBRL, voir Révision et exportation XBRL.
