Recommandations pour la mise en œuvre du multilinguisme

Voici les recommandations de mise en œuvre pour la prise en charge multilingue :

  • Pour mettre en place des versions traduites des notes d'information : Créez plusieurs versions des notes dans différentes langues avec des paramètres de visibilité.

  • Pour mettre en place des versions traduites pour les lettres : Créez plusieurs versions des sections de la lettre dans différentes langues avec des paramètres de visibilité.

  • Mettre en place des questions d'interrogation traduites et des procédures de liste de contrôle : Créez plusieurs versions des procédures et des questions d'interrogation dans différentes langues à l'aide de texte dynamique.

Pour des instructions détaillées, voir Langue du contenu. Un produit qui prend en charge les langues française et anglaise.

Notez que :

  • La fonction de support multilingue ne traduit pas automatiquement le contenu. Vous devez fournir les traductions pour les langues que vous souhaitez que votre produit SE prenne en charge.

  • Si vous supprimez une langue, le texte traduit que vous avez ajouté dans le modèle n'est pas supprimé.

  • En cas de report, la langue sélectionnée dans le fichier de la période en cours est reportée dans le fichier de la période suivante.

  • Les auteurs du cabinet ne peuvent pas ajouter ou supprimer des langues, mais ils peuvent effectuer toutes les autres modifications.

  • Les modifications apportées par l'auteur du cabinet à la langue du contenu ne sont pas remplacées lorsqu'il met à jour le modèle du cabinet avec la dernière modification apportée au modèle du produit.

    • Il existe toutefois une exception :

    Si les auteurs de l'entreprise rendent une langue active dans le modèle de l'entreprise, vous la rendez inactive dans le modèle du produit. Dans ce cas, vos paramètres remplacent ce qui se trouve dans le modèle de l'entreprise lorsque les auteurs de l'entreprise le mettent à jour - et la langue devient inactive.
