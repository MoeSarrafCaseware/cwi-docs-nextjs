Demander un produit

Pour demander un produit, clonez le ticket JIRA DIST-1925 .

Caseware International (CWI) vous informe lorsque votre produit SE est accessible.

Notez les points suivants lorsque vous donnez un nom à votre produit SE :

  • Ne pas utiliser "Cloud" dans le nom du produit. Votre matériel de marketing peut décrire le produit comme étant basé sur l'informatique en nuage, mais ne doit pas utiliser le terme "Cloud" dans le nom du produit.

  • Le produit doit avoir un nom unique afin d'éviter toute confusion dans la documentation marketing et en ligne pour chaque produit ou application. Par exemple, vous ne pouvez pas nommer votre produit SE "Audit". Le nom doit inclure un mot unique pour le distinguer des autres produits d'audit SE, tels que "Audit - Inde" ou "Audit - OBNL".

Ce que vous devez fournir au CWI

Veillez à copier le contenu de la description du ticket DIST-1925 et à compléter autant d'informations que possible avant de soumettre votre ticket.

La liste suivante résume les informations que vous devez fournir au CWI, avec une brève description et des exemples.

  • Description

    Une brève description de votre produit.

    Exemple : Nous voulons disposer d'un nouveau produit d'audit basé sur une nouvelle méthodologie d'audit, ce qui nous permettra de pénétrer un marché plus large.

  • Dates clés

    Indiquez les dates clés suivantes pour votre projet :

    • Development Start Date
    • Date de publication de la version bêta du client
    • Mise en production

  • Avez-vous besoin d'une réunion de lancement ?

    Indiquez si vous souhaitez ou non une réunion de lancement. Les options suivantes sont disponibles :

    • Oui, nous n'avons pas d'expérience en matière de développement de nuages et nous avons besoin de nouveaux sites de développement de nuages.
    • Oui, nous aimerions présenter la solution que nous sommes en train de construire et passer en revue notre analyse des lacunes.
    • Non, merci.

  • Analyse des lacunes

    Fournir une analyse des lacunes des plateformes Caseware requises pour votre produit.

  • Cas d'entreprise

    Expliquez brièvement la raison d'être du produit. Inclure la taille du marché potentiel et un flux de travail que les utilisateurs d'un produit minimum viable suivent.

  • Chef de produit

    Nom et adresse électronique du chef de produit.

    Exemple : Jane Smith, <jane.smith@caseware.com>

  • Développeur principal

    Nom et adresse électronique du principal développeur de produits.

    Exemple : John Smith, <john.smith@caseware.com>

  • Type

    Sélectionnez le type d'extraction. Les options suivantes sont disponibles :

    • Modèle d'engagement intelligent (e)
    • Plugin Collaborate (p)
    • Microservice (m)

    Exemple : Modèle d'engagement intelligent (e)

  • Nom du produit

    Un nom pour votre produit.

    Exemple : SDK Demo Engagement

  • Société de développement

    Indiquez l'URL de votre société de développement.

    Exemple : http://ca.cwcloudpartner.com/ca-develop

  • Modèle de base SE

    Fournir un modèle de base à copier pour le nouveau produit, le cas échéant.

    Si cela n'est pas possible, nous utiliserons l'engagement générique actuel du TPS (com.caseware.tps.e.generic) comme modèle de base.
