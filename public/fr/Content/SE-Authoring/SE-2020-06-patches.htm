SE 2020-06 correctifs

Voici un résumé des correctifs qui ont été déployés pour SE 2020-06.

2020-06-U2

Version : 2020 / 06 / 524

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U2.

  • [SE-21533] - Les colonnes de variance en pourcentage dans les tableaux dynamiques ne sont pas évaluées correctement lorsque des décimales sont utilisées comme séparateur de chiffres. The specified percentage is multiplied by 100.

  • [SE-21473] - Les documents ne s'impriment pas comme prévu pour les formats de page autres que Letter. La mise en page du PDF généré ne correspond pas à celle de l'aperçu avant impression.

  • [SE-20227] - Les liaisons de données (fx) faisant référence à des réponses de procédures de contact ne sont pas évaluées correctement.

  • [SE-21410] - Problèmes de performance lors de l'ouverture d'engagements pour des produits avec un grand nombre de formulaires SE builder.

2020-06-U3

Version : 2020 / 06 / 526

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U3.

  • [SE-21674] - Une erreur se produit lorsque les utilisateurs tentent d'imprimer les états financiers après avoir importé des transactions dans l'engagement.

  • [SE-21405] - Les liens de données (fx) référençant les réponses aux procédures de contact affichent les seconds prénoms dans le désordre.

2020-06-U4

Version : 2020 / 06 / 527

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U4.

  • Lorsque les utilisateurs impriment les états financiers,

    • [SE-21750] - Une erreur se produit si le document contient des graphiques d'analyse qui ont été reportés d'un fichier de l'année précédente avant la version SE 2020-06. Les états financiers ne s'impriment pas.

    • [SE-21783] - Le tableau des flux de trésorerie, défini dans statement.js, ne s'affiche pas comme prévu dans le PDF généré.

    • [SE-21446] - Les graphiques d'analyse ne s'affichent pas comme prévu dans le PDF généré.

  • [SE-21718] - Si les utilisateurs ajoutent une ligne de groupe personnalisé à un tableau dynamique, ferment le mode d'édition et le rouvrent ensuite, les types de ligne Balance de vérification et Entrée ne sont plus disponibles pour être ajoutés dans le groupe personnalisé.

  • [SE-21652] - Dans les tableaux dynamiques IFRS, lorsque les utilisateurs actualisent ou naviguent hors de la page,

    • Les données des lignes de dimensions sont supprimées

    • Les lignes de dimension qui sont définies pour afficher les données des dimensions de second niveau sont modifiées pour afficher les données de la dimension primaire correspondante.

  • [SE-21659] - Les utilisateurs sont redirigés vers la page Documents lorsqu'ils sélectionnent une transaction pour en afficher les détails si l'ID d'entrée de la transaction comprend une barre oblique (/).

2020-06-U5

Version : 2020 / 06 / 528

Un problème a été corrigé avec le déploiement de la SE 2020-06-U5.

[SE-21817] - Les tableaux dynamiques ne s'affichent pas comme prévu dans les états financiers ou les feuilles de calcul et une erreur se produit lorsque les utilisateurs tentent de les imprimer.

2020-06-U6

Version : 2020 / 06 / 530

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U6.

  • [SE-21837] - Les tableaux dynamiques qui utilisent des données consolidées et incluent des lignes d'entrée dans des groupes personnalisés ne s'affichent pas comme prévu dans les états financiers.

  • [SE-21801] - Les graphiques d'analyse ne s'affichent pas comme prévu. L'axe vertical ne permet pas d'afficher toutes les valeurs.

  • [SE-21671] - Lorsque les utilisateurs impriment des états financiers avec des sections d'analyse contenant des graphiques plus grands qu'une page, les graphiques ne sont pas mis à l'échelle pour s'adapter à la page.

2020-06-U7

Version : 2020 / 06 / 531

Un problème a été corrigé avec le déploiement de la SE 2020-06-U7.

[SE-21767] - Une erreur se produit lorsque les utilisateurs tentent d'utiliser l'option Copier à partir d'un autre produit dans le modèle de produit pour copier des états financiers qui incluent des conditions de visibilité de la consolidation .

2020-06-U8

Version : 2020 / 06 / 534

La SE 2020-06-U8 traite des améliorations qui atténuent le risque de pannes généralisées lorsque :

  • Les utilisateurs tentent de télécharger une copie de référence d'un engagement comportant plus de 600 documents

  • Print errors occur. La répartition de la charge des services d'impression a été améliorée pour garantir l'efficacité.

2020-06-U9

Version : 2020 / 06 / 538

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U9.

  • Lorsque les utilisateurs impriment les états financiers, le PDF généré ne s'affiche pas comme prévu. La liste suivante résume les problèmes liés à la production de PDF :

    • [SE-21947] - Problèmes de style liés à l'espacement des paragraphes dans les sections de texte, à la justification dans les cellules des tableaux dynamiques et aux marges des pages.

    • [SE-22012] - Les icônes de visibilité () s'affichent dans le PDF pour les listes dont la logique de visibilité est configurée.

    • [SE-22032] - Dans les dossiers d'engagement basés sur des modèles qui utilisent l'ancien tableau des flux de trésorerie, les lignes horizontales dans le tableau des flux de trésorerie ne s'affichent pas comme prévu.

    • [SE-21832] - Les liaisons de données (fx) référençant les réponses aux procédures sont doublées.

  • [SE-21926] - Les auteurs primaires rencontrent des problèmes de performance lors de l'édition de tableaux dynamiques dans les états financiers.

  • [SE-22073] - Dans les tableaux dynamiques, si les auteurs principaux font référence à des notes cachées, une virgule s'affiche avant le numéro de la note dans la colonne de référence de la note pour les utilisateurs engagés.

  • [SE-21972] - Une erreur se produit lorsque les utilisateurs tentent d'exporter des fichiers iXBRL et XBRL qui incluent des tableaux dynamiques avec des liens de données (fx).

  • [SE-21940] - Dans les cellules des tableaux dynamiques, les calculs personnalisés ajoutés dans les colonnes où la Somme des transactions est sélectionnée comme source de données ne fonctionnent pas correctement. Les valeurs calculées sont incorrectes.

  • [SE-21864] - Les étiquettes de diagnostic ne fonctionnent pas comme prévu dans les états financiers. Les comptes dont les soldes sont en désaccord ne sont pas signalés aux utilisateurs de l'engagement. L'indicateur d'écart de solde () ne s'affiche pas et les outils de révision du contrôle des soldes ne signalent pas l'écart.

2020-06-U10

Version : 2020 / 06 / 544

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U10.

  • [SE-22283] - Dans les états financiers et les formulaires SE Builder, la taille de la page revient automatiquement à Lettre une fois que la sortie PDF est produite, remplaçant la taille de la page existante.

  • [SE-22136] - Dans les états financiers, si un saut de page est la première section visible dans une zone, une page blanche est insérée dans le document.

  • [SE-22091] - Dans les listes de contrôle, le contenu des réponses aux procédures présentées sous forme de tableau ne s'affiche pas dans la sortie PDF ou dans la copie de référence. Les tableaux vides apparaissent dans le PDF.

  • [SE-21955] - Dans les missions comportant des données de balance de vérification volumineuses, les tentatives de téléchargement d'une copie de référence peuvent échouer et un fichier zip vide est généré.

  • [SE-22196] - Lors de l'affichage de documents PDF dans le fichier d'engagement, les fonctions de recherche et de défilement peuvent ne pas fonctionner comme prévu.

2020-06-U11

Version : 2020 / 06 / 549

Voici la liste des problèmes qui ont été corrigés avec le déploiement de SE 2020-06-U11.

  • [SE-22433] - Problèmes de style liés à la justification dans les tableaux dynamiques et à l'espacement lorsque les utilisateurs exportent un fichier iXBRL.

  • [SE-22337] - L'indicateur des espaces réservés dans les états financiers affiche un nombre incorrect d'espaces réservés restants.

  • [SE-22369] - Le report ne fonctionne pas comme prévu dans les grands états financiers. Les tableaux dynamiques figurant dans les notes ne sont pas reportés.

  • [SE-22253]- Dans les tableaux dynamiques, les cellules de tableau avec des montants négatifs sont indentées alors que les cellules avec des montants positifs ne le sont pas.

  • [SE-22270] - Lorsque les utilisateurs impriment les états financiers, les numéros de notes et les titres de notes correspondants s'affichent sur des pages séparées.

  • [SE-22333] - Lorsque les utilisateurs impriment les états financiers, les tableaux dynamiques et les sections de texte ne sont pas alignés.

  • [SE-22505] - Une erreur se produit lorsque les utilisateurs tentent d'utiliser Caseware Cloud Connector.
