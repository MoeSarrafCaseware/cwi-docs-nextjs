Conditions de visibilité

La liste suivante décrit les conditions de visibilité, les paramètres disponibles pour chaque condition et montre un exemple pour chaque condition.

RMM (risque d'anomalies significatives)

Sélectionnez un type d'assertion, un groupe de la balance générale et l'une des conditions suivantes :

  • RMM - le groupe sélectionné dispose d'un RMM pour ce type d'assertion
  • Pas de RMM - le groupe sélectionné n'a pas de RMM pour ce type d'assertion

Si le nouveau module de risque est activé dans votre produit, un ensemble différent de paramètres est disponible pour cette condition :

  • Sélectionner le type d'assertion
  • Sélectionner le domaine ou le cycle économique
  • Spécifier l'opération logique à comparer et à évaluer (=,<=,>=)
  • Sélectionner l'échelle RMM

Les valeurs de la liste des échelles RMM sont définies dans la configuration des échelles RMM dans les paramètres du produit.

Exemple

Si l' exhaustivité de l ' encaisse n'a pas de RMM, afficher cet élément.

Ou si vous avez activé le nouveau module de risque :

Si Précision pour Inventaire a RMM inférieur à Moyenalors cachez cet élément.

Valeur du formulaire

Vous pouvez inclure un ou plusieurs formulaires dans la configuration des conditions. Sélectionnez le nom du formulaire, un champ du formulaire sélectionné et une valeur pour le champ sélectionné.

Les options de valeur pour un champ sélectionné dépendent du type de champ. Par exemple, les champs numériques comportent des opérateurs logiques et des valeurs numériques, tandis que les champs de date comportent une icône de calendrier pour la sélection de la date.

Remarque : Si vous n'avez pas développé de formulaires à l'aide de SE Builder dans votre produit, les menus déroulants des paramètres pour cette condition sont désactivés.

Exemple

Si le champ T2S8 dans S8 - Déduction pour amortissement (DPA) est > 10 ou le champ T3S8 en S8 - Déduction pour amortissement (DPA) est > 10alors affichez ce poste.
