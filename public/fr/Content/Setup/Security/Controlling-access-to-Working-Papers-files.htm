Contrôle de l'accès aux fichiers Caseware Working Papers

Si votre organisation utilise la fonction de sécurité Working Papers sur Cloud, vous bénéficiez du meilleur des deux mondes. La sécurité des Working Papers combine la facilité de gestion des utilisateurs disponible sur le Cloud avec les paramètres de protection détaillés disponibles dans les fichiers des Working Papers.

La sécurité des Working Papers fonctionne de manière très similaire aux rôles de sécurité utilisés dans le Cloud pour gérer l'accès de vos utilisateurs et groupes. Tout d'abord, vous créez des rôles Working Papers dans Cloud, puis vous pouvez activer la sécurité Working Papers pour votre organisation. Une fois la sécurité activée, vous pouvez activer cette protection avancée sur les fichiers Working Papers que votre organisation a publiés sur Cloud, en attribuant les rôles Working Papers aux membres de votre personnel pour leur permettre d'accéder aux zones appropriées du fichier.

Pour importer des paramètres de configuration de sécurité à partir de Working Papers :

Si votre organisation utilise déjà les paramètres de protection disponibles dans les Working Papers, vous pouvez exporter vos données à partir de l'outil d'administration du magasin de données, puis importer ces paramètres dans Cloud.

Pour importer des paramètres de configuration de sécurité à partir de Working Papers :

  1. Assurez-vous que vous avez le rôle d' administrateur des paramètres ou des privilèges équivalents.

  2. Dans le menu Cloud, sélectionnez Paramètres.

  3. Select Working Papers | Working Papers Sécurité. La page Working Papers Security s'affiche.

  4. Cliquez sur le bouton Nouveau , puis sélectionnez Importer. La boîte de dialogue Importation s'affiche.

  5. Sélectionnez le bouton Sélectionner les fichiers .

  6. Sélectionnez le fichier .txt que vous avez exporté à partir de l'outil d'administration du magasin de données et sélectionnez Ouvrir. La boîte de dialogue Importation de personnel et de groupes s'affiche.

  7. Passez en revue les membres du personnel, les groupes et les rôles qui seront importés. Vous pouvez renommer n'importe quel groupe ou rôle et modifier leur description.

  8. Une fois que vous avez terminé de revoir le contenu de l'importation, sélectionnez Importer.

Les rôles de sécurité de votre fichier Working Papers sont créés dans le nuage. Vous créerez automatiquement de nouveaux profils de personnel pour tous les utilisateurs de l'importation qui n'avaient pas encore de profil, et vous créerez aussi automatiquement des groupes.

Pour créer des rôles de sécurité pour les Working Papers :

Si votre organisation n'utilise pas la protection des documents de travail, vous pouvez créer des rôles de sécurité pour les documents de travail dans le nuage. Ces rôles avancés utilisent la même liste de permissions que les Working Papers.

Pour créer des rôles de sécurité pour les Working Papers :

  1. Assurez-vous que vous avez le rôle d' administrateur des paramètres ou des privilèges équivalents.

  2. Dans le menu Cloud, sélectionnez Paramètres.

  3. Select Working Papers | Working Papers Sécurité. La page Working Papers Security s'affiche.

  4. Cliquez sur le bouton Nouveau , puis sélectionnez Créer. La boîte de dialogue Créer un rôle de document de travail s'affiche.

  5. Saisissez un nom de rôle. Vous pouvez également saisir une description.

  6. Le choix d'une catégorie sur la gauche affiche ses autorisations sur la droite. Sélectionnez toutes les autorisations relatives aux documents de travail qui s'appliquent à votre nouveau rôle.

  7. Si votre organisation utilise des niveaux d'accès CaseView, sélectionnez la catégorie de niveau d'accès des documents CaseView sur la gauche et sélectionnez un niveau d'accès approprié à votre rôle.

  8. Sélectionnez Enregistrer.

Vous avez maintenant créé le nouveau rôle de sécurité Working Papers. Une fois que vous avez créé quelques rôles correspondant au type de protection dont vous avez besoin, vous devriez être prêt à activer la sécurité des Working Papers.

Pour activer la sécurité des Working Papers :

Une fois les rôles de sécurité Working Papers créés sur Cloud, vous pouvez activer la sécurité Working Papers. Il s'agit d'un paramètre applicable à l'ensemble du système qui permet aux utilisateurs d'activer la protection avancée sur des fichiers spécifiques des Working Papers.

Pour activer la sécurité des Working Papers :

  1. Assurez-vous que vous avez le rôle d' administrateur des paramètres ou des privilèges équivalents.

  2. Dans le menu Cloud, sélectionnez Paramètres.

  3. Select Working Papers | Working Papers Sécurité. La page Working Papers Security s'affiche.

  4. Sélectionnez le bouton Désactivé .

  5. Sélectionnez Oui lorsque vous y êtes invité.

Vous avez maintenant activé la sécurité Working Papers pour l'ensemble de l'organisation, ce qui signifie que vous pouvez activer la protection avancée sur les fichiers Working Papers.

Pour activer la protection avancée sur un fichier Working Papers :

Une fois que la sécurité des Working Papers est activée pour l'organisation, les propriétaires des fichiers Working Papers publiés peuvent activer la protection avancée de ces fichiers. Cela vous permettra d'accorder l'accès aux membres du personnel en leur attribuant des rôles de sécurité Working Papers sur les fichiers publiés.

Pour activer la protection avancée sur un fichier Working Papers :

  1. Assurez-vous que vous avez le rôle de propriétaire ou des privilèges équivalents pour le fichier Working Papers.

  2. Dans le menu Cloud, sélectionnez Working Papers.

  3. Sélectionnez un fichier Working Papers, puis cliquez sur le bouton Partager ().

  4. Sélectionnez l'option Plus d'actions () | Activer la protection avancée.

  5. Sélectionnez un ou plusieurs utilisateurs à qui vous donnerez le rôle avancé pour ce fichier Working Papers.

  6. Sélectionnez le bouton Attribuer des rôles à (#) dans le coin inférieur gauche de la boîte de dialogue Partager, puis sélectionnez le rôle avancé applicable.

  7. Sélectionnez Partager.

Vous venez d'activer la protection avancée pour ce fichier Working Papers. Les utilisateurs que vous avez sélectionnés auront accès aux zones du fichier en fonction de leur rôle, et ils obtiendront toutes les autorisations correspondantes pour le document de travail.
