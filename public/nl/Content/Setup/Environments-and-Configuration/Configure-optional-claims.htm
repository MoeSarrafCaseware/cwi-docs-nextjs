Optionele claims configureren (Azure AD)

Voordat je het SSO-integratieproces afrondt, moet je het manifest bewerken om de given_name en family_name claims op te nemen. Als een medewerker SSO gebruikt, geven deze claims hun voor- en achternaam door aan Cloud.

Om optionele claims te configureren:

  1. Meld u aan bij uw Azure-portaal.

  2. Selecteer Azure Active Directory.

  3. Selecteer App-registratiesin het linker navigatiedeelvenster.

  4. Selecteer de Import Component uit de lijst.

  5. Selecteer Manifestin het linker navigatiedeelvenster.

  6. Bewerk de eigenschap optionalClaims als volgt:

    "optionalClaims": {
    
	"idToken": [
    
		{
    
			Naam "voornaam",
    
			"essentieel": waar
    
		},
    
		{
    
			Naam "family_name",
    
			"essentieel": waar
    
		}
    
	]
    
},

    Let op: U moet de bovenstaande informatie exact kopiëren om optionele claims correct te configureren.

  7. Selecteer Opslaan.

Volgende De antwoord-URL genereren (Okta)
